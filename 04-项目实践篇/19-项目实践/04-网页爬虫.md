# 项目四：网页爬虫

网页爬虫是互联网数据采集的核心工具，搜索引擎、价格比较网站、新闻聚合器背后都有爬虫的身影。本项目将从零构建一个结构清晰、有礼貌的网页爬虫，综合运用 HTTP 请求、HTML 解析、文件操作和面向对象等知识。

## 第一部分：项目概述

### 1.1 什么是网页爬虫

想象一位图书馆员工，他的任务是把图书馆里某个主题的书全部读一遍，然后摘录重点写成索引卡。他的工作流程是：

1. 从一张书单（URL 列表）出发
2. 取出第一本书（下载页面 HTML）
3. 阅读内容，摘录关键信息（解析 HTML，提取数据）
4. 把摘录保存到笔记本（写入文件）
5. 在书中发现了其他书的引用（发现新链接）
6. 把新书名加入书单，继续循环

网页爬虫做的正是这件事，只不过"书"变成了网页，"摘录"变成了结构化数据。

## 第三部分：核心概念讲解

### 3.1 HTML 基础结构

#### 概念说明

HTML（超文本标记语言）是网页的骨架。理解 HTML 结构是使用爬虫的前提——我们需要知道"要找的内容藏在哪个标签里"。

HTML 由**标签**（tag）构成，标签可以嵌套，形成树状结构：

```
┌─ html ───────────────────────────────────┐
│  ┌─ head ─────────────────────────────┐  │
│  │    <title>页面标题</title>          │  │
│  └────────────────────────────────────┘  │
│  ┌─ body ─────────────────────────────┐  │
│  │  ┌─ div class="article" ────────┐  │  │
│  │  │   <h1>文章标题</h1>           │  │  │
│  │  │   <p>第一段正文...</p>        │  │  │
│  │  │   <a href="/page2">下一页</a>│  │  │
│  │  └──────────────────────────────┘  │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

关键概念：
- **标签**：`<h1>标题</h1>`，有开始标签和结束标签
- **属性**：`<a href="/page2">`，`href` 是属性名，`"/page2"` 是属性值
- **嵌套关系**：标签可以包含其他标签，形成父子关系
- **class / id**：用于标识特定元素，是 CSS 选择器的基础

#### 示例代码

```python
# 这是一段示例 HTML，我们用它来理解结构
html_example = """
<html>
  <head>
    <title>Python 教程</title>
  </head>
  <body>
    <div class="article">
      <h1 id="main-title">欢迎学习 Python</h1>
      <p class="intro">Python 是一门简洁优雅的编程语言。</p>
      <ul>
        <li><a href="/chapter1">第一章</a></li>
        <li><a href="/chapter2">第二章</a></li>
      </ul>
      <img src="/logo.png" alt="Python Logo">
    </div>
  </body>
</html>
"""

# 用 BeautifulSoup 解析这段 HTML
from bs4 import BeautifulSoup

soup = BeautifulSoup(html_example, "html.parser")

# 查找标题标签
title = soup.find("title")
print(f"页面标题：{title.get_text()}")       # 输出：Python 教程

# 查找 h1 标签
h1 = soup.find("h1")
print(f"文章标题：{h1.get_text()}")          # 输出：欢迎学习 Python
print(f"h1 的 id：{h1['id']}")               # 输出：main-title

# 查找所有链接
links = soup.find_all("a")
for link in links:
    print(f"链接文字：{link.get_text()}，地址：{link['href']}")
# 输出：
# 链接文字：第一章，地址：/chapter1
# 链接文字：第二章，地址：/chapter2
```

### 3.2 BeautifulSoup 基础用法

#### 概念说明

BeautifulSoup 就像一个"翻译官"，把杂乱的 HTML 文本转化成我们可以方便查询的对象树。

主要查询方式有三种，各有适用场景：

```
┌──────────────────────────────────────────────────────────┐
│              BeautifulSoup 查询方式对比                   │
├──────────────┬───────────────────┬──────────────────────┤
│  方法         │  用途             │  示例                │
├──────────────┼───────────────────┼──────────────────────┤
│  find()      │  找第一个匹配元素  │  soup.find("h1")     │
├──────────────┼───────────────────┼──────────────────────┤
│  find_all()  │  找所有匹配元素   │  soup.find_all("a")  │
├──────────────┼───────────────────┼──────────────────────┤
│  select()    │  用 CSS 选择器    │  soup.select(".nav") │
└──────────────┴───────────────────┴──────────────────────┘
```

提取信息的方式：
- `.get_text()`：获取标签内的纯文本（去掉所有 HTML 标签）
- `tag['属性名']`：获取标签的属性值，如 `a['href']`、`img['src']`
- `tag.get('属性名')`：安全获取属性，不存在时返回 `None` 而不是报错

#### 示例代码

```python
from bs4 import BeautifulSoup

html = """
<div class="content">
  <h1>Python 官方文档</h1>
  <p class="summary">Python 是一门通用编程语言。</p>
  <nav class="links">
    <a href="https://docs.python.org/tutorial">教程</a>
    <a href="https://docs.python.org/library">标准库</a>
  </nav>
  <img src="/python-logo.png" alt="Python Logo">
</div>
"""

soup = BeautifulSoup(html, "html.parser")

# --- find()：查找第一个匹配的元素 ---
h1 = soup.find("h1")
print(h1.get_text())  # 输出：Python 官方文档

# 按 class 属性查找
summary = soup.find("p", class_="summary")
# 注意：Python 的 class 是关键字，BeautifulSoup 用 class_ 代替
print(summary.get_text())  # 输出：Python 是一门通用编程语言。

# --- find_all()：查找所有匹配的元素 ---
all_links = soup.find_all("a")
print(f"共找到 {len(all_links)} 个链接")

for link in all_links:
    text = link.get_text(strip=True)   # strip=True 去掉首尾空白
    href = link.get("href", "")        # 安全获取 href，默认空字符串
    print(f"  [{text}] → {href}")

# --- select()：使用 CSS 选择器 ---
# CSS 选择器语法更灵活，适合复杂查询
nav_links = soup.select("nav.links a")    # nav 标签中 class=links 下的所有 a
for link in nav_links:
    print(link["href"])

# 按 id 选择
# soup.select("#main-title")  等价于 soup.find(id="main-title")

# --- 获取属性 ---
img = soup.find("img")
print(img["src"])               # 输出：/python-logo.png
print(img.get("alt", "无描述"))  # 输出：Python Logo
```

### 3.3 requests.Session 会话保持

#### 概念说明

`requests.Session` 好比给爬虫办了一张"会员卡"。每次用 `Session` 发请求，都会：

1. **复用 TCP 连接**：不必每次重新握手建立连接，速度更快
2. **自动保持 Cookie**：登录一次后，后续请求自动携带登录凭证
3. **统一设置请求头**：只需设置一次 `User-Agent`，所有请求都会带上

```
┌─────────────────────────────────────────────────────────┐
│        requests.get() vs requests.Session               │
├──────────────────────────┬──────────────────────────────┤
│  每次 requests.get()     │  使用 Session                │
├──────────────────────────┼──────────────────────────────┤
│  建立新连接 → 关闭连接   │  建立一次连接 → 保持复用     │
│  不保存 Cookie           │  自动管理 Cookie             │
│  每次单独设置 Header     │  一次设置，全程生效          │
│  适合：偶尔发一两个请求  │  适合：爬取多个页面          │
└──────────────────────────┴──────────────────────────────┘
```

#### 示例代码

```python
import requests

# 方式一：不用 Session（每次都是全新的请求）
response1 = requests.get("https://httpbin.org/get")
response2 = requests.get("https://httpbin.org/get")
# 这两次请求之间没有任何关联，Cookie 不会共享

# 方式二：使用 Session（推荐爬虫使用）
session = requests.Session()

# 只需设置一次，所有请求都会自动带上这个 Header
session.headers.update({
    "User-Agent": "PythonLearnerBot/1.0 (学习用途)",
    "Accept-Language": "zh-CN,zh;q=0.9",
})

# 发送多个请求，Session 会自动处理 Cookie 和连接复用
response1 = session.get("https://httpbin.org/get")
response2 = session.get("https://httpbin.org/headers")  # 会带上同样的 Header

print(f"状态码：{response1.status_code}")  # 200 表示成功
print(f"内容类型：{response1.headers['Content-Type']}")

# 使用完毕后关闭（也可以用 with 语句自动关闭）
session.close()

# 更推荐的写法：用 with 语句自动关闭
with requests.Session() as session:
    session.headers["User-Agent"] = "PythonLearnerBot/1.0"
    response = session.get("https://httpbin.org/get")
    print(response.status_code)
# 离开 with 块后，session 自动关闭
```

### 3.4 礼貌爬取——速率限制

#### 概念说明

爬虫如果不加控制，可能每秒发送几十甚至几百个请求。对普通网站来说，这相当于在门口堵了几百个人同时要进去，服务器很快就会崩溃或者封禁你的 IP。

礼貌爬虫的核心原则是：**两次请求之间主动等待**。

```
┌─────────────────────────────────────────────────────────┐
│                   爬取速率建议                           │
├─────────────────────────┬───────────────────────────────┤
│  场景                   │  建议等待时间                 │
├─────────────────────────┼───────────────────────────────┤
│  一般网站               │  1 - 3 秒                    │
│  robots.txt 有指定      │  遵守 Crawl-delay 指令        │
│  大型商业网站           │  3 - 5 秒或更长              │
│  学习练习               │  2 秒以上，保持保守           │
└─────────────────────────┴───────────────────────────────┘
```

`time.sleep(秒数)` 让程序暂停指定时间，这是实现速率控制最简单的方式。

#### 示例代码

```python
import time
import random
import requests

def fetch_with_delay(urls: list, min_delay: float = 1.0, max_delay: float = 3.0):
    """
    带速率控制的批量请求

    使用随机延迟而不是固定延迟，原因：
    - 固定间隔的请求模式容易被识别为机器人
    - 随机延迟更接近人类浏览行为
    """
    results = []

    for i, url in enumerate(urls):
        print(f"正在获取第 {i+1}/{len(urls)} 个页面：{url}")

        try:
            response = requests.get(url, timeout=10)
            results.append({
                "url": url,
                "status": response.status_code,
                "content": response.text[:200],  # 只取前200字符作演示
            })
        except requests.RequestException as e:
            print(f"  请求失败：{e}")
            results.append({"url": url, "status": "error", "content": ""})

        # 不是最后一个 URL 时才需要等待
        if i < len(urls) - 1:
            delay = random.uniform(min_delay, max_delay)
            print(f"  等待 {delay:.1f} 秒后继续...")
            time.sleep(delay)

    return results

# 示例调用
urls = [
    "https://httpbin.org/get",
    "https://httpbin.org/headers",
]
# results = fetch_with_delay(urls, min_delay=1.0, max_delay=2.0)
```

### 3.5 URL 处理——相对 URL 与绝对 URL

#### 概念说明

网页中的链接分两种：

```
┌─────────────────────────────────────────────────────────────┐
│  当前页面：https://docs.python.org/zh-cn/3/tutorial/        │
├────────────────────┬────────────────────────────────────────┤
│  相对 URL          │  绝对 URL                             │
├────────────────────┼────────────────────────────────────────┤
│  /library/index    │  https://docs.python.org/library/     │
│  ../glossary       │  https://example.com/page             │
│  ./introduction    │  http://other-site.org/path           │
├────────────────────┼────────────────────────────────────────┤
│  需要"拼接"才能用  │  可以直接访问                        │
└────────────────────┴────────────────────────────────────────┘
```

`urllib.parse.urljoin(base, url)` 可以将相对 URL 正确地转换为绝对 URL，处理各种边界情况（比如 `../` 的路径跳转）。

#### 示例代码

```python
from urllib.parse import urljoin, urlparse

base_url = "https://docs.python.org/zh-cn/3/tutorial/"

# urljoin 自动处理各种相对 URL 的情况
print(urljoin(base_url, "/library/index"))
# 输出：https://docs.python.org/library/index
# （以 / 开头：从根路径开始）

print(urljoin(base_url, "../glossary"))
# 输出：https://docs.python.org/zh-cn/3/glossary
# （../ 跳到上一级目录）

print(urljoin(base_url, "introduction"))
# 输出：https://docs.python.org/zh-cn/3/tutorial/introduction
# （相对于当前路径）

print(urljoin(base_url, "https://example.com/page"))
# 输出：https://example.com/page
# （已经是绝对 URL，直接返回）

# urlparse 可以拆解 URL 的各个部分
parsed = urlparse("https://docs.python.org/zh-cn/3/tutorial/index.html?q=hello")
print(f"协议：{parsed.scheme}")    # https
print(f"域名：{parsed.netloc}")    # docs.python.org
print(f"路径：{parsed.path}")      # /zh-cn/3/tutorial/index.html
print(f"参数：{parsed.query}")     # q=hello

# 实际应用：判断链接是否在同一域名下
def is_same_domain(base_url: str, link_url: str) -> bool:
    """检查两个 URL 是否属于同一域名"""
    base_domain = urlparse(base_url).netloc
    link_domain = urlparse(link_url).netloc
    return base_domain == link_domain

print(is_same_domain(
    "https://docs.python.org/tutorial",
    "https://docs.python.org/library"
))  # True

print(is_same_domain(
    "https://docs.python.org/tutorial",
    "https://example.com/page"
))  # False
```

### 4.1 完整代码

将以下代码保存为 `scraper.py`：

```python
"""
网页爬虫 - 完整实现
项目四：网页爬虫

功能：
- 爬取指定网站的多个页面
- 提取标题、正文、链接、图片等信息
- 检查 robots.txt 遵守爬取规范
- 保存为 Markdown 和 JSON 格式
"""

import json
import time
import random
from datetime import datetime
from pathlib import Path
from urllib.parse import urljoin, urlparse
from urllib.robotparser import RobotFileParser
from typing import Optional, List, Dict, Any

import requests
from bs4 import BeautifulSoup


class WebScraper:
    """
    礼貌的网页爬虫

    设计理念：
    - 遵守 robots.txt 规范
    - 两次请求之间随机等待，避免给服务器造成压力
    - 自动去重，不重复爬取同一页面
    - 限定在同一域名内爬取
    """

    def __init__(
        self,
        base_url: str,
        output_dir: str = "output",
        min_delay: float = 1.5,
        max_delay: float = 3.0,
    ):
        """

        参数：
            base_url    起始 URL，爬虫从这里出发
            output_dir  保存结果的目录
            min_delay   两次请求之间的最短等待时间（秒）
            max_delay   两次请求之间的最长等待时间（秒）
        """
        self.base_url = base_url
        self.domain = urlparse(base_url).netloc  # 提取域名，限制爬虫不离开此域名
        self.output_dir = Path(output_dir)
        self.output_dir.mkdir(parents=True, exist_ok=True)  # 自动创建输出目录

        self.min_delay = min_delay
        self.max_delay = max_delay

        # 记录已经访问过的 URL，避免重复爬取
        self.visited_urls: set = set()

        # 记录爬取失败的 URL
        self.failed_urls: List[str] = []

        # 上次请求的时间戳，用于计算等待时间
        self._last_request_time: float = 0.0

        # 创建 Session，复用连接，统一设置请求头
        self.session = requests.Session()
        self.session.headers.update({
            # User-Agent 标识爬虫身份和用途，让网站管理员知道是谁在访问
            "User-Agent": (
                "PythonLearnerBot/1.0 "
                "(学习用途爬虫；仅访问公开内容；"
                "遵守 robots.txt)"
            ),
            "Accept": "text/html,application/xhtml+xml",
            "Accept-Language": "zh-CN,zh;q=0.9,en;q=0.8",
        })

        print(f"爬虫初始化完成")
        print(f"  起始地址：{self.base_url}")
        print(f"  限定域名：{self.domain}")
        print(f"  输出目录：{self.output_dir.resolve()}")
        print(f"  请求间隔：{self.min_delay} - {self.max_delay} 秒")

    def _respect_delay(self) -> None:
        """
        速率控制：确保两次请求之间有足够的间隔

        使用随机延迟而非固定延迟，原因：
        - 随机模式更像人类浏览行为
        - 固定间隔容易被识别并封禁
        """
        now = time.time()
        elapsed = now - self._last_request_time  # 距上次请求过了多少秒

        # 如果距离上次请求的时间还不够，则补充等待
        required_delay = random.uniform(self.min_delay, self.max_delay)
        if elapsed < required_delay:
            wait_time = required_delay - elapsed
            print(f"  礼貌等待 {wait_time:.1f} 秒...")
            time.sleep(wait_time)

        self._last_request_time = time.time()

    def check_robots_txt(self, url: str) -> bool:
        """
        检查 robots.txt 是否允许爬取指定 URL

        robots.txt 文件位于网站根目录，例如：
        https://www.python.org/robots.txt

        参数：
            url  要检查的目标 URL
        返回：
            True 表示允许爬取，False 表示禁止
        """
        parsed = urlparse(url)
        robots_url = f"{parsed.scheme}://{parsed.netloc}/robots.txt"

        # RobotFileParser 是 Python 标准库，专门用于解析 robots.txt
        rp = RobotFileParser()
        rp.set_url(robots_url)

        try:
            rp.read()  # 下载并解析 robots.txt
            allowed = rp.can_fetch(self.session.headers["User-Agent"], url)
            if not allowed:
                print(f"  [robots.txt 禁止] {url}")
            return allowed
        except Exception as e:
            # 如果无法获取 robots.txt（比如网络错误），保守起见允许继续
            print(f"  无法读取 robots.txt（{e}），默认允许")
            return True

    def fetch_page(self, url: str) -> Optional[BeautifulSoup]:
        """
        获取指定 URL 的页面内容，返回解析后的 BeautifulSoup 对象

        参数：
            url  要获取的页面地址
        返回：
            成功返回 BeautifulSoup 对象，失败返回 None
        """
        # 速率控制：先等待，再请求
        self._respect_delay()

        try:
            print(f"  正在获取：{url}")
            response = self.session.get(
                url,
                timeout=15,  # 超过 15 秒没有响应则放弃
            )

            # raise_for_status() 会在 HTTP 状态码为 4xx/5xx 时抛出异常
            # 例如 404（页面不存在）、403（拒绝访问）等
            response.raise_for_status()

            # 检查内容类型：只处理 HTML 页面，跳过 PDF、图片等
            content_type = response.headers.get("Content-Type", "")
            if "text/html" not in content_type:
                print(f"  非 HTML 内容（{content_type}），跳过")
                return None

            # 让 requests 根据响应头自动推断编码
            # 避免中文页面出现乱码
            response.encoding = response.apparent_encoding

            # 使用 html.parser 解析 HTML
            # lxml 更快，但需要额外安装；html.parser 是标准库自带的
            soup = BeautifulSoup(response.text, "html.parser")
            return soup

        except requests.exceptions.Timeout:
            print(f"  请求超时：{url}")
        except requests.exceptions.ConnectionError:
            print(f"  连接失败：{url}")
        except requests.exceptions.HTTPError as e:
            print(f"  HTTP 错误 {e.response.status_code}：{url}")
        except Exception as e:
            print(f"  未知错误：{e}")

        # 记录失败的 URL
        self.failed_urls.append(url)
        return None

    def _extract_title(self, soup: BeautifulSoup) -> str:
        """提取页面标题，按优先级尝试多个来源"""
        # 优先使用 <h1>，次选 <title> 标签
        h1 = soup.find("h1")
        if h1:
            return h1.get_text(strip=True)

        title_tag = soup.find("title")
        if title_tag:
            return title_tag.get_text(strip=True)

        return "（无标题）"

    def _extract_links(self, soup: BeautifulSoup, current_url: str) -> List[str]:
        """
        提取页面中所有链接，转换为绝对 URL
        并筛选出属于同一域名的链接
        """
        links = []

        for a_tag in soup.find_all("a", href=True):
            href = a_tag["href"].strip()

            # 跳过空链接、锚点链接、JavaScript 链接
            if not href or href.startswith("#") or href.startswith("javascript:"):
                continue

            # 将相对 URL 转换为绝对 URL
            # urljoin 能正确处理 ../、./、/path 等各种情况
            absolute_url = urljoin(current_url, href)

            # 只保留同一域名的链接（不爬到其他网站）
            if urlparse(absolute_url).netloc == self.domain:
                # 去掉 URL 中的锚点部分（#section1 这类）
                clean_url = absolute_url.split("#")[0]
                if clean_url and clean_url not in links:
                    links.append(clean_url)

        return links

    def _extract_content(self, soup: BeautifulSoup) -> str:
        """
        提取页面主要文字内容

        策略：
        1. 先尝试找 <article>、<main> 等语义化标签
        2. 找不到则使用整个 <body>
        3. 去掉导航、页脚等噪音内容
        """
        # 按优先级尝试找主要内容区域
        main_content = (
            soup.find("article") or
            soup.find("main") or
            soup.find(id="content") or
            soup.find(id="main") or
            soup.find(class_="content") or
            soup.find("body")
        )

        if not main_content:
            return ""

        # 删除不需要的标签（导航、脚本、样式表等）
        for unwanted in main_content.find_all(["nav", "script", "style", "footer", "header"]):
            unwanted.decompose()  # decompose() 从树中删除该元素

        # get_text() 提取纯文本，separator 指定段落之间的分隔符
        text = main_content.get_text(separator="\n", strip=True)

        # 简单清理：合并多个空行
        lines = [line for line in text.splitlines() if line.strip()]
        return "\n".join(lines)

    def _extract_meta_description(self, soup: BeautifulSoup) -> str:
        """提取 meta description（SEO 描述文字）"""
        # 网页头部通常有这样的标签：
        # <meta name="description" content="页面描述">
        meta = soup.find("meta", attrs={"name": "description"})
        if meta:
            return meta.get("content", "").strip()
        return ""

    def _extract_images(self, soup: BeautifulSoup, current_url: str) -> List[Dict[str, str]]:
        """提取页面中所有图片的 URL 和描述"""
        images = []

        for img in soup.find_all("img"):
            src = img.get("src", "").strip()
            if not src:
                continue

            # 将图片的相对地址转为绝对地址
            absolute_src = urljoin(current_url, src)
            alt_text = img.get("alt", "").strip()

            images.append({
                "src": absolute_src,
                "alt": alt_text,
            })

        return images

    def extract_info(self, soup: BeautifulSoup, url: str) -> Dict[str, Any]:
        """
        从 BeautifulSoup 对象中提取所有需要的信息

        参数：
            soup  解析好的 BeautifulSoup 对象
            url   当前页面的 URL（用于处理相对链接）
        返回：
            包含页面信息的字典
        """
        return {
            "url": url,
            "crawled_at": datetime.now().isoformat(),  # 记录爬取时间
            "title": self._extract_title(soup),
            "meta_description": self._extract_meta_description(soup),
            "content": self._extract_content(soup),
            "links": self._extract_links(soup, url),
            "images": self._extract_images(soup, url),
        }

    def save_to_markdown(self, info: Dict[str, Any]) -> Path:
        """
        将页面信息保存为 Markdown 格式

        Markdown 格式方便人类阅读，可以用任何文本编辑器打开
        """
        # 根据 URL 生成文件名，把特殊字符替换为下划线
        url_path = urlparse(info["url"]).path
        # 去掉开头的斜线，把路径分隔符换成下划线
        filename = url_path.strip("/").replace("/", "_") or "index"
        # 保证文件名合法（去掉不能在文件名中使用的字符）
        safe_filename = "".join(c if c.isalnum() or c in "-_." else "_" for c in filename)

        filepath = self.output_dir / f"{safe_filename}.md"

        # 构建 Markdown 内容
        lines = [
            f"# {info['title']}",
            "",
            f"> 来源：{info['url']}",
            f"> 爬取时间：{info['crawled_at']}",
            "",
        ]

        if info["meta_description"]:
            lines += [
                "## 页面描述",
                "",
                info["meta_description"],
                "",
            ]

        if info["content"]:
            lines += [
                "## 正文内容",
                "",
                info["content"][:3000],  # 限制内容长度，避免文件过大
                "",
            ]

        if info["images"]:
            lines += ["## 图片列表", ""]
            for img in info["images"][:10]:  # 最多显示 10 张图片
                alt = img["alt"] or "图片"
                lines.append(f"- ![{alt}]({img['src']})")
            lines.append("")

        if info["links"]:
            lines += ["## 发现的链接", ""]
            for link in info["links"][:20]:  # 最多显示 20 个链接
                lines.append(f"- {link}")
            lines.append("")

        filepath.write_text("\n".join(lines), encoding="utf-8")
        return filepath

    def save_to_json(self, info: Dict[str, Any]) -> Path:
        """
        将页面信息保存为 JSON 格式

        JSON 格式方便程序读取和后续处理，
        比如导入数据库、用 pandas 分析等
        """
        url_path = urlparse(info["url"]).path
        filename = url_path.strip("/").replace("/", "_") or "index"
        safe_filename = "".join(c if c.isalnum() or c in "-_." else "_" for c in filename)

        filepath = self.output_dir / f"{safe_filename}.json"

        # 只保存必要字段，正文内容截断，避免文件过大
        data = {
            "url": info["url"],
            "crawled_at": info["crawled_at"],
            "title": info["title"],
            "meta_description": info["meta_description"],
            "content_preview": info["content"][:500],  # 只保存前 500 字
            "links_count": len(info["links"]),
            "links": info["links"][:20],
            "images_count": len(info["images"]),
            "images": info["images"][:10],
        }

        # ensure_ascii=False：保留中文字符（不转义为 \uXXXX）
        # indent=2：格式化输出，方便阅读
        filepath.write_text(
            json.dumps(data, ensure_ascii=False, indent=2),
            encoding="utf-8",
        )
        return filepath

    def crawl(self, max_pages: int = 10) -> List[Dict[str, Any]]:
        """
        从 base_url 出发，爬取多个页面

        参数：
            max_pages  最多爬取多少个页面（防止爬虫失控）
        返回：
            所有成功爬取的页面信息列表
        """
        print(f"\n{'='*50}")
        print(f"开始爬取，目标：{self.base_url}")
        print(f"最多爬取 {max_pages} 个页面")
        print(f"{'='*50}\n")

        # 用列表模拟队列：待爬取的 URL 列表
        # 从起始 URL 开始
        url_queue: List[str] = [self.base_url]

        results = []

        while url_queue and len(results) < max_pages:
            # 从队列头部取出一个 URL
            current_url = url_queue.pop(0)

            # 跳过已经访问过的 URL（去重）
            if current_url in self.visited_urls:
                continue

            self.visited_urls.add(current_url)

            print(f"\n[{len(results)+1}/{max_pages}] 处理：{current_url}")

            # 检查 robots.txt
            if not self.check_robots_txt(current_url):
                print(f"  robots.txt 禁止，跳过")
                continue

            # 获取页面
            soup = self.fetch_page(current_url)
            if soup is None:
                continue

            # 提取信息
            info = self.extract_info(soup, current_url)

            # 保存结果
            md_path = self.save_to_markdown(info)
            json_path = self.save_to_json(info)

            print(f"  标题：{info['title']}")
            print(f"  发现 {len(info['links'])} 个链接，{len(info['images'])} 张图片")
            print(f"  已保存：{md_path.name}")

            results.append(info)

            # 把这个页面发现的新链接加入队列
            for link in info["links"]:
                if link not in self.visited_urls and link not in url_queue:
                    url_queue.append(link)

        # 打印爬取总结
        print(f"\n{'='*50}")
        print(f"爬取完成！")
        print(f"  成功：{len(results)} 个页面")
        print(f"  失败：{len(self.failed_urls)} 个页面")
        print(f"  结果保存在：{self.output_dir.resolve()}")
        print(f"{'='*50}")

        return results

    def __del__(self):
        """对象销毁时关闭 Session，释放网络连接"""
        if hasattr(self, "session"):
            self.session.close()


def main():
    """主函数：演示如何使用 WebScraper"""

    # 爬取 Python 官方文档（公开内容，适合练习）
    scraper = WebScraper(
        base_url="https://docs.python.org/zh-cn/3/tutorial/",
        output_dir="output/python_docs",
        min_delay=1.5,   # 最少等待 1.5 秒
        max_delay=3.0,   # 最多等待 3.0 秒
    )

    # 只爬取 5 个页面（学习练习，控制数量）
    results = scraper.crawl(max_pages=5)

    # 打印所有爬取到的标题
    print("\n爬取到的页面：")
    for i, info in enumerate(results, 1):
        print(f"  {i}. {info['title']}")
        print(f"     {info['url']}")


if __name__ == "__main__":
    main()

## 第六部分：进阶话题

### 6.1 为什么有些页面爬不到内容

你可能会遇到这种情况：用爬虫访问某个页面，拿到的 HTML 里几乎没有实际内容，只有一些 JavaScript 代码。

原因是**单页应用（SPA）**的普及。现代很多网站（如电商平台、社交媒体）会这样工作：

```
┌───────────────────────────────────────────────────────────┐
│              传统网页 vs 单页应用（SPA）                   │
├────────────────────────┬──────────────────────────────────┤
│  传统网页               │  单页应用（SPA）                │
├────────────────────────┼──────────────────────────────────┤
│  服务器直接返回         │  服务器返回空页面框架            │
│  完整的 HTML           │  浏览器执行 JavaScript           │
│                        │  JavaScript 再请求数据           │
│  requests 可以直接      │  requests 拿到的是空框架，       │
│  拿到全部内容          │  内容还没有被填充                 │
└────────────────────────┴──────────────────────────────────┘
```

解决这类问题，需要"真正运行浏览器"来执行 JavaScript。

### 6.2 Selenium 与 Playwright

这两个工具可以控制真实的浏览器（Chrome、Firefox 等），让 JavaScript 完整执行后再提取内容：

- **Selenium**：历史悠久，生态成熟，支持多种编程语言。安装配置稍显复杂，需要对应浏览器的驱动程序（ChromeDriver 等）
- **Playwright**：微软出品，较新，API 设计更现代，支持异步操作，配置更简单（`playwright install` 一键安装浏览器）

它们都可以做到：
- 等待页面 JavaScript 执行完毕
- 模拟用户点击、滚动、填写表单
- 截图、生成 PDF

如果你遇到内容爬不到的情况，可以考虑学习这两个工具。

### 6.3 异步爬虫的优势

本项目的爬虫是**顺序执行**的：请求页面 → 等待响应 → 处理 → 再请求下一页。爬虫大部分时间都在"等"——等网络响应。

**异步爬虫**可以同时发出多个请求，在等待一个响应的同时处理其他页面，大幅提升效率：

```
顺序爬虫：
  请求A ──等待── 处理A ──请求B ──等待── 处理B ──...
  总时间 = 每页时间之和

异步爬虫：
  请求A ─┐
  请求B ─┤  并发等待  ├── 处理A、B、C...
  请求C ─┘
  总时间 ≈ 单页最长时间
```

Python 的 `asyncio` + `aiohttp` 是实现异步爬虫的主流方案。如果你想了解异步编程的原理和实践，可以阅读本教程的：

- **第 18 章**：并发编程（asyncio 基础）
- **第 23 章**：asyncio 高级编程（事件循环、Task、队列）

### 练习 1：图片批量下载器

```
