## 第一部分：项目概述

### 1.1 为什么需要批量重命名？

你是否遇到过这样的情况：

- 手机导出了 500 张照片，文件名是 `IMG_0001.jpg`、`IMG_0002.jpg`……毫无意义
- 团队文档命名五花八门：`报告.docx`、`最终版报告.docx`、`最终版报告v2.docx`……
- 公司要求所有文件按日期命名：`2024-01-15_会议记录.docx`

手动一个一个改？500 张照片大概需要 1 个小时。写一个 Python 脚本？10 分钟搞定，以后还能复用。

**这就是编程的核心价值：让计算机替你做重复的事情。**

## 第二部分：核心概念讲解

### 2.1 pathlib.Path 基础

#### 概念说明

Python 处理文件路径有两种方式：

**旧方式**：`os.path` 模块
```python
import os
path = "/home/user/photos/DSC001.jpg"
filename = os.path.basename(path)      # DSC001.jpg
stem = os.path.splitext(filename)[0]   # DSC001
ext = os.path.splitext(filename)[1]    # .jpg
parent = os.path.dirname(path)         # /home/user/photos
```

**新方式**：`pathlib.Path` 对象（Python 3.4+ 推荐）

`pathlib` 把路径当作一个"对象"来处理，而不是普通字符串。类比：
- 旧方式就像用剪刀和胶水拼接字符串
- 新方式就像用积木，每块积木都有自己的功能

```
路径解析示意图：

  /home/user/photos/DSC001.jpg
  └────┬────┘└──┬──┘└───┬───┘
   parent   parent   name（文件名）
                     └──┬──┘└┬┘
                       stem suffix
```

常用属性和方法一览：

```
┌───────────────────┬──────────────────────────────────────────┐
│  属性 / 方法       │  说明                                     │
├───────────────────┼──────────────────────────────────────────┤
│  .name            │  文件名含扩展名：DSC001.jpg               │
│  .stem            │  文件名不含扩展名：DSC001                 │
│  .suffix          │  扩展名（含点）：.jpg                     │
│  .parent          │  父目录：/home/user/photos                │
│  .exists()        │  路径是否存在 → True / False             │
│  .is_dir()        │  是否是目录 → True / False               │
│  .is_file()       │  是否是文件 → True / False               │
│  .iterdir()       │  遍历目录下所有文件和子目录（不递归）      │
│  .glob("*.jpg")   │  按模式匹配文件，支持通配符               │
│  .rename(新路径)  │  重命名 / 移动文件                        │
│  .stat().st_mtime │  文件最后修改时间（时间戳）               │
└───────────────────┴──────────────────────────────────────────┘
```

#### 示例代码

```python
from pathlib import Path
from datetime import datetime

# 创建 Path 对象——只是在内存中描述一个路径，不会真正访问磁盘
photo = Path("/home/user/photos/DSC001.jpg")

# 拆解路径各部分
print(photo.name)    # 输出: DSC001.jpg
print(photo.stem)    # 输出: DSC001
print(photo.suffix)  # 输出: .jpg
print(photo.parent)  # 输出: /home/user/photos

# 拼接新路径：parent / "新文件名"
# 注意：/ 运算符在 Path 里是路径拼接，不是除法！
new_name = photo.parent / "vacation_DSC001.jpg"
print(new_name)  # 输出: /home/user/photos/vacation_DSC001.jpg

# 检查文件是否存在
if photo.exists():
    print("文件存在")
else:
    print("文件不存在")

# 遍历目录下所有 .jpg 文件
folder = Path("/home/user/photos")
for jpg_file in folder.glob("*.jpg"):
    print(jpg_file.name)

# 获取文件修改时间
mtime = photo.stat().st_mtime             # 时间戳（浮点数）
dt = datetime.fromtimestamp(mtime)        # 转成 datetime 对象
print(dt.strftime("%Y-%m-%d"))            # 格式化成 2024-01-15
```

### 2.2 dry_run 预览模式设计思想

#### 概念说明

`dry_run`（直译"空转运行"）是一种常见的安全设计模式：

**先让用户"看"到将要发生的事，再决定是否真的执行。**

类比：你要大量打印文件，好的打印机软件会先让你预览每一页效果，确认没问题才开始打印。如果直接按"打印"就开始，纸张浪费了也来不及后悔。

批量重命名同理：如果直接改，操作失误了文件就乱掉了。有了 `dry_run`：
1. 先以 `dry_run=True` 运行，打印出"计划改成什么样"
2. 用户确认没问题，再以 `dry_run=False` 真正执行

```
dry_run 工作流：

  调用函数(dry_run=True)
         │
         ▼
  计算新旧文件名映射
         │
         ▼
  ┌──────────────┐
  │  仅打印预览   │  ← 不碰磁盘，零风险
  └──────────────┘
         │
  用户确认 (y/n)？
         │
    y ───┘
         ▼
  调用函数(dry_run=False)
         │
         ▼
  ┌──────────────┐
  │  真正重命名   │  ← 写入磁盘
  └──────────────┘
```

这个模式在很多专业工具中都有：
- `rsync --dry-run`：预览文件同步结果
- `terraform plan`：预览云资源变更
- `git stash --dry-run`：预览会暂存哪些文件

#### 示例代码

```python
def add_prefix(self, prefix: str, dry_run: bool = True) -> list[tuple]:
    """
    为所有文件添加前缀。

    参数：
        prefix   - 要添加的前缀字符串，例如 "vacation_"
        dry_run  - True 表示只预览不执行，False 表示真正重命名
    返回：
        [(旧路径, 新路径), ...] 操作列表
    """
    changes = []

    for file_path in self.files:
        # 构造新文件名：前缀 + 原文件名
        new_name = prefix + file_path.name
        new_path = file_path.parent / new_name
        changes.append((file_path, new_path))

        if dry_run:
            # 预览模式：只打印，不执行
            print(f"  预览: {file_path.name}  →  {new_name}")
        else:
            # 执行模式：真正重命名
            file_path.rename(new_path)
            print(f"  完成: {file_path.name}  →  {new_name}")

    return changes


# 调用示例：先预览，再确认执行
renamer = FileRenamer("/home/user/photos")

print("=== 预览模式 ===")
renamer.add_prefix("vacation_", dry_run=True)   # 只打印，不改文件

answer = input("确认执行以上操作？(y/n): ")
if answer.lower() == "y":
    print("=== 执行重命名 ===")
    renamer.add_prefix("vacation_", dry_run=False)  # 真正执行
```

### 2.3 Path.rename() 的注意事项

#### 概念说明

`Path.rename()` 是重命名文件的核心方法，但有几个重要细节需要了解：

**原子性**：在同一个文件系统内重命名是"原子操作"——要么完全成功，要么完全失败，不会出现"改了一半"的中间状态。这是操作系统层面的保障。

**跨磁盘移动**：如果新路径和旧路径在不同的磁盘（例如从 C 盘移到 D 盘），`Path.rename()` 会失败。这种情况需要用 `shutil.move()`。

```
同一磁盘（rename 可用）：
  /home/user/photos/old.jpg  →  /home/user/photos/new.jpg
  └────────────────┘             └────────────────┘
       同一目录                       同一目录

跨磁盘（需要 shutil.move）：
  /mnt/disk1/old.jpg  →  /mnt/disk2/new.jpg
  └──────────┘            └──────────┘
    磁盘 1                   磁盘 2
```

**文件名冲突**：如果目标文件名已经存在，`Path.rename()` 在 Windows 上会报错，在 Linux/macOS 上会直接覆盖。跨平台开发时要提前检查。

#### 示例代码

```python
from pathlib import Path
import shutil

old_file = Path("/home/user/photos/DSC001.jpg")
new_file = Path("/home/user/photos/vacation_DSC001.jpg")

# 方式一：同一目录内重命名（推荐，速度快）
old_file.rename(new_file)

# 方式二：先检查目标是否已存在，避免覆盖
if new_file.exists():
    print(f"错误：目标文件 {new_file.name} 已存在，跳过")
else:
    old_file.rename(new_file)

# 方式三：跨磁盘移动（或者不确定是否跨盘时的保险写法）
shutil.move(str(old_file), str(new_file))

# 注意：rename() 返回新路径的 Path 对象
new_path = old_file.rename(new_file)
print(type(new_path))   # <class 'pathlib.PosixPath'>
print(new_path.name)    # vacation_DSC001.jpg
```

## 第三部分：完整代码实现

### 3.1 完整源代码

将以下代码保存为 `file_renamer.py`：

```python
"""
文件批量重命名工具
==================
支持：添加前缀/后缀、序号重命名、文本替换、按日期重命名、备份与撤销

用法：
    python file_renamer.py
"""

import json
import shutil
from datetime import datetime
from pathlib import Path


# ─────────────────────────────────────────────
#  FileRenamer 类：所有文件操作的核心
# ─────────────────────────────────────────────

class FileRenamer:
    """
    文件批量重命名工具类。

    使用方式：
        renamer = FileRenamer("/path/to/folder")
        renamer.list_files()
        renamer.add_prefix("vacation_", dry_run=False)
    """

    def __init__(self, directory: str):
        """
        初始化：指定要操作的目录。

        参数：
            directory - 目标目录的路径字符串，例如 "/home/user/photos"
        """
        self.directory = Path(directory)

        # 检查目录是否存在
        if not self.directory.exists():
            raise FileNotFoundError(f"目录不存在：{directory}")
        if not self.directory.is_dir():
            raise NotADirectoryError(f"路径不是目录：{directory}")

        # 加载目录下所有文件（不含子目录，不含隐藏文件）
        self.files: list[Path] = self._load_files()

    def _load_files(self) -> list[Path]:
        """
        内部方法：扫描目录，返回所有普通文件的路径列表。
        跳过：子目录、以 . 开头的隐藏文件、备份 JSON 文件本身。
        """
        result = []
        for item in sorted(self.directory.iterdir()):
            # 只收集普通文件
            if not item.is_file():
                continue
            # 跳过隐藏文件（如 .DS_Store）
            if item.name.startswith("."):
                continue
            # 跳过本工具自己生成的备份文件
            if item.name == "rename_backup.json":
                continue
            result.append(item)
        return result

    def list_files(self) -> None:
        """打印当前目录下所有文件，编号显示。"""
        if not self.files:
            print("  (目录为空，没有找到任何文件)")
            return

        print(f"\n目录：{self.directory}")
        print(f"共 {len(self.files)} 个文件：")
        print("─" * 50)
        for idx, f in enumerate(self.files, start=1):
            # 获取文件大小，转成人类可读格式
            size = f.stat().st_size
            size_str = _human_readable_size(size)
            print(f"  {idx:>3}. {f.name:<40} {size_str:>8}")
        print("─" * 50)

    def preview_changes(self, changes: list[tuple]) -> None:
        """
        打印重命名预览表格。

        参数：
            changes - [(旧路径, 新路径), ...] 列表
        """
        if not changes:
            print("  (没有需要变更的文件)")
            return

        print("\n重命名预览：")
        print(f"  {'旧文件名':<35} →  {'新文件名'}")
        print("  " + "─" * 70)
        for old_path, new_path in changes:
            old_name = old_path.name
            new_name = new_path.name
            # 如果名字没有变化，用灰色提示跳过
            if old_name == new_name:
                print(f"  {old_name:<35}    (无变化，跳过)")
            else:
                print(f"  {old_name:<35} →  {new_name}")
        print("  " + "─" * 70)

    # ─────────────────────────────────────────
    #  功能一：添加前缀
    # ─────────────────────────────────────────

    def add_prefix(self, prefix: str, dry_run: bool = True) -> list[tuple]:
        """
        为目录中所有文件的文件名添加前缀。

        例如：prefix="vacation_"
              DSC001.jpg  →  vacation_DSC001.jpg

        参数：
            prefix  - 前缀字符串
            dry_run - True=只预览，False=真正执行
        返回：
            [(旧路径, 新路径), ...] 操作列表
        """
        changes = []

        for file_path in self.files:
            new_name = prefix + file_path.name
            new_path = file_path.parent / new_name
            changes.append((file_path, new_path))

        if dry_run:
            self.preview_changes(changes)
        else:
            # 真正执行重命名
            renamed_count = 0
            for old_path, new_path in changes:
                if old_path.name == new_path.name:
                    continue  # 名字没变，跳过
                if new_path.exists():
                    print(f"  警告：{new_path.name} 已存在，跳过 {old_path.name}")
                    continue
                old_path.rename(new_path)
                renamed_count += 1
            # 重命名完成后刷新文件列表
            self.files = self._load_files()
            print(f"  完成！共重命名 {renamed_count} 个文件。")

        return changes

    # ─────────────────────────────────────────
    #  功能二：添加后缀
    # ─────────────────────────────────────────

    def add_suffix(self, suffix: str, dry_run: bool = True) -> list[tuple]:
        """
        为目录中所有文件的主文件名（stem）添加后缀，扩展名不变。

        例如：suffix="_v2"
              report.docx  →  report_v2.docx

        参数：
            suffix  - 后缀字符串（加在扩展名之前）
            dry_run - True=只预览，False=真正执行
        返回：
            [(旧路径, 新路径), ...] 操作列表
        """
        changes = []

        for file_path in self.files:
            # 在主文件名后、扩展名前插入后缀
            # 例如：stem="report", suffix=".docx" → "report_v2.docx"
            new_name = file_path.stem + suffix + file_path.suffix
            new_path = file_path.parent / new_name
            changes.append((file_path, new_path))

        if dry_run:
            self.preview_changes(changes)
        else:
            renamed_count = 0
            for old_path, new_path in changes:
                if old_path.name == new_path.name:
                    continue
                if new_path.exists():
                    print(f"  警告：{new_path.name} 已存在，跳过 {old_path.name}")
                    continue
                old_path.rename(new_path)
                renamed_count += 1
            self.files = self._load_files()
            print(f"  完成！共重命名 {renamed_count} 个文件。")

        return changes

    # ─────────────────────────────────────────
    #  功能三：序号重命名
    # ─────────────────────────────────────────

    def rename_with_numbering(
        self,
        base_name: str,
        start: int = 1,
        digits: int = 3,
        dry_run: bool = True
    ) -> list[tuple]:
        """
        用"基础名 + 序号"重命名所有文件，保留原扩展名。

        例如：base_name="photo", start=1, digits=3
              IMG_001.jpg  →  photo_001.jpg
              IMG_002.jpg  →  photo_002.jpg

        参数：
            base_name - 基础名字，例如 "photo"
            start     - 起始序号，默认 1
            digits    - 序号位数，默认 3（不足补零，如 001）
            dry_run   - True=只预览，False=真正执行
        返回：
            [(旧路径, 新路径), ...] 操作列表
        """
        changes = []

        for idx, file_path in enumerate(self.files, start=start):
            # zfill 补零：str(1).zfill(3) → "001"
            num_str = str(idx).zfill(digits)
            new_name = f"{base_name}_{num_str}{file_path.suffix}"
            new_path = file_path.parent / new_name
            changes.append((file_path, new_path))

        if dry_run:
            self.preview_changes(changes)
        else:
            # 序号重命名有个陷阱：如果新旧文件名有重叠（例如把 001 改成 002），
            # 直接按顺序重命名可能覆盖还没改的文件。
            # 解决方案：先全部改成临时名，再改成目标名。
            temp_paths = []

            # 第一步：改成临时名（加 __tmp__ 前缀）
            for old_path, new_path in changes:
                tmp_path = old_path.parent / ("__tmp__" + old_path.name)
                old_path.rename(tmp_path)
                temp_paths.append((tmp_path, new_path))

            # 第二步：从临时名改成目标名
            renamed_count = 0
            for tmp_path, new_path in temp_paths:
                if new_path.exists():
                    print(f"  警告：{new_path.name} 已存在，跳过")
                    # 还原临时文件名
                    original_name = tmp_path.name.removeprefix("__tmp__")
                    tmp_path.rename(tmp_path.parent / original_name)
                    continue
                tmp_path.rename(new_path)
                renamed_count += 1

            self.files = self._load_files()
            print(f"  完成！共重命名 {renamed_count} 个文件。")

        return changes

    # ─────────────────────────────────────────
    #  功能四：文本替换
    # ─────────────────────────────────────────

    def replace_text(
        self,
        old_text: str,
        new_text: str,
        dry_run: bool = True
    ) -> list[tuple]:
        """
        在所有文件名中将 old_text 替换为 new_text。

        例如：old_text="final", new_text="v2"
              report_final.docx  →  report_v2.docx

        参数：
            old_text - 要被替换的文本
            new_text - 替换后的文本
            dry_run  - True=只预览，False=真正执行
        返回：
            [(旧路径, 新路径), ...] 操作列表（仅含有变化的文件）
        """
        changes = []

        for file_path in self.files:
            # 只有文件名包含 old_text 时才处理
            if old_text not in file_path.name:
                continue
            new_name = file_path.name.replace(old_text, new_text)
            new_path = file_path.parent / new_name
            changes.append((file_path, new_path))

        if not changes:
            print(f"  没有找到包含 '{old_text}' 的文件名，无需修改。")
            return changes

        if dry_run:
            self.preview_changes(changes)
        else:
            renamed_count = 0
            for old_path, new_path in changes:
                if new_path.exists():
                    print(f"  警告：{new_path.name} 已存在，跳过 {old_path.name}")
                    continue
                old_path.rename(new_path)
                renamed_count += 1
            self.files = self._load_files()
            print(f"  完成！共重命名 {renamed_count} 个文件。")

        return changes

    # ─────────────────────────────────────────
    #  功能五：按日期重命名
    # ─────────────────────────────────────────

    def rename_by_date(
        self,
        date_format: str = "%Y-%m-%d",
        dry_run: bool = True
    ) -> list[tuple]:
        """
        根据文件最后修改日期重命名：日期_原文件名。

        例如：date_format="%Y-%m-%d"
              report.docx  →  2024-01-15_report.docx

        参数：
            date_format - 日期格式字符串，默认 "%Y-%m-%d"（如 2024-01-15）
            dry_run     - True=只预览，False=真正执行
        返回：
            [(旧路径, 新路径), ...] 操作列表
        """
        changes = []

        for file_path in self.files:
            # 获取文件修改时间戳，转成格式化日期字符串
            mtime = file_path.stat().st_mtime
            date_str = datetime.fromtimestamp(mtime).strftime(date_format)

            new_name = f"{date_str}_{file_path.name}"
            new_path = file_path.parent / new_name
            changes.append((file_path, new_path))

        if dry_run:
            self.preview_changes(changes)
        else:
            renamed_count = 0
            for old_path, new_path in changes:
                if old_path.name == new_path.name:
                    continue
                if new_path.exists():
                    print(f"  警告：{new_path.name} 已存在，跳过 {old_path.name}")
                    continue
                old_path.rename(new_path)
                renamed_count += 1
            self.files = self._load_files()
            print(f"  完成！共重命名 {renamed_count} 个文件。")

        return changes

    # ─────────────────────────────────────────
    #  功能六：备份当前文件名
    # ─────────────────────────────────────────

    def backup_names(self) -> Path:
        """
        将当前目录下所有文件名保存到 rename_backup.json。

        备份文件存放在被操作的目录中，格式如下：
        {
            "backup_time": "2024-01-15 10:00:00",
            "directory": "/path/to/dir",
            "files": [
                {"original": "DSC001.jpg", "current": "vacation_DSC001.jpg"}
            ]
        }

        注意：首次备份时 original 和 current 相同。
              撤销后再次备份会更新 current，但保留 original。

        返回：
            备份文件的 Path 对象
        """
        backup_path = self.directory / "rename_backup.json"

        # 如果备份文件已存在，读取已有的 original 信息
        # （这样多次操作后，original 仍然是最初的文件名）
        existing_originals: dict[str, str] = {}
        if backup_path.exists():
            try:
                with open(backup_path, "r", encoding="utf-8") as f:
                    existing_data = json.load(f)
                # 建立 current → original 的映射
                for entry in existing_data.get("files", []):
                    existing_originals[entry["current"]] = entry["original"]
            except (json.JSONDecodeError, KeyError):
                # 备份文件损坏，忽略，重新建立
                pass

        # 构建备份数据
        files_data = []
        for file_path in self.files:
            current_name = file_path.name
            # 如果这个文件在之前的备份里有记录，保留最初的 original
            original_name = existing_originals.get(current_name, current_name)
            files_data.append({
                "original": original_name,
                "current": current_name
            })

        backup_data = {
            "backup_time": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
            "directory": str(self.directory),
            "files": files_data
        }

        # 写入 JSON 文件（ensure_ascii=False 保证中文不被转义）
        with open(backup_path, "w", encoding="utf-8") as f:
            json.dump(backup_data, f, ensure_ascii=False, indent=2)

        print(f"  备份成功！已保存 {len(files_data)} 条记录到：")
        print(f"  {backup_path}")
        return backup_path

    # ─────────────────────────────────────────
    #  功能七：撤销重命名
    # ─────────────────────────────────────────

    def undo_rename(self) -> None:
        """
        从 rename_backup.json 读取备份，将文件恢复为 original 的名字。

        工作流程：
        1. 读取备份 JSON
        2. 对每条记录，把 current 文件改回 original 名字
        3. 重新扫描目录，更新 self.files
        """
        backup_path = self.directory / "rename_backup.json"

        # 检查备份文件是否存在
        if not backup_path.exists():
            print("  错误：找不到备份文件 rename_backup.json")
            print("  请先执行备份操作，再尝试撤销。")
            return

        # 读取备份数据
        try:
            with open(backup_path, "r", encoding="utf-8") as f:
                backup_data = json.load(f)
        except json.JSONDecodeError:
            print("  错误：备份文件格式损坏，无法读取。")
            return

        backup_time = backup_data.get("backup_time", "未知")
        files_info = backup_data.get("files", [])

        print(f"\n找到备份（时间：{backup_time}），共 {len(files_info)} 条记录。")
        print("将执行以下还原操作：")
        print(f"  {'当前文件名':<35} →  {'还原文件名'}")
        print("  " + "─" * 70)

        # 收集需要还原的文件
        restore_list = []
        for entry in files_info:
            original = entry.get("original", "")
            current = entry.get("current", "")

            if not original or not current:
                continue

            current_path = self.directory / current
            original_path = self.directory / original

            if current == original:
                # 文件名未变，无需还原
                continue

            if not current_path.exists():
                print(f"  警告：当前文件 {current} 不存在，跳过")
                continue

            print(f"  {current:<35} →  {original}")
            restore_list.append((current_path, original_path))

        print("  " + "─" * 70)

        if not restore_list:
            print("  所有文件名已是备份状态，无需还原。")
            return

        # 用户确认
        answer = input(f"\n确认还原以上 {len(restore_list)} 个文件？(y/n): ")
        if answer.lower() != "y":
            print("  已取消。")
            return

        # 执行还原
        restored_count = 0
        for current_path, original_path in restore_list:
            if original_path.exists():
                print(f"  警告：{original_path.name} 已存在，跳过 {current_path.name}")
                continue
            current_path.rename(original_path)
            restored_count += 1

        # 刷新文件列表
        self.files = self._load_files()
        print(f"\n  还原完成！共恢复 {restored_count} 个文件。")

def _human_readable_size(size_bytes: int) -> str:
    """将字节数转换成人类可读的文件大小字符串。"""
    for unit in ["B", "KB", "MB", "GB"]:
```

## 第五部分：安全机制说明

批量重命名是**不可逆操作**（文件系统没有回收站），因此本工具设计了三层安全保障：

```
┌─────────────────────────────────────────────────────────────┐
│                      三层安全保障                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第一层：dry_run 预览模式                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  每个操作函数默认 dry_run=True                        │   │
│  │  只计算和展示"将要做什么"，完全不碰磁盘               │   │
│  │  类比：打印预览，按"打印"前看看效果                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                          ▼                                  │
│  第二层：用户确认                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  预览后必须输入 "y" 才会真正执行                      │   │
│  │  其他任何输入（回车、n、误触）都会取消操作             │   │
│  │  类比：删除重要文件时系统弹出"确定要删除吗？"         │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                          ▼                                  │
│  第三层：备份与撤销                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  执行前调用 backup_names() 保存文件名快照             │   │
│  │  操作失误后调用 undo_rename() 一键还原                │   │
│  │  备份数据存为 JSON 文件，人类可读，便于检查           │   │
│  │  类比：Git 版本控制，随时可以 git revert              │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**推荐的操作流程**：

```
① 创建 FileRenamer 对象
       │
       ▼
② 调用 backup_names()  ← 养成操作前先备份的习惯
       │
       ▼
③ 调用操作函数（dry_run=True）预览效果
       │
       ├── 效果不对 → 重新设置参数，回到 ③
       │
       └── 效果满意 → 调用操作函数（dry_run=False）执行
                             │
                             ├── 结果有问题 → undo_rename() 还原
                             │
                             └── 结果满意 → 完成！
```

### 6.2 读取照片 EXIF 信息按拍摄日期重命名

**目标**：不用文件修改时间，而是读取照片的真实拍摄时间。

**提示**：使用 `Pillow` 库读取 EXIF 数据


```python
from PIL import Image
from PIL.ExifTags import TAGS

def get_photo_taken_date(image_path: Path) -> str | None:
    """读取 JPEG 照片的 EXIF 拍摄时间，返回格式如 '2024-01-15'。"""
    try:
        img = Image.open(image_path)
        exif_data = img._getexif()
        if exif_data:
            for tag_id, value in exif_data.items():
                tag = TAGS.get(tag_id, tag_id)
                if tag == "DateTimeOriginal":
                    # 原始格式：'2024:01:15 10:30:00'
                    dt = datetime.strptime(value, "%Y:%m:%d %H:%M:%S")
                    return dt.strftime("%Y-%m-%d")
    except Exception:
        pass
    return None
```

### 6.5 重命名冲突处理

**目标**：当目标文件名已存在时，自动处理冲突，而不是简单跳过。

常见策略：

```
┌───────────────────┬──────────────────────────────────────────┐
│  策略              │  示例                                     │
├───────────────────┼──────────────────────────────────────────┤
│  自动编号          │  photo.jpg → photo(1).jpg → photo(2).jpg  │
│  时间戳后缀        │  photo.jpg → photo_20240115103000.jpg     │
│  覆盖（危险！）   │  直接替换目标文件                          │
│  跳过             │  保留原文件名，打印警告（当前策略）         │
│  抛出异常         │  让调用者决定怎么处理                      │
└───────────────────┴──────────────────────────────────────────┘
```

```python
def _resolve_conflict(self, target_path: Path) -> Path:
    """如果目标路径已存在，自动在文件名后加 (1), (2), ... 直到不冲突。"""
    if not target_path.exists():
        return target_path

    stem = target_path.stem
    suffix = target_path.suffix
    parent = target_path.parent
    counter = 1

    while True:
        new_path = parent / f"{stem}({counter}){suffix}"
        if not new_path.exists():
            return new_path
        counter += 1
```
