# 09.02.01 - 图基础概念

**LangGraph：** 基于 Pregel 模型的状态图框架，用于构建复杂 Agent 工作流。

```
图架构：
┌──────────────────────────────────────┐
│  StateGraph                           │
│                                       │
│  State → 共享数据结构                 │
│  Nodes → 执行逻辑函数                 │
│  Edges → 控制流向路由                 │
│                                       │
│  Nodes 做工作，Edges 告下一步         │
└──────────────────────────────────────┘
```

### 为什么用它？

LangChain 的 Agent 底层是 LangGraph。理解 LangGraph 有助于：
- 自定义 Agent 行为
- 构建复杂工作流
- 实现多 Agent 协作

---

### 三要素详解

```
三要素：
┌──────────────────────────────────────┐
│  State   → 共享数据结构               │
│           TypedDict 或 Pydantic      │
│                                       │
│  Nodes   → 执行逻辑的函数             │
│           接收 State，返回更新        │
│                                       │
│  Edges   → 控制流向的路由             │
│           Normal 或 Conditional      │
└──────────────────────────────────────┘
```

**执行原理：**

LangGraph 使用消息传递机制，在离散的"super-step"中执行：
- 同一 super-step 的节点并行执行
- 不同 super-step 的节点顺序执行
- 节点收到消息变为 active，完成后投票 halt
- 所有节点 inactive 时图结束

---

### StateGraph：图类

`StateGraph` 是主要的图类，需要定义 State 后构建：

```python
from langgraph.graph import StateGraph
from typing_extensions import TypedDict

class State(TypedDict):
    messages: list[str]
    count: int

builder = StateGraph(State)
```

### 编译图

**必须编译图才能使用**：

```python
graph = builder.compile()
```

编译会：
- 检查结构（无孤立节点等）
- 配置 runtime 参数（checkpointer、breakpoints）

---

### 最简示例：Hello World

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# 第一步：定义 State
class State(TypedDict):
    message: str

# 第二步：定义节点函数
def hello_node(state: State) -> dict:
    return {"message": "Hello, LangGraph!"}

# 第三步：构建图
builder = StateGraph(State)
builder.add_node("hello", hello_node)
builder.add_edge(START, "hello")
builder.add_edge("hello", END)

# 第四步：编译和执行
graph = builder.compile()
result = graph.invoke({"message": ""})

print(result["message"])
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `TypedDict` | 定义 State 类型 | 类型提示，IDE 支持 |
| `add_node()` | 添加节点 | 定义执行单元 |
| `add_edge(START, "hello")` | 入口边 | 指定第一个执行的节点 |
| `add_edge("hello", END)` | 结束边 | 指定执行结束 |
| `builder.compile()` | 编译图 | 检查结构完整性 |
| `graph.invoke()` | 执行图 | 运行整个流程 |

---

### 详细示例：多节点流程

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    input: str
    processed: str
    output: str

def process_node(state: State) -> dict:
    return {"processed": state["input"].upper()}

def format_node(state: State) -> dict:
    return {"output": f"结果: {state['processed']}"}

builder = StateGraph(State)
builder.add_node("process", process_node)
builder.add_node("format", format_node)

builder.add_edge(START, "process")
builder.add_edge("process", "format")
builder.add_edge("format", END)

graph = builder.compile()
result = graph.invoke({"input": "hello", "processed": "", "output": ""})

print(result["output"])
```

## State：状态详解

State 是图的共享数据结构，定义所有节点可读写的数据。

### Schema：状态模式

使用 `TypedDict` 定义状态模式：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    messages: list[str]      # 消息历史
    user_id: str             # 用户标识
    documents: list[str]     # 文档列表
```

### Reducer：更新策略

Reducer 定义节点返回值如何更新 State：

```
Reducer 类型：
┌──────────────────────────────────────┐
│  默认 Reducer → 覆盖更新              │
│  operator.add → 列表追加              │
│  add_messages → 消息ID追踪追加        │
└──────────────────────────────────────┘
```

### 默认 Reducer

无 Reducer 时，返回值直接覆盖：

```python
class State(TypedDict):
    count: int        # 默认：覆盖
    name: str         # 默认：覆盖

# 节点返回 {"count": 5}
# State.count = 5（覆盖原值）
```

### 使用 operator.add

列表追加而非覆盖：

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    items: Annotated[list[str], add]  # 追加

# 节点返回 {"items": ["a"]}
# State.items = State.items + ["a"]
```

### add_messages

消息专用 Reducer，追踪 ID：

```python
from langchain.messages import AnyMessage
from langgraph.graph.message import add_messages
from typing import Annotated

class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
```

**add_messages 特性：**
- 新消息追加到列表
- 更新已有消息（相同 ID）
- 自动反序列化为 Message 对象

---

### MessagesState

预置的消息状态类型：

```python
from langgraph.graph import MessagesState

class State(MessagesState):
    documents: list[str]  # 扩展额外字段
```

## Nodes：节点详解

节点是 Python 函数，接收 State 并返回更新。

### 节点函数签名

```python
def node(state: State) -> dict:
    # 读取 state
    # 执行逻辑
    # 返回更新
    return {"key": new_value}
```

**参数：**
- `state` - 当前状态
- `config` - RunnableConfig（可选）
- `runtime` - Runtime 对象（可选）

### 最简节点

```python
def increment_node(state: State) -> dict:
    return {"count": state["count"] + 1}
```

### 访问 Runtime

```python
from langgraph.runtime import Runtime

def node_with_runtime(state: State, runtime: Runtime):
    user_id = runtime.context.user_id
    thread_id = runtime.execution_info.thread_id
    return {"processed_by": user_id}
```

## START 和 END

```
特殊节点：
┌──────────────────────────────────────┐
│  START → 图入口，用户输入起点         │
│  END   → 图终点，执行结束             │
└──────────────────────────────────────┘
```

```python
from langgraph.graph import START, END

builder.add_edge(START, "first_node")
builder.add_edge("last_node", END)
```

### 为什么用它？

```python
# 没有 LangGraph：手动管理流程
def run_workflow():
    result1 = step1(input)
    result2 = step2(result1)
    if condition:
        result3 = step3(result2)
    else:
        result3 = step4(result2)
    return result3

# 有 LangGraph：声明式定义
builder.add_node("step1", step1)
builder.add_node("step2", step2)
builder.add_conditional_edges("step2", route)
graph = builder.compile()
graph.invoke(input)
```
