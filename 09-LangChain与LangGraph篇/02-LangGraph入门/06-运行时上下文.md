# 09.02.06 - 运行时上下文

> 运行时上下文提供配置传递、递归限制和元数据管理能力

## 运行时上下文的三个维度

```
┌─────────────────────────────────────────────────────────┐
│                  运行时上下文                             │
│                                                         │
│  1. context_schema   —  跨图共享的只读配置               │
│  2. recursion_limit  —  防止无限循环的安全阀             │
│  3. config metadata  —  追踪执行信息的载体               │
└─────────────────────────────────────────────────────────┘
```

## context_schema — 共享配置

`context_schema` 允许你在编译图时指定一个**只读上下文类型**，所有节点都可以访问它但不需要它出现在 State 中。

### 为什么需要 context_schema

```python
# ❌ 不好的做法 — 把配置混入 State
class State(TypedDict):
    messages: list
    user_id: str          # 这不是对话的一部分
    api_key: str          # 这不应该出现在 State 中
    model_name: str       # 这是配置，不是数据

# ✅ 好的做法 — 配置与数据分离
class State(TypedDict):
    messages: list

class Context(TypedDict):
    user_id: str
    api_key: str
    model_name: str
```

### 定义和使用 context_schema

```python
from typing import TypedDict
from langgraph.graph import StateGraph

class State(TypedDict):
    messages: list

class Config(TypedDict):
    user_id: str
    model_name: str
    temperature: float

# 在构建时指定 context_schema
builder = StateGraph(State, context_schema=Config)
```

### 在节点中访问上下文

```python
from langgraph.config import get_config

def chat_node(state: State) -> dict:
    config = get_config()
    
    # 访问配置的字段
    user_id = config.get("user_id", "anonymous")
    model_name = config.get("model_name", "moonshot-v1-8k")
    temperature = config.get("temperature", 0.7)
    
    return {"messages": [...]}
```

### 完整示例

```python
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.config import get_config
from langchain.chat_models import init_chat_model
from langchain.messages import HumanMessage, AIMessage

class State(TypedDict):
    messages: Annotated[list, add_messages]

class Config(TypedDict):
    user_id: str
    model: str
    temperature: float

def personalized_chat(state: State) -> dict:
    config = get_config()
    user_id = config.get("user_id", "unknown")
    model_name = config.get("model", "moonshot:moonshot-v1-8k")
    temp = config.get("temperature", 0.7)
    
    model = init_chat_model(model_name, temperature=temp)
    
    messages = [
        ("system", f"你是为 {user_id} 服务的助手。"),
    ] + state["messages"]
    
    response = model.invoke(messages)
    return {"messages": [response]}

builder = StateGraph(State, context_schema=Config)
builder.add_node("chat", personalized_chat)
builder.add_edge(START, "chat")
builder.add_edge("chat", END)

graph = builder.compile()

# 运行时传入配置
result = graph.invoke(
    {"messages": [HumanMessage("你好")]},
    config={
        "configurable": {
            "user_id": "张三",
            "model": "moonshot:moonshot-v1-8k",
            "temperature": 0.3,
        }
    },
)
```

### context_schema 的特性

| 特性 | 说明 |
|------|------|
| 只读 | 节点不能修改上下文 |
| 全局 | 所有节点都可以访问 |
| 可选 | 调用时可以不传 |
| 默认值 | `get_config()` 返回空 dict，用 `.get()` 提供默认值 |
| 编译时验证 | 如果指定了 context_schema，invoke 时必须传 config |

## recursion_limit — 防止无限循环

### 为什么需要递归限制

```python
# 循环图 — 如果没有保护会无限执行
builder.add_edge("node_a", "node_b")
builder.add_edge("node_b", "node_a")  # 无限循环！
```

LangGraph 默认设置递归限制为 **25**：

```python
result = graph.invoke({"input": "test"})
# 如果循环超过 25 次，抛出 GraphRecursionError
```

### 自定义递归限制

```python
result = graph.invoke(
    {"input": "test"},
    config={"recursion_limit": 50},  # 增加到 50
)
```

### 使用场景

| 场景 | 推荐限制 | 原因 |
|------|---------|------|
| 简单线性流程 | 10 | 不可能达到限制 |
| Agent 工具循环 | 25 | 防止无限工具调用 |
| 复杂多轮对话 | 50 | 允许更多交互 |
| 批处理管道 | 10 | 固定步骤 |

### 递归限制的计算方式

```
递归限制 = 最多执行的节点调用次数

例如：
START → A → B → C → END
节点调用次数：3（A、B、C 各一次）

如果 C → A 形成循环：
START → A → B → C → A → B → C → END
节点调用次数：6

recursion_limit = 25 意味着最多 25 次节点调用
```

### 处理递归限制错误

```python
from langgraph.errors import GraphRecursionError

try:
    result = graph.invoke({"input": "test"})
except GraphRecursionError:
    print("执行步骤过多，可能陷入了无限循环")
    # 检查图的路由逻辑
```

## RemainingSteps — 在节点中检查剩余步数

```python
from langgraph.config import get_config

def safe_loop_node(state: State) -> Command:
    config = get_config()
    remaining = config.get("remaining_steps", 0)
    
    if remaining <= 2:
        # 快到了，提前退出
        return Command(update={"status": "timeout"}, goto=END)
    
    if state["is_done"]:
        return Command(update={}, goto=END)
    
    return Command(update={}, goto="safe_loop_node")
```

### RemainingSteps 的实际用法

```python
from langgraph.types import RemainingSteps

def monitor_node(state: State) -> dict:
    remaining = get_config().get("remaining_steps", 0)
    return {
        "log": [f"剩余步数: {remaining}"],
    }
```

## config — 传递元数据

### config 的结构

```python
config = {
    "configurable": {
        "user_id": "123",        # 自定义配置
        "thread_id": "456",      # 会话线程 ID
        "checkpoint_id": "789",  # 检查点 ID
    },
    "recursion_limit": 25,       # 递归限制
    "metadata": {
        "source": "api",         # 来源
        "run_id": "uuid-...",    # 运行 ID
    },
    "tags": ["production"],      # 标签
}
```

### 在节点中读取 config

```python
from langgraph.config import get_config

def node_with_metadata(state: State) -> dict:
    config = get_config()
    
    # 自定义配置
    user_id = config.get("configurable", {}).get("user_id", "anonymous")
    
    # 元数据
    metadata = config.get("metadata", {})
    source = metadata.get("source", "unknown")
    
    # 标签
    tags = config.get("tags", [])
    
    return {
        "user_id": user_id,
        "source": source,
    }
```

### 使用 tags 标记执行

```python
result = graph.invoke(
    {"input": "test"},
    config={
        "tags": ["user-facing", "production"],
    },
)
```

Tags 在 LangSmith 追踪中很有用：可以按标签过滤和搜索。

## 配置传递链

```
用户调用                    图执行                    节点访问
─────────                  ──────                   ────────
graph.invoke(              Pregel 引擎              get_config()
    {...},                 ──────────               ──────────
    config={                   │                    返回完整的
        configurable={    将 config 传入            config dict
            user_id="x"   每次节点调用
        }
    }
)
```

## 实战：带配置管理的多租户 Agent

```python
from typing import Annotated, Literal, TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command
from langgraph.graph.message import add_messages
from langgraph.config import get_config
from langchain.chat_models import init_chat_model
from langchain.messages import HumanMessage, AIMessage, SystemMessage

class State(TypedDict):
    messages: Annotated[list, add_messages]
    response_time: float
    model_used: str

class TenantConfig(TypedDict):
    tenant_id: str
    model: str
    temperature: float
    system_prompt: str
    max_tokens: int

def tenant_chat(state: State) -> dict:
    """根据租户配置调用模型"""
    import time
    config = get_config()
    cfg = config.get("configurable", {})
    
    tenant_id = cfg.get("tenant_id", "default")
    model = cfg.get("model", "moonshot:moonshot-v1-8k")
    temperature = cfg.get("temperature", 0.7)
    system_prompt = cfg.get("system_prompt", "你是一个有帮助的助手。")
    max_tokens = cfg.get("max_tokens", 4000)
    
    start_time = time.time()
    
    model_instance = init_chat_model(
        model,
        temperature=temperature,
        max_tokens=max_tokens,
    )
    
    messages = [
        SystemMessage(system_prompt),
    ] + state["messages"]
    
    response = model_instance.invoke(messages)
    
    elapsed = time.time() - start_time
    
    return {
        "messages": [response],
        "response_time": elapsed,
        "model_used": model,
    }

def route_by_tenant(state: State) -> Literal["tenant_chat"]:
    return "tenant_chat"

builder = StateGraph(State, context_schema=TenantConfig)
builder.add_node("tenant_chat", tenant_chat)
builder.add_edge(START, "tenant_chat")
builder.add_edge("tenant_chat", END)

graph = builder.compile()

# 租户 A — 客服场景
result_a = graph.invoke(
    {"messages": [HumanMessage("你好，我需要帮助")]},
    config={
        "configurable": {
            "tenant_id": "company_a",
            "model": "moonshot:moonshot-v1-8k",
            "temperature": 0.3,
            "system_prompt": "你是 Company A 的客服助手。礼貌、专业地回答。",
            "max_tokens": 2000,
        }
    },
)

# 租户 B — 创意写作场景
result_b = graph.invoke(
    {"messages": [HumanMessage("写一首关于 AI 的诗")]},
    config={
        "configurable": {
            "tenant_id": "company_b",
            "model": "moonshot:moonshot-v1-8k",
            "temperature": 0.9,
            "system_prompt": "你是 Company B 的创意写作助手。富有想象力和创意。",
            "max_tokens": 4000,
        }
    },
)
```

## 实战：带递归监控的循环 Agent

```python
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command
from langgraph.graph.message import add_messages
from langgraph.config import get_config
from langchain.chat_models import init_chat_model
from langchain.messages import HumanMessage, AIMessage

class LoopState(TypedDict):
    messages: Annotated[list, add_messages]
    loop_count: Annotated[int, add]
    remaining_info: str

def monitored_agent(state: LoopState) -> Command:
    """带递归限制监控的 Agent"""
    config = get_config()
    remaining = config.get("remaining_steps", 0)
    
    # 如果剩余步数太少，提前终止
    if remaining <= 3:
        return Command(
            update={
                "messages": [AIMessage(f"达到安全限制，还剩 {remaining} 步。停止执行。")],
                "loop_count": 1,
                "remaining_info": f"剩余步数: {remaining}",
            },
            goto=END,
        )
    
    # 正常执行
    model = init_chat_model("moonshot:moonshot-v1-8k")
    response = model.invoke(state["messages"])
    
    # 假设我们根据响应决定是否继续
    should_continue = len(response.content) > 20
    
    return Command(
        update={
            "messages": [response],
            "loop_count": 1,
            "remaining_info": f"剩余步数: {remaining - 1}",
        },
        goto="monitored_agent" if should_continue else END,
    )

builder = StateGraph(LoopState)
builder.add_node("agent", monitored_agent)
builder.add_edge(START, "agent")

graph = builder.compile()

result = graph.invoke(
    {"messages": [HumanMessage("请连续回复 5 次，每次只说一个词。")], "loop_count": 0, "remaining_info": ""},
    config={"recursion_limit": 15},
)
print(f"循环次数: {result['loop_count']}")
print(f"剩余信息: {result['remaining_info']}")
```

## 配置文件持久化

### 检查点配置

```python
from langgraph.checkpoint.memory import MemorySaver

# 内存检查点
checkpointer = MemorySaver()

graph = builder.compile(checkpointer=checkpointer)

# 使用 thread_id 恢复
result = graph.invoke(
    {"messages": [HumanMessage("你好")]},
    config={"configurable": {"thread_id": "conversation-1"}},
)

# 继续对话 — 自动加载之前的 State
result2 = graph.invoke(
    {"messages": [HumanMessage("继续")]},
    config={"configurable": {"thread_id": "conversation-1"}},
)
```

### 检查点类型

| 类型 | 适用场景 | 持久化 |
|------|---------|--------|
| `MemorySaver` | 开发测试 | 内存（进程重启丢失） |
| `SqliteSaver` | 单机部署 | SQLite 文件 |
| `PostgresSaver` | 生产环境 | PostgreSQL |

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)
```

## 小结

| 概念 | 说明 |
|------|------|
| `context_schema` | 定义只读上下文类型，与 State 分离 |
| `get_config()` | 在节点中访问运行时配置 |
| `configurable` | config 中的自定义配置字典 |
| `recursion_limit` | 防止无限循环（默认 25） |
| `RemainingSteps` | 节点中检查剩余执行步数 |
| `metadata` | 传递追踪元数据 |
| `tags` | 标记执行，用于过滤和搜索 |
| `checkpointer` | 持久化 State，支持恢复 |
| `thread_id` | 会话标识，用于多轮对话 |
