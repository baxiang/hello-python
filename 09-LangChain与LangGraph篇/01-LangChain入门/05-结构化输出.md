# 09.01.05 - 结构化输出

> 结构化输出让 LLM 返回格式化的数据，而不是自由文本，是构建可靠 AI 应用的关键

## 为什么需要结构化输出

LLM 默认返回自由文本，但实际应用中我们需要结构化数据：

```
┌────────────────────────────────────────────────────────────┐
│                    问题：自由文本不可控                      │
│                                                            │
│  提示词: "评价这段代码"                                     │
│                                                            │
│  回复1: "代码写得不错，逻辑清晰，但需要注意变量命名..."       │
│  回复2: "评分: 8/10\n优点: ...\n缺点: ..."                  │
│  回复3: "整体而言，这段代码的质量在中等水平以上。"            │
│                                                            │
│  → 每次回复格式不同，程序无法解析                           │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│                    解决：结构化输出                          │
│                                                            │
│  提示词: "评价这段代码" + 结构定义                           │
│                                                            │
│  回复: {"score": 8, "pros": ["逻辑清晰"], "cons": ["命名"]}  │
│                                                            │
│  → 格式固定，程序可以直接使用                                │
└────────────────────────────────────────────────────────────┘
```

## with_structured_output

### 基础用法（Pydantic）

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model

# 1. 定义输出结构
class CodeReview(BaseModel):
    """代码审查结果"""
    score: int = Field(description="评分，1-10 分")
    pros: list[str] = Field(description="优点列表")
    cons: list[str] = Field(description="缺点列表")
    suggestion: str = Field(description="改进建议")

# 2. 绑定结构化输出
model = init_chat_model("moonshot:moonshot-v1-8k")
structured_model = model.with_structured_output(CodeReview)

# 3. 调用
result = structured_model.invoke("评价: def add(a,b): return a+b")

print(f"评分: {result.score}")
print(f"优点: {result.pros}")
print(f"缺点: {result.cons}")
print(f"建议: {result.suggestion}")
```

### TypedDict 方式

```python
from typing_extensions import TypedDict
from langchain.chat_models import init_chat_model

class WeatherInfo(TypedDict):
    city: str
    temperature: int
    condition: str
    humidity: int

model = init_chat_model("deepseek:deepseek-chat")
structured_model = model.with_structured_output(WeatherInfo)

result = structured_model.invoke("查询北京的天气")
print(result)
# {'city': '北京', 'temperature': 25, 'condition': '晴', 'humidity': 45}
```

### JSON Schema 方式

```python
from langchain.chat_models import init_chat_model

schema = {
    "title": "PersonInfo",
    "description": "个人信息",
    "type": "object",
    "properties": {
        "name": {"type": "string", "description": "姓名"},
        "age": {"type": "integer", "description": "年龄"},
        "hobbies": {
            "type": "array",
            "items": {"type": "string"},
            "description": "爱好列表",
        },
    },
    "required": ["name", "age"],
}

model = init_chat_model("moonshot:moonshot-v1-8k")
structured_model = model.with_structured_output(schema)

result = structured_model.invoke("介绍一下你自己")
print(result)
```

## Pydantic 详解

### 字段约束

```python
from pydantic import BaseModel, Field
from typing import Literal

class ProductReview(BaseModel):
    """产品评价"""
    product_name: str = Field(description="产品名称")
    rating: int = Field(
        description="评分",
        ge=1,   # >= 1
        le=5,   # <= 5
    )
    sentiment: Literal["positive", "negative", "neutral"] = Field(
        description="情感倾向"
    )
    summary: str = Field(
        description="评价摘要",
        max_length=200,  # 最大长度
    )
    tags: list[str] = Field(
        description="标签列表",
        min_length=1,    # 最少 1 个
        max_length=5,    # 最多 5 个
    )
```

### 嵌套结构

```python
from pydantic import BaseModel, Field

class Address(BaseModel):
    """地址"""
    province: str = Field(description="省份")
    city: str = Field(description="城市")
    detail: str = Field(description="详细地址")

class Person(BaseModel):
    """完整个人信息"""
    name: str = Field(description="姓名")
    age: int = Field(description="年龄")
    address: Address = Field(description="居住地址")
    contacts: list[str] = Field(description="联系方式列表")

model = init_chat_model("moonshot:moonshot-v1-32k")
structured_model = model.with_structured_output(Person)

result = structured_model.invoke(
    "描述一个住在广州市天河区的人，名字叫小明，30岁"
)
print(result)
```

## 多模式输出

### 方式1: 返回多个候选

```python
from pydantic import BaseModel, Field
from typing import Literal

class SentimentAnalysis(BaseModel):
    """情感分析结果"""
    sentiment: Literal["positive", "negative", "neutral"] = Field(
        description="整体情感"
    )
    confidence: float = Field(description="置信度，0-1")
    reasons: list[str] = Field(description="判断理由")

model = init_chat_model("moonshot:moonshot-v1-8k")
structured_model = model.with_structured_output(SentimentAnalysis)

reviews = [
    "这个产品太棒了，超出预期！",
    "质量很差，用了两天就坏了。",
    "一般般吧，价格还算合理。",
]

for review in reviews:
    result = structured_model.invoke(review)
    print(f"评论: {review}")
    print(f"情感: {result.sentiment} (置信度: {result.confidence})")
    print(f"原因: {result.reasons}")
    print()
```

### 方式2: 选择格式

```python
from pydantic import BaseModel, Field
from typing import Union, Literal

class DirectAnswer(BaseModel):
    """直接回答"""
    type: Literal["direct"] = "direct"
    content: str = Field(description="回答内容")

class SearchNeeded(BaseModel):
    """需要搜索"""
    type: Literal["search"] = "search"
    query: str = Field(description="搜索关键词")
    reason: str = Field(description="为什么需要搜索")

class Clarification(BaseModel):
    """需要澄清"""
    type: Literal["clarify"] = "clarify"
    question: str = Field(description="需要澄清的问题")

ResponseType = Union[DirectAnswer, SearchNeeded, Clarification]

model = init_chat_model("deepseek:deepseek-chat")
structured_model = model.with_structured_output(ResponseType)

questions = [
    "Python 中 list 和 tuple 的区别是什么？",  # 可以直接回答
    "今天北京的天气怎么样？",                     # 需要搜索
    "你昨天说的那件事后来怎么样了？",             # 需要澄清
]

for q in questions:
    result = structured_model.invoke(q)
    print(f"问: {q}")
    print(f"类型: {result.type}")
    print(f"详情: {result}")
    print()
```

## 实际应用示例

### 示例1: 信息提取

```python
from pydantic import BaseModel, Field

class ContactInfo(BaseModel):
    """联系信息"""
    name: str = Field(description="姓名")
    phone: str = Field(description="电话号码")
    email: str = Field(description="电子邮箱")
    company: str = Field(description="公司名称")

text = """
你好，我是张三，来自阿里巴巴技术部。
我的电话是 138-0013-8000，
工作邮箱是 zhangsan@alibaba.com。
"""

model = init_chat_model("moonshot:moonshot-v1-8k")
extractor = model.with_structured_output(ContactInfo)

result = extractor.invoke(f"从以下文本中提取联系信息:\n\n{text}")
print(f"姓名: {result.name}")
print(f"电话: {result.phone}")
print(f"邮箱: {result.email}")
print(f"公司: {result.company}")
```

### 示例2: 分类任务

```python
from pydantic import BaseModel, Field
from typing import Literal

class TextCategory(BaseModel):
    """文本分类"""
    category: Literal[
        "科技", "体育", "财经", "娱乐", "教育", "医疗",
        "政治", "社会", "国际", "其他"
    ] = Field(description="文章类别")
    confidence: float = Field(description="分类置信度，0-1")
    keywords: list[str] = Field(description="关键词列表")

model = init_chat_model("deepseek:deepseek-chat")
classifier = model.with_structured_output(TextCategory)

articles = [
    "苹果公司发布新款 iPhone，搭载 A18 芯片...",
    "中国队在亚洲杯决赛中以 3:0 战胜对手...",
    "央行宣布降准 0.25 个百分点，释放流动性...",
]

for article in articles:
    result = classifier.invoke(f"分类以下文章: {article}")
    print(f"类别: {result.category} ({result.confidence:.0%})")
    print(f"关键词: {result.keywords}")
    print()
```

### 示例3: 数据验证与清洗

```python
from pydantic import BaseModel, Field, field_validator

class CleanData(BaseModel):
    """清洗后的数据"""
    name: str = Field(description="名称")
    price: float = Field(description="价格，必须为正数")
    category: str = Field(description="分类")

    @field_validator("price")
    @classmethod
    def price_must_be_positive(cls, v):
        if v <= 0:
            raise ValueError("价格必须为正数")
        return v

    @field_validator("name")
    @classmethod
    def name_must_not_be_empty(cls, v):
        if not v.strip():
            raise ValueError("名称不能为空")
        return v.strip()

raw_data = [
    "商品: iPhone 15, 价格: 5999元, 分类: 手机",
    "名称: , 价格: -100, 分类: 电子产品",  # 有问题的数据
    "商品: 华为 Mate 60, 价格: 6999元, 分类: 手机",
]

model = init_chat_model("moonshot:moonshot-v1-8k")
cleaner = model.with_structured_output(CleanData)

for raw in raw_data:
    try:
        result = cleaner.invoke(f"从以下文本中提取并清洗数据: {raw}")
        print(f"清洗成功: {result}")
    except Exception as e:
        print(f"清洗失败: {e}")
```

### 示例4: 批量结构化输出

```python
from pydantic import BaseModel, Field

class QAPair(BaseModel):
    """问答对"""
    question: str = Field(description="问题")
    answer: str = Field(description="答案")

class QAList(BaseModel):
    """问答列表"""
    qa_pairs: list[QAPair] = Field(description="问答对列表")

text = """
Python 中的 GIL（全局解释器锁）是 CPython 解释器中的一个互斥锁，
它保证同一时刻只有一个线程执行 Python 字节码。这意味着即使
在多核 CPU 上，Python 的多线程程序也无法真正实现并行计算。
但是 I/O 密集型任务仍然可以从多线程中受益。
"""

model = init_chat_model("moonshot:moonshot-v1-32k")
qa_extractor = model.with_structured_output(QAList)

result = qa_extractor.invoke(f"从以下文本中提取所有问答对:\n\n{text}")

for qa in result.qa_pairs:
    print(f"Q: {qa.question}")
    print(f"A: {qa.answer}")
    print()
```

## 格式化 Prompt

有时候 LLM 会忽略结构定义，需要配合格式化提示：

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model
from langchain.messages import SystemMessage, HumanMessage

class Summary(BaseModel):
    """文本摘要"""
    title: str = Field(description="标题，不超过 20 字")
    key_points: list[str] = Field(description="关键要点，3-5 条")
    summary: str = Field(description="一句话摘要")

model = init_chat_model("moonshot:moonshot-v1-8k")
structured_model = model.with_structured_output(Summary)

# 格式化提示词
text = "人工智能正在改变各行各业..."

response = structured_model.invoke([
    SystemMessage(
        "你是一个专业的文本分析助手。"
        "请严格按照要求的 JSON 格式输出结果。"
    ),
    HumanMessage(f"请为以下文本生成结构化摘要:\n\n{text}"),
])

print(f"标题: {response.title}")
print(f"要点: {response.key_points}")
print(f"摘要: {response.summary}")
```

## 流式结构化输出

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model

class LongAnswer(BaseModel):
    """长回答"""

model = init_chat_model("moonshot:moonshot-v1-32k")
structured_model = model.with_structured_output(LongAnswer)

# 流式调用

):
    print(f"部分结果: {chunk}")
```

## 常见错误与解决

### 错误1: 结构过于复杂

```python
# 不好：嵌套太深
class BadStructure(BaseModel):
    data: dict[str, dict[str, dict[str, list[dict[str, str]]]]]

# 好：扁平化设计
class GoodStructure(BaseModel):
    items: list[str] = Field(description="项目列表")
    categories: list[str] = Field(description="分类列表")
    mappings: dict[str, str] = Field(description="键值对映射")
```

### 错误2: 字段描述不清晰

```python
# 不好：描述模糊
class BadField(BaseModel):
    value: str = Field(description="值")

# 好：具体描述
class GoodField(BaseModel):
    value: str = Field(
        description="具体的数值，包含单位，如 '25°C'、'100km/h'"
    )
```

### 错误3: 类型不匹配

```python
# 模型可能返回字符串 "25" 而不是整数 25
class WeatherData(BaseModel):
    temperature: int = Field(description="温度（整数）")  # 明确说明类型

# 使用 Literal 限制选择范围
from typing import Literal

class WeatherCondition(BaseModel):
    condition: Literal["晴", "多云", "阴", "雨", "雪"] = Field(
        description="天气状况"
    )
```

## 结构化输出 vs 工具调用

```
┌─────────────────────────────────────────────────────────┐
│            结构化输出 vs 工具调用                         │
├────────────────────┬────────────────────────────────────┤
│ 结构化输出          │ 工具调用                            │
├────────────────────┼────────────────────────────────────┤
│ 返回结构化数据      │ 调用外部函数                       │
│ 不需要执行代码      │ 需要实际执行代码                    │
│ 适合数据提取        │ 适合获取实时信息                    │
│ 格式固定            │ 可以组合多个工具                   │
│ with_structured_   │ bind_tools                         │
│ output()           │                                    │
└────────────────────┴────────────────────────────────────┘
```

**何时使用结构化输出**：
- 信息提取（从文本中提取字段）
- 分类任务（将文本分类到预定义类别）
- 数据格式化（统一输出格式）
- 评估判断（打分、评价）

**何时使用工具调用**：
- 需要执行外部操作（搜索、计算、查询）
- 需要获取实时数据（天气、股价）
- 需要与用户系统交互（数据库、API）

**组合使用**：

```python
# 先用工具获取数据
weather = weather_tool.invoke({"city": "北京"})

# 再用结构化输出格式化结果
class FormattedWeather(BaseModel):
    city: str
    temperature: int
    suggestion: str

formatter = model.with_structured_output(FormattedWeather)
result = formatter.invoke(
    f"将以下天气信息格式化: {weather}"
)
```

## 小结

| 要点 | 说明 |
|------|------|
| `with_structured_output()` | 核心方法，让模型返回结构化数据 |
| Pydantic | 最推荐的定义方式，支持类型验证和约束 |
| TypedDict | 轻量级替代方案 |
| JSON Schema | 最灵活，可以定义任意结构 |
| 字段描述 | 描述越详细，输出越准确 |
| 多模式 | 使用 `Union` 让模型选择不同的输出格式 |
| 嵌套 | 支持任意深度的嵌套结构 |
