# 09.01.03 - 流式与批量

> 流式输出改善用户体验，批量调用提升吞吐量，掌握这两种模式是生产环境的必修课

## 调用模式概览

```
┌─────────────────────────────────────────────────────────────────┐
│                        LLM 调用模式                               │
├──────────────┬──────────────┬─────────────┬──────────────────────┤
│ 模式          │ 方法          │ 返回类型     │ 适用场景              │
├──────────────┼──────────────┼─────────────┼──────────────────────┤
│ 单次调用      │ invoke()     │ AIMessage   │ 简单问答、调试        │
│ 流式调用      │ stream()     │ Iterator    │ 聊天界面、长文本生成  │
│ 批量调用      │ batch()      │ List        │ 并行处理多个独立请求  │
│ 异步完成      │ batch_as_    │ Iterator    │ 需要处理完成顺序      │
│              │ completed()  │             │                      │
│ 事件流        │ astream_     │ AsyncIter   │ 复杂工具调用、Agent  │
│              │ events()     │             │                      │
└──────────────┴──────────────┴─────────────┴──────────────────────┘
```

## stream — 流式输出

### 基础用法

流式调用逐块返回响应，用户不需要等待全部内容生成完成：

```python
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

# 流式输出一篇长文章
for chunk in model.stream("请写一篇关于人工智能发展的短文，约500字"):
    print(chunk.content, end="", flush=True)

print()  # 最后换行
```

### Chunk 的结构

每个 chunk 是一个 AIMessageChunk 对象：

```python
from langchain.chat_models import init_chat_model

model = init_chat_model("deepseek:deepseek-chat")

for chunk in model.stream("解释 Python 的 GIL"):
    print(f"--- chunk 属性 ---")
    print(f"content: {chunk.content!r}")       # 当前块的文本
    print(f"type: {type(chunk).__name__}")     # AIMessageChunk
    print(f"id: {chunk.id}")                   # 响应 ID（可能为 None）
```

```
典型输出：
chunk 1: content='G'
chunk 2: content='IL'
chunk 3: content='（'
chunk 4: content='Glo'
chunk 5: content='bal '
chunk 6: content='Inter'
...
```

### 累积完整响应

```python
from langchain.chat_models import init_chat_model
from langchain.messages import AIMessageChunk

model = init_chat_model("moonshot:moonshot-v1-8k")

# 累积所有 chunk
full_response = None
for chunk in model.stream("写一首关于编程的短诗"):
    if full_response is None:
        full_response = chunk
    else:
        full_response += chunk  # AIMessageChunk 支持加法

print(f"\n\n完整响应: {full_response.content}")
print(f"Token 使用: {full_response.usage_metadata}")
```

### 带进度指示的流式

```python
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

total_chars = 0
start_time = time.time()

print("正在生成回答...\n")
for chunk in model.stream("详细介绍 Python 中装饰器的用法和原理"):
    print(chunk.content, end="", flush=True)
    total_chars += len(chunk.content)

elapsed = time.time() - start_time
print(f"\n\n生成完成: {total_chars} 字符, 耗时 {elapsed:.1f} 秒")
print(f"速度: {total_chars / elapsed:.0f} 字符/秒")
```

### 异步流式调用

```python
import asyncio
from langchain.chat_models import init_chat_model

model = init_chat_model("deepseek:deepseek-chat")

async def stream_async():
    async for chunk in model.astream("异步流式调用示例"):
        print(chunk.content, end="", flush=True)

asyncio.run(stream_async())
```

## astream_events — 事件流

`astream_events` 提供更细粒度的事件流，适合调试和复杂 Agent：

### 基础事件流

```python
import asyncio
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

async def show_events():
    async for event in model.astream_events("你好，请介绍一下自己", version="v2"):
        print(f"事件: {event['event']}")
        print(f"名称: {event.get('name', 'N/A')}")
        if event["event"] == "on_chat_model_stream":
            print(f"数据: {event['data']['chunk'].content[:50]}")
        print("---")

asyncio.run(show_events())
```

### 事件类型详解

```
┌─────────────────────────────────────────────────────────┐
│                  事件类型及触发时机                        │
├─────────────────────────┬───────────────────────────────┤
│ on_chat_model_start     │ 模型开始生成时                  │
│ on_chat_model_stream    │ 每个 chunk 到达时               │
│ on_chat_model_end       │ 模型完成生成时                  │
│ on_tool_start           │ 工具开始执行时                  │
│ on_tool_end             │ 工具执行完成时                  │
│ on_chain_start          │ 链/Agent 开始运行时             │
│ on_chain_end            │ 链/Agent 完成运行时             │
│ on_retriever_start      │ 检索器开始工作时                │
│ on_retriever_end        │ 检索器完成工作时                │
└─────────────────────────┴───────────────────────────────┘
```

### 工具调用事件流

```python
import asyncio
from langchain.chat_models import init_chat_model
from langchain.tools import tool

@tool
def search(query: str) -> str:
    """搜索信息"""
    return f"搜索结果: {query}"

model = init_chat_model("moonshot:moonshot-v1-8k")
model_with_tools = model.bind_tools([search])

async def tool_events():
    async for event in model_with_tools.astream_events(
        "搜索一下 Python 3.12 有什么新特性",
        version="v2",
    ):
        kind = event["event"]
        name = event.get("name", "")

        if kind == "on_chat_model_stream":
            data = event["data"]["chunk"]
            if data.content:
                print(f"[AI] {data.content}", end="", flush=True)
            if data.tool_calls:
                print(f"\n[工具调用] {data.tool_calls}")

        elif kind == "on_tool_start":
            print(f"\n[工具开始] {event['data'].get('input', {})}")

        elif kind == "on_tool_end":
            print(f"[工具完成] {event['data'].get('output', '')}")

        print()

asyncio.run(tool_events())
```

### 过滤特定事件

```python
async def filter_events():
    """只关注模型流式事件"""
    async for event in model.astream_events("写一首诗", version="v2"):
        if event["event"] != "on_chat_model_stream":
            continue
        content = event["data"]["chunk"].content
        if content:
            print(content, end="", flush=True)

asyncio.run(filter_events())
```

## batch — 批量调用

### 基础批量调用

`batch` 并发发送多个请求，等待所有完成后返回结果：

```python
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

# 批量处理多个问题
questions = [
    "什么是 Python 的 GIL？",
    "什么是列表推导式？",
    "什么是装饰器？",
    "什么是上下文管理器？",
    "什么是描述符？",
]

responses = model.batch(questions)

for question, response in zip(questions, responses):
    if isinstance(response, Exception):
        print(f"Q: {question}")
        print(f"错误: {response}")
    else:
        print(f"Q: {question}")
        print(f"A: {response.content[:100]}...\n")
```

### 批量调用参数

```python
responses = model.batch(
    questions,
    config=None,              # 可选配置列表
    return_exceptions=True,   # 返回异常而不是抛出
    batch_size=None,          # 批次大小（None=全部并行）
)
```

```
┌───────────────────────────────────────────────────────┐
│                batch 执行流程                          │
├───────────────────────────────────────────────────────┤
│                                                       │
│  输入: [Q1, Q2, Q3, Q4, Q5]                           │
│       ↓                                               │
│  ┌──────────────────────────────────────┐             │
│  │  并发发送所有请求到 API                │             │
│  │  Q1 ──→  ●                           │             │
│  │  Q2 ──→  ●                           │             │
│  │  Q3 ──→  ●                           │             │
│  │  Q4 ──→  ●                           │             │
│  │  Q5 ──→  ●                           │             │
│  └──────────────────────────────────────┘             │
│       ↓                                               │
│  等待最慢的那个完成                                      │
│       ↓                                               │
│  输出: [A1, A2, A3, A4, A5]  (顺序对应输入)            │
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 处理批量异常

```python
# return_exceptions=True: 异常作为结果返回
results = model.batch(
    ["正常问题", "可能导致错误的内容" * 10000],
    return_exceptions=True,
)

for i, result in enumerate(results):
    if isinstance(result, Exception):
        print(f"请求 {i} 失败: {result}")
    else:
        print(f"请求 {i} 成功: {result.content[:50]}")
```

### 限制并发数

```python
# 控制并发请求数量，避免触发 API 限流
responses = model.batch(
    ["问题" + str(i) for i in range(20)],
    batch_size=5,  # 每次只并发 5 个
    return_exceptions=True,
)
```

## batch_as_completed — 按完成顺序返回

### 基础用法

与 `batch` 不同，`batch_as_completed` 按完成顺序返回，不需要等待最慢的请求：

```python
from langchain.chat_models import init_chat_model
import time

model = init_chat_model("moonshot:moonshot-v1-8k")

questions = [
    "写一首关于春天的短诗",      # 短响应
    "写一篇关于人工智能的论文",   # 长响应
    "列出 5 个质数",            # 短响应
]

# 按完成顺序处理
start = time.time()
for idx, response in model.batch_as_completed(
    questions,
    return_exceptions=True,
):
    elapsed = time.time() - start
    if isinstance(response, Exception):
        print(f"[{elapsed:.1f}s] 请求 {idx} 失败: {response}")
    else:
        print(f"[{elapsed:.1f}s] 请求 {idx} 完成: {response.content[:30]}...")
```

### batch vs batch_as_completed 对比

```
batch（按输入顺序返回）：
  输入: [短问题, 长问题, 短问题]
  发送: 同时发出 3 个请求
  等待: 等待最慢的「长问题」完成
  返回: [短答案, 长答案, 短答案]  ← 顺序与输入对应

batch_as_completed（按完成顺序返回）：
  输入: [短问题, 长问题, 短问题]
  发送: 同时发出 3 个请求
  返回: (0, 短答案)  ← 第 1 个完成
         (2, 短答案)  ← 第 2 个完成
         (1, 长答案)  ← 第 3 个完成（最慢）
```

### 异步批量完成

```python
import asyncio
from langchain.chat_models import init_chat_model

model = init_chat_model("deepseek:deepseek-chat")

async def async_batch():
    results = await model.abatch([
        "什么是异步编程？",
        "什么是协程？",
        "什么是事件循环？",
    ])
    for r in results:
        print(r.content[:50])

asyncio.run(async_batch())
```

## 性能对比与选择

### 不同模式性能特征

| 模式 | 延迟 | 吞吐量 | 内存 | 适用场景 |
|------|------|--------|------|---------|
| `invoke` | 高（等待全部生成） | 低 | 低 | 调试、简单问答 |
| `stream` | 低（首字即显示） | 中 | 低 | 聊天界面、长文本 |
| `batch` | 中（等最慢的完成） | 高 | 中 | 批量处理、数据管道 |
| `batch_as_completed` | 低（先完成先处理） | 高 | 中 | 异构请求、超时敏感 |
| `astream_events` | 低 | 中 | 中 | Agent、工具调用链 |

### 场景选择决策树

```
你的需求是什么？
│
├── 需要逐字显示给用户？
│   └──→ 使用 stream() 或 astream()
│
├── 要处理很多独立问题？
│   ├── 需要按顺序处理完成的？
│   │   └──→ 使用 batch_as_completed()
│   └── 需要按输入顺序对应？
│       └──→ 使用 batch()
│
├── 需要观察工具调用/链内部事件？
│   └──→ 使用 astream_events()
│
└── 只是简单调用？
    └──→ 使用 invoke()
```

### 示例1: 批量翻译

```python
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

texts = [
    "Hello, welcome to Python programming.",
    "The quick brown fox jumps over the lazy dog.",
    "Practice makes perfect.",
    "Time is money.",
    "Knowledge is power.",
]

# 批量翻译
prompts = [f"将以下英文翻译成中文: {text}" for text in texts]

    if isinstance(result, Exception):
        print(f"翻译失败: {text}")

        print(f"原文: {text}")
```

## 流式错误处理

### 网络超时处理

```python
from langchain.chat_models import init_chat_model

model = init_chat_model(
    "moonshot:moonshot-v1-8k",
    timeout=60,       # 60 秒超时
    max_retries=3,    # 自动重试
)

try:
    for chunk in model.stream("写一篇很长的文章" * 10):
        print(chunk.content, end="", flush=True)
except Exception as e:
    print(f"\n流式调用失败: {e}")
```

### 流式中断恢复

```python
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

collected = []

try:
    for chunk in model.stream("写一篇关于 Python 历史的文章"):
        collected.append(chunk.content)
        print(chunk.content, end="", flush=True)
except KeyboardInterrupt:
    print(f"\n\n用户中断，已收集 {len(collected)} 个 chunk")
    full_text = "".join(collected)
    print(f"已收集内容长度: {len(full_text)}")
```

## 小结

| 要点 | 说明 |
|------|------|
| `stream()` | 逐块返回响应，适合实时显示 |
| `astream_events()` | 细粒度事件流，适合调试和 Agent |
| `batch()` | 并发批量处理，按输入顺序返回 |
| `batch_as_completed()` | 并发批量处理，按完成顺序返回 |
| `return_exceptions=True` | 批量调用时返回异常而非抛出 |
| 异步方法 | `astream()`, `abatch()`, `abatch_as_completed()` |
| 选择依据 | 实时显示用 stream，批处理用 batch，调试用 astream_events |
