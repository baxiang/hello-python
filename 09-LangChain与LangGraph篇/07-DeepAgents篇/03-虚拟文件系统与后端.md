# 09.07.03 - 虚拟文件系统与后端

> 选择并配置可插拔的文件系统后端

## 概述

Deep Agents 通过虚拟文件系统工具（`ls`、`read_file`、`write_file`、`edit_file`、`glob`、`grep`）让 Agent 能管理上下文。这些工具操作的是**可插拔的后端（Backend）**，而不是直接读写磁盘。

```
文件系统工具
    ↓
后端接口（BackendProtocol）
    ↓
┌─────────┬──────────┬───────┬────────┐
│ State   │ Filesys  │ Store │ Sandbox│
│ Backend │ Backend  │Backend │ Backend│
└─────────┴──────────┴───────┴────────┘
```

`read_file` 原生支持图片文件（`.png`、`.jpg`、`.jpeg`、`.gif`、`.webp`），所有后端均返回多模态内容块。

## 后端类型一览

| 后端 | 存储位置 | 跨线程 | 执行命令 | 适用场景 |
|------|---------|--------|---------|---------|
| `StateBackend` | LangGraph 状态 | ❌ | ❌ | 开发测试、临时工作区 |
| `FilesystemBackend` | 本地磁盘 | ✅ | ❌ | 本地开发项目 |
| `LocalShellBackend` | 本地磁盘 | ✅ | ✅ | 本地编码助手 |
| `StoreBackend` | LangGraph Store | ✅ | ❌ | 长期记忆、生产部署 |
| `CompositeBackend` | 组合多种 | 按需 | 按需 | 混合存储方案 |
| Sandbox 后端 | 隔离环境 | 按需 | ✅ | 安全代码执行 |

## StateBackend（默认）

```python
from deepagents import create_deep_agent

# 默认使用 StateBackend
agent = create_deep_agent(model="moonshot:moonshot-v1-8k")

# 显式指定（等价于上面）
from deepagents.backends import StateBackend
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=StateBackend(),
)
```

### 工作原理

- 文件存储在 LangGraph agent 状态中
- **仅在当前 thread 内持久**（通过 Checkpointer 在多轮对话中保持）
- 子代理写入的文件在子代理结束后**仍保留在状态中**
- 自动清理大型工具输出（接近 85% 上下文窗口时）

### 适用场景

- Agent 的临时草稿纸
- 中间结果暂存
- 不需要跨会话持久化的场景

### 初始化文件

通过 `invoke` 的 `files` 参数预填内容：

```python
from deepagents.backends.utils import create_file_data

result = agent.invoke(
    {
        "messages": [{"role": "user", "content": "查看配置"}],
        "files": {
            "/config/settings.txt": create_file_data("theme=dark\nlang=zh"),
            "/data/template.md": create_file_data("# 报告模板\n\n请在此填写..."),
        },
    },
    config={"configurable": {"thread_id": "session-1"}},
)
```

## FilesystemBackend（本地磁盘）

```python
from deepagents import create_deep_agent
from deepagents.backends import FilesystemBackend

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=FilesystemBackend(
        root_dir="/absolute/path/to/project",  # 必须是绝对路径
        virtual_mode=True,                      # 启用虚拟模式
    ),
)
```

### 虚拟模式（virtual_mode=True）

启用后，Agent 被限制在 `root_dir` 下操作：
- 禁止 `..` 路径穿越
- 禁止 `~` 扩展
- 禁止 `root_dir` 外的绝对路径

> ⚠️ **默认 `virtual_mode=False` 即使设置了 `root_dir` 也不提供任何安全保障！务必显式启用。**

### 安全警告

> **FilesystemBackend 授予 Agent 直接读写文件的权限。**
>
> **适用场景：**
> - 本地开发 CLI（编码助手）
> - CI/CD 沙箱
>
> **不适用场景：**
> - Web 服务器或 HTTP API — 改用 `StateBackend`、`StoreBackend` 或沙箱
>
> **安全风险：**
> - Agent 可读取任何可访问文件，包括 `.env`、凭证
> - 结合网络工具可能导致 SSRF 泄露密钥
> - 文件修改不可逆
>
> **建议防护：**
> 1. 始终启用 `virtual_mode=True`
> 2. 启用人类审核（HITL）审查敏感操作
> 3. 排除密钥文件所在路径
> 4. 生产环境改用沙箱后端

## LocalShellBackend（本地 Shell）

```python
from deepagents import create_deep_agent
from deepagents.backends import LocalShellBackend

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=LocalShellBackend(
        root_dir=".",
        env={"PATH": "/usr/bin:/bin"},  # 限制可执行路径
    ),
)
```

### 功能

- 包含 `FilesystemBackend` 的所有文件操作
- 额外提供 `execute` 工具，可执行 shell 命令
- 命令在主机上直接执行（**无沙箱**）

### 参数

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `timeout` | 120 | 命令超时秒数 |
| `max_output_bytes` | 100000 | 最大输出字节数 |
| `env` | `{}` | 环境变量 |
| `inherit_env` | `True` | 是否继承主机环境变量 |

### 安全警告

> **LocalShellBackend 授予 Agent 直接文件读写 + 无限制 Shell 执行的权限。**
>
> - 命令以当前用户权限运行
> - 可访问系统任何路径（`virtual_mode` 对 Shell 命令无效）
> - 命令消耗 CPU、内存、磁盘无上限
>
> **强烈建议：**
> 1. 启用人类审核（HITL）
> 2. 仅在受控开发环境使用
> 3. 生产环境改用沙箱后端

## StoreBackend（跨线程持久存储）

```python
from deepagents import create_deep_agent
from deepagents.backends import StoreBackend
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()  # 生产环境换成 Redis/PostgreSQL

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=StoreBackend(
        namespace=lambda rt: (rt.server_info.user.identity,),
    ),
    store=store,
)
```

### 工作原理

- 文件存储在 LangGraph `BaseStore` 中
- **跨线程持久化**（不同 thread_id 可访问相同数据）
- 支持语义搜索
- 支持命名空间隔离

### 命名空间工厂

`namespace` 参数控制数据隔离粒度：

```python
from deepagents.backends import StoreBackend

# 按用户隔离：每个用户独立存储
backend = StoreBackend(
    namespace=lambda rt: (rt.server_info.user.identity,),
)

# 按助手隔离：同一助手的所有用户共享
backend = StoreBackend(
    namespace=lambda rt: (rt.server_info.assistant_id,),
)

# 按线程隔离：每个会话独立
backend = StoreBackend(
    namespace=lambda rt: (rt.execution_info.thread_id,),
)

# 组合隔离：用户 + 助手
backend = StoreBackend(
    namespace=lambda rt: (rt.server_info.assistant_id, rt.server_info.user.identity),
)

# 添加后缀区分不同用途
backend = StoreBackend(
    namespace=lambda rt: (rt.server_info.user.identity, "filesystem"),
)
```

### 部署到 LangSmith 时

```python
# 省略 store 参数，平台自动提供
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=StoreBackend(
        namespace=lambda rt: (rt.server_info.user.identity,),
    ),
)
```

## CompositeBackend（组合路由）

```python
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=CompositeBackend(
        default=StateBackend(),              # 默认：临时存储
        routes={
            "/memories/": StoreBackend(),    # 记忆：跨线程持久
        },
    ),
    store=InMemoryStore(),
)
```

### 路由规则

| 路径 | 路由到 | 持久性 |
|------|--------|-------|
| `/workspace/plan.md` | StateBackend | 线程内临时 |
| `/memories/agent.md` | StoreBackend | 跨线程永久 |

### 规则

- 更长前缀优先匹配（`/memories/projects/` 优先于 `/memories/`）
- `ls`、`glob`、`grep` 聚合所有后端结果
- StoreBackend 路由需要确保 `store` 已提供

### 混合多种后端

```python
from deepagents.backends import CompositeBackend, StateBackend, FilesystemBackend, StoreBackend

composite = CompositeBackend(
    default=StateBackend(),
    routes={
        "/memories/": StoreBackend(
            namespace=lambda rt: (rt.server_info.user.identity,),
        ),
        "/workspace/": FilesystemBackend(
            root_dir="/path/to/project",
            virtual_mode=True,
        ),
    },
)
```

## 自定义后端

实现 `BackendProtocol` 接口：

```python
    BackendProtocol, WriteResult, EditResult,
    LsResult, ReadResult, GrepResult, GlobResult,
    FileInfo, GrepMatch, FileData,
)

class S3Backend(BackendProtocol):
    """基于 S3 的自定义后端"""
    
    def __init__(self, bucket: str, prefix: str = ""):
        self.bucket = bucket
        self.prefix = prefix.rstrip("/")
    
    def _key(self, path: str) -> str:
        return f"{self.prefix}{path}"
    
    def ls(self, path: str) -> LsResult:
        # 列出路径下的文件
        # 返回 LsResult(entries=[FileInfo(path="file.txt", is_dir=False, size=100)])
        ...
    
    def read(self, file_path: str, offset: int = 0, limit: int = 2000) -> ReadResult:
        # 读取文件内容
        # 成功返回 ReadResult(file_data=FileData(content="...", encoding="utf-8"))
        # 失败返回 ReadResult(error="Error: File '/x' not found")
        ...
    
    def write(self, file_path: str, content: str) -> WriteResult:
        # 仅创建，不可覆盖
        # 成功返回 WriteResult(path=file_path)
        # 冲突返回 WriteResult(error="File already exists")
        ...
    
    def edit(self, file_path: str, old_string: str, new_string: str, replace_all: bool = False) -> EditResult:
        # 读取 → 替换 → 写入
        # old_string 必须唯一（除非 replace_all=True）
        # 返回 EditResult(occurrences=1) 或 EditResult(error="String not found")
        ...
    
    def glob(self, pattern: str, path: str = "/") -> GlobResult:
        # 文件模式匹配
        # 返回 GlobResult(matches=[FileInfo(...)])
        ...
    
    def grep(self, pattern: str, path: str | None = None, glob: str | None = None) -> GrepResult:
        # 文件内容搜索
        # 返回 GrepResult(matches=[GrepMatch(path="file.txt", line=5, text="matched")])
        ...
```

### 后端协议参考

| 方法 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `ls(path)` | 路径 | `LsResult` | 列出目录，返回 FileInfo 列表 |
| `read(file_path, offset, limit)` | 路径 + 偏移 | `ReadResult` | 读取文件内容 |
| `write(file_path, content)` | 路径 + 内容 | `WriteResult` | 创建文件（仅创建） |
| `edit(file_path, old, new, replace_all)` | 路径 + 替换 | `EditResult` | 编辑文件 |
| `glob(pattern, path)` | 模式 + 路径 | `GlobResult` | 文件名匹配 |
| `grep(pattern, path, glob)` | 模式 + 可选路径 | `GrepResult` | 文件内容搜索 |

### Policy Wrapper（通用策略包装器）

```python
from deepagents.backends.protocol import (
    BackendProtocol, WriteResult, EditResult,
    LsResult, ReadResult, GrepResult, GlobResult,
)

class PolicyWrapper(BackendProtocol):
    """通用策略包装器，适用于任何后端"""
    
    def __init__(self, inner: BackendProtocol, deny_prefixes: list[str] | None = None):
        self.inner = inner
        self.deny_prefixes = [p if p.endswith("/") else p + "/" for p in (deny_prefixes or [])]
    
    def _deny(self, path: str) -> bool:
        return any(path.startswith(p) for p in self.deny_prefixes)
    
    def ls(self, path: str) -> LsResult:
        return self.inner.ls(path)
    
    def read(self, file_path: str, offset: int = 0, limit: int = 2000) -> ReadResult:
        return self.inner.read(file_path, offset=offset, limit=limit)
    
    def grep(self, pattern: str, path: str | None = None, glob: str | None = None) -> GrepResult:
        return self.inner.grep(pattern, path, glob)
    
    def glob(self, pattern: str, path: str = "/") -> GlobResult:
        return self.inner.glob(pattern, path)
    
    def write(self, file_path: str, content: str) -> WriteResult:
        if self._deny(file_path):
            return WriteResult(error=f"写入不允许: {file_path}")
        return self.inner.write(file_path, content)
    
    def edit(self, file_path: str, old_string: str, new_string: str, replace_all: bool = False) -> EditResult:
        if self._deny(file_path):
            return EditResult(error=f"编辑不允许: {file_path}")
        return self.inner.edit(file_path, old_string, new_string, replace_all)
```

## 后端选择决策树

```
需要跨线程持久化？
├─ 是 → 需要执行 Shell 命令？
│       ├─ 是 → 需要隔离安全？
│       │       ├─ 是 → Sandbox 后端
│       │       └─ 否 → LocalShellBackend
│       └─ 否 → 需要组合不同存储？
│               ├─ 是 → CompositeBackend
│               └─ 否 → StoreBackend
└─ 否 → 需要执行 Shell 命令？
        ├─ 是 → LocalShellBackend
        └─ 否 → FilesystemBackend（或默认 StateBackend）
```

## 小结

| 要点 | 说明 |
|------|------|
| StateBackend | 默认，线程内临时存储 |
| FilesystemBackend | 本地磁盘，需启用 virtual_mode |
| LocalShellBackend | 本地磁盘 + Shell，无沙箱 |
| StoreBackend | 跨线程持久，支持命名空间隔离 |
| CompositeBackend | 组合多种后端，路径路由 |
| 自定义后端 | 实现 BackendProtocol 接口 |
| 后端工厂 | 已废弃，直接传递后端实例 |
