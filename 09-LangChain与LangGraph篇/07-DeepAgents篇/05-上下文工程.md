# 09.07.05 - 上下文工程

> 控制 Agent 的上下文访问和管理方式

## 什么是上下文工程

上下文工程是**在正确的时机以正确的格式提供正确的信息**，让 Agent 能可靠完成任务。

```
上下文工程五维度：
┌──────────────────────────────────────────────┐
│ 1. 输入上下文    → 系统提示词、Memory、Skills  │
│ 2. 运行时上下文  → 用户元数据、API 密钥         │
│ 3. 上下文压缩    → 自动卸载 + 自动摘要         │
│ 4. 上下文隔离    → 子代理隔离重型工作           │
│ 5. 长期记忆      → 跨会话持久化               │
└──────────────────────────────────────────────┘
```

## 输入上下文（Input Context）

启动时提供给 Agent 的静态信息，组成最终系统提示词。

### 系统提示词

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    system_prompt=(
        "你是专业研究助手，专注于科学文献分析。"
        "始终引用来源。使用子代理并行研究不同主题。"
    ),
)
```

### Memory 文件

Memory 文件（`AGENTS.md`）**始终加载**到系统提示词中：

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    memory=["/project/AGENTS.md"],
)
```

> 与 Skills 不同，Memory 没有渐进式加载，每次启动都注入。保持 Memory 精简。

### Skills

Skills **按需加载**：Agent 启动时只读取 frontmatter 描述，判断相关后才加载完整内容：

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    skills=["/skills/research/", "/skills/web-search/"],
)
```

### Skills vs Memory

| | Skills | Memory |
|---|--------|--------|
| **目的** | 按需能力，渐进式加载 | 始终加载的持久上下文 |
| **加载时机** | Agent 判断相关时 | Agent 启动时 |
| **文件** | `SKILL.md` | `AGENTS.md` |
| **适用** | 任务特定指令 | 项目规范、偏好 |
| **Token 占用** | 按需加载，节省 Token | 始终占用 |

## 运行时上下文（Runtime Context）

每次调用时传入的动态配置，不自动包含在模型提示词中。

```python
from dataclasses import dataclass
from deepagents import create_deep_agent
from langchain.tools import tool, ToolRuntime

@dataclass
class Context:
    user_id: str
    api_key: str

@tool
def fetch_user_data(query: str, runtime: ToolRuntime[Context]) -> str:
    """获取当前用户的数据"""
    user_id = runtime.context.user_id
    return f"用户 {user_id} 的数据: {query}"

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[fetch_user_data],
    context_schema=Context,
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "获取我的近期活动"}]},
    context=Context(user_id="user-123", api_key="sk-..."),
)
```

**Runtime 可访问：**
- `runtime.context` — 用户传入的上下文
- `runtime.store` — 长期存储
- `runtime.stream_writer` — 流式输出
- `runtime.execution_info` — thread_id 等执行信息

> 运行时上下文**自动传播到所有子代理**。

## 上下文压缩（Context Compression）

长时间运行的任务会产生大量工具输出和对话历史。Deep Agents 内置两种压缩机制：

### 自动卸载（Offloading）

当工具调用输入或结果超过 **20,000 token** 时：

#### 大型工具输入

```
文件写入操作
  ↓
完整的文件内容保留在对话历史中
  ↓
当上下文达到 85% 窗口限制时
  ↓
自动截断旧的工具调用
  ↓
替换为文件路径指针
```

#### 大型工具结果

```
工具返回大量数据（>20,000 token）
  ↓
自动保存到配置的 Backend
  ↓
替换为文件路径引用 + 前 10 行预览
  ↓
Agent 使用 read_file/grep 按需读取详情
```

### 自动摘要（Summarization）

当上下文达到模型 `max_input_tokens` 的 **85%** 且没有更多可卸载内容时：

```
对话历史过长
  ↓
1. LLM 生成结构化摘要
   - 会话意图
   - 已创建的产物
   - 下一步计划
  ↓
2. 完整原始消息写入文件系统
  ↓
3. 摘要替换历史
  ↓
4. 保留最近 10% 的 token 作为近期上下文
```

**配置参数：**
- 触发阈值：85% 的 `max_input_tokens`
- 保留近期：10% token
- 降级方案：170,000 token 触发 / 保留 6 条消息

### 摘要工具

默认摘要在 85% 时自动触发。可额外启用摘要工具，让 Agent 在任务间隙主动触发：

```python
from deepagents import create_deep_agent
from deepagents.backends import StateBackend
from deepagents.middleware.summarization import (
    create_summarization_tool_middleware,
)

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    middleware=[
        create_summarization_tool_middleware("moonshot:moonshot-v1-8k", StateBackend()),
    ],
)
```

## 上下文隔离（子代理）

子代理是上下文隔离的核心机制：

```
主 Agent 上下文：
┌─────────────────────────────────────────┐
│ 系统提示词                               │
│ 用户输入                                 │
│ write_todos: 分解任务                     │
│ → task("research-agent", "研究量子计算")   │
│ ← 500 字摘要（干净！）                    │
└─────────────────────────────────────────┘

子代理上下文（隔离）：
┌─────────────────────────────────────────┐
│ 子代理系统提示词                          │
│ 研究任务                                 │
│ internet_search × 10（大量结果）          │
│ write_file 保存原始数据                   │
│ 综合分析                                  │
│ → 返回 500 字摘要                         │
└─────────────────────────────────────────┘
```

**隔离的好处：**
- 主 Agent 上下文不被大型工具输出污染
- 每个子代理有独立的新鲜上下文
- 主 Agent 可专注于协调

**最佳实践：**
1. 指示子代理返回简洁结果
2. 使用文件系统存储大型数据
3. 写清晰的子代理 system_prompt

```python
research_subagent = {
    "name": "researcher",
    "description": "研究主题",
    "system_prompt": """你是研究助手。
    重要：只返回必要的摘要（500 字以内）。
    不要包含原始搜索结果或详细工具输出。
    将大型结果保存到 /data/ 目录。""",
    "tools": [internet_search],
}
```

## 长期记忆

### 跨线程持久化

默认文件系统（StateBackend）只在线程内持久。长期记忆需要 `CompositeBackend`：

```python
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    store=InMemoryStore(),
    backend=CompositeBackend(
        default=StateBackend(),
        routes={"/memories/": StoreBackend()},
    ),
    system_prompt="""当用户告诉你偏好时，保存到 /memories/preferences.txt
以便在未来对话中记住。""",
)
```

### 工作流

```
1. 用户："我喜欢详细的解释"
   ↓
2. Agent 调用 write_file → /memories/preferences.txt
   ↓
3. 数据通过 StoreBackend 持久化
   ↓
4. 新线程启动
   ↓
5. MemoryMiddleware 自动加载 /memories/preferences.txt
   ↓
6. Agent "记住" 了用户的偏好
```

## 最佳实践

| # | 实践 | 说明 |
|---|------|------|
| 1 | **精简输入上下文** | Memory 只放始终相关的内容，Skills 放任务特定指令 |
| 2 | **子代理处理重型工作** | 多步骤、输出重的任务委派给子代理 |
| 3 | **使用文件系统** | 大型输出保存到文件，Agent 用 read_file/grep 按需读取 |
| 4 | **记录长期记忆结构** | 在 system_prompt 中说明 /memories/ 的内容和使用方式 |
| 5 | **传递运行时上下文** | 用户元数据、API 密钥等通过 `context` 传入 |

## 小结

| 维度 | 机制 | 何时使用 |
|------|------|---------|
| 输入上下文 | 系统提示词 + Memory + Skills | Agent 启动时 |
| 运行时上下文 | context_schema + invoke(context=) | 每次调用 |
| 上下文压缩 | 自动卸载 + 自动摘要 | 上下文接近窗口限制 |
| 上下文隔离 | 子代理 | 多步骤、大型输出的任务 |
| 长期记忆 | CompositeBackend + StoreBackend | 跨会话持久化 |
