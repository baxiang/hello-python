# 09.07.07 - 权限控制与安全

> 声明式控制 Agent 的文件系统访问权限

## 权限系统概述

权限系统通过声明式规则控制 Agent 的内置文件工具（`ls`、`read_file`、`write_file`、`edit_file`、`glob`、`grep`）能访问哪些路径。

```
Agent 调用 write_file
  ↓
权限检查（第一个匹配的规则）
  ├─ mode="deny" → 拒绝，返回错误
  └─ mode="allow" → 放行，执行写入
```

> ⚠️ 权限仅作用于内置文件工具。自定义工具、沙箱的 `execute` 不受限制。

## 规则结构

每个 `FilesystemPermission` 包含三个字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `operations` | `["read"]` / `["write"]` / `["read", "write"]` | 适用的操作 |
| `paths` | `list[str]` | Glob 路径模式（支持 `**` 递归和 `{a,b}` 多选） |
| `mode` | `"allow"` / `"deny"` | 允许或拒绝（默认 `"allow"`） |

**评估规则：第一个匹配生效（First-Match-Wins）。没有匹配的规则 → 允许。**

## 基本用法

### 只读 Agent

```python
from deepagents import create_deep_agent, FilesystemPermission

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    permissions=[
        FilesystemPermission(
            operations=["write"],
            paths=["/**"],
            mode="deny",
        ),
    ],
)
```

### 隔离到工作目录

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    permissions=[
        # 1. 允许 /workspace/ 下的读写
        FilesystemPermission(
            operations=["read", "write"],
            paths=["/workspace/**"],
            mode="allow",
        ),
        # 2. 拒绝其他所有
        FilesystemPermission(
            operations=["read", "write"],
            paths=["/**"],
            mode="deny",
        ),
    ],
)
```

### 保护特定文件

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    permissions=[
        # 1. 先拒绝敏感文件
        FilesystemPermission(
            operations=["read", "write"],
            paths=["/workspace/.env", "/workspace/examples/**"],
            mode="deny",
        ),
        # 2. 允许 workspace 其他文件
        FilesystemPermission(
            operations=["read", "write"],
            paths=["/workspace/**"],
            mode="allow",
        ),
        # 3. 拒绝其他所有
        FilesystemPermission(
            operations=["read", "write"],
            paths=["/**"],
            mode="deny",
        ),
    ],
)
```

### 只读记忆

```python
from deepagents import create_deep_agent, FilesystemPermission
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(
                namespace=lambda rt: (rt.server_info.user.identity,),
            ),
            "/policies/": StoreBackend(
                namespace=lambda rt: (rt.context.org_id,),
            ),
        },
    ),
    permissions=[
        # 禁止修改记忆和政策文件
        FilesystemPermission(
            operations=["write"],
            paths=["/memories/**", "/policies/**"],
            mode="deny",
        ),
    ],
)
```

### 全部拒绝（白名单模式基础）

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    permissions=[
        FilesystemPermission(
            operations=["read", "write"],
            paths=["/**"],
            mode="deny",
        ),
    ],
)
# 然后按需添加 allow 规则
```

## 规则顺序

> **规则按声明顺序评估，第一个匹配的生效。** 更具体的规则必须放在更宽泛的规则之前。

```python
# ✅ 正确：先拒绝 .env，再允许 workspace
permissions=[
    FilesystemPermission(
        operations=["read", "write"],
        paths=["/workspace/.env"],
        mode="deny",
    ),
    FilesystemPermission(
        operations=["read", "write"],
        paths=["/workspace/**"],
        mode="allow",
    ),
    FilesystemPermission(
        operations=["read", "write"],
        paths=["/**"],
        mode="deny",
    ),
]

# ❌ 错误：/workspace/** 先匹配到 .env，deny 永远不会触发
permissions=[
    FilesystemPermission(
        operations=["read", "write"],
        paths=["/workspace/**"],
        mode="allow",     # .env 被这条规则放行了
    ),
    FilesystemPermission(
        operations=["read", "write"],
        paths=["/workspace/.env"],
        mode="deny",      # 永远不会到达这里
    ),
]
```

## 子代理权限

子代理**默认继承**父代理权限。可通过 `permissions` 参数**完全替换**：

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    permissions=[
        FilesystemPermission(operations=["read", "write"], paths=["/workspace/**"], mode="allow"),
        FilesystemPermission(operations=["read", "write"], paths=["/**"], mode="deny"),
    ],
    subagents=[
        {
            "name": "auditor",
            "description": "只读代码审查",
            "system_prompt": "审查代码问题",
            "permissions": [
                # 完全替换：只读
                FilesystemPermission(operations=["write"], paths=["/**"], mode="deny"),
                FilesystemPermission(operations=["read"], paths=["/workspace/**"], mode="allow"),
                FilesystemPermission(operations=["read"], paths=["/**"], mode="deny"),
            ],
        },
    ],
)
```

## 与 CompositeBackend 的交互

使用沙箱默认后端时，权限路径**必须**限定在已知的路由前缀下：

```python
from deepagents.backends import CompositeBackend

composite = CompositeBackend(
    default=sandbox,                    # 沙箱默认后端
    routes={"/memories/": memories_backend},
)

# ✅ 有效：权限限定在 /memories/ 路由
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=composite,
    permissions=[
        FilesystemPermission(
            operations=["write"],
            paths=["/memories/**"],
            mode="deny",
        ),
    ],
)

# ❌ 无效：/workspace/** 命中沙箱默认路径
# ❌ 无效：/** 覆盖所有路由
```

## 后端策略钩子

对于超出路径匹配的需求（速率限制、审计日志、内容检查），使用后端策略钩子：

### 继承后端类

```python
from deepagents.backends.filesystem import FilesystemBackend
from deepagents.backends.protocol import WriteResult, EditResult

class GuardedBackend(FilesystemBackend):
    def __init__(self, *, deny_prefixes: list[str], **kwargs):
        super().__init__(**kwargs)
        self.deny_prefixes = [p if p.endswith("/") else p + "/" for p in deny_prefixes]

    def write(self, file_path: str, content: str) -> WriteResult:
        if any(file_path.startswith(p) for p in self.deny_prefixes):
            return WriteResult(error=f"不允许写入: {file_path}")
        return super().write(file_path, content)

    def edit(self, file_path: str, old_string: str, new_string: str, replace_all: bool = False) -> EditResult:
        if any(file_path.startswith(p) for p in deny_prefixes):
            return EditResult(error=f"不允许编辑: {file_path}")
        return super().edit(file_path, old_string, new_string, replace_all)
```

### 通用包装器

```python
from deepagents.backends.protocol import (
    BackendProtocol, WriteResult, EditResult,
    LsResult, ReadResult, GrepResult, GlobResult,
)

class PolicyWrapper(BackendProtocol):
    def __init__(self, inner: BackendProtocol, deny_prefixes: list[str] | None = None):
        self.inner = inner
        self.deny_prefixes = [p if p.endswith("/") else p + "/" for p in (deny_prefixes or [])]

    def _deny(self, path: str) -> bool:
        return any(path.startswith(p) for p in self.deny_prefixes)

    def ls(self, path: str) -> LsResult:
        return self.inner.ls(path)

    def read(self, file_path: str, offset: int = 0, limit: int = 2000) -> ReadResult:
        return self.inner.read(file_path, offset=offset, limit=limit)

    def grep(self, pattern: str, path: str | None = None, glob: str | None = None) -> GrepResult:
        return self.inner.grep(pattern, path, glob)

    def glob(self, pattern: str, path: str = "/") -> GlobResult:
        return self.inner.glob(pattern, path)

    def write(self, file_path: str, content: str) -> WriteResult:
        if self._deny(file_path):
            return WriteResult(error=f"不允许写入: {file_path}")
        return self.inner.write(file_path, content)

    def edit(self, file_path: str, old_string: str, new_string: str, replace_all: bool = False) -> EditResult:
        if self._deny(file_path):
            return EditResult(error=f"不允许编辑: {file_path}")
        return self.inner.edit(file_path, old_string, new_string, replace_all)
```

## 权限 vs 后端钩子

| | 权限（permissions） | 后端钩子（Policy Hooks） |
|---|---|---|
| **适用** | 路径级允许/拒绝规则 | 自定义验证逻辑 |
| **控制** | 内置文件工具 | 内置 + 自定义工具 |
| **功能** | allow / deny | 速率限制、审计日志、内容检查 |
| **配置** | `FilesystemPermission` 列表 | 继承或包装后端类 |

## 小结

| 要点 | 说明 |
|------|------|
| 规则评估 | 第一个匹配生效（First-Match-Wins） |
| 默认行为 | 没有匹配的规则 → 允许 |
| 规则顺序 | 更具体的规则放在前面 |
| 子代理 | 默认继承，可完全替换 |
| 沙箱后端 | 权限路径必须限定在已知路由 |
| 扩展控制 | 后端策略钩子用于自定义验证 |
