# 09.07.01 - Deep Agents 概述与快速入门

> 构建能规划、使用子代理、管理文件系统的深度 Agent

## Deep Agents 是什么

`deepagents` 是 LangChain 生态中的独立库，构建在 [LangChain](https://docs.langchain.com/oss/python/langchain/overview) 核心构建块和 [LangGraph](https://docs.langchain.com/oss/python/langgraph/overview) 运行时之上。它提供了一个 **"Agent Harness"（代理容器）**，在标准工具调用循环的基础上，内置了大量开箱即用的能力。

### 三层架构

```
┌────────────────────────────────────────────────────────┐
│  LangChain     → 核心构建块                              │
│                  模型接口、工具、中间件、消息系统           │
│                                                        │
│  LangGraph     → 执行运行时                              │
│                  持久化执行、流式、人类审核、检查点          │
│                                                        │
│  Deep Agents   → Agent Harness（完整方案）               │
│                  任务规划、文件系统、子代理、长期记忆        │
│                  沙箱执行、权限控制、Skills                │
└────────────────────────────────────────────────────────┘
```

### 与普通 Agent 的区别

| 特性 | `create_agent` | `create_deep_agent` |
|------|---------------|--------------------|
| 工具调用循环 | ✅ | ✅ |
| 任务规划（write_todos） | ❌ | ✅ 内置 |
| 虚拟文件系统（ls/read/write/edit） | ❌ | ✅ 内置 |
| 子代理（task 工具） | 需手动搭建 | ✅ 内置 |
| 上下文自动卸载 | ❌ | ✅ 超过 20K token 自动存储 |
| 自动摘要 | ❌ | ✅ 85% 窗口限制触发 |
| 沙箱执行 | ❌ | ✅ 支持 |
| 权限控制 | ❌ | ✅ 声明式规则 |
| Skills 系统 | ❌ | ✅ 渐进式加载 |

### 何时使用

| 场景 | 推荐方案 |
|------|---------|
| 简单单次问答 | `model.invoke()` |
| 带工具的 Agent | `create_agent` |
| 自定义工作流 | LangGraph 原生 API |
| **复杂多步骤任务** | ✅ `create_deep_agent` |
| **需要管理大量上下文** | ✅ `create_deep_agent` |
| **需要子代理隔离** | ✅ `create_deep_agent` |
| **需要沙箱代码执行** | ✅ `create_deep_agent` |

## 快速入门

### 最简示例

```python
from deepagents import create_deep_agent

def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    return f"{city} 天气晴朗，25°C"

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[get_weather],
    system_prompt="你是一个友好的助手",
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "北京天气怎么样？"}]}
)
print(result["messages"][-1].content)
```

**发生了什么：**
1. Agent 收到用户问题
2. LLM 决定调用 `get_weather` 工具
3. 工具返回结果
4. LLM 生成最终回复

### 研究 Agent 示例

```python
import os
from typing import Literal
from deepagents import create_deep_agent

# 模拟搜索工具
def internet_search(
    query: str,
    max_results: int = 5,
    topic: Literal["general", "news", "finance"] = "general",
) -> list[dict]:
    """运行网络搜索获取相关信息"""
    return [
        {
            "title": f"搜索结果 {i + 1}: {query}",
            "content": f"关于「{query}」的相关信息（类型：{topic}）",
        }
        for i in range(max_results)
    ]

# 系统提示词
research_prompt = """\
你是专业研究员。你的任务：
1. 使用 write_todos 分解研究任务
2. 使用 internet_search 收集信息
3. 将大型结果保存到文件系统（/data/ 目录）
4. 撰写结构化研究报告
"""

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[internet_search],
    system_prompt=research_prompt,
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "什么是 LangGraph？"}]}
)
print(result["messages"][-1].content)
```

### Agent 自动执行流程

Deep Agent 收到任务后会自动执行以下流程：

```
用户输入: "什么是 LangGraph？"
    ↓
┌─ write_todos ────────────────────────────┐
│ 1. 搜索 LangGraph 核心概念               │
│ 2. 搜索 LangGraph 与 LangChain 的关系     │
│ 3. 搜索实际应用案例                       │
│ 4. 撰写研究报告                           │
└──────────────────────────────────────────┘
    ↓
┌─ internet_search ────────────────────────┐
│ 查询: "LangGraph 核心概念"               │
│ 返回 5 条结果                             │
│ 如果结果过大 → 自动保存到 /data/          │
└──────────────────────────────────────────┘
    ↓
┌─ 自动上下文管理 ─────────────────────────┐
│ 超过 20,000 token → 卸载到文件系统       │
│ 超过 85% 窗口 → 自动摘要 + 保留最近消息   │
└──────────────────────────────────────────┘
    ↓
┌─ 撰写报告 ───────────────────────────────┐
│ 综合所有发现，生成结构化报告               │
│ 保存到 /data/research_report.md           │
└──────────────────────────────────────────┘
    ↓
最终回复用户
```

## create_deep_agent 参数详解

```python
create_deep_agent(
    # 核心参数
    model: str | BaseChatModel | None = None,     # 模型
    tools: Sequence[BaseTool | Callable] | None = None,  # 工具
    *,
    system_prompt: str | SystemMessage | None = None,    # 系统提示词
    middleware: Sequence[AgentMiddleware] = (),           # 中间件
    subagents: Sequence[SubAgent | CompiledSubAgent] | None = None,  # 子代理
    skills: list[str] | None = None,                      # 技能路径
    memory: list[str] | None = None,                      # 记忆文件路径
    response_format: ResponseFormat | type | dict | None = None,  # 结构化输出
    backend: BackendProtocol | BackendFactory | None = None,      # 文件系统后端
    interrupt_on: dict[str, bool | InterruptOnConfig] | None = None,  # 人类审核
    store: BaseStore | None = None,                       # 长期存储
    checkpointer: BaseCheckpointer | None = None,         # 检查点
    ...
) -> CompiledStateGraph
```

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `model` | `str` 或模型实例 | 模型标识（`provider:model` 格式）或模型对象 |
| `tools` | `list` | 自定义工具列表 |
| `system_prompt` | `str` 或 `SystemMessage` | 自定义系统提示词 |
| `middleware` | `list[AgentMiddleware]` | 中间件（扩展功能） |
| `subagents` | `list[SubAgent]` | 子代理配置 |
| `skills` | `list[str]` | 技能文件路径 |
| `memory` | `list[str]` | 记忆文件路径（如 `AGENTS.md`） |
| `backend` | `BackendProtocol` | 文件系统后端 |
| `interrupt_on` | `dict` | 人类审核配置 |
| `store` | `BaseStore` | 跨线程存储 |
| `checkpointer` | `BaseCheckpointer` | 状态检查点 |

## 内置能力

Deep Agents 默认包含以下内置中间件和工具：

### 内置中间件

| 中间件 | 功能 | 触发条件 |
|--------|------|---------|
| `TodoListMiddleware` | 管理待办任务清单 | 始终启用 |
| `FilesystemMiddleware` | 虚拟文件系统操作 | 始终启用 |
| `SubAgentMiddleware` | 子代理创建和协调 | 始终启用 |
| `SummarizationMiddleware` | 自动摘要长对话 | 85% 窗口限制触发 |
| `PatchToolCallsMiddleware` | 修复中断的工具调用 | 始终启用 |
| `AnthropicPromptCachingMiddleware` | Anthropic 缓存优化 | 仅 Anthropic 模型 |
| `MemoryMiddleware` | 跨会话记忆加载 | 配置 `memory` 时启用 |
| `SkillsMiddleware` | 按需 Skills 加载 | 配置 `skills` 时启用 |
| `HumanInTheLoopMiddleware` | 人类审核 | 配置 `interrupt_on` 时启用 |

### 内置工具

| 工具 | 功能 | 适用场景 |
|------|------|---------|
| `write_todos` | 创建和管理任务清单 | 复杂任务分解 |
| `ls` | 列出目录内容 | 文件系统浏览 |
| `read_file` | 读取文件（支持图片） | 查看文件内容 |
| `write_file` | 创建文件 | 保存数据/报告 |
| `edit_file` | 编辑文件内容 | 修改已有文件 |
| `glob` | 文件模式匹配搜索 | 批量查找文件 |
| `grep` | 文件内容搜索 | 搜索关键词 |
| `task` | 委派任务给子代理 | 上下文隔离 |
| `execute` | 执行 shell 命令 | 仅沙箱后端可用 |

## 流式输出

Deep Agents 内置流式支持，可实时观察 Agent 执行过程：

```python
# 流式查看 Agent 执行
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "研究 Python 最佳实践"}]},
    stream_mode="values",
):
    latest = chunk["messages"][-1]
    if latest.content:
        print(f"Agent: {latest.content}", end="", flush=True)
    elif latest.tool_calls:
        print(f"\n[调用工具: {[tc['name'] for tc in latest.tool_calls]}]")
```

### 子代理流式

```python
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "复杂研究任务"}]},
    stream_mode=["messages", "updates"],
    subgraphs=True,  # 包含子代理信息
    version="v2",
):
    if chunk["type"] == "messages":
        token, metadata = chunk["data"]
        agent_name = metadata.get("lc_agent_name")  # 识别是哪个 Agent
        if agent_name:
            print(f"🤖 {agent_name}: {token.text}")
```

## 结构化输出

Deep Agents 支持让 Agent 返回结构化的 Pydantic 对象：

```python
from pydantic import BaseModel, Field
from deepagents import create_deep_agent

class WeatherReport(BaseModel):
    location: str = Field(description="地点")
    temperature: float = Field(description="温度(°C)")
    condition: str = Field(description="天气状况")
    humidity: int = Field(description="湿度百分比")

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=WeatherReport,
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "北京天气怎么样？"}]
})
print(result["structured_response"])
# WeatherReport(location='北京', temperature=25.0, condition='晴朗', humidity=48)
```

## 连接韧性

LangChain 模型自动重试失败的 API 请求，使用指数退避策略：

| 错误类型 | 是否重试 | 说明 |
|---------|---------|------|
| 网络错误 | ✅ | 连接超时、DNS 解析失败 |
| 429（速率限制） | ✅ | API 频率限制 |
| 5xx（服务器错误） | ✅ | 服务端内部错误 |
| 401（未授权） | ❌ | API 密钥无效 |
| 404（未找到） | ❌ | 模型不存在 |

**默认重试 6 次**，可通过 `max_retries` 调整：

```python
from langchain.chat_models import init_chat_model
from deepagents import create_deep_agent

agent = create_deep_agent(
    model=init_chat_model(
        model="moonshot:moonshot-v1-8k",
        max_retries=10,  # 网络不稳定时增加到 10-15
        timeout=120,     # 增加超时时间
    ),
)
```
