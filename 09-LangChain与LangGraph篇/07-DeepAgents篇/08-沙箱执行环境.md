# 09.07.08 - 沙箱执行环境

> 在隔离环境中安全执行代码

## 为什么需要沙箱

```
没有沙箱：
┌──────────────────────────────────────┐
│  Agent 直接在主机执行                 │
│  - 可读取 .env、credentials           │
│  - 可执行任意 shell 命令              │
│  - 文件系统修改不可逆                 │
│  - 可能安装恶意包                     │
└──────────────────────────────────────┘

有沙箱：
┌──────────────────────────────────────┐
│  Agent 在隔离沙箱执行                 │
│  - 无法访问主机文件                   │
│  - 命令在隔离环境执行                 │
│  - 可随时销毁重建                     │
│  - 不影响主机系统                     │
└──────────────────────────────────────┘
```

## 沙箱 vs 其他后端

| | 普通后端 | 沙箱后端 |
|---|--------|---------|
| 文件工具 | ✅ | ✅ |
| `execute` 工具 | ❌ | ✅ |
| 隔离性 | ❌ | ✅ |
| 适用场景 | 本地开发 | 安全代码执行 |

## 支持的沙箱提供商

| 提供商 | 包名 | 说明 |
|--------|------|------|
| Modal | `langchain-modal` | 云端沙箱 |
| Daytona | `langchain-daytona` | 开发环境沙箱 |
| Runloop | `langchain-runloop` | 远程开发沙箱 |
| LangSmith | `langsmith[sandbox]` | 私有 Beta |

## 基本使用

### Daytona

```bash
uv add langchain-daytona
```

```python
from daytona import Daytona
from deepagents import create_deep_agent
from langchain_daytona import DaytonaSandbox

sandbox = Daytona().create()
backend = DaytonaSandbox(sandbox=sandbox)

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    system_prompt="你是 Python 编码助手，可以使用沙箱。",
    backend=backend,
)

try:
    result = agent.invoke(
        {"messages": [{"role": "user", "content": "创建 Python 包并运行 pytest"}]}
    )
    print(result["messages"][-1].content)
finally:
    sandbox.stop()
```

## 沙箱生命周期

### 线程级（默认）

每个对话获得独立的沙箱。沙箱创建后在同一 thread 复用。对话清理或 TTL 到期后销毁。

**适用：** 数据分析 Bot，每次对话都从干净环境开始。

### 助手级

同一助手的所有线程共享一个沙箱。文件、依赖、Git 仓库跨对话持久。

**适用：** 编码助手，维护长期工作区。

> ⚠️ 助手级沙箱会随时间积累文件、包。配置 TTL 或定期重置。

### 按对话生命周期

```python
from daytona import CreateSandboxFromSnapshotParams, Daytona
from deepagents import create_deep_agent
from langchain_daytona import DaytonaSandbox
from langchain_core.utils.uuid import uuid7

client = Daytona()
thread_id = str(uuid7())

# 按 thread_id 查找或创建
try:
    sandbox = client.find_one(labels={"thread_id": thread_id})
except Exception:
    params = CreateSandboxFromSnapshotParams(
        labels={"thread_id": thread_id},
        auto_delete_interval=3600,  # 1 小时空闲后自动删除
    )
    sandbox = client.create(params)

backend = DaytonaSandbox(sandbox=sandbox)
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    backend=backend,
    system_prompt="你是编码助手。",
)

try:
    result = agent.invoke(
        {"messages": [{"role": "user", "content": "创建 hello.py 并运行"}]},
        config={"configurable": {"thread_id": thread_id}},
    )
except Exception:
    client.delete(sandbox)
    raise
```

## 两种集成模式

### Agent 在沙箱内

Agent 运行在沙箱内部，通过网络通信。

**优点：**
- 接近本地开发体验
- Agent 与环境紧密耦合

**缺点：**
- API 密钥必须在沙箱内（安全风险）
- 更新需要重建镜像
- 需要 WebSocket/HTTP 通信基础设施

### 沙箱作为工具

Agent 在主机运行，通过 API 调用沙箱工具。

**优点：**
- 即时更新 Agent 代码
- API 密钥留在主机
- 沙箱故障不影响 Agent 状态
- 可并行多个沙箱
- 按执行时间付费

**缺点：**
- 每次执行有网络延迟

> 本文档使用"沙箱作为工具"模式。

## 沙箱工作原理

### 隔离边界

| 隔离 | 说明 |
|------|------|
| 文件系统 | Agent 无法读取主机文件 |
| 环境变量 | Agent 无法访问主机环境变量 |
| 进程 | Agent 无法干扰主机进程 |

| **不隔离** | 说明 |
|---------|------|
| 上下文注入 | 控制输入的攻击者可指令 Agent 在沙箱内执行任意命令 |
| 网络外泄 | 未限制网络时，Agent 可通过 HTTP/DNS 外泄数据 |

### execute 方法

沙箱后端只需实现 `execute()`，其他文件操作都构建在其上：

```python
# 直接调用
result = backend.execute("python --version")
print(result.output)
# Python 3.11.0
# [Command succeeded with exit code 0]

# 命令失败
result = backend.execute("foobar")
print(result.output)
# bash: foobar: command not found
# [Command failed with exit code 127]
```

### 文件传输 API

```
Agent 文件系统工具                    应用代码
(read_file, write_file, execute)    (upload_files, download_files)
         ↓                                  ↓
     沙箱文件系统 ◄────── 提供者原生 API ─────► 主机
```

| 用途 | 方式 |
|------|------|
| Agent 在沙箱内读写 | `read_file`、`write_file`、`execute` |
| 预填沙箱（应用侧） | `backend.upload_files()` |
| 获取产物（应用侧） | `backend.download_files()` |

```python
# 预填沙箱
backend.upload_files([
    ("/src/index.py", b"print('Hello')\n"),
    ("/pyproject.toml", b"[project]\nname = 'my-app'\n"),
])

# Agent 工作...

# 获取产物
results = backend.download_files(["/src/index.py", "/output.txt"])
for result in results:
    if result.content is not None:
        print(f"{result.path}: {result.content.decode()}")
```

## 安全注意事项

> ⚠️ **永远不要在沙箱中放置密钥！**
> 
> API 密钥、Token、数据库凭证等一旦注入沙箱，都可能被上下文注入攻击的 Agent 读取并外泄。

### 安全的密钥处理方式

**推荐：密钥留在沙箱外**
```python
# 定义沙箱外的工具处理认证
@tool
def call_api(endpoint: str) -> str:
    """调用 API（密钥在主机）"""
    headers = {"Authorization": f"Bearer {os.environ['SECRET_KEY']}"}
    response = requests.get(endpoint, headers=headers)
    return response.text
```

**不推荐：注入密钥到沙箱**
```python
# ❌ 危险：密钥在沙箱内可被读取
backend.execute("export SECRET_KEY=sk-123 && python run.py")
```

### 通用最佳实践

- 审查沙箱输出再执行
- 不需要时阻止网络访问
- 使用中间件过滤敏感模式
- 将沙箱内产生的一切视为不可信输入

## 小结

| 要点 | 说明 |
|------|------|
| 沙箱提供 | 文件系统工具 + `execute` 工具 |
| 提供商 | Modal、Daytona、Runloop、LangSmith |
| 生命周期 | 线程级（默认）或助手级 |
| 文件传输 | `upload_files()` / `download_files()` |
| 安全 | 不在沙箱内放密钥，使用外部工具认证 |
