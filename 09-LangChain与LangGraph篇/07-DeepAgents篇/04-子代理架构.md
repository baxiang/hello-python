# 09.07.04 - 子代理架构

> 使用子代理隔离上下文、委派专业任务

## 为什么需要子代理

### 上下文膨胀问题

```
没有子代理：
┌──────────────────────────────────────────┐
│  用户问题                                  │
│    ↓                                     │
│  Agent 搜索 1 → 结果 5000 token           │
│  Agent 搜索 2 → 结果 5000 token           │
│  Agent 搜索 3 → 结果 5000 token           │
│  Agent 搜索 4 → 结果 5000 token           │
│  Agent 搜索 5 → 结果 5000 token           │
│    ↓                                     │
│  25000 token 中间结果挤占上下文窗口        │
│  最终报告质量下降                           │
└──────────────────────────────────────────┘

使用子代理：
┌──────────────────────────────────────────┐
│  用户问题                                  │
│    ↓                                     │
│  主 Agent → task("研究量子计算")           │
│    ↓                                     │
│  ┌─ 子代理（独立上下文） ─────────────────┐ │
│  │ 搜索 1-5（全部在子代理内部）             │ │
│  │ 分析数据                                │ │
│  │ 返回 500 字摘要                         │ │
│  └───────────────────────────────────────┘ │
│    ↓                                     │
│  主 Agent 只收到 500 字摘要                │
│  上下文干净，专注于协调                     │
└──────────────────────────────────────────┘
```

### 何时使用

| 场景 | 推荐 | 原因 |
|------|------|------|
| 多步骤复杂任务 | ✅ 子代理 | 隔离中间结果 |
| 需要专业指令的领域 | ✅ 子代理 | 独立系统提示 |
| 不同模型能力组合 | ✅ 子代理 | 子代理可用不同模型 |
| 保持主 Agent 上下文干净 | ✅ 子代理 | 大型工具输出不污染主上下文 |
| 简单单次查询 | ❌ | 开销大于收益 |
| 需要保留中间状态 | ❌ | 子代理只返回最终结果 |

## 子代理配置

### SubAgent（字典式定义）

```python
import os
from typing import Literal
from deepagents import create_deep_agent

# 搜索工具
def internet_search(
    query: str,
    max_results: int = 5,
    topic: Literal["general", "news", "finance"] = "general",
) -> list[dict]:
    """运行网络搜索"""
    return [
        {"title": f"结果{i+1}: {query}", "content": f"..."}
        for i in range(max_results)
    ]

# 定义子代理
research_subagent = {
    "name": "research-agent",
    "description": "深度研究问题，使用网络搜索收集信息并综合分析",
    "system_prompt": """你是专业研究员。
1. 分解研究问题为可搜索的关键词
2. 使用 internet_search 收集信息
3. 综合发现为简洁摘要（500 字以内）
4. 引用来源
""",
    "tools": [internet_search],
    "model": "deepseek:deepseek-chat",  # 可选：覆盖主 Agent 模型
}

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    subagents=[research_subagent],
)
```

### 参数详解

| 参数 | 类型 | 必需 | 说明 | 继承关系 |
|------|------|------|------|---------|
| `name` | `str` | ✅ | 唯一标识符，用于 `task` 工具调用 | 必须唯一 |
| `description` | `str` | ✅ | 功能描述，主 Agent 据此决定何时委派 | — |
| `system_prompt` | `str` | ✅ | 子代理指令 | ❌ 不继承 |
| `tools` | `list` | ✅ | 可用工具 | ❌ 不继承 |
| `model` | `str`/模型 | ❌ | 使用的模型 | ✅ 默认继承 |
| `middleware` | `list` | ❌ | 中间件 | ❌ 不继承 |
| `interrupt_on` | `dict` | ❌ | 人类审核 | ✅ 默认继承 |
| `skills` | `list[str]` | ❌ | 技能路径 | ❌ 不继承 |
| `permissions` | `list` | ❌ | 权限规则 | ✅ 默认继承 |

### CompiledSubAgent（编译图式定义）

适用于复杂自定义工作流：

```python
from deepagents import create_deep_agent, CompiledSubAgent
from langchain.agents import create_agent

# 创建自定义 LangGraph 图
custom_graph = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[internet_search],
    system_prompt="专业数据分析助手。分析数据并返回结构化结果。",
)

# 作为子代理
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    subagents=[
        CompiledSubAgent(
            name="data-analyzer",
            description="专业数据分析，处理复杂数据集",
            runnable=custom_graph,  # 必须是 .compile() 后的图
        ),
    ],
)
```

## 通用子代理（General-Purpose）

Deep Agents **始终**内置一个 `general-purpose` 子代理：

| 特性 | 说明 |
|------|------|
| 系统提示 | 与主 Agent 相同 |
| 工具 | 与主 Agent 相同 |
| 模型 | 与主 Agent 相同（可覆盖） |
| Skills | 自动继承 |

### 覆盖通用子代理

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[internet_search],
    subagents=[
        {
            "name": "general-purpose",  # 替换默认
            "description": "通用子代理，处理复杂多步骤任务",
            "system_prompt": "你是通用助手。返回简洁结果。",
            "tools": [internet_search],
            "model": "deepseek:deepseek-chat",  # 使用不同模型
        },
    ],
)
```

### 何时使用

主 Agent 委派不需要专业行为的复杂任务：

```python
# 主 Agent 委派给通用子代理
# task(name="general-purpose", task="搜索量子计算最新进展并总结")
# 子代理内部执行所有搜索，只返回摘要
```

## Skills 继承

| 子代理类型 | Skills 继承 | 说明 |
|-----------|-----------|------|
| 通用子代理 | ✅ 自动继承 | 与主 Agent 共享 skills |
| 自定义子代理 | ❌ 不继承 | 需用 `skills` 参数单独配置 |

```python
research_subagent = {
    "name": "researcher",
    "description": "研究助手",
    "system_prompt": "你是研究员。",
    "tools": [internet_search],
    "skills": ["/skills/research/", "/skills/web-search/"],  # 自定义技能
}

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    skills=["/skills/main/"],  # 主 Agent 和通用子代理用这些
    subagents=[research_subagent],  # researcher 只用 /skills/research/
)
```

## 子代理流式输出

```python
# 创建带名字的 Agent
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    subagents=[research_subagent],
    name="main-agent",
)

# 流式追踪，识别数据来源
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "研究 Python 最佳实践"}]},
    stream_mode=["messages", "updates"],
    subgraphs=True,
    version="v2",
):
    if chunk["type"] == "messages":
        token, metadata = chunk["data"]
        agent_name = metadata.get("lc_agent_name")
        if agent_name:
            print(f"🤖 {agent_name}: {token.text}")
```

## 子代理上下文管理

### 运行时上下文传播

```python
from dataclasses import dataclass
from deepagents import create_deep_agent
from langchain.messages import HumanMessage
from langchain.tools import tool, ToolRuntime

@dataclass
class Context:
    user_id: str
    session_id: str

@tool
def get_user_data(query: str, runtime: ToolRuntime[Context]) -> str:
    """获取当前用户的数据"""
    user_id = runtime.context.user_id  # 子代理中也可访问
    return f"用户 {user_id} 的数据: {query}"

research_subagent = {
    "name": "researcher",
    "description": "为当前用户进行研究",
    "system_prompt": "你是研究助手。",
    "tools": [get_user_data],
}

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    subagents=[research_subagent],
    context_schema=Context,
)

# context 自动传播到子代理
result = agent.invoke(
    {"messages": [HumanMessage("查找我的研究记录")]},
    context=Context(user_id="user-123", session_id="abc"),
)
```

### 识别调用 Agent

```python
from langchain.tools import tool, ToolRuntime

@tool
def shared_lookup(query: str, runtime: ToolRuntime) -> str:
    """共享查询工具"""
    agent_name = runtime.config.get("metadata", {}).get("lc_agent_name")
    if agent_name == "researcher":
        return strict_lookup(query)
    return general_lookup(query)
```

## 多子代理协作模式

```python
from deepagents import create_deep_agent

subagents = [
    {
        "name": "data-collector",
        "description": "从各种来源收集原始数据",
        "system_prompt": "收集全面的主题数据",
        "tools": [web_search, api_call, database_query],
    },
    {
        "name": "data-analyzer",
        "description": "分析收集的数据提取关键洞察",
        "system_prompt": "分析数据并提取关键洞察",
        "tools": [statistical_analysis],
    },
    {
        "name": "report-writer",
        "description": "基于分析结果撰写专业报告",
        "system_prompt": "从洞察创建专业报告",
        "tools": [format_document],
    },
]

agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    system_prompt="你协调数据分析和报告。使用子代理处理专业任务。",
    subagents=subagents,
)
```

**工作流：**
1. 主 Agent 创建高级计划
2. 委派数据收集给 `data-collector`
3. 结果传递给 `data-analyzer`
4. 洞察发送给 `report-writer`
5. 编译最终输出

## 故障排查

### 子代理没被调用

**问题：** 主 Agent 自己做工作而不是委派。

**解决：**
1. 使 description 更具体
2. 在 system_prompt 中指示委派

```python
agent = create_deep_agent(
    model="moonshot:moonshot-v1-8k",
    system_prompt="""...

重要：对于复杂任务，使用 task() 工具委派给子代理。
这能保持你的上下文干净并提升结果质量。""",
    subagents=[...]
)
```

### 上下文仍然膨胀

**解决：**
1. 指示子代理返回简洁结果
2. 使用文件系统存储大数据

```python
system_prompt="""...

重要：返回必要的摘要。
不要包含原始数据、中间搜索结果或详细工具输出。
响应控制在 500 字以内。"""
```

### 选错了子代理

**解决：** 在 description 中明确区分：

```python
subagents = [
    {
        "name": "quick-researcher",
        "description": "用于简单快速的研究问题，需要 1-2 次搜索。用于基本事实或定义。",
    },
    {
        "name": "deep-researcher",
        "description": "用于复杂、深入的研究，需要多次搜索、综合和分析。用于全面报告。",
    }
]
```

## 小结

| 要点 | 说明 |
|------|------|
| 上下文隔离 | 子代理在独立上下文中工作，主 Agent 只收到最终结果 |
| 定义方式 | 字典式 SubAgent 或 CompiledSubAgent |
| 通用子代理 | 始终可用，可自定义覆盖 |
| Skills 继承 | 通用子代理自动继承，自定义子代理不继承 |
| 上下文传播 | 运行时 context 自动传递给所有子代理 |
| 描述清晰 | description 决定主 Agent 何时委派 |
| 工具最小化 | 只给子代理需要的工具 |
