# 09.03.02 - 工具上下文

> ToolRuntime：工具执行时的上下文信息，提供状态访问、存储、流式输出等能力

## 为什么需要工具上下文

工具执行时需要访问运行时信息：

```
工具执行的上下文需求：
┌────────────────────────────────────────────────┐
│  工具函数                                      │
│                                                 │
│  需要访问：                                     │
│  ├── 当前对话状态（消息历史）                   │
│  ├── 长期存储（用户偏好）                       │
│  ├── 流式输出（实时反馈）                       │
│  ├── 执行信息（工具调用ID）                     │
│  └── 服务器信息（MCP场景）                      │
│                                                 │
│  ToolRuntime 提供所有这些能力                   │
└────────────────────────────────────────────────┘
```

## ToolRuntime 概览

```python
from langchain.tools import tool, ToolRuntime

@tool
def my_tool(query: str, runtime: ToolRuntime) -> str:
    """使用运行时上下文的工具"""
    
    # 可用的属性
    runtime.state          # 当前状态（短期记忆）
    runtime.context        # 运行时配置
    runtime.store          # 长期存储
    runtime.stream_writer  # 流式输出写入器
    runtime.execution_info # 执行信息
    runtime.server_info    # 服务器信息（MCP）
    
    return "结果"
```

**ToolRuntime 属性表：**

| 属性 | 类型 | 用途 |
|------|------|------|
| `state` | `dict` | 访问和更新 Agent 的当前状态 |
| `context` | `Any` | 运行时上下文（由 context_schema 定义） |
| `store` | `BaseStore` | 长期存储，跨会话持久化 |
| `stream_writer` | `Callable` | 向客户端发送流式事件 |
| `execution_info` | `ToolExecutionInfo` | 当前工具执行的元数据 |
| `server_info` | `MCPServerInfo` | MCP 服务器信息 |
| `tool_call_id` | `str` | 当前工具调用的唯一标识 |

## 访问 State

### 读取状态

```python
from langchain.tools import tool, ToolRuntime
from langchain.messages import HumanMessage, AIMessage

@tool
def get_conversation_summary(runtime: ToolRuntime) -> str:
    """获取当前对话摘要"""
    messages = runtime.state.get("messages", [])
    
    user_count = sum(1 for m in messages if isinstance(m, HumanMessage))
    ai_count = sum(1 for m in messages if isinstance(m, AIMessage))
    
    return f"对话历史：{user_count} 轮用户消息，{ai_count} 轮 AI 回复"
```

### 读取最后一条用户消息

```python
@tool
def get_last_user_message(runtime: ToolRuntime) -> str:
    """获取用户最后一条消息"""
    messages = runtime.state.get("messages", [])
    
    # 倒序查找第一条 HumanMessage
    for message in reversed(messages):
        if isinstance(message, HumanMessage):
            if isinstance(message.content, str):
                return message.content
            else:
                # 多模态内容
                for part in message.content:
                    if part.get("type") == "text":
                        return part["text"]
    
    return "未找到用户消息"
```

@tool
def get_time(runtime: ToolRuntime) -> str:

## 访问 Store

### Store 是什么

Store 是 LangGraph 提供的长期存储接口：

```
Store 结构：
┌────────────────────────────────────────────┐
│  BaseStore                                 │
│                                             │
│  操作：                                     │
│  ├── put(namespace, key, value)   # 写入   │
│  ├── get(namespace, key)          # 读取   │
│  ├── delete(namespace, key)       # 删除   │
│  └── search(namespace, prefix)    # 搜索   │
│                                             │
│  命名空间：用于数据分组                     │
│  键：唯一标识                               │
│  值：任意可序列化数据                       │
└────────────────────────────────────────────┘
```

---

### 基本操作

```python
@tool
def save_user_preference(
    key: str,
    value: str,
    runtime: ToolRuntime,
) -> str:
    """保存用户偏好设置"""
    store = runtime.store
    user_id = runtime.context.user_id
    
    # 写入：命名空间 + 键
    store.put(("users", user_id, "preferences"), key, value)
    
    return f"偏好 '{key}' 已保存"
```

---

### 读取操作

```python
@tool
def get_user_preference(key: str, runtime: ToolRuntime) -> str:
    """获取用户偏好设置"""
    store = runtime.store
    user_id = runtime.context.user_id
    
    # 读取
    result = store.get(("users", user_id, "preferences"), key)
    
    if result is not None:
        return result.value
    return f"未找到偏好设置 '{key}'"
```

---

### 搜索操作

```python
@tool
def list_user_preferences(runtime: ToolRuntime) -> str:
    """列出所有用户偏好"""
    store = runtime.store
    user_id = runtime.context.user_id
    
    # 搜索指定命名空间下的所有键
    results = store.search(
        ("users", user_id, "preferences"),
        prefix="",
    )
    
    preferences = []
    for item in results:
        preferences.append(f"{item.key}: {item.value}")
    
    if preferences:
        return "用户偏好:\n" + "\n".join(preferences)
    return "暂无偏好设置"
```

### 命名空间设计

```python
# 推荐的命名空间结构：
#
# ("users", user_id, "preferences")  → 用户偏好
# ("users", user_id, "history")      → 用户历史
# ("sessions", session_id, "data")   → 会话数据
# ("global", "config")               → 全局配置
#
# 示例代码：

@tool
def save_chat_history(summary: str, runtime: ToolRuntime) -> str:
    """保存聊天摘要到历史"""
    store = runtime.store
    user_id = runtime.context.user_id
    session_id = runtime.execution_info.thread_id
    
    store.put(
        ("users", user_id, "history"),
        session_id,
        {"summary": summary, "timestamp": "2024-01-01"},
    )
    
    return "聊天历史已保存"
```

## Stream Writer

### 什么是 Stream Writer

允许工具在执行过程中向客户端发送实时反馈：

```
流式输出流程：
┌────────────────────────────────────────────┐
│  工具执行                                   │
│                                             │
│  writer("正在查询数据库...")   ← 即时反馈  │
│  writer("找到 5 条结果")       ← 即时反馈  │
│  writer("正在格式化...")       ← 即时反馈  │
│                                             │
│  return "最终结果"             ← 最终返回  │
└────────────────────────────────────────────┘
```

---

### 基本使用

```python
@tool
def get_weather(city: str, runtime: ToolRuntime) -> str:
    """获取城市天气（带进度反馈）"""
    writer = runtime.stream_writer
    
    writer(f"正在查询 {city} 的天气...")
    
    # 模拟耗时操作
    import time
    time.sleep(1)
    
    writer("数据获取成功")
    writer("正在格式化结果...")
    
    return f"{city} 天气晴朗，25°C"
```

---

### 多步骤任务反馈

```python
@tool
def generate_report(report_type: str, runtime: ToolRuntime) -> str:
    """生成报告（多步骤）"""

    # 收集数据逻辑
    
    writer("步骤 2/4: 分析数据...")
    # 分析逻辑
    
    writer("步骤 3/4: 生成图表...")
    # 图表生成
    
    writer("步骤 4/4: 格式化报告...")
    # 格式化
    
    return f"{report_type} 报告已生成"
```

@tool
def process_data(file_name: str, runtime: ToolRuntime) -> str:

### 基于执行信息做缓存

```python
@tool
def expensive_query(query: str, runtime: ToolRuntime) -> str:
    """耗时查询（带执行追踪）"""
    info = runtime.execution_info
    
    # 记录到状态以便追踪
    runtime.state.setdefault("tool_usage", []).append({
        "tool": "expensive_query",
        "query": query,
        "thread_id": info.thread_id,
        "run_id": info.run_id,
    })
    
    # 执行耗时操作
    return f"查询结果: {query}"
```

## 综合示例：完整工具

结合所有 Runtime 能力：

```python
from dataclasses import dataclass
from langchain.tools import tool, ToolRuntime

@dataclass
class Context:
    user_id: str

@tool
def comprehensive_search(
    query: str,
    max_results: int = 5,
    runtime: ToolRuntime = None,
) -> str:
    """综合搜索工具：带进度、状态更新、长期存储"""
    writer = runtime.stream_writer
    store = runtime.store
    user_id = runtime.context.user_id
    
    # 1. 流式反馈
    writer(f"开始搜索: {query}")
    
    # 2. 检查缓存
    cache_key = f"search:{query}"
    cached = store.get(("search_cache", user_id), cache_key)
    if cached:
        writer("使用缓存结果")
        return cached.value
    
    # 3. 执行搜索
    writer("正在搜索...")
    results = _do_search(query, max_results)  # 假设的搜索函数
    writer(f"找到 {len(results)} 条结果")
    
    # 4. 更新短期状态
    runtime.state["last_search"] = query
    runtime.state["last_results_count"] = len(results)
    
    # 5. 缓存结果
    store.put(
        ("search_cache", user_id),
        cache_key,
        results,
    )
    
    # 6. 记录使用情况
    runtime.state.setdefault("search_history", []).append({
        "query": query,
        "timestamp": "2024-01-01",
        "results": len(results),
    })
    
    return f"搜索结果: {results}"
```

## 常见错误

### 错误1：忘记声明 runtime 参数

```python
# ❌ 错误：想使用 runtime 但没有声明
@tool
def my_tool(query: str) -> str:
    """工具"""
    print(runtime.state)  # NameError: runtime 未定义

# ✅ 正确：声明 runtime 参数
@tool
def my_tool(query: str, runtime: ToolRuntime) -> str:
    """工具"""
    print(runtime.state)
```

---

### 错误2：runtime 类型错误

```python
# ❌ 错误：runtime 参数类型不对
@tool
def my_tool(query: str, runtime: str) -> str:
    """工具"""
    # runtime 被框架注入为 ToolRuntime，不是 str
```

---

### 错误3：在没有 context_schema 时访问 context

```python
# Agent 没有设置 context_schema
agent = create_agent(model, tools=[my_tool])

@tool
def my_tool(runtime: ToolRuntime) -> str:
    ctx = runtime.context  # 可能为 None
    return ctx.user_id     # AttributeError
```

**修复：**

```python
@tool
def my_tool(runtime: ToolRuntime) -> str:
    ctx = runtime.context
    if ctx is None:
        return "未提供上下文"
    return ctx.user_id
```

---

### 错误4：Store 命名空间不规范

```python
# ❌ 错误：命名空间不清晰
store.put(("data"), key, value)

# ✅ 正确：使用分层命名空间
store.put(("users", user_id, "preferences"), key, value)
```

## 小结

| 要点 | 说明 |
|------|------|
| ToolRuntime | 工具执行时的上下文对象，通过 `runtime` 参数注入 |
| state | 访问和更新 Agent 的短期状态（对话级别） |
| context | 运行时配置，由 context_schema 定义 |
| store | 长期存储，跨会话持久化数据 |
| stream_writer | 向客户端发送实时进度反馈 |
| execution_info | 执行元数据（thread_id, run_id） |
| server_info | MCP 服务器信息 |
| tool_call_id | 当前工具调用的唯一标识 |
