# 09.03.04 - 系统提示词

> Agent 的系统提示词配置：静态提示词、多模态提示词、动态提示词

## 为什么需要系统提示词

系统提示词定义 Agent 的行为准则：

```
提示词的作用：
┌────────────────────────────────────────────┐
│  没有系统提示词：                           │
│  - Agent 行为不可预测                       │
│  - 回答风格不一致                           │
│  - 不知道何时使用工具                       │
│                                             │
│  有系统提示词：                             │
│  - 定义角色和专业领域                       │
│  - 规范回答格式                             │
│  - 指导工具使用时机                         │
│  - 设置语言和语气                           │
└────────────────────────────────────────────┘
```

## 基础提示词

### 字符串提示词

最简单的方式：

```python
from langchain.agents import create_agent
from langchain.tools import tool

@tool
def search(query: str) -> str:
    """搜索信息"""
    return f"搜索结果: {query}"

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[search],
    system_prompt="你是一个专业的研究助手。使用搜索工具获取最新信息。"
                  "回答要简洁准确，引用信息来源。",
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "搜索最新的AI进展"}]
})
```

### 提示词设计原则

```python
# ✅ 好的提示词
system_prompt = (
    "你是一个专业的编程助手，专注于 Python 开发。"
    "使用工具获取最新文档和最佳实践。"
    "回答时：\n"
    "1. 先给出直接答案\n"
    "2. 提供代码示例\n"
    "3. 解释关键概念\n"
    "避免使用过于专业的术语，除非用户明确要求。"
)

# ❌ 差的提示词
system_prompt = "你是一个助手。"  # 太模糊
```

### 提示词结构

```python
system_prompt = """
# 角色
你是一个专业的数据分析助手。

# 能力
- 使用 calculate 工具执行数学运算
- 使用 search 工具查找统计资料

# 行为准则
1. 遇到数学问题必须使用 calculate 工具
2. 回答要包含计算过程
3. 最终答案用粗体标记

# 限制
- 不要编造数据
- 不确定的信息要标注"可能"
"""

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[calculate, search],
    system_prompt=system_prompt,
)
```

## SystemMessage 提示词

### 基本用法

```python
from langchain.messages import SystemMessage

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[search],
    system_prompt=SystemMessage(content="你是一个搜索助手。"),
)
```

### 多模态内容

SystemMessage 支持富内容：

```python
from langchain.messages import SystemMessage

agent = create_agent(
    model="anthropic:claude-sonnet-4-20250514",
    system_prompt=SystemMessage(
        content=[
            {"type": "text", "text": "你是一个文学分析助手。"},
            {
                "type": "text",
                "text": "分析以下小说的主题和人物关系：\n\n" + novel_text,
            },
        ]
    ),
)
```

**支持的内容类型：**

| 类型 | 说明 | 适用模型 |
|------|------|----------|
| `text` | 纯文本 | 所有模型 |
| `image_url` | 图片URL | 多模态模型 |
| `image` | Base64图片 | 多模态模型 |

### 内容缓存

对于大段内容，使用缓存降低成本：

```python
from langchain.messages import SystemMessage

agent = create_agent(
    model="anthropic:claude-sonnet-4-20250514",
    system_prompt=SystemMessage(
        content=[
            {
                "type": "text",
                "text": "<用户手册全文>",
                "cache_control": {"type": "ephemeral"},
            },
            {
                "type": "text",
                "text": "基于手册回答用户问题。",
            },
        ]
    ),
)
```

**缓存说明：**

| 属性 | 说明 |
|------|------|
| `cache_control` | 告诉提供商缓存此内容 |
| `type: "ephemeral"` | 临时缓存，会话内有效 |
| 适用提供商 | Anthropic 支持，其他提供商可能不支持 |

---

### 缓存工作原理

```
无缓存：
┌──────────────────────────────────────┐
│  每次请求：                           │
│  用户手册(50K tokens) + 问题         │
│  → 按 50K tokens 计费                │
└──────────────────────────────────────┘

有缓存：
┌──────────────────────────────────────┐
│  第一次：用户手册 + 问题              │
│  → 按 50K tokens 计费，缓存手册      │
│                                      │
│  第二次：缓存命中 + 问题              │
│  → 只按新 tokens 计费（节省 80%+）   │
└──────────────────────────────────────┘
```

## 动态提示词

### @dynamic_prompt 装饰器

运行时根据状态生成提示词：

```python
from langchain.agents.middleware import dynamic_prompt

@dynamic_prompt
def role_based_prompt(request) -> str:
    """根据用户角色生成提示词"""
    ctx = request.runtime.context
    user_role = ctx.role if ctx else "user"
    
    base_prompt = "你是一个有帮助的助手。"
    
    if user_role == "expert":
        return f"{base_prompt} 用户是专业人士，提供详细的技术回答。"
    elif user_role == "beginner":
        return f"{base_prompt} 用户是初学者，用简单语言解释，避免术语。"
    elif user_role == "child":
        return f"{base_prompt} 用户是儿童，用有趣的例子解释。"
    
    return base_prompt

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[search],
    middleware=[role_based_prompt],
)
```

### 基于时间的提示词

```python
from datetime import datetime

@dynamic_prompt
def time_aware_prompt(request) -> str:
    """根据时间调整提示词"""
    hour = datetime.now().hour
    
    if hour < 6:
        return "现在是深夜，回答要简洁，避免冗长解释。"
    elif hour < 12:
        return "现在是上午，用户可能刚起床，回答要清晰直接。"
    elif hour < 18:
        return "现在是下午，可以提供详细的回答。"
    else:
        return "现在是晚上，回答要友好轻松。"
```

### 基于对话历史的提示词

```python
@dynamic_prompt
def context_aware_prompt(request) -> str:
    """根据对话历史调整提示词"""
    messages = request.state.get("messages", [])
    
    # 计算对话轮数
    turn_count = len([m for m in messages if m.type == "human"])
    
    if turn_count == 0:
        return "这是对话的第一轮，热情问候并了解用户需求。"
    elif turn_count < 3:
        return "对话刚开始，继续了解用户问题。"
    else:
        return f"已经对话了 {turn_count} 轮，回顾之前的内容，保持一致性。"
```

### 基于工具使用的提示词

```python
@dynamic_prompt
def tool_aware_prompt(request) -> str:
    """根据工具使用情况调整提示词"""
    messages = request.state.get("messages", [])
    
    # 检查是否使用了工具
    tool_used = any(
        hasattr(m, "tool_calls") and m.tool_calls
        for m in messages
    )
    
    if tool_used:
        return (
            "你已经使用了工具获取信息。"
            "基于工具返回的结果回答用户问题。"
            "如果工具结果不完整，考虑使用更多工具。"
        )
    
    return "你还没有使用工具。如果用户问题需要外部信息，请使用工具。"
```

### 组合多个动态提示词

```python
@dynamic_prompt
def prompt_1(request) -> str:
    ctx = request.runtime.context
    return f"用户角色: {ctx.role}" if ctx else ""

@dynamic_prompt
def prompt_2(request) -> str:
    messages = request.state.get("messages", [])
    return f"对话轮数: {len(messages)}"

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[search],
    middleware=[prompt_1, prompt_2],  # 多个动态提示词
)
```

## Request 对象详解

动态提示词的 `request` 参数包含丰富信息：

| 属性 | 类型 | 说明 |
|------|------|------|
| `request.state` | `dict` | Agent 当前状态 |
| `request.runtime` | `ToolRuntime` | 运行时上下文 |
| `request.tools` | `list[BaseTool]` | 可用工具列表 |
| `request.model` | `BaseChatModel` | 当前模型 |
| `request.override()` | 方法 | 覆盖请求参数 |

---

### Request 使用示例

```python
@dynamic_prompt
def detailed_prompt(request) -> str:
    """详细的动态提示词"""
    
    # 访问状态
    messages = request.state.get("messages", [])
    
    # 访问上下文
    ctx = request.runtime.context
    user_id = ctx.user_id if ctx else "unknown"
    
    # 访问工具列表
    tool_names = [t.name for t in request.tools]
    
    return (
        f"用户 {user_id} 的助手。\n"
        f"可用工具: {', '.join(tool_names)}\n"
        f"对话历史: {len(messages)} 条消息。\n"
        f"请使用合适的工具回答用户问题。"
    )
```

## 常见错误

### 错误1：提示词过长

```python
# ❌ 错误：提示词包含过多细节
system_prompt = "你是一个助手。当用户问天气时，使用 get_weather 工具..."
# + 500行工具使用说明

# ✅ 正确：简洁明了
system_prompt = "你是一个天气助手。使用 get_weather 工具获取天气信息。"
# 模型能从工具描述中理解何时使用工具
```

---

### 错误2：提示词与工具描述冲突

```python
# ❌ 错误：提示词说使用 search，但工具名是 web_search
system_prompt = "使用 search 工具搜索信息"

@tool("web_search")
def search(query: str) -> str:
    """搜索"""

# ✅ 正确：一致
system_prompt = "使用 web_search 工具搜索信息"
```

---

### 错误3：忘记动态提示词的返回值

```python
# ❌ 错误：没有返回值
@dynamic_prompt
def my_prompt(request) -> str:
    messages = request.state.get("messages", [])
    print(len(messages))  # 只有打印，没有 return

# ✅ 正确
@dynamic_prompt
def my_prompt(request) -> str:
    messages = request.state.get("messages", [])
    return f"对话历史: {len(messages)} 条消息"
```

---

### 错误4：动态提示词引发异常

```python
# ❌ 错误：没有处理 None
@dynamic_prompt
def risky_prompt(request) -> str:
    ctx = request.runtime.context
    return f"用户: {ctx.user_id}"  # ctx 可能为 None

# ✅ 正确
@dynamic_prompt
def safe_prompt(request) -> str:
    ctx = request.runtime.context
    if ctx:
        return f"用户: {ctx.user_id}"
    return "匿名用户"
```

## 小结

| 要点 | 说明 |
|------|------|
| 字符串提示词 | 最简单的方式，直接传入字符串 |
| SystemMessage | 支持多模态内容和缓存控制 |
| 内容缓存 | Anthropic 模型支持，降低大内容成本 |
| @dynamic_prompt | 运行时根据状态生成提示词 |
| Request 对象 | 包含 state、runtime、tools 等信息 |
| 中间件组合 | 多个动态提示词可以同时使用 |
| 设计原则 | 简洁明确，不与工具描述冲突 |
