# 09.03.06 - 结构化输出

> 使用 ToolStrategy 和 ProviderStrategy 实现 Agent 的结构化输出

## 为什么需要结构化输出

LLM 默认返回非结构化文本：

```
非结构化输出 vs 结构化输出：
┌────────────────────────────────────────────┐
│  非结构化（默认）：                         │
│  "用户的姓名是张三，邮箱是                  │
│   zhang@example.com，电话是 13800138000"    │
│                                             │
│  → 难以程序化处理                           │
│                                             │
│  结构化：                                   │
│  {                                          │
│    "name": "张三",                          │
│    "email": "zhang@example.com",            │
│    "phone": "13800138000"                   │
│  }                                          │
│                                             │
│  → 可直接用于后续处理                       │
└────────────────────────────────────────────┘
```

## ToolStrategy

### 基本用法

通过工具调用实现结构化输出，适用于所有支持工具的模型：

```python
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy
from pydantic import BaseModel, Field

class ContactInfo(BaseModel):
    """联系人信息"""
    name: str = Field(description="姓名")
    email: str = Field(description="邮箱地址")
    phone: str = Field(description="电话号码")

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ToolStrategy(ContactInfo),
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "提取联系方式：张三，zhang@example.com，13800138000"
    }]
})

# 获取结构化响应
contact = result["structured_response"]
print(f"姓名: {contact.name}")
print(f"邮箱: {contact.email}")
print(f"电话: {contact.phone}")
```

---

### ToolStrategy 工作原理

```
ToolStrategy 流程：
┌────────────────────────────────────────────┐
│  1. 创建一个工具，其参数 schema 为          │
│     用户定义的 Pydantic 模型                │
│                                             │
│  2. 在提示词中指示模型调用该工具，          │
│     传入提取的数据                          │
│                                             │
│  3. 工具返回原始数据                        │
│                                             │
│  4. 解析为 Pydantic 模型实例                │
│                                             │
│  5. 存入 result["structured_response"]      │
└────────────────────────────────────────────┘
```

---

### 带工具的结构化输出

```python
from langchain.tools import tool

@tool
def search_people(name: str) -> str:
    """搜索人员信息"""
    return f"{name}，邮箱: name@example.com，电话: 138****0000"

class ContactInfo(BaseModel):
    name: str
    email: str
    phone: str

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[search_people],
    response_format=ToolStrategy(ContactInfo),
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "查找张三的联系方式"
    }]
})

# Agent 会：
# 1. 调用 search_people 工具
# 2. 从结果中提取信息
# 3. 调用结构化输出工具
contact = result["structured_response"]
```

---

### 多个结构化输出

```python
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy
from typing import List

class ContactInfo(BaseModel):
    name: str
    email: str

class ContactList(BaseModel):
    contacts: List[ContactInfo]

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ToolStrategy(ContactList),
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "提取所有联系方式：张三 zhang@example.com，李四 li@example.com"
    }]
})

contact_list = result["structured_response"]
for c in contact_list.contacts:
    print(f"{c.name}: {c.email}")
```

## ProviderStrategy

### 基本用法

使用提供商原生结构化输出（更可靠）：

```python
from langchain.agents.structured_output import ProviderStrategy

class ContactInfo(BaseModel):
    name: str
    email: str
    phone: str

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ProviderStrategy(ContactInfo),
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "提取联系方式：张三，zhang@example.com，13800138000"
    }]
})

contact = result["structured_response"]
```

---

### ProviderStrategy vs ToolStrategy

| 特性 | ToolStrategy | ProviderStrategy |
|------|-------------|------------------|
| 实现方式 | 通过工具调用 | 使用提供商原生功能 |
| 兼容性 | 所有支持工具的模型 | 仅支持结构化输出的模型 |
| 可靠性 | 依赖模型遵循指令 | 通常更可靠 |
| 速度 | 需要额外工具调用 | 通常更快 |

---

### 自动选择

直接传入 schema 时自动使用 ProviderStrategy（如果支持）：

```python
class ContactInfo(BaseModel):
    name: str
    email: str

# 自动选择 ProviderStrategy
agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ContactInfo,
)
```

## Union 类型

### 多种输出格式

```python
from typing import Union

class WeatherReport(BaseModel):
    """天气报告"""
    city: str
    temperature: int
    conditions: str

class ErrorResponse(BaseModel):
    """错误响应"""
    error: str
    suggestion: str

# 联合类型
WeatherResult = Union[WeatherReport, ErrorResponse]

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ToolStrategy(WeatherResult),
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "查询北京天气"
    }]
})

response = result["structured_response"]
if isinstance(response, WeatherReport):
    print(f"{response.city}: {response.temperature}°C")
else:
    print(f"错误: {response.error}")
```

### 实际场景：搜索或错误

class SearchResult(BaseModel):

## 错误处理

### 结构化输出失败

```python
try:
    result = agent.invoke({
        "messages": [{
            "role": "user",
            "content": "提取联系方式"
        }]
    })
    contact = result["structured_response"]
except Exception as e:
    print(f"结构化输出失败: {e}")
    # 回退到普通响应
    print(result["messages"][-1].content)
```

---

### 验证错误

```python
from pydantic import ValidationError

try:
    contact = result["structured_response"]
except ValidationError as e:
    print(f"数据验证失败: {e}")
    # 可能模型返回的格式不符合 schema
```

---

### 自定义工具消息

当结构化输出使用 ToolStrategy 时，可以自定义工具返回消息：

```python
from langchain.agents.structured_output import ToolStrategy
from langchain.messages import ToolMessage

class ContactInfo(BaseModel):
    name: str
    email: str

def custom_output_handler(data: dict) -> ToolMessage:
    """自定义输出处理"""
    return ToolMessage(
        content=f"已提取: {data.get('name', '未知')}",
        tool_call_id=data.get("_tool_call_id"),
    )

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ToolStrategy(
        ContactInfo,
        output_handler=custom_output_handler,
    ),
)
```

## 实际示例

### 示例1：信息提取

```python
from pydantic import BaseModel, Field
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy

class PersonInfo(BaseModel):
    """人员信息"""
    name: str = Field(description="姓名")
    age: int = Field(description="年龄")
    occupation: str = Field(description="职业")
    city: str = Field(description="所在城市")

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ToolStrategy(PersonInfo),
)

text = "我叫王明，今年32岁，是一名软件工程师，目前居住在杭州。"

result = agent.invoke({
    "messages": [{"role": "user", "content": f"从以下文本提取信息：{text}"}]
})

person = result["structured_response"]
print(f"姓名: {person.name}")
print(f"年龄: {person.age}")
print(f"职业: {person.occupation}")
print(f"城市: {person.city}")
```

## 常见错误

### 错误1：模型不支持

```python
# ❌ 错误：模型不支持结构化输出
agent = create_agent(
    model="某些不支持的模型",
    response_format=ProviderStrategy(ContactInfo),
)
# 可能抛出异常或忽略

# ✅ 修复：使用 ToolStrategy（兼容性更好）
agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    response_format=ToolStrategy(ContactInfo),
)
```

---

### 错误2：Schema 过于复杂

```python
# ❌ 错误：嵌套太深，模型难以理解
class VeryComplex(BaseModel):
    level1: dict[str, dict[str, dict[str, list[dict]]]]

# ✅ 修复：扁平化设计
class Simple(BaseModel):
    name: str
    value: float
    tags: list[str]
```

---

### 错误3：缺少 Field 描述

```python
# ❌ 错误：没有描述，模型不知道字段含义
class Contact(BaseModel):
    name: str
    email: str

# ✅ 正确：每个字段都有描述
class Contact(BaseModel):
    name: str = Field(description="用户全名")
    email: str = Field(description="电子邮箱地址")
```

---

### 错误4：期望不存在的字段

```python
# ❌ 错误：模型可能不会填充所有字段
class Contact(BaseModel):
    name: str
    email: str
    phone: str  # 文本中没有电话号码

# ✅ 修复：使用 Optional
class Contact(BaseModel):
    name: str
    email: str
    phone: Optional[str] = Field(default=None)
```

## 小结

| 要点 | 说明 |
|------|------|
| ToolStrategy | 通过工具调用实现，兼容所有模型 |
| ProviderStrategy | 使用提供商原生功能，更可靠 |
| 自动选择 | 直接传入 schema 自动选择 ProviderStrategy |
| Union 类型 | 支持多种输出格式 |
| 错误处理 | 捕获 ValidationError 处理格式错误 |
| Schema 设计 | 字段要有描述，避免过于复杂 |
| 获取结果 | `result["structured_response"]` |
