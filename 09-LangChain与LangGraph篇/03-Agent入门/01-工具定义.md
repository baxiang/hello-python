# 09.03.01 - 工具定义

> LangChain 中工具（Tool）是 Agent 的"双手"——让大模型能够执行真实世界的操作

## 什么是工具

Agent 本身只是一个会思考的大脑，它需要工具来与世界交互：

```
Agent（大脑）
    │
    ├── 思考：我需要查天气 → 调用天气工具
    ├── 思考：我需要算数学 → 调用计算器工具
    ├── 思考：我需要查数据库 → 调用查询工具
    └── 思考：我需要发邮件 → 调用邮件工具
```

工具本质上是一个 **函数 + 元数据** 的组合：

```
工具结构
┌──────────────────────────────────────────┐
│  Tool                                    │
├──────────────────────────────────────────┤
│  name: str        → "get_weather"        │
│  description: str → "查询指定城市天气"    │
│  args_schema:     → Pydantic/JSON Schema │
│  func: callable   → 实际执行的函数        │
└──────────────────────────────────────────┘
```

大模型通过工具的 `name` 和 `description` 来决定调用哪个工具，通过 `args_schema` 来构造正确的参数。

## @tool 装饰器基础

### 最简用法

`@tool` 装饰器将普通 Python 函数转换为 LangChain 工具：

```python
from langchain_core.tools import tool

@tool
def add(a: int, b: int) -> int:
    """将两个数字相加。"""
    return a + b

# 查看工具信息
print(f"工具名称: {add.name}")
print(f"工具描述: {add.description}")
print(f"参数Schema: {add.args_schema}")

# 调用工具
result = add.invoke({"a": 3, "b": 5})
print(f"结果: {result}")  # 输出: 8
```

关键点：

| 要素 | 说明 |
|------|------|
| 函数名 | 自动成为工具名称（转换为 snake_case） |
| 类型注解 | 自动生成 args_schema |
| 文档字符串 | **第一行**成为工具描述 |
| 返回值 | 自动转为字符串返回给模型 |

### 文档字符串的格式

文档字符串的**第一行**是工具描述，后续内容可作为详细说明：

```python
@tool
def search_database(query: str, limit: int = 10) -> str:
    """搜索数据库记录。

    在用户表中搜索匹配的记录，返回 JSON 格式的结果。
    支持模糊匹配，最多返回 limit 条记录。

    Args:
        query: 搜索关键词
        limit: 最大返回数量，默认 10

    Returns:
        JSON 格式的搜索结果
    """
    # 模拟搜索
    return f'[{{"name": "张三", "score": 95}}]'
```

模型看到的描述是："搜索数据库记录。"

## 自定义工具名称

### 使用 @tool("custom_name")

函数名不适合做工具名时，可以显式指定：

```python
@tool("calculate_sum")
def my_add_func(a: int, b: int) -> int:
    """将两个数字相加。"""
    return a + b

print(f"工具名称: {my_add_func.name}")  # 输出: calculate_sum
```

### 为什么需要自定义名称？

```
场景对比
┌─────────────────────────────────────────────────────────┐
│ 函数名                     → 工具名                      │
├─────────────────────────────────────────────────────────┤
│ _internal_calc            → calculate（去掉下划线）      │
│ calculateTwoNumbers       → calculate_two_numbers        │
│ 天气查询                   → invalid（中文，会报错）      │
│ get_user_info_from_db     → get_user_info（自定义更好）  │
└─────────────────────────────────────────────────────────┘
```

常见需要自定义名称的场景：

```python
# 1. 内部函数（前导下划线会被移除）
@tool("get_weather")
def _internal_weather_api(city: str) -> str:
    """获取指定城市的天气信息。"""
    return f"{city} 晴天，25°C"

# 2. 缩写更友好
@tool("calc")
def calculate_complex_formula(x: float, y: float) -> float:
    """计算复杂数学公式的结果。"""
    return x ** 2 + y ** 2

# 3. 与现有函数名冲突
@tool("send_email_v2")
def send_email(to: str, subject: str, body: str) -> str:
    """发送邮件。"""
    return f"邮件已发送至 {to}"
```

## 自定义工具描述

### 使用 description 参数

当文档字符串不够清晰或需要动态描述时：

```python
@tool(description="查询实时天气信息，返回温度、湿度、风速等数据")
def get_weather(city: str) -> str:
    # 即使没有文档字符串，description 也会覆盖
    return f"{city}: 25°C, 湿度 60%"

print(f"工具描述: {get_weather.description}")
# 输出: 查询实时天气信息，返回温度、湿度、风速等数据
```

### description 与文档字符串的优先级

```python
@tool(description="这是自定义描述")
def func1(x: int) -> int:
    """这是文档字符串描述。"""
    return x

@tool
def func2(x: int) -> int:
    """这是文档字符串描述。"""
    return x

print(f"func1 描述: {func1.description}")  # 自定义描述优先
print(f"func2 描述: {func2.description}")  # 使用文档字符串
```

```
优先级顺序（从高到低）
┌──────────────────────────────────────────────┐
│ 1. description 参数      → 最高优先级        │
│ 2. 文档字符串第一行       → 默认行为          │
│ 3. 无描述                → 工具可能不被选中  │
└──────────────────────────────────────────────┘
```

### 好的描述 vs 差的描述

```python
# ❌ 差的描述——太模糊
@tool
def get_data(x: str) -> str:
    """获取数据。"""
    return "some data"

# ✅ 好的描述——具体、包含用途
@tool
def get_weather(city: str) -> str:
    """查询指定城市的实时天气，包括温度、湿度和风向。"""
    return f"{city}: 25°C, 湿度 60%, 东南风"

# ❌ 差的描述——包含实现细节
@tool
def calculate_price(price: float) -> float:
    """使用 requests 库调用 API 然后乘以 1.2 的税率。"""
    return price * 1.2

# ✅ 好的描述——关注功能而非实现
@tool
def calculate_price(price: float) -> float:
    """计算商品含税价格，税率为 20%。"""
    return price * 1.2
```

## Pydantic args_schema

### 为什么需要 args_schema

```
args_schema 的作用
┌───────────────────────────────────────────────────────────────┐
│                       Agent 循环                              │
│                                                               │
│  模型输出: {"tool": "get_weather", "args": {"city": "北京"}}   │
│                          ↓                                    │
│              args_schema 验证参数                              │
│              ┌─────────┬─────────┬──────────┐                 │
│              │ 类型检查 │ 必填检查 │ 范围检查 │                 │
│              └─────────┴─────────┴──────────┘                 │
│                          ↓                                    │
│              验证通过 → 执行工具函数                           │
│              验证失败 → 返回错误给模型                         │
└───────────────────────────────────────────────────────────────┘
```

### 基础 Pydantic Schema

```python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class WeatherInput(BaseModel):
    city: str = Field(description="城市名称，如：北京、上海")
    unit: str = Field(
        default="celsius",
        description="温度单位",
        enum=["celsius", "fahrenheit"],
    )

@tool(args_schema=WeatherInput)
def get_weather(city: str, unit: str = "celsius") -> str:
    """查询指定城市的天气。"""
    temp = 25 if unit == "celsius" else 77
    return f"{city}: {temp}°{'C' if unit == 'celsius' else 'F'}"

# 验证参数
print(f"Schema: {get_weather.args_schema.model_json_schema()}")
```

### Field 的详细用法

```python
from pydantic import BaseModel, Field
from typing import Literal

class SearchInput(BaseModel):
    query: str = Field(
        description="搜索关键词",
        min_length=1,
        max_length=100,
    )
    category: Literal["news", "product", "user"] = Field(
        default="news",
        description="搜索类别",
    )
    limit: int = Field(
        default=10,
        description="最大返回数量",
        ge=1,   # greater than or equal
        le=100, # less than or equal
    )

@tool(args_schema=SearchInput)
def search_content(
    query: str,
    category: str = "news",
    limit: int = 10,
) -> str:
    """搜索内容。"""
    return f"在 {category} 中搜索 '{query}'，返回 {limit} 条结果"
```

### 嵌套 Pydantic 模型

```python
from pydantic import BaseModel, Field
from typing import Optional

class LocationInput(BaseModel):
    lat: float = Field(description="纬度", ge=-90, le=90)
    lng: float = Field(description="经度", ge=-180, le=180)

class NearbySearchInput(BaseModel):
    location: LocationInput = Field(description="搜索中心点坐标")
    radius: int = Field(description="搜索半径（米）", ge=100, le=50000)
    keyword: Optional[str] = Field(default=None, description="关键词过滤")

@tool(args_schema=NearbySearchInput)
def search_nearby(location: dict, radius: int, keyword: str | None = None) -> str:
    """搜索附近的地点。"""
    lat = location["lat"]
    lng = location["lng"]
    return f"在 ({lat}, {lng}) 附近 {radius} 米范围内搜索"
```

## JSON Schema args_schema

### 直接使用字典定义

不想引入 Pydantic 时，可以直接用 JSON Schema 字典：

```python
from langchain_core.tools import tool

weather_schema = {
    "type": "object",
    "properties": {
        "city": {
            "type": "string",
            "description": "城市名称",
        },
        "unit": {
            "type": "string",
            "description": "温度单位",
            "enum": ["celsius", "fahrenheit"],
            "default": "celsius",
        },
    },
    "required": ["city"],
}

@tool(args_schema=weather_schema)
def get_weather(city: str, unit: str = "celsius") -> str:
    """查询天气。"""
    temp = 25 if unit == "celsius" else 77
    return f"{city}: {temp}°{'C' if unit == 'celsius' else 'F'}"
```

### JSON Schema vs Pydantic 对比

```
对比维度          Pydantic args_schema         JSON Schema dict
─────────────── ──────────────────────────── ──────────────────────────
类型安全        ✅ 运行时类型验证             ⚠️  仅作为描述传递给模型
IDE 支持        ✅ 自动补全、类型提示         ❌  无
嵌套结构        ✅ 嵌套 BaseModel 自动处理    ⚠️  需手动编写嵌套 schema
约束验证        ✅ Field(ge, le, regex...)   ⚠️  需手动写 JSON Schema 约束
依赖安装        需要 pydantic                无需额外依赖
适用场景        复杂参数、强类型项目          简单工具、快速原型
```

```python
# 复杂 JSON Schema 示例（等价于 Pydantic 嵌套模型）
nearby_schema = {
    "type": "object",
    "properties": {
        "location": {
            "type": "object",
            "properties": {
                "lat": {"type": "number", "description": "纬度", "minimum": -90, "maximum": 90},
                "lng": {"type": "number", "description": "经度", "minimum": -180, "maximum": 180},
            },
            "required": ["lat", "lng"],
            "description": "搜索中心点坐标",
        },
        "radius": {
            "type": "integer",
            "description": "搜索半径（米）",
            "minimum": 100,
            "maximum": 50000,
        },
        "keyword": {
            "type": ["string", "null"],
            "description": "关键词过滤",
        },
    },
    "required": ["location", "radius"],
}

@tool(args_schema=nearby_schema)
def search_nearby(location: dict, radius: int, keyword: str | None = None) -> str:
    """搜索附近地点。"""
    return f"在 ({location['lat']}, {location['lng']}) 附近 {radius}m 搜索"
```

## 保留参数名称

### config 和 runtime

LangChain 保留了一些参数名，用于传递运行时上下文：

```python
from langchain_core.tools import tool
from langchain_core.runnables import RunnableConfig

@tool
def search_with_config(query: str, config: RunnableConfig) -> str:
    """搜索内容，使用配置。"""
    # 从 config 获取运行时信息
    metadata = config.get("metadata", {})
    user_id = metadata.get("user_id", "anonymous")
    return f"用户 {user_id} 搜索了 '{query}'"

@tool
def search_with_runtime(query: str, runtime: dict) -> str:
    """搜索内容，使用运行时信息。"""
    # runtime 包含运行时上下文
    callbacks = runtime.get("callbacks")
    return f"搜索 '{query}'，callbacks: {callbacks}"
```

### 保留参数列表

```
保留参数
┌──────────────────────────────────────────────────────────────────┐
│ 参数名      │ 类型                  │ 用途                       │
├──────────────────────────────────────────────────────────────────┤
│ config      │ RunnableConfig        │ 运行时配置（metadata等）   │
│ runtime     │ dict                  │ 运行时上下文                │
└──────────────────────────────────────────────────────────────────┘

⚠️  这些参数不会出现在 args_schema 中，模型不会为它们传值。
   它们由 LangChain 框架自动注入。
```

### 正确使用 config

```python
from langchain_core.tools import tool
from langchain_core.runnables import RunnableConfig

@tool
def get_user_profile(user_id: str, config: RunnableConfig | None = None) -> str:
    """获取用户资料。"""
    # config 是可选的，不影响工具调用
    if config and config.get("metadata"):
        caller = config["metadata"].get("caller_id")
        return f"用户 {user_id} 的资料（由 {caller} 查询）"
    return f"用户 {user_id} 的资料"

# 直接调用时不需要传 config
result = get_user_profile.invoke({"user_id": "123"})
print(result)
```

### 常见错误

```python
# ❌ 错误：把 config 当作普通参数写入文档字符串
@tool
def bad_tool(query: str, config: RunnableConfig) -> str:
    """
    搜索内容。

    Args:
        query: 搜索关键词
        config: 运行配置  ← 模型会困惑，但 config 不会出现在 schema 中
    """
    return "result"

# ✅ 正确：config 不需要在文档字符串中描述
@tool
def good_tool(query: str, config: RunnableConfig | None = None) -> str:
    """搜索内容。"""
    return "result"
```

## 服务端工具使用

### 工具与服务端解耦

工具定义和执行可以在不同地方：

```
工具执行流程
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  客户端（模型）           工具注册中心          服务端执行       │
│  ┌──────────┐           ┌──────────┐          ┌──────────┐     │
│  │ 决定调用  │ ──────→   │ 查找工具 │ ──────→  │ 执行函数 │     │
│  │ get_weather│          │ 获取schema│          │ 返回结果 │     │
│  └──────────┘           └──────────┘          └──────────┘     │
│                                                                 │
│  模型只看到 name + description，不需要知道实现细节               │
└─────────────────────────────────────────────────────────────────┘
```

### 使用 tool 函数创建可序列化工具

```python
from langchain_core.tools import tool

# 普通 @tool 装饰器创建的工具可以直接使用
@tool
def calculator(expression: str) -> str:
    """计算数学表达式。"""
    return str(eval(expression))

# 获取工具的结构化表示（可用于 API 传输）
tool_def = calculator.tool_call
print(tool_def)
# {'name': 'calculator', 'args': {'expression': '...'}, 'id': '...'}
```

### 工具绑定到模型

```python
from langchain_core.tools import tool
from langchain.chat_models import init_chat_model

@tool
def get_weather(city: str) -> str:
    """查询指定城市的天气。"""
    return f"{city}: 晴天，25°C"

@tool
def get_time(timezone: str = "Asia/Shanghai") -> str:
    """获取指定时区的当前时间。"""
    from datetime import datetime
    import zoneinfo
    tz = zoneinfo.ZoneInfo(timezone)
    now = datetime.now(tz)
    return now.strftime("%Y-%m-%d %H:%M:%S")

model = init_chat_model("moonshot:moonshot-v1-8k")

# 绑定工具到模型
model_with_tools = model.bind_tools([get_weather, get_time])

# 模型会自动选择工具
response = model_with_tools.invoke("北京现在几点？天气怎么样？")
print(f"响应: {response.content}")
print(f"工具调用: {response.tool_calls}")
```

### 完整 Agent 工具执行示例

```python
from langchain_core.tools import tool
from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent

@tool
def get_weather(city: str) -> str:
    """查询指定城市的实时天气。"""
    # 实际项目中应调用真实 API
    weather_data = {
        "北京": "晴天，25°C，湿度 40%",
        "上海": "多云，22°C，湿度 65%",
        "广州": "阵雨，28°C，湿度 80%",
        "深圳": "晴天，30°C，湿度 70%",
    }
    return weather_data.get(city, f"暂无 {city} 的天气数据")

@tool
def get_time(city: str = "上海") -> str:
    """获取指定城市的当前时间。"""
    from datetime import datetime
    import zoneinfo
    tz_map = {
        "北京": "Asia/Shanghai",
        "上海": "Asia/Shanghai",
        "广州": "Asia/Shanghai",
        "深圳": "Asia/Shanghai",
        "东京": "Asia/Tokyo",
        "纽约": "America/New_York",
        "伦敦": "Europe/London",
    }
    tz_name = tz_map.get(city, "Asia/Shanghai")
    tz = zoneinfo.ZoneInfo(tz_name)
    now = datetime.now(tz)
    return f"{city}: {now.strftime('%Y-%m-%d %H:%M:%S')}"

# 创建 Agent
model = init_chat_model("moonshot:moonshot-v1-8k")
agent = create_react_agent(model, [get_weather, get_time])

# 运行 Agent
result = agent.invoke({
    "messages": [{"role": "user", "content": "北京现在的天气和时间？"}]
})

# 查看完整对话
for msg in result["messages"]:
    print(f"--- {type(msg).__name__} ---")
    print(msg.content)
    if hasattr(msg, "tool_calls") and msg.tool_calls:
        print(f"工具调用: {msg.tool_calls}")
```

## 工具返回值

### 返回字符串（最常见）

```python
@tool
def get_weather(city: str) -> str:
    """查询天气。"""
```

### 使用 Command 返回（LangGraph）

在 LangGraph 中，工具可以返回 `Command` 来控制图流程：

```python
from langchain_core.tools import tool
from langgraph.types import Command

@tool
def escalate_to_human(query: str) -> Command:
    """当问题超出处理能力时，转接人工客服。"""
    return Command(
        goto="human_agent",  # 跳转到 human_agent 节点
        update={"query": query, "status": "escalated"},
    )
```

### 处理工具错误

```python
@tool
def divide(a: float, b: float) -> str:
    """除法运算。"""
    if b == 0:
        return "错误：除数不能为零"  # ✅ 返回错误信息字符串
    return str(a / b)

def divide_strict(a: float, b: float) -> str:

    if b == 0:

    return str(a / b)
```

## 完整实战示例

### 示例1：天气查询 Agent

```python
"""天气查询 Agent 示例，展示 @tool 的完整用法。"""

from langchain_core.tools import tool
from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from pydantic import BaseModel, Field
from typing import Literal

# 1. 定义输入 Schema
class WeatherInput(BaseModel):
    city: str = Field(description="城市名称，如：北京、上海、广州")
    unit: Literal["celsius", "fahrenheit"] = Field(
        default="celsius",
        description="温度单位",
    )

# 2. 定义工具
@tool(args_schema=WeatherInput)
def get_weather(city: str, unit: str = "celsius") -> str:
    """查询指定城市的实时天气信息。"""
    # 模拟天气数据
    weather_db = {
        "北京": {"temp_c": 25, "condition": "晴天", "humidity": 40},
        "上海": {"temp_c": 22, "condition": "多云", "humidity": 65},
        "广州": {"temp_c": 28, "condition": "阵雨", "humidity": 80},
        "深圳": {"temp_c": 30, "condition": "晴天", "humidity": 70},
    }

    data = weather_db.get(city)
    if not data:
        return f"暂无 {city} 的天气数据，请尝试其他城市"

    temp = data["temp_c"]
    if unit == "fahrenheit":
        temp = temp * 9 / 5 + 32
        unit_symbol = "°F"
    else:
        unit_symbol = "°C"

    return (
        f"{city} 天气\n"
        f"  温度: {temp}{unit_symbol}\n"
        f"  天气: {data['condition']}\n"
        f"  湿度: {data['humidity']}%"
    )

@tool
def convert_temperature(
    value: float,
    from_unit: Literal["C", "F"] = "C",
    to_unit: Literal["C", "F"] = "F",
) -> str:
    """转换温度单位。"""
    if from_unit == to_unit:
        return f"{value}°{from_unit}"

    if from_unit == "C":
        result = value * 9 / 5 + 32
    else:
        result = (value - 32) * 5 / 9

    return f"{value}°{from_unit} = {result:.1f}°{to_unit}"

# 3. 创建并运行 Agent
model = init_chat_model("moonshot:moonshot-v1-8k")
agent = create_react_agent(model, [get_weather, convert_temperature])

# 测试
if __name__ == "__main__":
    questions = [
        "北京今天天气怎么样？",
        "把 100 摄氏度转成华氏度",
        "上海和广州哪个更热？",
    ]

    for q in questions:
        print(f"\n{'='*50}")
        print(f"问题: {q}")
        print(f"{'='*50}")

        result = agent.invoke({
            "messages": [{"role": "user", "content": q}]
        })

        # 打印最后一条 AI 消息
        for msg in reversed(result["messages"]):
            if msg.__class__.__name__ == "AIMessage":
                print(f"回答: {msg.content}")
                break
```

### 示例2：计算器工具集合

```python
"""计算器工具集合，展示多种 args_schema 用法。"""

from langchain_core.tools import tool
from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from pydantic import BaseModel, Field
import math

# 1. 简单参数
@tool
def add(a: float, b: float) -> str:
    """加法运算。"""
    return f"{a} + {b} = {a + b}"

# 2. JSON Schema args_schema
multiply_schema = {
    "type": "object",
    "properties": {
        "a": {"type": "number", "description": "第一个数字"},
        "b": {"type": "number", "description": "第二个数字"},
    },
    "required": ["a", "b"],
}

@tool(args_schema=multiply_schema)
def multiply(a: float, b: float) -> str:
    """乘法运算。"""
    return f"{a} × {b} = {a * b}"

# 3. 带约束的 Pydantic
class PowerInput(BaseModel):
    base: float = Field(description="底数")
    exponent: float = Field(description="指数", ge=-100, le=100)

@tool(args_schema=PowerInput)
def power(base: float, exponent: float) -> str:
    """幂运算。"""
    result = base ** exponent
    return f"{base} ^ {exponent} = {result}"

# 4. 自定义名称 + 描述
@tool("sqrt", description="计算一个非负数的平方根")
def square_root(number: float) -> str:
    if number < 0:
        return "错误：不能对负数开平方"
    return f"√{number} = {math.sqrt(number):.4f}"

# 5. 组合使用
model = init_chat_model("moonshot:moonshot-v1-8k")
tools = [add, multiply, power, square_root]
agent = create_react_agent(model, tools)

if __name__ == "__main__":
    result = agent.invoke({
        "messages": [{"role": "user", "content": "计算 2 的 10 次方，然后开平方是多少？"}]
    })

    for msg in result["messages"]:
        print(f"--- {type(msg).__name__} ---")
        if hasattr(msg, "content"):
            print(msg.content)
        if hasattr(msg, "tool_calls") and msg.tool_calls:
            print(f"工具调用: {msg.tool_calls}")
```

### 示例3：带 config 的工具

```python
"""使用 config 传递用户上下文。"""

from langchain_core.tools import tool
from langchain_core.runnables import RunnableConfig
from langchain.chat_models import init_chat_model
from langgraph.prebuilt import create_react_agent
from datetime import datetime

@tool
def get_greeting(config: RunnableConfig | None = None) -> str:
    """根据当前时间返回问候语。"""
    hour = datetime.now().hour

    if hour < 12:
        greeting = "早上好"
    elif hour < 18:
        greeting = "下午好"
    else:
        greeting = "晚上好"

    # 从 config 获取用户姓名
    user_name = "用户"
    if config and config.get("metadata"):
        user_name = config["metadata"].get("user_name", "用户")

    return f"{greeting}，{user_name}！"

@tool
def get_user_timezone(user_id: str, config: RunnableConfig | None = None) -> str:
    """获取用户的时区设置。"""
    # 模拟用户时区数据
    timezone_db = {
        "001": "Asia/Shanghai",
        "002": "America/New_York",
        "003": "Europe/London",
    }

    tz = timezone_db.get(user_id, "Asia/Shanghai")
    return f"用户 {user_id} 的时区: {tz}"

# 使用带 config 的工具需要特殊方式
model = init_chat_model("moonshot:moonshot-v1-8k")
agent = create_react_agent(model, [get_greeting, get_user_timezone])

if __name__ == "__main__":
```

## 常见错误与排查

### 错误1：缺少类型注解

```python
# ❌ 错误：没有类型注解
@tool
def bad_tool(a, b):
    """加法。"""
    return a + b

# TypeError: 无法推断参数类型
```

```python
# ✅ 正确：添加类型注解
@tool
def good_tool(a: int, b: int) -> int:
    """加法。"""
    return a + b
```

### 错误2：文档字符串格式错误

```python
# ❌ 错误：文档字符串不在第一行
@tool
def bad_tool(city: str) -> str:
    x = 1  # 代码在文档字符串前面
    """查询天气。"""  # 这不是文档字符串！
    return "weather"

# 工具描述为空！
```

```python
# ✅ 正确：文档字符串在函数体第一行
@tool
def good_tool(city: str) -> str:
    """查询天气。"""
    x = 1
    return "weather"
```

### 错误3：参数名与 schema 不匹配

```python
# ❌ 错误：函数参数名与 schema 不一致
class Input(BaseModel):
    city_name: str = Field(description="城市")

@tool(args_schema=Input)
def get_weather(city: str) -> str:  # 应该是 city_name
    """查询天气。"""
    return f"{city}: sunny"
```

```python
# ✅ 正确：参数名匹配
@tool(args_schema=Input)
def get_weather(city_name: str) -> str:
    """查询天气。"""
    return f"{city_name}: sunny"
```

### 错误4：使用保留参数名作为普通参数

```python
# ❌ 错误：config 是保留参数，不应出现在 schema 描述中
@tool
def search(query: str, config: RunnableConfig) -> str:
    """
    搜索内容。

    Args:
        query: 关键词
        config: 配置信息  ← 不要这样描述
    """
    return "result"
```

## 小结

| 要点 | 说明 |
|------|------|
| `@tool` 装饰器 | 将普通函数转为工具，自动提取名称和描述 |
| 自定义名称 | `@tool("custom_name")` 覆盖函数名 |
| 自定义描述 | `description` 参数优先于文档字符串 |
| Pydantic schema | `args_schema=BaseModel` 提供强类型验证 |
| JSON Schema | `args_schema=dict` 轻量级方案，无需 Pydantic |
| 保留参数 | `config` 和 `runtime` 由框架自动注入 |
| 返回值 | `str` 最常用，`dict` 自动转 JSON，`Command` 控制流程 |
| 命名规范 | `snake_case`，动词开头，简洁明确，不用中文 |

```
工具定义检查清单
┌──────────────────────────────────────────────────────────────────┐
│ □ 使用 snake_case 命名                                           │
│ □ 文档字符串第一行是清晰的描述                                    │
│ □ 所有参数都有类型注解                                            │
│ □ 复杂输入使用 Pydantic args_schema                               │
│ □ 需要时 config 标记为 Optional                                   │
│ □ 返回值可被序列化为字符串                                        │
│ □ 描述关注功能而非实现细节                                        │
└──────────────────────────────────────────────────────────────────┘
```
