# 09.04.07 - 结构化输出进阶

> 使用 Union 类型、ToolStrategy、ProviderStrategy 控制 LLM 输出格式

## 为什么需要结构化输出

```
结构化输出场景：
┌──────────────────────────────────────┐
│  场景1：Union 类型                    │
│  查询: "北京的天气"                   │
│  输出: WeatherData 或 ErrorResponse  │
│                                       │
│  场景2：代码生成                      │
│  查询: "写一个排序函数"               │
│  输出: CodeResponse（含代码和解释）   │
│                                       │
│  场景3：数据提取                      │
│  查询: 从文本提取人名和地址           │
│  输出: ExtractedData（结构化字段）    │
│                                       │
│  场景4：验证循环                      │
│  输出不符合 schema 时自动重试         │
└──────────────────────────────────────┘
```

### 结构化 vs 非结构化输出

| 特性 | 非结构化输出 | 结构化输出 |
|------|------------|-----------|
| 格式 | 自由文本 | Pydantic 模型 |
| 解析 | 需要正则/LLM 提取 | 直接访问字段 |
| 验证 | 困难 | 自动验证 |
| 类型安全 | 无 | 强类型 |
| 适用场景 | 对话、创意 | 数据提取、API 响应 |

## Union 类型

### 基本用法

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model
from langchain.structured_output import OutputStrategy

class WeatherData(BaseModel):
    """天气数据"""
    city: str = Field(description="城市名称")
    temperature: float = Field(description="温度（摄氏度）")
    condition: str = Field(description="天气状况", enum=["晴", "阴", "雨", "雪"])

class ErrorResponse(BaseModel):
    """错误响应"""
    error: str = Field(description="错误信息")
    suggestion: str = Field(description="建议")

# 使用 Union 类型
llm = init_chat_model("deepseek:deepseek-chat")
structured_llm = llm.with_structured_output(WeatherData | ErrorResponse)

# 调用
result = structured_llm.invoke("北京今天天气怎么样？")
print(type(result))  # WeatherData 或 ErrorResponse

if isinstance(result, WeatherData):
    print(f"城市: {result.city}")
    print(f"温度: {result.temperature}°C")
    print(f"天气: {result.condition}")
elif isinstance(result, ErrorResponse):
    print(f"错误: {result.error}")
    print(f"建议: {result.suggestion}")
```

### 多类型 Union

```python
from pydantic import BaseModel, Field
from typing import Literal

class CodeSolution(BaseModel):
    """代码解决方案"""
    language: Literal["python", "javascript", "java"]
    code: str
    explanation: str

class ExplanationOnly(BaseModel):
    """纯解释"""
    concept: str
    explanation: str
    examples: list[str]

class Clarification(BaseModel):
    """需要澄清"""
    question: str
    missing_info: list[str]

# 多类型 Union
ResponseUnion = CodeSolution | ExplanationOnly | Clarification

llm = init_chat_model("deepseek:deepseek-chat")
structured_llm = llm.with_structured_output(ResponseUnion)

result = structured_llm.invoke("写一个快速排序算法")

if isinstance(result, CodeSolution):
    print(f"语言: {result.language}")
    print(f"代码:\n{result.code}")
elif isinstance(result, ExplanationOnly):
    print(f"概念: {result.concept}")
elif isinstance(result, Clarification):
    print(f"问题: {result.question}")
```

## ToolStrategy

### ToolStrategy 工作原理

```
ToolStrategy 流程：
┌──────────────────────────────────────┐
│  1. 将 Pydantic 类型转换为工具定义    │
│  2. 添加工具到模型调用               │
│  3. 模型调用工具返回结构化数据       │
│  4. 解析工具调用结果为 Pydantic 对象 │
└──────────────────────────────────────┘
```

### 基本 ToolStrategy

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model
from langchain.structured_output import ToolStrategy

class ProductInfo(BaseModel):
    """产品信息"""
    name: str = Field(description="产品名称")
    price: float = Field(description="价格")
    category: str = Field(description="分类")
    in_stock: bool = Field(description="是否有货")

# 使用 ToolStrategy
llm = init_chat_model("deepseek:deepseek-chat")
strategy = ToolStrategy(schema=ProductInfo)
structured_llm = llm.with_structured_output(ProductInfo, strategy=strategy)

result = structured_llm.invoke(
    "iPhone 15 Pro 的价格是 7999 元，属于手机类别，目前有货。"
)

print(f"产品名称: {result.name}")
print(f"价格: {result.price}")
print(f"分类: {result.category}")
print(f"是否有货: {result.in_stock}")
```

### ToolStrategy 自定义错误处理

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model
from langchain.structured_output import ToolStrategy
from langchain.messages import ToolMessage

class ExtractedEntity(BaseModel):
    """提取的实体"""
    entity_type: str = Field(description="实体类型", enum=["人名", "地名", "组织"])
    entity_value: str = Field(description="实体值")
    confidence: float = Field(description="置信度 0-1")

def handle_extraction_error(error: Exception) -> str:
    """自定义错误处理"""
    return f"提取失败: {str(error)}。请重新分析文本并提取实体。"

# 使用自定义错误处理
strategy = ToolStrategy(
    schema=ExtractedEntity,
    error_handler=handle_extraction_error,
)

llm = init_chat_model("deepseek:deepseek-chat")
structured_llm = llm.with_structured_output(ExtractedEntity, strategy=strategy)

result = structured_llm.invoke(
    "张三住在北京市朝阳区，在北京大学工作。"
)

print(f"实体类型: {result.entity_type}")
print(f"实体值: {result.entity_value}")
print(f"置信度: {result.confidence}")
```

## ProviderStrategy

### ProviderStrategy 工作原理

```
ProviderStrategy 流程：
┌──────────────────────────────────────┐
│  1. 使用提供商原生的结构化输出能力   │
│  2. 如 OpenAI 的 response_format     │
│  3. 或 Anthropic 的 tool_use         │
│  4. 或 Google 的 response_schema     │
└──────────────────────────────────────┘
```

### 基本 ProviderStrategy

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model
from langchain.structured_output import ProviderStrategy

class Recipe(BaseModel):
    """食谱"""
    name: str = Field(description="菜名")
    ingredients: list[str] = Field(description="食材列表")
    steps: list[str] = Field(description="步骤")
    cook_time_minutes: int = Field(description="烹饪时间（分钟）")

# 使用 ProviderStrategy（利用提供商原生能力）
llm = init_chat_model("openai:gpt-4o")
strategy = ProviderStrategy(schema=Recipe)
structured_llm = llm.with_structured_output(Recipe, strategy=strategy)

result = structured_llm.invoke("番茄炒蛋的做法")

print(f"菜名: {result.name}")
print(f"食材: {result.ingredients}")
print(f"步骤: {result.steps}")
print(f"时间: {result.cook_time_minutes} 分钟")
```

### Strict Mode

```python
from pydantic import BaseModel, Field
from langchain.chat_models import init_chat_model
from langchain.structured_output import ProviderStrategy

class ContactInfo(BaseModel):
    """联系信息"""
    name: str = Field(description="姓名")
    email: str = Field(description="邮箱")
    phone: str = Field(description="电话")

# Strict Mode：严格要求输出符合 schema
strategy = ProviderStrategy(
    schema=ContactInfo,
    strict=True,  # 启用严格模式
)

llm = init_chat_model("openai:gpt-4o")
structured_llm = llm.with_structured_output(ContactInfo, strategy=strategy)

result = structured_llm.invoke(
    "联系人：张三，邮箱：zhangsan@example.com，电话：13800138000"
)

print(f"姓名: {result.name}")
print(f"邮箱: {result.email}")
print(f"电话: {result.phone}")
```

### Strict Mode 对比

| 特性 | 非严格模式 | 严格模式 |
|------|-----------|---------|
| schema 遵守 | 尽量遵守 | 必须遵守 |
| 输出格式 | 可能偏离 schema | 严格符合 schema |
| 适用提供商 | 所有 | 仅支持严格模式的提供商 |
| 错误率 | 较低 | 可能较高（如果 schema 复杂） |
| 推荐场景 | 一般场景 | 关键数据提取 |

## 验证循环

### 自动验证重试

```python
from pydantic import BaseModel, Field, field_validator
from langchain.chat_models import init_chat_model

class ValidatedResponse(BaseModel):
    """带验证的响应"""
    answer: str = Field(description="回答内容")
    confidence: float = Field(description="置信度 0-1", ge=0, le=1)
    sources: list[str] = Field(description="引用来源", min_length=1)
    
    @field_validator("confidence")
    @classmethod
    def validate_confidence(cls, v: float) -> float:
        if v < 0 or v > 1:
            raise ValueError("置信度必须在 0-1 之间")
        return v
    
    @field_validator("sources")
    @classmethod
    def validate_sources(cls, v: list[str]) -> list[str]:
        if not v:
            raise ValueError("必须提供至少一个来源")
        return v

def validate_loop(llm, prompt, schema, max_retries=3):
    """验证循环：失败时重试"""
    structured_llm = llm.with_structured_output(schema)
    
    for attempt in range(max_retries):
        try:
            result = structured_llm.invoke(prompt)
            # 验证通过
            return result
        except Exception as e:
            print(f"验证失败 (尝试 {attempt + 1}/{max_retries}): {e}")
            if attempt == max_retries - 1:
                raise
            
            # 添加错误信息到提示，重试
            prompt = f"""
            {prompt}
            
            上一次尝试失败: {e}
            请确保输出符合 schema 要求。
            """
    
    raise ValueError(f"验证失败超过 {max_retries} 次")

# 使用
llm = init_chat_model("deepseek:deepseek-chat")
result = validate_loop(
    llm,
    "Python 中 list 和 tuple 有什么区别？",
    ValidatedResponse,
)

print(f"回答: {result.answer}")
print(f"置信度: {result.confidence}")
print(f"来源: {result.sources}")
```

### 自定义验证逻辑

```python
from pydantic import BaseModel, Field, model_validator
from langchain.chat_models import init_chat_model

class BusinessReport(BaseModel):
    """商业报告"""
    title: str = Field(description="报告标题")
    summary: str = Field(description="摘要")
    key_findings: list[str] = Field(description="关键发现", min_length=3)
    recommendations: list[str] = Field(description="建议", min_length=2)
    
    @model_validator(mode="after")
    def validate_report(self) -> "BusinessReport":
        """自定义验证逻辑"""
        if len(self.summary) < 50:
            raise ValueError("摘要至少需要 50 字")
        
        if len(self.key_findings) < 3:
            raise ValueError("至少需要 3 个关键发现")
        
        if len(self.recommendations) < 2:
            raise ValueError("至少需要 2 条建议")
        
        return self

# 使用验证循环
llm = init_chat_model("deepseek:deepseek-chat")

def generate_valid_report(prompt: str, max_retries=3):
    """生成并验证报告"""
    for attempt in range(max_retries):
        try:
            structured_llm = llm.with_structured_output(BusinessReport)
            result = structured_llm.invoke(prompt)
            return result
        except Exception as e:
            print(f"报告验证失败 (尝试 {attempt + 1}/{max_retries}): {e}")
            if attempt == max_retries - 1:
                raise
            
            prompt = f"""
            {prompt}
            
            上一次尝试不符合要求: {e}
            请确保：
            - 摘要至少 50 字
            - 至少 3 个关键发现
            - 至少 2 条建议
            """
    
    raise ValueError("生成有效报告失败")

# 使用
result = generate_valid_report("写一份关于 Python 市场的商业报告")
print(f"标题: {result.title}")
print(f"摘要: {result.summary[:100]}...")
print(f"关键发现: {result.key_findings}")
print(f"建议: {result.recommendations}")
```

## 复杂结构化输出

### 嵌套模型

```python
from pydantic import BaseModel, Field
from typing import list
from langchain.chat_models import init_chat_model

class Address(BaseModel):
    """地址"""
    street: str = Field(description="街道")
    city: str = Field(description="城市")
    province: str = Field(description="省份")
    postal_code: str = Field(description="邮编")

class Person(BaseModel):
    """人员信息"""
    name: str = Field(description="姓名")
    age: int = Field(description="年龄", ge=0, le=150)
    address: Address = Field(description="地址")
    phone: str = Field(description="电话")

class CompanyInfo(BaseModel):
    """公司信息"""
    company_name: str = Field(description="公司名称")
    employees: list[Person] = Field(description="员工列表")
    headquarters: Address = Field(description="总部地址")

# 使用嵌套模型
llm = init_chat_model("deepseek:deepseek-chat")
structured_llm = llm.with_structured_output(CompanyInfo)

result = structured_llm.invoke(
    "腾讯公司总部在深圳南山区，员工张三（30岁，住北京朝阳区）"
    "和李四（28岁，住上海浦东新区）。"
)

print(f"公司: {result.company_name}")
print(f"总部: {result.headquarters.city}")
for emp in result.employees:
    print(f"  员工: {emp.name}, 年龄: {emp.age}")
    print(f"    地址: {emp.address.city}")
```

from pydantic import BaseModel, Field

## 策略选择指南

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| 简单结构化输出 | ToolStrategy | 兼容所有提供商 |
| 需要原生性能 | ProviderStrategy | 利用提供商优化 |
| 关键数据提取 | ProviderStrategy + Strict | 最高准确性 |
| 复杂嵌套模型 | ToolStrategy | 更好的灵活性 |
| Union 类型 | ToolStrategy | 工具调用天然支持 |
| 验证循环 | 自定义 + ToolStrategy | 灵活错误处理 |

---

## 策略对比

### ToolStrategy vs ProviderStrategy

| 特性 | ToolStrategy | ProviderStrategy |
|------|-------------|-----------------|
| 兼容性 | 所有 LLM 提供商 | 仅支持结构化输出的提供商 |
| 实现方式 | 工具调用模拟 | 原生 response_format |
| 速度 | 稍慢（多一次工具调用） | 快（原生支持） |
| 成本 | 稍高 | 较低 |
| 严格模式 | 不支持 | 支持（部分提供商） |
| 复杂类型 | 支持好 | 支持好 |

### 自动策略选择

```python
from langchain.chat_models import init_chat_model
from langchain.structured_output import ToolStrategy, ProviderStrategy

def smart_structured_output(llm, schema, strict=False):
    """智能选择策略"""
    # 检查是否支持 ProviderStrategy
    if hasattr(llm, 'supports_structured_output') and llm.supports_structured_output:
        strategy = ProviderStrategy(schema=schema, strict=strict)
        print("使用 ProviderStrategy（原生）")
    else:
        strategy = ToolStrategy(schema=schema)
        print("使用 ToolStrategy（兼容模式）")
    
    return llm.with_structured_output(schema, strategy=strategy)

# 使用
llm = init_chat_model("deepseek:deepseek-chat")
structured_llm = smart_structured_output(llm, TaskList)
result = structured_llm.invoke("列出今天的任务")
```

## 小结

| 要点 | 说明 |
|------|------|
| Union 类型 | `SchemaA | SchemaB` 支持多种输出格式 |
| ToolStrategy | 通过工具调用实现结构化输出 |
| ProviderStrategy | 利用提供商原生结构化能力 |
| Strict Mode | 严格要求输出符合 schema |
| 验证循环 | 失败时自动重试并修复 |
| 嵌套模型 | Pydantic 嵌套定义复杂结构 |
| 策略选择 | 根据兼容性和性能需求选择 |
