# 09.04.01 - 中间件概述

> LangChain Agent 中间件系统：拦截和增强 Agent 执行的核心机制

## 为什么需要中间件

在 Agent 执行过程中，我们经常需要在不修改核心逻辑的情况下：

```
需求场景：
┌──────────────────────────────────────┐
│  日志追踪    → 记录每次模型调用       │
│  输入过滤    → 检测敏感词/PII         │
│  输出过滤    → 格式化响应内容         │
│  模型切换    → 根据复杂度选择模型     │
│  工具控制    → 动态启用/禁用工具      │
│  人工审核    → 关键操作需要确认       │
│  自动摘要    → 长对话压缩上下文       │
└──────────────────────────────────────┘
```

中间件（Middleware）提供了一套钩子（hooks），让我们在每个执行阶段插入自定义逻辑。

---

## Agent 执行循环（Agent Loop）

理解中间件的前提是理解 Agent 的执行流程：

```
Agent Loop：
┌──────────────────────────────────────┐
│                                       │
│  ┌──────────────────────────────┐    │
│  │  @before_agent               │    │
│  │  拦截输入，可修改 State       │    │
│  └──────────────┬───────────────┘    │
│                 │                     │
│  ┌──────────────▼───────────────┐    │
│  │  @wrap_model_call            │    │
│  │  包装模型调用                 │    │
│  │  ├─ 修改 prompt/messages     │    │
│  │  ├─ 修改 tools               │    │
│  │  ├─ 修改 model               │    │
│  │  └─ 修改 response_format     │    │
│  └──────────────┬───────────────┘    │
│                 │                     │
│  ┌──────────────▼───────────────┐    │
│  │  model.invoke(prompt)        │    │
│  │  ↓                           │    │
│  │  返回 AI 响应                │    │
│  │  ├─ 最终回复                  │    │
│  │  └─ 工具调用请求              │    │
│  └──────────────┬───────────────┘    │
│                 │                     │
│         有工具调用？                   │
│         ┌───┴────┐                   │
│        是        否                   │
│         │        │                     │
│  ┌──────▼──┐   ┌▼──────────────┐    │
│  │@wrap_   │   │@after_agent   │    │
│  │tool_call│   │处理最终响应    │    │
│  └─────┬───┘   └───────────────┘    │
│        │                             │
│  ┌─────▼───┐                         │
│  │tool.    │                         │
│  │invoke() │                         │
│  └─────┬───┘                         │
│        │                             │
│        └───── 循环回到 @before_agent  │
│                                       │
└──────────────────────────────────────┘
```

**执行流程说明：**

| 步骤 | 说明 | 中间件钩子 |
|------|------|-----------|
| 1 | Agent 接收用户输入 | `@before_agent` |
| 2 | 构建模型请求（prompt + tools） | `@wrap_model_call` |
| 3 | 调用 LLM | 模型内部 |
| 4 | 解析响应（文本 or 工具调用） | `@after_model` |
| 5 | 如有工具调用，执行工具 | `@wrap_tool_call` |
| 6 | 工具结果返回给模型 | 循环到步骤 2 |
| 7 | 无工具调用，返回最终响应 | `@after_agent` |

## 中间件钩子总览

LangChain 提供四种核心钩子：

### @before_agent

在 Agent 每次迭代开始前执行：

```python
from langchain.agents.middleware import before_agent

@before_agent
def log_input(state, runtime):
    """记录每次用户输入"""
    last_message = state["messages"][-1]
    print(f"用户输入: {last_message.content[:100]}")
    # 返回 None 继续执行，返回 dict 修改 State
    return None
```

**用途：**
- 输入验证和过滤
- 日志记录
- 注入额外上下文到 State
- 提前终止（返回修改后的 State）

### @after_agent

在 Agent 完成所有迭代后执行：

```python
from langchain.agents.middleware import after_agent

@after_agent
def log_output(state, runtime):
    """记录最终输出"""
    last_message = state["messages"][-1]
    print(f"AI 回复: {last_message.content[:100]}")
    return None
```

**用途：**
- 输出格式化和过滤
- PII 检测和处理
- 最终日志记录
- 跳转到特定节点（`can_jump_to`）

### @wrap_model_call

包装模型调用，可在调用前后执行逻辑：

```python
from langchain.agents.middleware import wrap_model_call, ModelRequest, ModelResponse

@wrap_model_call
def log_model_call(request: ModelRequest, handler) -> ModelResponse:
    """记录模型调用详情"""
    print(f"调用模型: {request.model}")
    print(f"消息数量: {len(request.state['messages'])}")
    print(f"可用工具: {[t.name for t in request.tools]}")
    
    # 调用实际模型
    response = handler(request)
    
    print(f"响应内容: {response.content[:100]}...")
    return response
```

**用途：**
- 动态模型选择
- 动态工具过滤
- 动态 response_format
- 请求/响应日志
- 缓存

### @wrap_tool_call

包装工具调用：

```python
from langchain.agents.middleware import wrap_tool_call, ToolCallRequest

@wrap_tool_call
def log_tool_call(request: ToolCallRequest, handler):
    """记录工具调用"""
    print(f"调用工具: {request.tool_call['name']}")
    print(f"工具参数: {request.tool_call['args']}")
    
    try:
        result = handler(request)
        print(f"工具结果: {result[:200]}")
        return result
    except Exception as e:
        print(f"工具错误: {e}")
        raise
```

**用途：**
- 工具调用日志
- 错误处理和重试
- 工具权限控制
- 工具结果缓存

## 预置中间件列表

LangChain 内置多种常用中间件：

### SummarizationMiddleware

自动摘要长对话，防止超出上下文窗口：

```python
from langchain.agents.middleware import SummarizationMiddleware

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[search_tool, calc_tool],
    middleware=[
        SummarizationMiddleware(
            model="moonshot:moonshot-v1-8k",
            token_limit=4000,    # 超过此 token 数触发摘要
            max_tokens=500,      # 摘要最大 token
        ),
    ],
)
```

**工作原理：**

```
SummarizationMiddleware 流程：
┌──────────────────────────────────────┐
│  每次 @before_agent 执行时：          │
│                                       │
│  1. 计算当前消息总 token 数            │
│  2. 如果超过 token_limit：            │
│     ├─ 保留系统消息                   │
│     ├─ 保留最近 N 条消息              │
│     └─ 其余消息压缩为摘要             │
│  3. 用摘要替换旧消息                  │
└──────────────────────────────────────┘
```

**配置选项：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `model` | `str` | 必需 | 用于生成摘要的模型 |
| `token_limit` | `int` | 4000 | 触发摘要的 token 阈值 |
| `max_tokens` | `int` | 500 | 摘要最大 token 数 |
| `strategy` | `str` | `"last"` | 摘要策略（last/first） |

### HumanInTheLoopMiddleware

在指定工具调用前暂停，等待人工审核：

```python
from langchain.agents.middleware import HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[send_email, delete_file, search_web],
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={
                "send_email": True,      # 发送邮件需审核
                "delete_file": True,     # 删除文件需审核
                "search_web": False,     # 搜索无需审核
            }
        ),
    ],
    checkpointer=InMemorySaver(),  # 需要 checkpointer 支持中断
)

# 第一次调用 - 遇到审核工具会中断
config = {"configurable": {"thread_id": "session-1"}}
result = agent.invoke(
    {"messages": [{"role": "user", "content": "发送邮件给 boss"}]},
    config,
)

# 检查是否中断
if result.get("__interrupt__"):
    print("等待审核...")
    # 审核通过
    result = agent.invoke(Command(resume={"type": "approve"}), config)
```

**审核流程：**

```
HumanInTheLoopMiddleware 流程：
┌──────────────────────────────────────┐
│  用户: "发送邮件给 boss"              │
│       ↓                              │
│  Agent 决定调用 send_email            │
│       ↓                              │
│  Middleware 检测到 interrupt_on=True  │
│       ↓                              │
│  中断执行，保存状态                    │
│       ↓                              │
│  返回中断信息给调用者                 │
│       ↓                              │
│  调用者显示审核界面                   │
│       ↓                              │
│  用户批准/拒绝                        │
│       ↓                              │
│  Command(resume=...) 恢复执行         │
└──────────────────────────────────────┘
```

### ModelFallbackMiddleware

模型调用失败时自动降级到备用模型：

```python
from langchain.agents.middleware import ModelFallbackMiddleware

agent = create_agent(
    model="deepseek:deepseek-chat",
    tools=[search_tool],
    middleware=[
        ModelFallbackMiddleware(
            fallbacks=[
                "moonshot:moonshot-v1-8k",
                "openai:gpt-4o-mini",
            ],
        ),
    ],
)
```

**降级策略：**

```
ModelFallbackMiddleware：
┌──────────────────────────────────────┐
│  主模型: deepseek-chat               │
│       ↓ 失败                         │
│  备用1: moonshot-v1-8k               │
│       ↓ 失败                         │
│  备用2: gpt-4o-mini                  │
│       ↓ 失败                         │
│  抛出最终错误                         │
└──────────────────────────────────────┘
```

### ToolRetryMiddleware

工具调用失败时自动重试：

```python
from langchain.agents.middleware import ToolRetryMiddleware

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[api_tool],
    middleware=[
        ToolRetryMiddleware(
            max_retries=3,             # 最大重试次数
            backoff_factor=2,          # 退避因子
        ),
    ],
)
```

## 中间件执行顺序

多个中间件按添加顺序执行：

```python
agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[tool1, tool2],
    middleware=[
        LoggingMiddleware(),          # 1. 最先执行
        InputGuardrailMiddleware(),   # 2. 输入过滤
        SummarizationMiddleware(),    # 3. 上下文摘要
        HumanInTheLoopMiddleware(),   # 4. 人工审核
    ],
)
```

**执行顺序图解：**

```
中间件链式调用：
┌──────────────────────────────────────┐
│  @before_agent 执行顺序：             │
│  LoggingMiddleware.before_agent       │
│    ↓                                  │
│  InputGuardrailMiddleware.before_agent│
│    ↓                                  │
│  SummarizationMiddleware.before_agent │
│    ↓                                  │
│  HumanInTheLoopMiddleware.before_agent│
│    ↓                                  │
│  实际 Agent 执行                     │
│    ↓                                  │
│  @after_agent 执行顺序（反向）：       │
│  HumanInTheLoopMiddleware.after_agent │
│    ↓                                  │
│  SummarizationMiddleware.after_agent  │
│    ↓                                  │
│  InputGuardrailMiddleware.after_agent │
│    ↓                                  │
│  LoggingMiddleware.after_agent        │
└──────────────────────────────────────┘
```

**重要规则：**
- `@before_agent` 按声明顺序执行
- `@after_agent` 按相反顺序执行
- `@wrap_model_call` 嵌套执行（洋葱模型）
- `@wrap_tool_call` 嵌套执行（洋葱模型）

## 中间件 vs LangGraph 节点

| 特性 | Middleware | LangGraph Node |
|------|-----------|----------------|
| 作用范围 | 每次 Agent 迭代 | 图的特定节点 |
| 修改能力 | State/Model/Tools | 仅 State |
| 适用场景 | 横切关注点 | 业务逻辑 |
| 组合性 | 多个中间件链式 | 节点按边连接 |
| 调试难度 | 较复杂（嵌套） | 较简单（线性） |

**选择建议：**

| 需求 | 推荐方案 |
|------|---------|
| 日志追踪 | Middleware |
| 输入验证 | Middleware |
| 动态模型选择 | Middleware |
| 业务流程控制 | LangGraph Node |
| 条件分支 | LangGraph Node |
| 并行执行 | LangGraph Node |

## 中间件与 State 的关系

中间件可以读取和修改 Agent 的 State：

```python
from langchain.agents.middleware import before_agent

@before_agent
def inject_context(state, runtime):
    """在 State 中注入额外上下文"""
    # 读取当前 State
    message_count = len(state["messages"])
    
    # 修改 State（返回 dict）
    if message_count > 5:
        return {
            "context": "用户已进行多轮对话，请保持上下文连贯"
        }
    
    return None  # 不修改 State
```

**State 修改规则：**

| 返回值 | 效果 |
|--------|------|
| `None` | 不修改 State，继续执行 |
| `dict` | 合并到当前 State |
| 抛出异常 | 终止执行 |

## 中间件与 Runtime 的关系

Runtime 提供运行时上下文：

```python
from langchain.agents.middleware import before_agent
from dataclasses import dataclass

@dataclass
class AppContext:
    user_id: str
    user_role: str

@before_agent
def check_permissions(state, runtime):
    """检查用户权限"""
    user_role = runtime.context.user_role
    
    if user_role == "guest":
        # 游客只能使用公开工具
        return {"allowed_tools": ["public_search"]}
    
    return None
```

**Runtime 可用信息：**

| 属性 | 说明 |
|------|------|
| `runtime.context` | 自定义上下文对象 |
| `runtime.store` | 长期记忆存储 |
| `runtime.config` | 运行配置 |

## 最佳实践

### 1. 中间件职责单一

```python
# ✅ 好的做法：每个中间件负责一件事
@before_agent
def log_input(state, runtime):
    ...

@before_agent
def validate_input(state, runtime):
    ...

# ❌ 不好的做法：一个中间件做所有事
@before_agent
def do_everything(state, runtime):
    log_input(state)
    validate_input(state)
    inject_context(state)
    ...
```

### 2. 中间件顺序合理

```python
# ✅ 推荐顺序
middleware = [
    LoggingMiddleware(),          # 日志最先
    InputGuardrailMiddleware(),   # 安全验证
    SummarizationMiddleware(),    # 上下文管理
    CustomMiddleware(),           # 业务逻辑
    HumanInTheLoopMiddleware(),   # 人工审核最后
]
```

### 3. 错误处理

```python
@wrap_tool_call
def safe_tool_call(request, handler):
    try:
        return handler(request)
    except Exception as e:
        # 记录错误
        print(f"工具调用失败: {e}")
        # 返回友好错误信息
        from langchain.messages import ToolMessage
        return ToolMessage(
            content="工具调用失败，请稍后重试",
            tool_call_id=request.tool_call["id"],
        )
```

## 小结

| 要点 | 说明 |
|------|------|
| Agent Loop | before → wrap_model → model → wrap_tool → tool → 循环 |
| 四个钩子 | `@before_agent`, `@after_agent`, `@wrap_model_call`, `@wrap_tool_call` |
| 预置中间件 | SummarizationMiddleware, HumanInTheLoopMiddleware, ModelFallbackMiddleware, ToolRetryMiddleware |
| 执行顺序 | before 按顺序，after 反顺序，wrap 嵌套 |
| State 修改 | 返回 dict 合并，返回 None 不修改 |
| 选择原则 | 横切关注点用 Middleware，业务流程用 LangGraph Node |
