# 09.05.03 - 中断与审核

> interrupt() 函数、Command(resume=)、审批模式、审核和编辑

## 中断概述

中断（Interrupt）暂停图执行，等待外部输入（Human-in-the-loop）：

```
中断流程：
┌──────────────────────────────────────┐
│  执行 → interrupt() → 暂停           │
│           ↓                          │
│  外部输入 → Command(resume) → 继续   │
└──────────────────────────────────────┘
```

### 中断使用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| 审批 | 关键操作需要人工批准 | 转账、删除数据 |
| 纠正 | 人工纠正 AI 输出 | 修改回复内容 |
| 确认 | 确认敏感操作 | 发送邮件、发布 |
| 验证 | 验证用户输入 | 年龄、邮箱确认 |
| 编辑 | 编辑中间状态 | 修改生成的内容 |

## interrupt() 函数

### 基本用法

```python
from langgraph.types import interrupt

def approval_node(state: State):
    """审批节点"""
    approved = interrupt("是否批准此操作？")
    
    if approved:
        return {"status": "approved"}
    return {"status": "rejected"}
```

### interrupt() 执行过程

| 步骤 | 说明 |
|------|------|
| 1 | 图执行暂停在 interrupt() 调用点 |
| 2 | State 通过 checkpointer 保存 |
| 3 | interrupt() 的值返回给调用者 |
| 4 | 图等待，直到 Command(resume) 恢复 |
| 5 | resume 值成为 interrupt() 的返回值 |

### 传递复杂数据

```python
from langgraph.types import interrupt

def review_node(state: State):
    """审核节点"""
    result = interrupt({
        "question": "请审核此内容",
        "content": state["generated_content"],
        "options": ["approve", "reject", "edit"],
    })
    
    if result == "approve":
        return {"status": "approved"}
    elif result == "reject":
        return {"status": "rejected"}
    elif result == "edit":
        # 获取编辑后的内容
        edited = interrupt("请输入编辑后的内容")
        return {"content": edited, "status": "edited"}
```

## 最简审批示例

```python
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, START, END
from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import MemorySaver

class State(TypedDict):
    action: str
    status: Literal["pending", "approved", "rejected"]

def approval_node(state: State) -> Command[Literal["proceed", "cancel"]]:
    """审批节点"""
    decision = interrupt({
        "question": "是否批准？",
        "action": state["action"],
    })
    
    return Command(goto="proceed" if decision else "cancel")

def proceed_node(state: State):
    return {"status": "approved"}

def cancel_node(state: State):
    return {"status": "rejected"}

# 构建图
builder = StateGraph(State)
builder.add_node("approval", approval_node)
builder.add_node("proceed", proceed_node)
builder.add_node("cancel", cancel_node)
builder.add_edge(START, "approval")
builder.add_edge("proceed", END)
builder.add_edge("cancel", END)

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "approval-123"}}

# 第一次调用 - 中断
initial = graph.invoke(
    {"action": "转账100元", "status": "pending"},
    config
)
print(initial["__interrupt__"])
# 输出: (Interrupt(value={'question': '是否批准？', 'action': '转账100元'}),)

# 恢复执行（批准）
resumed = graph.invoke(Command(resume=True), config)
print(resumed["status"])  # "approved"
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `interrupt()` | 暂停执行 | 等待人工输入 |
| `Command(resume=True)` | 恢复执行 | 传入回答继续 |
| `checkpointer` | 状态保存 | 中断时保存状态 |
| `thread_id` | 会话标识 | 区分不同对话 |

## 审核和编辑模式

### 审核模式

```python
from langgraph.types import interrupt

def content_review(state: State):
    """内容审核"""
    generated = state["generated_content"]
    
    review_result = interrupt({
        "type": "review",
        "content": generated,
        "question": "内容是否合适？",
        "options": ["approve", "reject", "modify"],
    })
    
    if review_result == "approve":
        return {"status": "approved"}
    elif review_result == "reject":
        return {"status": "rejected"}
    elif review_result == "modify":
        # 进入编辑模式
        modified = interrupt("请输入修改后的内容")
        return {"generated_content": modified, "status": "modified"}
    
    return {"status": "unknown"}
```

### 编辑模式

```python
def edit_node(state: State):
    """编辑节点"""
    current = state["content"]
    
    # 显示当前内容
    edit_request = interrupt({
        "type": "edit",
        "current": current,
        "question": "请编辑内容",
    })
    
    # edit_request 可以是完整的新内容
    # 也可以是编辑指令
    if isinstance(edit_request, dict) and "action" in edit_request:
        if edit_request["action"] == "replace":
            return {"content": edit_request["new_content"]}
        elif edit_request["action"] == "append":
            return {"content": current + edit_request["text"]}
    
    # 直接返回新内容
    return {"content": edit_request}
```

## 工具中的中断

```python
from langchain.tools import tool
from langgraph.types import interrupt

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """发送邮件"""
    # 中断等待审核
    response = interrupt({
        "action": "send_email",
        "to": to,
        "subject": subject,
        "body": body,
        "message": "批准发送？",
    })
    
    if response.get("action") == "approve":
        # 允许覆盖参数
        final_to = response.get("to", to)
        final_subject = response.get("subject", subject)
        return f"邮件已发送给 {final_to}"
    
    return "邮件发送已取消"

# 恢复时可覆盖参数
graph.invoke(
    Command(resume={
        "action": "approve",
        "subject": "更新后的主题",
    }),
    config=config,
)
```

## 验证输入循环

```python
def get_age_node(state: State):
    """获取年龄（带验证）"""
    prompt = "请输入年龄"
    
    while True:
        answer = interrupt(prompt)
        
        if isinstance(answer, int) and answer > 0:
            return {"age": answer}
        
        prompt = f"'{answer}' 不是有效年龄，请输入正整数"
```

## 流式处理中断

```python
async def handle_with_interrupts(graph, initial_input, config):
    """流式处理中断"""
    current_input = initial_input
    
    while True:
        async for chunk in graph.astream(
            current_input,
            stream_mode=["messages", "updates"],
            config=config,
            version="v2",
        ):
            if chunk["type"] == "updates":
                if "__interrupt__" in chunk["data"]:
                    interrupt_info = chunk["data"]["__interrupt__"][0].value
                    
                    # 获取用户响应
                    user_response = get_user_input(interrupt_info)
                    current_input = Command(resume=user_response)
                    break
        else:
            # 正常完成
            break
```

## 中断规则

### 注意事项

| 规则 | 说明 |
|------|------|
| 不要 try/except 包裹 | interrupt 通过异常实现，会被捕获 |
| 不要条件跳过 interrupt | 恢复时顺序严格基于索引 |
| 不要传递复杂对象 | 仅 JSON 可序列化值 |
| interrupt 前的操作应幂等 | 恢复时节点重新执行 |

```python
def safe_node(state: State):
    """安全的中断节点"""
    # interrupt 前的操作应该幂等
    # 恢复时会重新执行整个节点
    
    # ✅ 好的做法：先计算，再中断
    result = expensive_calculation(state)
    approved = interrupt("批准此结果？")
    
    if approved:
        # 幂等操作
        db.upsert_user(user_id=state["user_id"], status="approved")
    
    return {"approved": approved, "result": result}

def unsafe_node(state: State):
    """不安全的中断节点"""
    # ❌ 不好的做法：中断后执行非幂等操作
    approved = interrupt("批准？")
    
    if approved:
        db.insert_record(...)  # 恢复时会重复插入
```

## 静态中断（调试）

`interrupt_before` / `interrupt_after`：用于调试，不推荐用于 HITL。

```python
graph = builder.compile(
    interrupt_before=["node_a"],  # 在 node_a 之前暂停
    interrupt_after=["node_b"],   # 在 node_b 之后暂停
    checkpointer=checkpointer,
)

graph.invoke(inputs, config=config)  # 在 node_a 前暂停
graph.invoke(None, config=config)    # 继续
```

## HumanInTheLoopMiddleware

LangChain Agent 预置中间件：

```python
from langchain.agents import create_agent
from langchain.agents.middleware import HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import MemorySaver

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[send_email, delete_file],
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={
                "send_email": True,
                "delete_file": True,
            }
        ),
    ],
    checkpointer=MemorySaver(),
)
```

## 中断总结

| 要点 | 说明 |
|------|------|
| interrupt() | 动态暂停，可在任意位置 |
| Command(resume) | 恢复执行，值成为 interrupt 返回值 |
| checkpointer | 必需，保存中断状态 |
| thread_id | 必需，标识会话 |
| 多中断 | 用 ID 映射恢复值（见 04-多中断处理） |
| 工具中断 | 在 tool 内调用 interrupt |
| 审核模式 | approve/reject/modify |
| 编辑模式 | 修改中间状态 |
| 规则 | 幂等操作、JSON 可序列化、不跳过 |
