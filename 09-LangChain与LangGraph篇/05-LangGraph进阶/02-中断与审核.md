# 09.05.02 - 中断与审核

**Interrupt：** 暂停图执行，等待外部输入（Human-in-the-loop）。

```
中断流程：
┌──────────────────────────────────────┐
│  执行 → interrupt() → 暂停           │
│           ↓                          │
│  外部输入 → Command(resume) → 继续   │
└──────────────────────────────────────┘
```

### 为什么用它？

- 审批关键操作（转账、删除）
- 人工纠正 AI 输出
- 确认敏感操作
- 验证用户输入

---

### interrupt() 函数

```python
from langgraph.types import interrupt

def approval_node(state: State):
    approved = interrupt("是否批准此操作？")
    
    if approved:
        return {"status": "approved"}
    return {"status": "rejected"}
```

**interrupt() 执行过程：**

| 步骤 | 说明 |
|------|------|
| 1 | 图执行暂停在 interrupt() 调用点 |
| 2 | State 通过 checkpointer 保存 |
| 3 | interrupt() 的值返回给调用者 |
| 4 | 图等待，直到 Command(resume) 恢复 |
| 5 | resume 值成为 interrupt() 的返回值 |

---

### 最简示例：审批流程

```python
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, START, END
from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import MemorySaver

class State(TypedDict):
    action: str
    status: Literal["pending", "approved", "rejected"]

def approval_node(state: State) -> Command[Literal["proceed", "cancel"]]:
    decision = interrupt({
        "question": "是否批准？",
        "action": state["action"],
    })
    
    return Command(goto="proceed" if decision else "cancel")

def proceed_node(state: State):
    return {"status": "approved"}

def cancel_node(state: State):
    return {"status": "rejected"}

builder = StateGraph(State)
builder.add_node("approval", approval_node)
builder.add_node("proceed", proceed_node)
builder.add_node("cancel", cancel_node)
builder.add_edge(START, "approval")
builder.add_edge("proceed", END)
builder.add_edge("cancel", END)

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "approval-123"}}
initial = graph.invoke({"action": "转账100元", "status": "pending"}, config)
print(initial["__interrupt__"])

resumed = graph.invoke(Command(resume=True), config)
print(resumed["status"])  # "approved"
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `interrupt()` | 暂停执行 | 等待人工输入 |
| `Command(resume=True)` | 恢复执行 | 传入回答继续 |
| `checkpointer` | 状态保存 | 中断时保存状态 |
| `thread_id` | 会话标识 | 区分不同对话 |

### 多中断处理

**并行节点同时中断：**

```python
from typing import Annotated, TypedDict
from operator import add

class State(TypedDict):
    vals: Annotated[list[str], add]

def node_a(state):
    answer = interrupt("question_a")
    return {"vals": [f"a:{answer}"]}

def node_b(state):
    answer = interrupt("question_b")
    return {"vals": [f"b:{answer}"]}

builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_edge(START, "a")
builder.add_edge(START, "b")  # 并行执行
builder.add_edge("a", END)
builder.add_edge("b", END)

graph = builder.compile(checkpointer=MemorySaver())
config = {"configurable": {"thread_id": "1"}}
result = graph.invoke({"vals": []}, config)

# 处理多个中断
resume_map = {
    i.id: f"answer for {i.value}"
    for i in result["__interrupt__"]
}
result = graph.invoke(Command(resume=resume_map), config)
```

### 工具中的中断

```python
from langchain.tools import tool
from langgraph.types import interrupt

@tool
def send_email(to: str, subject: str, body: str):
    """发送邮件"""
    response = interrupt({
        "action": "send_email",
        "to": to,
        "subject": subject,
        "body": body,
        "message": "批准发送？",
    })
    
    if response.get("action") == "approve":
        final_to = response.get("to", to)
        return f"邮件已发送给 {final_to}"
    return "邮件已取消"
```

**恢复时可覆盖参数：**

```python
graph.invoke(
    Command(resume={"action": "approve", "subject": "更新后的主题"}),
    config=config,
)
```

### 验证输入循环

```python
def get_age_node(state: State):
    prompt = "请输入年龄"
    
    while True:
        answer = interrupt(prompt)
        
        if isinstance(answer, int) and answer > 0:
            return {"age": answer}
        
        prompt = f"'{answer}' 不是有效年龄，请输入正数"
```

### interrupt 规则

**注意事项：**

| 规则 | 说明 |
|------|------|
| 不要 try/except 包裹 | interrupt 通过异常实现，会被捕获 |
| 不要条件跳过 interrupt | 恢复时顺序严格基于索引 |
| 不要传递复杂对象 | 仅 JSON 可序列化值 |
| interrupt 前的操作应幂等 | 恢复时节点重新执行 |

```python
def node_a(state: State):
    approved = interrupt("批准？")
    
    if approved:
        db.upsert_user(user_id=state["user_id"], status="approved")
    
    return {"approved": approved}
```

### 静态中断（调试）

**interrupt_before / interrupt_after：** 用于调试，不推荐用于 HITL。

```python
graph = builder.compile(
    interrupt_before=["node_a"],
    interrupt_after=["node_b"],
    checkpointer=checkpointer,
)

graph.invoke(inputs, config=config)  # 在 node_a 前暂停
graph.invoke(None, config=config)    # 继续
```

### HumanInTheLoopMiddleware

**LangChain Agent 预置中间件：**

```python
from langchain.agents import create_agent
from langchain.agents.middleware import HumanInTheLoopMiddleware

agent = create_agent(
    model="moonshot:moonshot-v1-8k",
    tools=[send_email, delete_file],
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={"send_email": True, "delete_file": True}
        ),
    ],
    checkpointer=MemorySaver(),
)
```

### 中断总结

| 要点 | 说明 |
|------|------|
| interrupt() | 动态暂停，可在任意位置 |
| Command(resume) | 恢复执行，值成为 interrupt 返回值 |
| checkpointer | 必需，保存中断状态 |
| thread_id | 必需，标识会话 |
| 多中断 | 用 ID 映射恢复值 |
| 工具中断 | 在 tool 内调用 interrupt |
