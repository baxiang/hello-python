# 09.05.01 - 记忆管理

**记忆系统：** Agent 执行过程中的上下文保存机制，分为短期记忆（对话内）和长期记忆（跨对话）。

```
记忆架构：
┌──────────────────────────────────────┐
│  短期记忆 (State)                     │
│  ├─ 当前对话上下文                    │
│  ├─ thread-scoped                    │
│  ├─ Checkpointer 持久化               │
│                                       │
│  长期记忆 (Store)                     │
│  ├─ 跨对话共享                        │
│  ├─ 自定义 namespace                  │
│  ├─ semantic/episodic/procedural     │
└──────────────────────────────────────┘
```

---

### 为什么需要记忆？

- 保持多轮对话上下文
- 记住用户偏好和历史
- 跨会话个性化服务
- 从过往经验学习

---

## 短期记忆 (State)

短期记忆是图执行过程中的共享状态，通过 Checkpointer 持久化到特定 thread。

```
短期记忆：
┌──────────────────────────────────────┐
│  State     → 图状态定义               │
│  Thread    → 会话标识                 │
│  Checkpointer → 状态持久化            │
│                                       │
│  invoke → 更新 State                  │
│  resume → 从 checkpoint 恢复          │
└──────────────────────────────────────┘
```

### MessagesState

LangGraph 提供预置的 `MessagesState`，自动处理消息追加：

```python
from langgraph.graph import MessagesState

class State(MessagesState):
    user_id: str          # 用户标识
    documents: list[str]  # 相关文档
    summary: str          # 当前摘要
```

**MessagesState 内置：**
- `messages: Annotated[list[AnyMessage], add_messages]`
- 自动追踪消息 ID
- 支持消息更新和追加

### 最简示例：带记忆的对话

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.checkpoint.memory import MemorySaver
from langchain_openai import ChatOpenAI

def chat_node(state: MessagesState):
    model = ChatOpenAI(model="gpt-4o-mini")
    response = model.invoke(state["messages"])
    return {"messages": [response]}

builder = StateGraph(MessagesState)
builder.add_node("chat", chat_node)
builder.add_edge(START, "chat")
builder.add_edge("chat", END)

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 同一个 thread 保持上下文
config = {"configurable": {"thread_id": "user-123"}}

# 第一轮
result1 = graph.invoke({"messages": ["你好"]}, config)

# 第二轮（记住上一轮）
result2 = graph.invoke({"messages": ["我们之前聊了什么？"]}, config)
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `MessagesState` | 消息状态 | 预置类型，自动处理消息追加 |
| `MemorySaver()` | 内存持久化 | 开发用，生产用 SQLite/PostgreSQL |
| `thread_id` | 会话标识 | 区分不同对话 |
| `checkpointer` | 状态保存 | 中断和恢复依赖它 |

### 消息管理策略

长对话会超出模型上下文窗口，需要管理策略：

```
管理策略：
┌──────────────────────────────────────┐
│  Trimming   → 截断旧消息              │
│  Summarizing → 摘要压缩               │
│  Sliding    → 滑动窗口                │
└──────────────────────────────────────┘
```

### Trimming：截断策略

限制消息数量：

```python
from langchain.messages import trim_messages

trimmed = trim_messages(
    messages,
    max_tokens=4000,      # 最大 token
    strategy="last",      # 保留最近的
    token_counter=len,    # 计数器
)
```

### Summarizing：摘要策略

将旧消息压缩为摘要：

```python
def summarize_node(state: State):
    if len(state["messages"]) > 10:
        summary = model.invoke("总结以下对话: " + str(state["messages"][:5]))
        return {
            "messages": [SystemMessage(f"历史摘要: {summary.content}")] + state["messages"][-5:]
        }
    return state
```

## 长期记忆 (Store)

长期记忆跨对话共享，存储在自定义 namespace 中。

```
长期记忆结构：
┌──────────────────────────────────────┐
│  Store.put(namespace, key, value)    │
│  Store.get(namespace, key)           │
│  Store.search(namespace, query)      │
│                                       │
│  namespace → 类似文件夹               │
│  key      → 类似文件名                │
│  value    → JSON 文档                 │
└──────────────────────────────────────┘
```

### 最简示例：保存用户信息

```python
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()

# 存储用户偏好
namespace = ("user", "123", "preferences")
store.put(namespace, "profile", {
    "language": "zh-CN",
    "interests": ["Python", "AI"],
})

# 读取用户偏好
profile = store.get(namespace, "profile")
print(profile.value)
```

### 详细示例：带长期记忆的 Agent

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.store.memory import InMemoryStore
from langgraph.checkpoint.memory import MemorySaver
from langchain_openai import ChatOpenAI
from langchain.messages import HumanMessage

store = InMemoryStore()

def chat_node(state: MessagesState, store: InMemoryStore):
    # 第一步：获取用户历史偏好
    user_id = state.get("user_id", "default")
    preferences = store.get(("user", user_id), "preferences")
    
    # 第二步：构建个性化上下文
    context = ""
    if preferences:
        context = f"用户偏好: {preferences.value}"
    
    # 第三步：调用模型
    model = ChatOpenAI(model="gpt-4o-mini")
    messages = state["messages"]
    if context:
        messages = [HumanMessage(context)] + messages
    
    response = model.invoke(messages)
    
    # 第四步：更新用户偏好（如需要）
    if "我喜欢" in state["messages"][-1].content:
        store.put(("user", user_id), "preferences", {"likes": "X"})
    
    return {"messages": [response]}

builder = StateGraph(MessagesState)
builder.add_node("chat", chat_node)
builder.add_edge(START, "chat")
builder.add_edge("chat", END)

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer, store=store)
```

## 记忆类型详解

LangGraph 支持三种记忆类型，类比人类记忆：

### Semantic Memory：语义记忆

存储事实和知识：

```
语义记忆：
┌──────────────────────────────────────┐
│  存储内容 → 事实、概念               │
│  人类类比 → 学校学到的知识           │
│  Agent 应用 → 用户偏好、事实         │
│                                       │
│  Profile   → 单一文档持续更新         │
│  Collection → 多文档集合              │
└──────────────────────────────────────┘
```

**Profile 模式：**

```python
# 单一文档，持续更新
namespace = ("user", "123")
store.put(namespace, "profile", {
    "name": "张三",
    "language": "zh-CN",
    "interests": ["Python"],
})

# 更新时需要合并旧数据
old = store.get(namespace, "profile")
new_profile = {**old.value, "interests": old.value["interests"] + ["AI"]}
store.put(namespace, "profile", new_profile)
```

**Collection 模式：**

```python
# 多文档，易于扩展
namespace = ("user", "123", "memories")
store.put(namespace, "mem_1", {"fact": "用户喜欢 Python"})
store.put(namespace, "mem_2", {"fact": "用户使用 macOS"})

# 搜索相关记忆
results = store.search(namespace, query="编程偏好")
```

### Episodic Memory：事件记忆

存储过往经验和行动：

```
事件记忆：
┌──────────────────────────────────────┐
│  存储内容 → 过往事件                 │
│  人类类比 → 第一次骑车经历           │
│  Agent 应用 → 任务执行历史           │
│                                       │
│  Few-shot  → 从示例学习              │
└──────────────────────────────────────┘
```

```python
# 存储成功任务执行
namespace = ("agent", "tasks", "successful")
store.put(namespace, "task_1", {
    "input": "搜索天气",
    "steps": ["调用天气API", "格式化输出"],
    "output": "北京晴，25°C",
})

# 作为 few-shot 示例使用
examples = store.search(namespace, query="天气")
```

### Procedural Memory：程序记忆

存储执行规则和指令：

```
程序记忆：
┌──────────────────────────────────────┐
│  存储内容 → 执行规则                 │
│ 人类类比 → 骑车的肌肉记忆            │
│  Agent 应用 → 系统提示词             │
│                                       │
│  Reflection → 自我反思优化           │
└──────────────────────────────────────┘
```

```python
# 存储和更新系统提示词
def update_instructions(state, store):
    namespace = ("agent", "instructions")
    
    # 获取当前指令
    current = store.get(namespace, "system_prompt")
    
    # 根据用户反馈优化
    model = ChatOpenAI(model="gpt-4o-mini")
    new_prompt = model.invoke(f"""
    当前指令: {current.value}
    用户反馈: {state['feedback']}
    请优化指令使其更好地响应用户需求。
    """)
    
    store.put(namespace, "system_prompt", {"instructions": new_prompt.content})
```

## 写入时机

记忆可以在两种时机写入：

### In the hot path（热路径）

实时写入，用户可见：

```python
def chat_node(state, store):
    # 响应用户同时写入记忆
    response = model.invoke(state["messages"])
    
    # 立即保存重要信息
    if is_important(state["messages"][-1]):
        store.put(("user", user_id), f"mem_{time}", extract_fact(state))
    
    return {"messages": [response]}
```

**优点：** 实时更新、用户透明
**缺点：** 增加延迟、任务复杂

### In the background（后台）

异步写入，不影响主流程：

```python
# 后台任务定期总结对话
async def background_memory_task():
    while True:
        await asyncio.sleep(60)  # 每分钟
        conversations = get_recent_conversations()
        for conv in conversations:
            summary = summarize(conv)
            store.put(("memories", conv.user_id), f"ep_{conv.id}", summary)
```

**优点：** 无延迟、任务分离
**缺点：** 可能延迟、触发时机需设计

## Store 搜索

Store 支持语义搜索和内容过滤：

```python
# 语义搜索（需要嵌入函数）
results = store.search(
    ("user", "123", "memories"),
    query="编程偏好",  # 语义相似度
)

# 内容过滤
results = store.search(
    ("user", "123"),
    filter={"category": "preferences"},  # 字段匹配
)

# 组合搜索
results = store.search(
    ("user", "123"),
    query="Python",
    filter={"type": "fact"},
)
```

## 为什么用它？

```python
# 没有记忆：每次对话独立
# 用户: "我喜欢 Python"
# AI: "好的"
# 用户: "推荐课程"
# AI: 不记得用户喜欢 Python

# 有记忆：个性化服务
# 用户: "我喜欢 Python"
# AI: 记住偏好
# 用户: "推荐课程"
# AI: "既然你喜欢 Python，推荐..."
```
