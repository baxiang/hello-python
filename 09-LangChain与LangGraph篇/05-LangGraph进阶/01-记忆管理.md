# 09.05.01 - 记忆管理

> MessagesState、add_messages reducer、短期记忆、消息裁剪

## 记忆系统概述

Agent 的记忆分为短期记忆和长期记忆：

```
记忆架构：
┌──────────────────────────────────────┐
│  短期记忆 (State)                     │
│  ├─ 当前对话上下文                    │
│  ├─ thread-scoped                    │
│  ├─ Checkpointer 持久化               │
│  └─ MessagesState + add_messages      │
│                                       │
│  长期记忆 (Store)                     │
│  ├─ 跨对话共享                        │
│  ├─ 自定义 namespace                  │
│  ├─ semantic/episodic/procedural     │
│  └─ 详见 02-长期记忆.md              │
└──────────────────────────────────────┘
```

### 为什么需要记忆？

| 场景 | 无记忆 | 有记忆 |
|------|--------|--------|
| 多轮对话 | 每次独立，不记得上文 | 保持上下文连贯 |
| 用户偏好 | 每次重新询问 | 记住偏好，个性化服务 |
| 任务进度 | 不知道之前的步骤 | 跟踪任务状态 |
| 中断恢复 | 无法恢复 | 从断点继续 |

## 短期记忆基础

### MessagesState

LangGraph 提供预置的 `MessagesState`，自动处理消息追加：

```python
from langgraph.graph import MessagesState

# 直接使用
class State(MessagesState):
    """自定义状态，继承 MessagesState"""
    user_id: str          # 用户标识
    documents: list[str]  # 相关文档
    summary: str          # 当前摘要
```

**MessagesState 内置字段：**

| 字段 | 类型 | Reducer | 说明 |
|------|------|---------|------|
| `messages` | `list[AnyMessage]` | `add_messages` | 对话消息列表 |

**add_messages reducer 行为：**

| 操作 | 效果 |
|------|------|
| 追加新消息 | 添加到列表末尾 |
| 更新已有消息 | 根据 ID 替换 |
| 删除消息 | 传入空 ID 的消息 |

### 最简对话示例

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.checkpoint.memory import MemorySaver
from langchain.chat_models import init_chat_model

def chat_node(state: MessagesState):
    """对话节点"""
    model = init_chat_model("moonshot:moonshot-v1-8k")
    response = model.invoke(state["messages"])
    return {"messages": [response]}

# 构建图
builder = StateGraph(MessagesState)
builder.add_node("chat", chat_node)
builder.add_edge(START, "chat")
builder.add_edge("chat", END)

# 编译（带记忆）
checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 同一个 thread 保持上下文
config = {"configurable": {"thread_id": "user-123"}}

# 第一轮
result1 = graph.invoke({"messages": ["你好，我叫张三"]}, config)
print(result1["messages"][-1].content)

# 第二轮（记得上一轮）
result2 = graph.invoke({"messages": ["我叫什么名字？"]}, config)
print(result2["messages"][-1].content)
# 输出：你叫张三
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `MessagesState` | 消息状态 | 预置类型，自动处理消息追加 |
| `MemorySaver()` | 内存持久化 | 开发用，生产用 SQLite/PostgreSQL |
| `thread_id` | 会话标识 | 区分不同对话 |
| `checkpointer` | 状态保存 | 多轮对话依赖它 |

## add_messages Reducer 详解

### 追加消息

```python
# 默认行为：追加
state = {"messages": []}

update = {"messages": [HumanMessage("你好")]}
# 结果: messages = [HumanMessage("你好")]

update = {"messages": [AIMessage("你好！有什么可以帮你？")]}
# 结果: messages = [HumanMessage("你好"), AIMessage("你好！...")]
```

### 更新消息

```python
from langchain.messages import HumanMessage, AIMessage, ToolMessage

# 通过 ID 更新消息
tool_call_id = "call_abc123"

# 先有工具调用
state = {"messages": [
    HumanMessage("查询天气"),
    AIMessage(content="", tool_calls=[{"id": tool_call_id, "name": "weather", "args": {"city": "北京"}}]),
]}

# 添加工具结果（通过 ID 关联）
update = {"messages": [ToolMessage(content="北京晴，25°C", tool_call_id=tool_call_id)]}
# 结果: ToolMessage 关联到对应的 ToolCall
```

### 删除消息

```python
from langchain.messages import RemoveMessage

# 删除特定消息
msg_id = "msg_abc123"
update = {"messages": [RemoveMessage(id=msg_id)]}
# 结果: 对应 ID 的消息被删除
```

## 消息管理策略

长对话会超出模型上下文窗口，需要管理策略：

```
管理策略：
┌──────────────────────────────────────┐
│  Trimming   → 截断旧消息              │
│  Summarizing → 摘要压缩               │
│  Sliding    → 滑动窗口                │
│  Deletion   → 条件删除                │
└──────────────────────────────────────┘
```

### Trimming：截断策略

```python
from langchain.messages import trim_messages, SystemMessage, HumanMessage, AIMessage

messages = [
    SystemMessage("你是助手"),
    HumanMessage("消息1"),
    AIMessage("回复1"),
    HumanMessage("消息2"),
    AIMessage("回复2"),
    # ... 更多消息
]

# 截断到最近的消息，保留系统消息
trimmed = trim_messages(
    messages,
    max_tokens=4000,       # 最大 token 数
    strategy="last",       # 保留最近的
    token_counter=len,     # 简单计数器
    include_system=True,   # 保留系统消息
    allow_partial=False,   # 不允许部分消息
)

print(f"原始: {len(messages)} 条")
print(f"截断后: {len(trimmed)} 条")
```

### trim_messages 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `max_tokens` | `int` | 必需 | 最大 token 数 |
| `strategy` | `str` | `"last"` | `"last"` 保留最近，`"first"` 保留最早 |
| `token_counter` | `callable` | `len` | token 计数函数 |
| `include_system` | `bool` | `True` | 是否保留系统消息 |
| `allow_partial` | `bool` | `False` | 是否允许截断单条消息 |
| `start_on` | `type` | `None` | 从哪种消息类型开始 |
| `end_on` | `type` | `None` | 到哪种消息类型结束 |

### 在节点中使用 Trimming

```python
from langgraph.graph import MessagesState
from langchain.messages import trim_messages, SystemMessage
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")

def chat_with_trimming(state: MessagesState):
    """带截断的对话"""
    messages = state["messages"]
    
    # 如果消息过多，进行截断
    if len(messages) > 20:
        # 分离系统消息
        system_msgs = [m for m in messages if m.type == "system"]
        
        # 截断非系统消息
        non_system = [m for m in messages if m.type != "system"]
        trimmed = trim_messages(
            non_system,
            max_tokens=4000,
            strategy="last",
            token_counter=len,
        )
        
        # 合并
        messages = system_msgs + trimmed
    
    response = model.invoke(messages)
    return {"messages": [response]}
```

### Summarizing：摘要策略

```python
from langgraph.graph import MessagesState
from langchain.messages import SystemMessage
from langchain.chat_models import init_chat_model

model = init_chat_model("moonshot:moonshot-v1-8k")
summary_model = init_chat_model("moonshot:moonshot-v1-8k")

def chat_with_summarization(state: MessagesState):
    """带摘要的对话"""
    messages = state["messages"]
    
    # 如果消息过多，生成摘要
    if len(messages) > 15:
        # 分离系统消息
        system_msgs = [m for m in messages if m.type == "system"]
        
        # 取旧消息生成摘要
        old_messages = messages[1:10]  # 跳过系统消息，取前 10 条
        
        summary_prompt = f"请总结以下对话的要点：\n\n" + "\n".join(
            f"{m.type}: {m.content}" for m in old_messages
        )
        
        summary_response = summary_model.invoke(summary_prompt)
        summary_msg = SystemMessage(f"对话历史摘要：{summary_response.content}")
        
        # 保留最近 5 条消息
        recent_messages = messages[-5:]
        
        # 合并
        messages = system_msgs + [summary_msg] + recent_messages
    
    response = model.invoke(messages)
    return {"messages": [response]}
```

### Sliding Window：滑动窗口

```python
from langgraph.graph import MessagesState
from langchain.messages import SystemMessage
from langchain.chat_models import init_chat_model

WINDOW_SIZE = 10  # 窗口大小

def chat_with_sliding_window(state: MessagesState):
    """滑动窗口对话"""
    messages = state["messages"]
    
    # 分离系统消息
    system_msgs = [m for m in messages if m.type == "system"]
    non_system = [m for m in messages if m.type != "system"]
    
    # 只保留最近的 N 条消息
    if len(non_system) > WINDOW_SIZE:
        non_system = non_system[-WINDOW_SIZE:]
    
    messages = system_msgs + non_system
    
    model = init_chat_model("moonshot:moonshot-v1-8k")
    response = model.invoke(messages)
    return {"messages": [response]}
```

## 自定义 Reducer

### 基本 Reducer

```python
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END

def add_reducer(existing: list[str], new: list[str]) -> list[str]:
    """自定义追加 Reducer"""
    return existing + new

class State(TypedDict):
    items: Annotated[list[str], add_reducer]

def add_item_node(state: State):
    return {"items": ["新条目"]}

builder = StateGraph(State)
builder.add_node("add", add_item_node)
builder.add_edge(START, "add")
builder.add_edge("add", END)

graph = builder.compile()

result = graph.invoke({"items": ["条目1"]})
print(result["items"])  # ['条目1', '新条目']
```

### 覆盖 Reducer

```python
def replace_reducer(existing: str, new: str) -> str:
    """覆盖 Reducer"""
    return new

class State(TypedDict):
    current_topic: Annotated[str, replace_reducer]

def update_topic(state: State):
    return {"current_topic": "Python 编程"}
```

### 合并 Reducer

```python
def merge_reducer(existing: dict, new: dict) -> dict:
    """合并 Reducer"""
    return {**existing, **new}

class State(TypedDict):
    metadata: Annotated[dict, merge_reducer]

def update_metadata(state: State):
    return {"metadata": {"key1": "value1"}}

def update_more(state: State):
    return {"metadata": {"key2": "value2"}}
```

## 消息类型

### LangChain 消息类型

| 类型 | 角色 | 说明 |
|------|------|------|
| `SystemMessage` | system | 系统提示词 |
| `HumanMessage` | user | 用户输入 |

msg = AIMessage(

## 最佳实践

### 1. 合理设置窗口大小

```python
# 简单对话：10 条消息
# 复杂任务：20 条消息
# 代码生成：15 条消息

WINDOW_SIZES = {
    "chat": 10,
    "coding": 15,
    "analysis": 20,
}
```

### 2. 保留关键消息

```python
def smart_trim(messages):
    """智能截断：保留重要消息"""
    system_msgs = [m for m in messages if m.type == "system"]
    
    # 保留带工具调用的消息（重要上下文）
    tool_messages = [m for m in messages if hasattr(m, 'tool_calls') and m.tool_calls]
    tool_results = [m for m in messages if m.type == "tool"]
    
    # 其余消息按时间截断
    other = [m for m in messages if m.type not in ("system", "tool") and not (hasattr(m, 'tool_calls') and m.tool_calls)]
    
    # 保留最近 10 条
    recent_other = other[-10:]
    
    return system_msgs + tool_messages + tool_results + recent_other
```

### 3. 监控上下文使用

```python
def estimate_tokens(messages):
    """估算 token 数"""
    return sum(len(m.content) * 0.5 for m in messages)  # 粗略估算

def check_context_usage(state: MessagesState):
    """检查上下文使用率"""
    messages = state["messages"]
    tokens = estimate_tokens(messages)
    
    MAX_TOKENS = 8000  # moonshot-v1-8k 的上下文限制
    
    usage_percent = (tokens / MAX_TOKENS) * 100
    print(f"上下文使用率: {usage_percent:.1f}%")
    
    if usage_percent > 80:
        print("警告：上下文即将耗尽，建议截断或摘要")
    
    return usage_percent
```

## 小结

| 要点 | 说明 |
|------|------|
| MessagesState | 预置消息状态，内置 `add_messages` reducer |
| add_messages | 追加、更新、删除消息 |
| 管理策略 | Trimming（截断）、Summarizing（摘要）、Sliding Window（滑动窗口） |
| trim_messages | 控制最大 token 数，保留系统消息 |
| 自定义 Reducer | 使用 `Annotated[type, reducer_fn]` 定义 |
| 最佳实践 | 合理窗口大小、保留关键消息、监控上下文使用 |
