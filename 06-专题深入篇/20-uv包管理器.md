# 第 20 章 - uv 包管理器（详细版）

## 第一部分：uv 简介与安装

### 1.1 什么是 uv

**uv** 是由 Astral 团队（也是 Ruff 的开发者）开发的超快速 Python 包管理器和项目管理工具。

#### uv 的特点

| 特性 | 说明 |
|------|------|
| 🚀 **超快速** | 使用 Rust 编写，比 pip 快 10-100 倍 |
| 📦 **一体化** | 替代 pip、pip-tools、virtualenv、pyenv |
| 🔒 **可靠** | 确定性安装、锁文件支持 |
| 🎯 **现代** | 支持 pyproject.toml、uv.lock |
| 🔄 **兼容** | 完全兼容 PyPI、支持 pip 命令 |

#### uv vs pip vs poetry

```
┌─────────────────┬──────────┬──────────┬──────────┐
│     功能        │    uv    │   pip    │ poetry   │
├─────────────────┼──────────┼──────────┼──────────┤
│ 包安装速度      │ ⚡⚡⚡     │ ⚡        │ ⚡⚡      │
│ 虚拟环境管理    │ ✅        │ ❌       │ ✅       │
│ Python 版本管理 │ ✅        │ ❌       │ ❌       │
│ 锁文件          │ ✅        │ ❌       │ ✅       │
│ pyproject.toml  │ ✅        │ ❌       │ ✅       │
│ 依赖解析        │ ⚡⚡⚡     │ ⚡        │ ⚡⚡      │
│ 学习曲线        │ 低        │ 低        │ 中       │
└─────────────────┴──────────┴──────────┴──────────┘
```

### 1.2 安装 uv

#### macOS / Linux 安装

```bash
# 方法 1：官方安装脚本（推荐）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 方法 2：使用 Homebrew（macOS）
brew install uv

# 方法 3：使用 pip
pip install uv

# 方法 4：使用 pipx（推荐，隔离安装）
pipx install uv

# 方法 5：使用 cargo（如果有 Rust）
cargo install uv

# 验证安装
uv --version

# 查看 uv 路径
which uv
```

#### Windows 安装

```powershell
# 方法 1：PowerShell 安装脚本（推荐）
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# 方法 2：使用 winget
winget install uv

# 方法 3：使用 pip
pip install uv

# 方法 4：使用 pipx
pipx install uv

# 验证安装
uv --version

# 查看 uv 路径
where uv
```

#### 安装后配置

```bash
# uv 默认安装位置
# macOS/Linux: ~/.local/bin/uv
# Windows: %USERPROFILE%\.local\bin\uv

# 确保 uv 在 PATH 中
# 添加到家目录的 shell 配置文件

# ~/.zshrc 或 ~/.bashrc
export PATH="$HOME/.local/bin:$PATH"

# 使配置生效
source ~/.zshrc  # 或 source ~/.bashrc

# 启用自动补全
# zsh
uv generate-shell-completion zsh >> ~/.zsh/_uv

# bash
uv generate-shell-completion bash >> ~/.bash_completion.d/uv

# fish
uv generate-shell-completion fish >> ~/.config/fish/completions/uv.fish
```

### 1.3 uv 更新与卸载

```bash
# 更新 uv
uv self update

# 或
uvx run uv self update

# 查看版本
uv --version

# 卸载 uv
# macOS/Linux
rm ~/.local/bin/uv

# Windows
del %USERPROFILE%\.local\bin\uv.exe
```

## 第二部分：uv 项目管理

### 2.1 创建新项目

```bash
# 创建新项目（交互式）
uv init my-project

# 创建当前目录项目
uv init

# 创建指定 Python 版本的项目
uv init --python 3.12

# 创建非包项目（应用）
uv init --kind project my-app

# 创建包项目（库）
uv init --kind lib my-library

# 项目结构
my-project/
├── pyproject.toml    # 项目配置
├── uv.lock           # 依赖锁文件
├── README.md
├── .python-version   # Python 版本文件
└── src/
    └── my_project/
        └── __init__.py
```

### 2.2 pyproject.toml 详解

```toml
[project]
# 基本信息
name = "my-project"
version = "0.1.0"
description = "我的 Python 项目"
readme = "README.md"
requires-python = ">=3.10"  # 最低 Python 版本
license = { text = "MIT" }
authors = [
    { name = "Your Name", email = "your@email.com" }
]

# 依赖
dependencies = [
    "requests>=2.31.0",
    "click>=8.0.0",
]

# 可选依赖
[project.optional-dependencies]
dev = [
    "pytest>=7.0.0",
    "black>=23.0.0",
    "ruff>=0.1.0",
    "mypy>=1.0.0",
]
docs = [
    "sphinx>=6.0.0",
    "sphinx-rtd-theme>=1.0.0",
]

# 项目入口（命令行工具）
[project.scripts]
my-command = "my_project.cli:main"

# 构建系统
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

# uv 特定配置
[tool.uv]
# 开发依赖（推荐方式）
dev-dependencies = [
    "pytest>=7.0.0",
    "ruff>=0.1.0",
]

# Python 版本
python-version = "3.12"

# 包索引配置
[[tool.uv.index]]
name = "pypi"
url = "https://pypi.org/simple"

# 私有源
[[tool.uv.index]]
name = "private"
url = "https://pypi.mycompany.com/simple"
```

### 2.3 uv 项目管理命令

```bash
# ========== 项目初始化 ==========

# 创建新项目
uv init my-project
cd my-project

# ========== 依赖管理 ==========

# 添加依赖
uv add requests              # 添加最新
uv add "requests>=2.31.0"    # 指定版本
uv add "django>=4.0,<5.0"    # 版本范围
uv add pandas numpy scipy    # 多个包
uv add --dev pytest          # 开发依赖
uv add --optional docs sphinx  # 可选依赖

# 移除依赖
uv remove requests
uv remove --dev pytest

# 更新依赖
uv lock --upgrade           # 更新所有依赖
uv lock --upgrade requests  # 更新特定包
uv lock --upgrade-package requests  # 同上

# 查看依赖树
uv tree                     # 显示依赖树
uv tree --depth 2           # 限制深度
uv tree --invert            # 反向显示

# ========== 虚拟环境 ==========

# 创建虚拟环境
uv venv

# 创建指定 Python 版本的虚拟环境
uv venv --python 3.11
uv venv --python 3.12.0

# 创建指定位置的虚拟环境
uv venv .venv
uv venv /path/to/venv

# 查看虚拟环境信息
uv venv --info

# 删除虚拟环境
rm -rf .venv

# 激活虚拟环境
# macOS/Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate

# ========== 运行命令 ==========

# 在虚拟环境中运行
uv run python main.py
uv run pytest
uv run my-command

# 使用特定 Python 版本运行
uv run --python 3.11 python main.py

# 运行脚本
uv run script.py arg1 arg2

# ========== 同步环境 ==========

# 同步依赖（类似 pip install -r）
uv sync

# 仅安装生产依赖
uv sync --no-dev

# 仅安装开发依赖
uv sync --only-dev

# 使用特定 Python 版本
uv sync --python 3.11

# ========== 包管理 ==========

# 列出已安装的包
uv pip list

# 查看包信息
uv pip show requests

# 检查过时包
uv pip list --outdated

# 检查依赖
uv pip check
```

## 第三部分：uv pip 兼容模式

### 3.1 uv 作为 pip 替代品

```bash
# ========== 基本用法 ==========

# 安装包（替代 pip install）
uv pip install requests
uv pip install "django>=4.0"
uv pip install -r requirements.txt

# 卸载包（替代 pip uninstall）
uv pip uninstall requests
uv pip uninstall -r requirements.txt

# 列出包（替代 pip list）
uv pip list
uv pip list --outdated
uv pip list --format json

# 显示包信息（替代 pip show）
uv pip show requests
uv pip show django numpy

# ========== requirements.txt ==========

# 从 requirements.txt 安装
uv pip install -r requirements.txt

# 生成 requirements.txt
uv pip freeze > requirements.txt

# 导出所有依赖
uv export > requirements.txt
uv export --format requirements-txt > requirements.txt

# 导出开发依赖
uv export --dev > requirements-dev.txt

# 导出可选依赖
uv export --extra docs > requirements-docs.txt

# ========== 索引源配置 ==========

# 使用自定义源
uv pip install requests --index-url https://pypi.tuna.tsinghua.edu.cn/simple

# 添加额外源
uv pip install package --extra-index-url https://private.pypi.org/simple

# 配置默认源（永久）
# ~/.config/uv/uv.toml 或 ~/.uv/uv.toml
[pip]
index-url = "https://pypi.tuna.tsinghua.edu.cn/simple"

# ========== 缓存管理 ==========

# 查看缓存位置
uv cache dir

# 清理缓存
uv cache clean

# 清理特定包缓存
uv cache clean requests

# 缓存信息
uv cache info
```

### 3.2 国内镜像源配置

```bash
# 清华源
uv pip install requests \
  --index-url https://pypi.tuna.tsinghua.edu.cn/simple

# 阿里源
uv pip install requests \
  --index-url https://mirrors.aliyun.com/pypi/simple/

# 中科大源
uv pip install requests \
  --index-url https://pypi.mirrors.ustc.edu.cn/simple/

# 永久配置（推荐）
# 创建或编辑配置文件
# macOS/Linux: ~/.config/uv/uv.toml
# Windows: %APPDATA%\uv\uv.toml

cat > ~/.config/uv/uv.toml << EOF
[pip]
index-url = "https://pypi.tuna.tsinghua.edu.cn/simple"
EOF

# 验证配置
uv pip install requests  # 应该使用清华源
```

### 5.1 工作区（Workspaces）

```bash
# ========== 创建工作区 ==========

# 创建根项目
uv init my-workspace
cd my-workspace

# 添加子项目（包）
uv add --workspace packages/package-a
uv add --workspace packages/package-b

# 工作区结构
my-workspace/
├── pyproject.toml       # 根配置
├── uv.lock
└── packages/
    ├── package-a/
    │   ├── pyproject.toml
    │   └── src/
    └── package-b/
        ├── pyproject.toml
        └── src/

# ========== 工作区依赖 ==========

# 根 pyproject.toml
[project]
name = "my-workspace"

[tool.uv.workspace]
members = ["packages/*"]

# 子包可以依赖同一工作区中的其他包
# packages/package-b/pyproject.toml
[project]
name = "package-b"
dependencies = [
    "package-a",  # 自动解析为工作区版本
]
```

## 第六部分：CI/CD 集成

### 6.1 GitHub Actions

```yaml
# .github/workflows/ci.yml

name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.10', '3.11', '3.12']

    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v3

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version-file: '.python-version'

      - name: Install dependencies
        run: uv sync --dev

      - name: Run tests
        run: uv run pytest

      - name: Run lint
        run: uv run ruff check .

      - name: Run type check
        run: uv run mypy src/

  build:
    runs-on: ubuntu-latest
    needs: test

    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v3

      - name: Build package
        run: uv build

      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/
```

### 6.2 Docker 集成

```dockerfile
# ========== 基础镜像 ==========

# 使用官方 Python 镜像
FROM python:3.12-slim

# 安装 uv
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv

WORKDIR /app

# 复制项目文件
COPY pyproject.toml uv.lock ./

# 同步依赖
RUN uv sync --frozen

# 复制源代码
COPY . .

# 运行应用
CMD ["uv", "run", "python", "main.py"]

# ========== 多阶段构建 ==========

FROM python:3.12-slim as builder

COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv
WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen

FROM python:3.12-slim

COPY --from=builder /app/.venv /app/.venv
COPY --from=builder /app/src /app/src

ENV PATH="/app/.venv/bin:$PATH"

CMD ["python", "main.py"]
```

## 第十部分：实战示例

### 10.1 完整的 Web 应用示例

```bash
# 1. 创建项目
uv init web-app
cd web-app

# 2. 设置 Python 版本
echo "3.12" > .python-version

# 3. 添加 Web 框架依赖
uv add "fastapi>=0.100.0" "uvicorn>=0.23.0" "pydantic>=2.0.0"

# 4. 添加数据库依赖
uv add "sqlalchemy>=2.0.0" "alembic>=1.12.0" "asyncpg>=0.29.0"

# 5. 添加开发依赖
uv add --dev pytest pytest-asyncio httpx ruff black mypy

# 6. 创建目录结构
mkdir -p src/web_app api models tests

# 7. 创建主应用
cat > src/web_app/main.py << 'EOF'
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
async def root():
    return {"message": "Hello, World!"}

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}
EOF

# 8. 运行应用
uv run uvicorn src.web_app.main:app --reload
```

### 10.2 命令行工具示例

```bash
# 1. 创建项目
uv init my-cli

# 2. 添加依赖
uv add click rich

# 3. 配置命令行入口
# 编辑 pyproject.toml
cat >> pyproject.toml << 'EOF'

[project.scripts]
my-cli = "my_cli.cli:main"
EOF

# 4. 创建 CLI
cat > src/my_cli/cli.py << 'EOF'
import click
from rich.console import Console

console = Console()

@click.group()
def main():
    """My CLI Tool"""
    pass

@main.command()
@click.option('--name', default='World', help='Name to greet')
def hello(name):
    """Say hello"""
    console.print(f"[bold green]Hello, {name}![/bold green]")

if __name__ == '__main__':
    main()
EOF

# 5. 同步并运行
uv sync
uv run my-cli hello --name Alice
```
