## 第一部分 - print 调试

### 1.1 基础 print

```python
# ❌ 不推荐：难以清理
def process_data(data):
    print("DEBUG: data =", data)
    result = []
    for item in data:
        print("DEBUG: processing", item)
        result.append(item * 2)
    print("DEBUG: result =", result)
    return result

# ✅ 推荐：使用 logging
import logging
logger = logging.getLogger(__name__)

def process_data(data):
    logger.debug("Processing data: %s", data)
    result = []
    for item in data:
        logger.debug("Processing item: %s", item)
        result.append(item * 2)
    logger.debug("Result: %s", result)
    return result
```

### 1.2 f-string 调试

```python
# Python 3.8+
name = "Alice"
age = 30

# 快速调试
print(f"{name=}")  # 输出：name='Alice'
print(f"{age=}")   # 输出：age=30
print(f"{name=}, {age=}")  # 输出：name='Alice', age=30

# 表达式调试
numbers = [1, 2, 3]
print(f"{len(numbers)=}")  # 输出：len(numbers)=3
```

## 第二部分 - assert 断言

### 2.1 基础用法

```python
def calculate_discount(price: float, discount: float) -> float:
    # 参数验证
    assert price >= 0, "Price must be non-negative"
    assert 0 <= discount <= 1, "Discount must be between 0 and 1"
    
    final_price = price * (1 - discount)
    
    # 结果验证
    assert final_price >= 0, "Final price must be non-negative"
    assert final_price <= price, "Final price must not exceed original"
    
    return final_price

# 类型检查（开发环境）
def process_user(user: dict) -> None:
    assert isinstance(user, dict), "User must be a dict"
    assert "id" in user, "User must have an id"
    assert "name" in user, "User must have a name"
```

### 2.2 注意事项

```python
# ❌ 不要这样做：assert 可能被禁用
# python -O 会禁用 assert
def validate_input(data):
    assert check_data(data)  # 生产环境可能不执行
    return process(data)

# ✅ 正确做法：使用显式检查
def validate_input(data):
    if not check_data(data):
        raise ValueError("Invalid data")
    return process(data)
```

## 第三部分 - pdb 调试器

### 3.1 基础用法

```python
import pdb

def buggy_function(x, y):
    result = x + y
    pdb.set_trace()  # 设置断点
    result = result * 2
    return result

# 常用命令
# n (next) - 执行下一行
# s (step) - 进入函数
# c (continue) - 继续执行
# l (list) - 显示代码
# p (print) - 打印变量
# pp (pretty print) - 美化打印
# q (quit) - 退出调试
# h (help) - 帮助
# w (where) - 显示堆栈
# u (up) - 向上移动栈帧
# d (down) - 向下移动栈帧
```

### 3.2 breakpdb 快捷方式

```python
# Python 3.7+
def process_data(data):
    breakpoint()  # 等价于 import pdb; pdb.set_trace()
    return data * 2

# 条件断点
for i in range(100):
    if i == 50:
        breakpoint()
    process(i)
```

### 3.3 命令行调试

```bash
# 使用 pdb 运行脚本
python -m pdb script.py

# 调试中执行代码
(Pdb) x  # 查看变量 x
(Pdb) x = 10  # 修改变量
(Pdb) !print(x)  # 执行 Python 命令
(Pdb) where  # 查看调用栈
(Pdb) up  # 向上移动
(Pdb) down  # 向下移动
(Pdb) list  # 显示代码
(Pdb) next  # 下一行
(Pdb) step  # 进入函数
(Pdb) return  # 执行到函数返回
(Pdb) continue  # 继续执行
```

## 第五部分 - 异常调试

### 5.1 捕获和记录异常

```python
import logging
import traceback

logger = logging.getLogger(__name__)

def risky_operation():
    try:
        result = do_something()
        return result
    except Exception as e:
        # 记录完整堆栈
        logger.exception("Operation failed: %s", e)
        # 或手动获取堆栈
        error_trace = traceback.format_exc()
        logger.error("Traceback:\n%s", error_trace)
        raise
```

### 5.2 上下文管理器调试

```python
from contextlib import contextmanager
import logging
import time

logger = logging.getLogger(__name__)

@contextmanager
def debug_timer(operation_name: str):
    """记录操作执行时间"""
    start = time.time()
    logger.debug("Starting %s", operation_name)
    try:
        yield
    except Exception as e:
        logger.error("%s failed after %.2fs: %s", 
                    operation_name, time.time() - start, e)
        raise
    else:
        logger.debug("%s completed in %.2fs", 
                    operation_name, time.time() - start)

# 使用
with debug_timer("database query"):
    result = db.query("SELECT * FROM users")
```

### 6.2 日志调试（生产环境）

```python
import logging
import sys
from functools import wraps

# 配置详细日志
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(funcName)s:%(lineno)d - %(message)s',
    handlers=[
        logging.StreamHandler(sys.stdout),
        logging.FileHandler('debug.log')
    ]
)

logger = logging.getLogger(__name__)

def debug_log(func):
    """调试装饰器"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        logger.debug("Calling %s with args=%s, kwargs=%s", 
                    func.__name__, args, kwargs)
        try:
            result = func(*args, **kwargs)
            logger.debug("%s returned %s", func.__name__, result)
            return result
        except Exception as e:
            logger.exception("%s raised exception: %s", func.__name__, e)
            raise
    return wrapper

@debug_log
def process_order(order_id: int):
    logger.debug("Processing order %s", order_id)
    # ...
```

## 第七部分 - 调试最佳实践

### 7.1 调试清单

```markdown
## 调试步骤

1. **复现问题**
   - [ ] 确认问题可以稳定复现
   - [ ] 记录复现步骤
   - [ ] 准备测试数据

2. **定位问题**
   - [ ] 查看错误信息和堆栈
   - [ ] 检查相关日志
   - [ ] 使用断点缩小范围

3. **分析原因**
   - [ ] 理解代码预期行为
   - [ ] 对比实际行为
   - [ ] 找出差异原因

4. **修复验证**
   - [ ] 编写测试用例
   - [ ] 修复代码
   - [ ] 运行测试
   - [ ] 回归测试
```

### 7.2 常见调试场景

```python
# 场景 1：变量值不符合预期
def debug_variable():
    x = get_value()
    assert isinstance(x, int), f"Expected int, got {type(x)}"
    assert x >= 0, f"Expected non-negative, got {x}"

# 场景 2：函数返回值错误
def debug_function():
    result = process(data)
    expected = ...
    if result != expected:
        import pdb; pdb.set_trace()

# 场景 3：性能问题
def debug_performance():
    import time
    start = time.time()
    process_large_data()
    elapsed = time.time() - start
    print(f"Elapsed: {elapsed:.2f}s")
```

## 最佳实践清单

### ✅ 推荐做法

1. 使用 logging 而非 print
2. 在生产环境禁用 assert
3. 使用断言验证前置条件
4. 记录异常时包含完整堆栈
5. 使用上下文管理器记录执行时间

### ❌ 避免

1. 在生产代码中遗留断点
2. 使用空的 except 块
3. 忽略异常不处理
4. 在生产环境使用 pdb
