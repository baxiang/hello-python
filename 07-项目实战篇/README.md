# 项目实战篇

13个真实项目，按难度梯度组织，从零基础到产业级实战。

---

## 学习路径

```
Level-1 入门级 (3个项目)
    ↓  适合初学者，1-2天完成
Level-2 进阶级 (3个项目)
    ↓  需要基础，3-5天完成
Level-3 高级 (4个项目)
    ↓  需要进阶，1-2周完成
Level-4 专业级 (1个项目)
    ↓  接近产业级，2-4周完成
```

## 学习建议

### 1. 按顺序学习
不要跳级，每个项目都有知识点依赖。建议学习顺序：
```
Level-1 (全部完成)
    ↓
Level-2 (至少完成2个)
    ↓
Level-3 (根据兴趣选择)
    ↓
Level-4 (挑战自我)
```

### 2. 自己实现
- 先看文档理解需求
- 独立设计和实现
- 遇到困难再参考示例代码
- 完成后对比学习

### 3. 扩展功能
完成基础功能后，尝试：
- 添加新功能
- 优化用户体验
- 提高代码质量
- 编写测试用例

### 4. 提交代码
使用Git管理代码，养成好习惯：
```bash
# 初始化项目
git init

# 功能开发
git add .
git commit -m "feat: 实现XX功能"

# 保持提交历史清晰
```

## 常见问题

### Q: 项目太多，应该先做哪个？
**A:** 按 Level 顺序，每个 Level 完成至少 2 个项目再进入下一级。

### Q: 卡住了怎么办？
**A:** 
1. 回顾相关章节知识点
2. 查看示例代码的关键部分
3. 不要完全复制，理解思路后自己实现

### Q: 如何判断自己掌握了？
**A:** 能够不看示例代码独立实现，并且能添加新功能。

### Q: Level-4太难怎么办？
**A:** Level-4是挑战项目，难度较大。可以先完成Level-3的所有项目，积累经验后再挑战。
