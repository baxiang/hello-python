### 反模式对比示例

**示例：列表推导式**

```markdown
#### 反模式：不要这样做

```python
# ❌ 用列表推导式只为了遍历
[print(x) for x in items]  # 创建了无用列表

# 问题：
# 1. 创建临时列表浪费内存
# 2. 返回 [None, None, ...] 无意义结果
# 3. 语义不清晰——这不是"推导"而是"遍历"
```

```python
# ✅ 正确做法：用 for 循环遍历
for x in items:
    print(x)

# 或用生成器表达式（如果需要惰性执行）
print(x) for x in items  # 不创建列表
```

#### 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| 列表推导式用于**创建新列表** | 语义清晰，返回有意义的结果 | `[x * 2 for x in items]` |
| for 循环用于**遍历执行操作** | 不创建临时对象，意图明确 | `for x in items: print(x)` |
| 生成器表达式用于**惰性处理** | 节省内存，适合大数据 | `(x * 2 for x in items)` |
```

#### Python 如何实现

字典使用**哈希表**实现：

```
字典内部结构：
┌──────────────────────────────────────────────────┐
│  哈希表（开放寻址法）                              │
│                                                   │
│  indices  entries                                 │
│  [0] ───→  [hash, key, value]                    │
│  [1] ───→  None（空槽）                           │

│                                                   │
│  查找流程：                                        │
│  1. hash(key) 计算哈希值                          │
│  2. hash % table_size 得到索引                    │
│  3. 检查该位置是否匹配                             │
│  4. 冲突时线性探测下一个位置                       │
└──────────────────────────────────────────────────┘
```

```python
# 演示哈希冲突
d = {}
d['a'] = 1  # hash('a') 可能与另一个键冲突
d['b'] = 2  # Python 自动处理冲突

# 查看字典内部（Python 3.3+）
import sys
print(sys.getsizeof(d))  # 字典内存占用

#### 性能考量

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| `d[key]` 查找 | O(1) 平均 | 哈希直接定位 |
| `d[key] = v` 插入 | O(1) 平均 | 哈希定位后写入 |
| `key in d` 成员检查 | O(1) 平均 | 比 list 的 O(n) 快得多 |
| 遍历 `for k in d` | O(n) | 访问所有键 |

**注意：** 哈希冲突时性能退化到 O(n)，但 Python 3.3+ 优化后极少发生。

| 开放寻址 | 内存紧凑，缓存友好 | 比 Python 3.6 前节省 ~25% 内存 |

## 完整示例：列表推导式

```markdown
### 列表推导式

**定义：** 用简洁语法从可迭代对象创建新列表。

#### L1 理解层：语法结构

```
语法结构：
┌──────────────────────────────────────────────────────┐
│  [表达式 for 变量 in 可迭代对象 if 条件]              │
│                                                       │
│  [x * 2      for x   in range(5)   if x > 0]         │
│   ↑          ↑        ↑              ↑               │
│   结果值     遍历变量  数据源         过滤条件         │
│                                                       │
│  等价于：                                              │
│  result = []                                          │
│  for x in range(5):                                   │
│      if x > 0:                                        │
│          result.append(x * 2)                         │
└──────────────────────────────────────────────────────┘
```

**最简示例：**

```python
# 创建平方列表
squares = [x ** 2 for x in range(5)]
# 结果：[0, 1, 4, 9, 16]
```

**详细示例：**

```python
# 带条件过滤
even_squares = [x ** 2 for x in range(10) if x % 2 == 0]
# 结果：[0, 4, 16, 36, 64]

# 嵌套推导（二维矩阵）
matrix = [[i * j for j in range(3)] for i in range(3)]
# 结果：[[0, 0, 0], [0, 1, 2], [0, 2, 4]]
```

**关键代码说明：**

| 代码 | 含义 |
|------|------|
| `[x ** 2 for x in range(5)]` | 遍历 0-4，每个值平方，组成列表 |
| `if x % 2 == 0` | 只保留偶数 |
| `[[i * j for j in range(3)] for i in range(3)]` | 外层遍历 i，内层遍历 j |

#### L2 实践层：最佳实践

**推荐做法：**

| 做法 | 原因 | 示例 |
|------|------|------|
| 用于**创建新列表** | 语义清晰，返回有意义结果 | `[x.strip() for x in lines]` |
| 保持**简单** | 可读性优先 | 单层优于嵌套 |
| 复杂逻辑用**for循环** | 可读性更好 | 多步骤、有异常处理时 |

**反模式：不要这样做：**

```python
# ❌ 只为了遍历（不创建有意义列表）
[print(x) for x in items]  # 创建 [None, None, ...]

# ❌ 过于复杂（可读性差）
result = [func(x) for x in [g(y) for y in range(10) if cond(y)] if check(x)]

# ❌ 有副作用时用推导式
[file.close() for file in open_files]  # 副作用不该隐藏在推导式里
```

```python
# ✅ 正确做法

# 遍历执行操作 → 用 for 循环
for x in items:
    print(x)

# 复杂逻辑 → 用 for 循环展开
result = []
for y in range(10):
    if cond(y):
        x = g(y)
        if check(x):
            result.append(func(x))

# 有副作用 → 用 for 循环
for file in open_files:
    file.close()
```

**适用场景：**

| 场景 | 是否推荐 | 原因 |
|------|---------|------|
| 简单变换 | ✅ 推荐 | `[x.strip() for x in lines]` |
| 简单过滤 | ✅ 推荐 | `[x for x in items if x > 0]` |
| 执行操作（打印、写入） | ❌ 不推荐 | 用 for 循环 |
| 嵌套超过 2 层 | ❌ 不推荐 | 用 for 循环展开 |
| 有异常处理 | ❌ 不推荐 | 推导式难以处理异常 |

#### L3 专家层：底层原理

**Python 如何实现：**

列表推导式在 Python 3 中有特殊优化：

```python
# 字节码层面：推导式有专门指令
import dis
dis.dis("[x for x in range(5)]")

# 输出包含 BUILD_LIST、GET_ITER、FOR_ITER 等专门指令
# 比等价 for 循环更高效
```

**性能对比：**

| 方式 | 时间 | 内存 | 说明 |
|------|------|------|------|
| `[x * 2 for x in items]` | 快 | 新列表 | 字节码优化 |
| `for x in items: result.append(x * 2)` | 稍慢 | 新列表 | 每次调用 append 方法 |
| `(x * 2 for x in items)` | 最快（惰性） | 无列表 | 生成器，按需生成 |

**性能测试：**

```python
import timeit

# 推导式
timeit.timeit("[x * 2 for x in range(1000)]", number=10000)
# 约 0.3 秒

# for 循环
timeit.timeit("""
result = []
for x in range(1000):
    result.append(x * 2)
""", number=10000)
# 约 0.5 秒（慢 60%+）
```

**设计动机：**

| Python 设计选择 | 原因 |
|-----------------|------|
| 推导式有专门字节码 | 性能优化，避免 append 方法调用开销 |
| 生成器表达式 `(x for x in items)` | 大数据场景节省内存 |
| Python 2 中推导式泄露变量 | 已修复（Python 3 推导式有独立作用域） |

**知识关联：**

```

### 文档字符串规范

使用 Google 风格：

```python
def calculate_average(
    numbers: list[float],
    exclude_zeros: bool = False
) -> float | None:
    """计算数值列表的平均值。

    Args:
        numbers: 数值列表。
        exclude_zeros: 是否排除零值，默认 False。

    Returns:
        平均值，空列表返回 None。

    Example:
        >>> calculate_average([1, 2, 3])
        2.0
    """
    if not numbers:
        return None
    
    filtered = [n for n in numbers if not exclude_zeros or n != 0]
    return sum(filtered) / len(filtered) if filtered else None
```

### 反模式清单

教程代码必须避免以下反模式：

| 反模式 | 问题 | 正确做法 |
|--------|------|---------|
| 裸 `except:` | 吞掉所有异常 | 捕获具体异常 |
| `import *` | 污染命名空间 | 显式导入 |
| 可变默认参数 | 数据累积 | 用 `None` |
| 全局变量 | 隐式依赖 | 用类/函数封装 |
| 过深嵌套 | 可读性差 | 提前返回（guard clauses） |

```python
# ❌ 过深嵌套
def process(data):
    if data:
        if data.valid:
            if data.ready:
                if data.has_permission:
                    return do_work(data)
    return None

# ✅ 提前返回
def process(data):
    if not data or not data.valid or not data.ready or not data.has_permission:
        return None
    return do_work(data)
```
