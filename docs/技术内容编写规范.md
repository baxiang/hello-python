# 技术内容编写规范

本文档定义 Python 学习教程的技术内容编写标准，确保读者能够快速理解概念。

---

## 核心重点

每个技术概念必须包含：

| 重点 | 说明 | 必须 |
|------|------|------|
| **语法结构图** | ASCII 图展示语法组成，让读者一眼看懂 | ✅ 必须 |
| **代码示例** | 最简示例 → 渐进复杂 → 实际应用 | ✅ 必须 |
| **代码解释** | 核心/复杂代码需要单独的技术说明 | ✅ 必须 |
| **为什么用它** | 对比"有 vs 无"，展示价值 | ❓ 可选 |

"为什么用它"根据内容价值灵活决定：
- 新语法概念（如 `list[T]`）→ 需要对比
- 已知概念的具体语法（如正则元字符）→ 不需要对比

---

## 标准模板

```markdown
**概念名称：** 一句话定义。

```
语法结构（重点）：
┌──────────────────────────────────────┐
│  关键语法                              │
│                                       │
│  示例 → 解释                           │
└──────────────────────────────────────┘
```

### 最简示例

```python
# 最简用法
...
```

### 详细示例

```python
# 实际应用
...
```

### 代码解释（针对核心/复杂代码）

如果示例代码包含核心概念或复杂逻辑，必须单独解释：

```markdown
**关键代码说明：**

| 代码片段 | 含义 | 为什么这样写 |
|----------|------|-------------|
| `xxx` | 解释 | 原因 |

或使用分段注释：

```python
# 第一步：xxx
# ↓ 这行代码的作用是...
代码...

# 第二步：xxx
# ↓ 关键点：...
代码...
```
```

## 代码示例的处理规则

### 何时需要代码解释？

| 情况 | 是否需要 | 处理方式 |
|------|---------|---------|
| 最简示例（1-3 行） | ❌ 不需要 | 代码本身已足够清晰 |
| 核心/关键代码 | ✅ 必须解释 | 用表格或注释分段说明 |
| 复杂逻辑（多步骤） | ✅ 必须解释 | 分步骤注释 |
| 嵌套/组合结构 | ✅ 必须解释 | 拆解结构说明 |

### 示例对比

**❌ 不好的写法（缺少解释）：**

```python
def first(items: list[T]) -> T:
    return items[0]
```

问题：`list[T]` 和 `T` 是什么？读者不理解。

**✅ 好的写法（有解释）：**

```python
def first(items: list[T]) -> T:
    #     ↑          ↑
    #     参数类型    返回类型
    #     list[T]：T类型的列表
    #     T：返回值与元素类型相同
    return items[0]
```

**关键代码说明：**

| 代码 | 含义 |
|------|------|
| `list[T]` | 参数是 T 类型元素的列表 |
| `-> T` | 返回值类型与元素类型相同（类型保持） |
| `items[0]` | 取第一个元素，类型自动推断为 T |

## 何时需要"为什么用它"

| 情况 | 是否需要 | 原因 |
|------|---------|------|
| 新语法概念（类型提示、泛型） | ✅ 需要 | 读者不理解价值 |
| 已知概念的语法（正则元字符） | ❌ 不需要 | 读者已知道正则是什么 |
| 替代方案（defaultdict vs dict） | ✅ 需要 | 需要对比优劣 |
| 功能介绍（Counter.most_common） | ❌ 不需要 | 功能显而易见 |

## 具体示例

### ✅ 需要对比的写法

```markdown
**列表 list[T]：**

`list[T]` 表示元素类型为 T 的列表。

```
语法结构：
┌──────────────────────────────────────┐
│  list[元素类型]                        │
│                                       │
│  list[int]   → 整数列表               │
│  list[str]   → 字符串列表             │
└──────────────────────────────────────┘
```

**为什么用它？**

```python
# 没有：看不出元素类型
numbers = [1, 2, 3]  # 整数？字符串？

# 有：一目了然
numbers: list[int] = [1, 2, 3]
```

### 最简示例

```python
numbers: list[int] = [1, 2, 3]
names: list[str] = ["张三", "李四"]
```
```

### ✅ 不需要对比的写法

```markdown
**正则表达式元字符 `.`：**

`.` 匹配任意单个字符（除换行符）。

```
语法：
┌──────────────────────────────────────┐
│  .                                    │
│                                       │
│  匹配：任意单个字符                    │
│  不匹配：换行符                        │
└──────────────────────────────────────┘
```

### 最简示例

```python
import re

# a.c 匹配 a + 任意字符 + c
re.search('a.c', 'abc')   # 匹配 "abc"
re.search('a.c', 'a1c')   # 匹配 "a1c"
re.search('a.c', 'ac')    # 不匹配（缺中间字符）
```
```

### ✅ 核心/复杂代码需要解释的写法

```markdown
**泛型函数：**

泛型函数可以处理任意类型，同时保持类型一致性。

```
语法结构：
┌──────────────────────────────────────┐
│  def func(items: list[T]) -> T:      │
│      ↑            ↑       ↑          │
│      函数名     参数类型  返回类型    │
└──────────────────────────────────────┘
```

### 最简示例

```python
from typing import TypeVar

T = TypeVar('T')  # 定义类型变量

def first(items: list[T]) -> T:
    return items[0]
```

### 关键代码解释

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `TypeVar('T')` | 定义类型变量 T | T 是占位符，使用时替换为具体类型 |
| `list[T]` | 参数是 T 类型元素的列表 | 不限制具体类型，任意类型都可以 |
| `-> T` | 返回值类型为 T | 保证返回类型与元素类型一致 |

**类型推断示例：**

```python
# 使用时自动推断类型
nums = [1, 2, 3]
first_num = first(nums)  # T 替换为 int，first_num 类型是 int

words = ["a", "b"]
first_word = first(words)  # T 替换为 str，first_word 类型是 str
```
