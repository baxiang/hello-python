# 第 21 章 - 类型提示（详细版）

> 类型提示是 Python 3.5+ 引入的特性，用于在代码中声明变量、函数参数和返回值的类型。它不是用来运行程序做检查的，而是给开发者和类型检查工具（如 mypy、pyright）看的，帮助我们在运行代码之前就发现潜在的类型错误。

---

## 一、为什么要用类型提示

### 1.1 没有类型提示的问题

```python
# 看下面的函数，你能一眼看出它接收什么参数、返回什么吗？
def process(data):
    return data * 2

# 调用时：
process(5)        # 返回 10
process("hello")  # 返回 "hellohello"
process([1, 2])   # 返回 [1, 2, 1, 2]
```

没有类型提示时：
- 读代码的人不知道函数 expects 什么类型
- 写错了只有运行时才知道
- IDE 无法提供准确的智能补全

### 1.2 加上类型提示后

```python
def process(data: int) -> int:
    return data * 2

# 现在一目了然：
# - 接收一个 int 参数
# - 返回一个 int
# - 如果传了字符串，类型检查器会提前报警
```

### 1.3 类型提示的好处

| 好处 | 说明 |
|------|------|
| 🔍 **早期发现错误** | 类型检查工具在运行前就能发现类型错误 |
| 📖 **更好的文档** | 代码即文档，一看就知道函数接收什么返回什么 |
| 💡 **智能补全** | IDE 能提供更准确的代码补全 |
| 🛠️ **重构更安全** | 修改代码时类型检查器可以找出潜在问题 |

### 1.4 常用类型检查工具

| 工具 | 特点 |
|------|------|
| mypy | 最流行的静态类型检查器 |
| pyright | Microsoft 开发，速度快 |
| pylance | VS Code 默认的 Python 语言引擎 |

安装工具：
```bash
# 使用 uv 安装（推荐）
uv add --dev mypy pyright

# 或使用 pip
pip install mypy pyright
```

## 二、基础类型注解

### 2.1 基本类型

```python
# 变量类型注解
name: str = "Alice"
age: int = 25
height: float = 1.68
is_student: bool = False

# 无初始值的注解（先声明类型，稍后赋值）
count: int
items: list

# 稍后赋值
count = 0
items = []
```

### 2.2 容器类型

> **Python 3.9+** 可以直接使用 `list`、`dict` 等内置类型，之前的版本需要从 `typing` 导入 `List`、`Dict` 等。

```python
# Python 3.9+ 推荐写法（使用内置类型）
numbers: list[int] = [1, 2, 3]
scores: dict[str, int] = {"Alice": 90, "Bob": 85}
unique_ids: set[int] = {1, 2, 3}

# Python 3.9 之前的写法（现在仍常见）
from typing import List, Dict, Set

numbers: List[int] = [1, 2, 3]
scores: Dict[str, int] = {"Alice": 90, "Bob": 85}
```

### 2.3 元组类型（Tuple）

元组类型注解有两种形式：

```python
from typing import Tuple

# 形式 1：固定长度元组 - 指定每个位置的类型
coord: Tuple[float, float] = (10.5, 20.3)     # 必须 2 个元素
person: Tuple[str, int, bool] = ("Alice", 25, True)  # 必须 3 个元素

# 形式 2：可变长度元组 - 所有元素是同一类型
# 注意："可变长度"指的是类型注解可接受不同长度的元组，
#       但元组对象本身仍然是不可变的（immutable）
nums1: Tuple[int, ...] = ()              # 0 个元素
nums2: Tuple[int, ...] = (1,)            # 1 个元素
nums3: Tuple[int, ...] = (1, 2, 3)       # 3 个元素
nums4: Tuple[int, ...] = (1, 2, 3, 4, 5) # 多个元素

# Python 3.9+ 简化写法
coord: tuple[float, float] = (10.5, 20.3)
nums: tuple[int, ...] = (1, 2, 3)
```

### 2.4 函数类型注解

```python
# 基本函数注解
def greet(name: str) -> str:
    """打招呼"""
    return f"Hello, {name}!"

def add(a: int, b: int) -> int:
    """相加"""
    return a + b

# 多个参数，带默认值
def create_user(
    name: str,
    age: int,
    email: str,
    is_active: bool = True
) -> dict:
    """创建用户字典"""
    return {
        "name": name,
        "age": age,
        "email": email,
        "is_active": is_active
    }

# 无返回值（返回 None 也要标注）
def log_message(message: str) -> None:
    """记录日志，不返回值"""
    print(f"[LOG] {message}")
```

## 三、常用特殊类型

### 3.1 Optional：可能是 None

`Optional[X]` 表示"可能是 X 类型，也可能是 None"。

```python
from typing import Optional

# 查找用户，找到返回用户名，找不到返回 None
def find_user(user_id: int) -> Optional[str]:
    users = {1: "Alice", 2: "Bob"}
    return users.get(user_id)

# 使用示例
user = find_user(1)  # 可能是 "Alice"，也可能是 None

# 处理 Optional 值
if user is not None:
    print(user.upper())  # 这里 user 被确认为 str
else:
    print("用户不存在")
```

**Python 3.10+ 简化写法**：

```python
# Optional[str] 等价于 str | None
def find_user(user_id: int) -> str | None:
    users = {1: "Alice", 2: "Bob"}
    return users.get(user_id)
```

### 3.2 Union：多种类型之一

`Union[A, B]` 表示"可以是 A 类型，也可以是 B 类型"。

```python
from typing import Union

# 可以接受整数或字符串
def process_value(value: Union[int, str]) -> str:
    return str(value)

# 调用
process_value(10)      # ✅
process_value("hello") # ✅
```

**Python 3.10+ 简化写法**：

```python
# Union[int, str] 可以写成 int | str
def process_value(value: int | str) -> str:
    return str(value)

# 多个类型联合
def parse(data: int | str | float | bool) -> str:
    return str(data)
```

### 3.3 Literal：限制为特定字面值

`Literal` 用于限制变量只能是某些具体的值。

```python
from typing import Literal

# 状态只能是这两个字符串之一
def set_status(status: Literal["active", "inactive"]) -> None:
    print(f"状态：{status}")

# 调用
set_status("active")    # ✅
set_status("inactive")  # ✅
# set_status("unknown")  # ❌ 类型检查器会报错

# 限制为特定数字
def set_level(level: Literal[1, 2, 3, 4, 5]) -> None:
    print(f"等级：{level}")

set_level(3)   # ✅
# set_level(6) # ❌ 类型检查器会报错
```

### 3.4 Any：任意类型（慎用）

`Any` 是"逃生舱口"类型，表示放弃类型检查。

```python
from typing import Any

# 接受并返回任意类型
def process_any(data: Any) -> Any:
    return data

# 这些调用都是合法的
process_any(1)              # ✅
process_any("hello")        # ✅
process_any([1, 2, 3])      # ✅
```

**常见使用场景**：

```python
# 场景 1：解析 JSON（结构未知）
import json
from typing import Any

def parse_json(json_str: str) -> Any:
    return json.loads(json_str)

# 场景 2：处理混合类型列表
def sum_mixed(numbers: list[Any]) -> float:
    total = 0.0
    for n in numbers:
        if isinstance(n, (int, float)):
            total += n
    return total
```

> **建议**：尽量避免使用 `Any`，优先使用具体类型。如果必须使用，添加注释说明原因。

### 3.5 其他特殊类型

```python
from typing import NoReturn, Final, ClassVar

# NoReturn：永不返回（抛出异常或无限循环）
def raise_error(message: str) -> NoReturn:
    raise RuntimeError(message)

# Final：常量，不能修改
MAX_SIZE: Final = 100
PI: Final[float] = 3.14159
# MAX_SIZE = 200  # ❌ 类型错误

# Final 类：不能被继承
from typing import final

@final
class BaseConfig:
    """不能被继承"""
    pass

# class DevConfig(BaseConfig):  # ❌ 类型错误
#     pass

# ClassVar：类变量（不是实例变量）
class MyClass:
    class_attr: ClassVar[int] = 0  # 类变量
    instance_attr: int = 0         # 实例变量
```

## 四、进阶类型

### 4.1 TypedDict：带类型的字典

`TypedDict` 用于定义字典的键和类型。

```python
from typing import TypedDict, NotRequired

# 定义用户字典类型
class UserDict(TypedDict):
    id: int
    name: str
    email: str
    age: NotRequired[int]  # 可选字段（Python 3.11+）

# 使用
user: UserDict = {
    "id": 1,
    "name": "Alice",
    "email": "alice@example.com",
    # "age" 可选
}

# 类型检查
# 以下会报错：
# - 缺少必需字段 "email"
# - "id" 类型错误
# invalid_user: UserDict = {
#     "id": "not_an_int",  # ❌
#     "name": "Bob"
#     # 缺少 "email" ❌
# }
```

### 4.2 NamedTuple：带类型的命名元组

```python
from typing import NamedTuple

# 方式 1：类语法（推荐）
class Person(NamedTuple):
    name: str
    age: int
    email: str | None = None

# 使用
person = Person("Alice", 25)
print(person.name)   # Alice
print(person.age)    # 25
print(person.email)  # None

# 方式 2：函数语法
Point = NamedTuple('Point', [('x', float), ('y', float)])
p = Point(1.5, 2.5)
print(p.x, p.y)  # 1.5 2.5
```

### 4.3 Callable：函数类型

`Callable` 用于注解接收函数作为参数或返回函数的情况。

```python
from typing import Callable

# Callable[[参数类型...], 返回类型]

# 接收一个函数作为参数
def apply_operation(
    x: int,
    operation: Callable[[int], int]
) -> int:
    """应用操作"""
    return operation(x)

def double(n: int) -> int:
    return n * 2

result = apply_operation(5, double)  # 10

# 多个参数的函数
def combine(
    a: int,
    b: int,
    func: Callable[[int, int], int]
) -> int:
    return func(a, b)

def add(x: int, y: int) -> int:
    return x + y

result = combine(3, 4, add)  # 7

# 实际示例：过滤列表
from typing import List

def filter_list(
    items: List[int],
    predicate: Callable[[int], bool]
) -> List[int]:
    """过滤列表"""
    return [item for item in items if predicate(item)]

def is_even(n: int) -> bool:
    return n % 2 == 0

numbers = [1, 2, 3, 4, 5, 6]
evens = filter_list(numbers, is_even)  # [2, 4, 6]

# 使用 lambda
odds = filter_list(numbers, lambda x: x % 2 != 0)  # [1, 3, 5]
```

## 五、泛型和 TypeVar

### 5.1 TypeVar 基础

`TypeVar` 用于创建"类型变量"，表示某种未知但一致的类型。

```python
from typing import TypeVar, List

# 定义类型变量
T = TypeVar('T')

# 泛型函数：接收什么类型的列表，返回该类型的元素
def first_element(items: List[T]) -> T:
    """获取列表第一个元素"""
    return items[0]

# 类型推断
result = first_element([1, 2, 3])      # T 推断为 int
result = first_element(["a", "b"])     # T 推断为 str
```

### 5.2 约束 TypeVar

限制 `TypeVar` 只能是某些特定类型。

```python
from typing import TypeVar

# 限制为 int 或 float
Number = TypeVar('Number', int, float)

def add_numbers(a: Number, b: Number) -> Number:
    return a + b

add_numbers(1, 2)        # ✅ int
add_numbers(1.5, 2.5)    # ✅ float
# add_numbers(1, 2.5)    # ❌ 类型不匹配

# 有界 TypeVar：必须是某类型的子类
S = TypeVar('S', bound=str)  # 必须是 str 或其子类

def process_string(s: S) -> S:
    return s.upper()
```

### 5.3 泛型类

```python
from typing import Generic, TypeVar, List, Optional

T = TypeVar('T')

class Stack(Generic[T]):
    """泛型栈"""

    def __init__(self) -> None:
        self.items: List[T] = []

    def push(self, item: T) -> None:
        self.items.append(item)

    def pop(self) -> Optional[T]:
        if self.items:
            return self.items.pop()
        return None

    def peek(self) -> Optional[T]:
        if self.items:
            return self.items[-1]
        return None

    def is_empty(self) -> bool:
        return len(self.items) == 0

# 使用
int_stack = Stack[int]()
int_stack.push(1)
int_stack.push(2)
print(int_stack.pop())  # 2

str_stack = Stack[str]()
str_stack.push("hello")
str_stack.push("world")
print(str_stack.pop())  # world
```

### 5.4 Protocol：结构化类型

`Protocol` 用于定义"接口"，关注对象能做什么，而不是对象是什么类型。

```python
from typing import Protocol

# 定义协议
class Drawable(Protocol):
    """可绘制协议"""

    def draw(self) -> None:
        ...

    def get_size(self) -> tuple:
        ...

# 任何实现了这些方法的类都符合 Drawable 协议
class Circle:
    def __init__(self, radius: float):
        self.radius = radius

    def draw(self) -> None:
        print(f"绘制圆形，半径={self.radius}")

    def get_size(self) -> tuple:
        return (self.radius * 2, self.radius * 2)

class Square:
    def __init__(self, side: float):
        self.side = side

    def draw(self) -> None:
        print(f"绘制正方形，边长={self.side}")

    def get_size(self) -> tuple:
        return (self.side, self.side)

# 函数接受任何符合协议的类
def render(shape: Drawable) -> None:
    shape.draw()
    print(f"尺寸：{shape.get_size()}")

render(Circle(5))   # ✅
render(Square(10))  # ✅
```

## 六、类型检查实践

### 6.1 使用 mypy 检查代码

```bash
# 检查单个文件
mypy main.py

# 检查目录
mypy src/

# 详细输出
mypy --verbose main.py
```

```toml
[[tool.mypy.overrides]]

ignore_missing_imports = true
```

### 6.2 处理类型检查错误

```python
# 方式 1：修复代码（推荐）
def add(a: int, b: int) -> int:
    return a + b

# 方式 2：忽略特定行（必要时使用）
result = add(1, "2")  # type: ignore[arg-type]

# mypy: disable-error-code="annotation-unchecked"
```

## 七、最佳实践

### 7.1 推荐做法（DO）

```python
# 1. 为公共 API 添加完整类型注解
def calculate_total(
    prices: list[float],
    tax_rate: float = 0.1
) -> float:
    return sum(prices) * (1 + tax_rate)

# 2. 使用 Optional 表示可能为 None
def find_user(user_id: int) -> str | None:
    ...

# 3. 为复杂返回类型使用 TypedDict
from typing import TypedDict

class ApiResponse(TypedDict):
    status: int
    data: dict
    message: str

# 4. 不要注解局部变量（让类型推断工作）
def func():
    x = 1      # ✅ 不需要 x: int
    y = "hello"  # ✅ 不需要 y: str

# 5. self 和 cls 不需要注解
class MyClass:
    def method(self) -> None:  # ✅ 不需要 self: "MyClass"
        ...
```

### 7.2 避免的做法（DON'T）

```python
# 1. 不要过度使用 Any
# ❌
def process(data: Any) -> Any:
    ...

# ✅
def process(data: dict[str, int]) -> list[int]:
    ...

# 2. 不要使用可变默认值
# ❌
def add_item(item: int, items: list[int] = []) -> list[int]:
    items.append(item)
    return items

# ✅
def add_item(item: int, items: list[int] | None = None) -> list[int]:
    if items is None:
        items = []
    items.append(item)
    return items
```

## 八、实战练习

### 练习 1：为现有代码添加类型

```python
# 原始代码（无类型）
def calculate_average(numbers):
    if not numbers:
        return None
    return sum(numbers) / len(numbers)

# 添加类型注解后
from typing import Optional

def calculate_average(numbers: list[int | float]) -> Optional[float]:
    if not numbers:
        return None
    return sum(numbers) / len(numbers)
```

### 练习 2：定义 API 响应类型

```python
from typing import TypedDict, Literal, Optional

# 成功响应
class ApiSuccess(TypedDict):
    success: Literal[True]
    data: dict

# 失败响应
class ApiError(TypedDict):
    success: Literal[False]
    message: str

# 联合类型
ApiResponse = ApiSuccess | ApiError
```

## 总结

| 类别 | 类型注解 | 示例 |
|------|---------|------|
| 基本类型 | `int`, `str`, `float`, `bool` | `x: int = 1` |
| 空值 | `None`, `Optional[T]` | `name: str \| None` |
| 任意类型 | `Any` | `data: Any` |
| 容器 | `list[T]`, `dict[K,V]`, `tuple[...]` | `nums: list[int]` |
| 联合 | `Union[A,B]`, `A \| B` | `value: int \| str` |
| 字面量 | `Literal[...]` | `status: Literal["ok"]` |
| 函数 | `Callable[[A], R]` | `func: Callable[[int], str]` |
| 泛型 | `TypeVar`, `Generic[T]` | `class Box(Generic[T])` |
| 协议 | `Protocol` | `class Drawable(Protocol)` |
| TypedDict | `TypedDict` | `class User(TypedDict)` |

类型提示让 Python 代码更可靠、更易维护！
