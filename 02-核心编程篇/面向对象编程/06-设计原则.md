# 设计原则

本章讲解面向对象设计的基本原则，包括组合与继承的选择和最佳实践。

---

## 组合 vs 继承

### 概念说明

| | 继承 | 组合 |
|---|---|---|
| **关系** | "is-a"（是一个） | "has-a"（有一个） |
| **示例** | 狗 is-a 动物 | 汽车 has-a 发动机 |
| **代码** | `class Dog(Animal):` | `self.engine = Engine()` |
| **耦合度** | 高 | 低 |

### 继承示例

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def breathe(self):
        print(f"{self.name} 在呼吸")

class Dog(Animal):
    def bark(self):
        print(f"{self.name} 在叫")

# Dog is-a Animal
dog = Dog("Buddy")
dog.breathe()  # 继承的方法
dog.bark()     # 自己的方法
```

### 组合示例

```python
class Engine:
    """发动机类"""
    def start(self):
        return "发动机启动"

class Wheel:
    """轮子类"""
    def rotate(self):
        return "轮子转动"

class Car:
    """汽车类 - 使用组合"""
    def __init__(self, brand):
        self.brand = brand
        # 组合：汽车"有"发动机和轮子
        self.engine = Engine()
        self.wheels = [Wheel() for _ in range(4)]

    def drive(self):
        result = [f"{self.brand} 汽车："]
        result.append(self.engine.start())
        for wheel in self.wheels:
            result.append(wheel.rotate())
        return "\n".join(result)


# 使用
car = Car("特斯拉")
print(car.drive())
# 特斯拉 汽车：
# 发动机启动
# 轮子转动
# 轮子转动
# 轮子转动
# 轮子转动
```

### 设计原则

```
┌─────────────────────────────────────────────────────────────┐
│              组合 vs 继承 选择指南                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   使用继承：                                                 │
│   ✓ 确实存在 "is-a" 关系                                   │
│   ✓ 需要重写或扩展父类行为                                 │
│   ✓ 例如：Dog is-a Animal                                  │
│                                                             │
│   使用组合：                                                 │
│   ✓ 存在 "has-a" 关系                                      │
│   ✓ 需要灵活替换组件                                       │
│   ✓ 需要降低耦合度                                         │
│   ✓ 例如：Car has-a Engine                                 │
│                                                             │
│   设计原则：                                                 │
│   ✓ 组合优于继承                                            │
│   ✓ 继承层次不宜过深（建议不超过 3 层）                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## SOLID 原则简介

### 五大设计原则

```
┌─────────────────────────────────────────────────────────────┐
│              SOLID 原则                                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   S - 单一职责原则（Single Responsibility）                 │
│       一个类只做一件事                                      │
│                                                             │
│   O - 开放封闭原则（Open/Closed）                           │
│       对扩展开放，对修改封闭                                │
│                                                             │
│   L - 里氏替换原则（Liskov Substitution）                   │
│       子类可以替换父类                                      │
│                                                             │
│   I - 接口隔离原则（Interface Segregation）                 │
│       接口要小而专一                                        │
│                                                             │
│   D - 依赖倒置原则（Dependency Inversion）                  │
│       依赖抽象而非具体实现                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 单一职责示例

```python
# ❌ 违反单一职责：一个类做太多事
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

    def save(self):
        # 保存到数据库
        pass

    def send_email(self, message):
        # 发送邮件
        pass

    def generate_report(self):
        # 生成报告
        pass


# ✅ 遵循单一职责：每个类只做一件事
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

class UserRepository:
    def save(self, user):
        # 保存到数据库
        pass

class EmailService:
    def send(self, user, message):
        # 发送邮件
        pass

class ReportGenerator:
    def generate(self, user):
        # 生成报告
        pass
```

## 最佳实践

```
┌─────────────────────────────────────────────────────────────┐
│              面向对象最佳实践                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   命名规范：                                                 │
│   ✓ 类名用大驼峰：MyClassName                               │
│   ✓ 方法和属性用小写+下划线：my_method                      │
│   ✓ 私有属性用双下划线：__private                           │
│                                                             │
│   设计原则：                                                 │
│   ✓ 优先使用组合而非继承                                    │
│   ✓ 单一职责：一个类只做一件事                              │
│   ✓ 开放封闭：对扩展开放，对修改封闭                        │
│                                                             │
│   避免：                                                     │
│   ✗ 上帝类（一个类做太多事）                                │
│   ✗ 过度继承（建议不超过 3 层）                             │
│   ✗ 直接访问私有属性                                        │
│   ✗ 使用可变对象作为类属性                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
