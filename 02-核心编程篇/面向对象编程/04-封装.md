# 封装

本章讲解 Python 面向对象编程中的封装机制，包括私有属性、私有方法和 @property 装饰器。

---

## 什么是封装

### 概念说明

**封装** 是面向对象编程的核心特性之一，它将数据（属性）和操作数据的方法绑定在一起，并隐藏内部实现细节。

```
┌─────────────────────────────────────────────────────────────┐
│                      封装的作用                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 数据隐藏                                               │
│      隐藏内部实现细节，只暴露必要的接口                     │
│                                                             │
│   2. 数据保护                                               │
│      防止外部直接修改内部数据，保证数据安全                 │
│                                                             │
│   3. 降低耦合                                               │
│      内部实现改变不影响外部调用                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 私有属性和方法

### 命名约定

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.__balance = balance  # 私有属性（双下划线）
        self.__transaction_count = 0

    def deposit(self, amount):
        """存款 - 公共接口"""
        if amount > 0:
            self.__balance += amount
            self.__transaction_count += 1
            print(f"存入{amount}，余额：{self.__balance}")
            return True
        print("存款金额必须大于 0")
        return False

    def withdraw(self, amount):
        """取款 - 公共接口"""
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            self.__transaction_count += 1
            print(f"取出{amount}，余额：{self.__balance}")
            return True
        print("余额不足")
        return False

    def get_balance(self):
        """查询余额 - 公共接口"""
        return self.__balance

    def __validate_amount(self, amount):
        """私有方法：验证金额"""
        return amount > 0


# 使用
account = BankAccount("张三", 1000)

# ✅ 通过公共接口访问
print(account.get_balance())  # 1000
account.deposit(500)
account.withdraw(200)

# ❌ 无法直接访问私有属性
# print(account.__balance)  # AttributeError!

# 可以通过名称改写访问（但不推荐）
print(account._BankAccount__balance)  # 1300
```

### 访问级别

| 命名方式 | 访问级别 | 说明 |
|----------|----------|------|
| `name` | 公共 | 可以直接访问 |
| `_name` | 受保护 | 约定不直接访问（单下划线） |
| `__name` | 私有 | 名称改写，外部无法直接访问 |

## @property 装饰器

### 概念说明

`@property` 让方法像属性一样调用，可以添加验证逻辑。

```python
class Person:
    def __init__(self, age):
        self.__age = age  # 私有属性

    @property
    def age(self):
        """getter 方法：获取 age"""
        return self.__age

    @age.setter
    def age(self, value):
        """setter 方法：设置 age"""
        if 0 < value < 150:
            self.__age = value
        else:
            raise ValueError("年龄必须在 0-150 之间")

    @property
    def age_group(self):
        """只读属性：年龄段"""
        if self.__age < 18:
            return "未成年"
        elif self.__age < 60:
            return "成年人"
        else:
            return "老年人"


# 使用
p = Person(25)

# 调用 getter（像访问属性一样）
print(p.age)  # 25

# 调用 setter
p.age = 26  # 自动调用 setter 方法

# 调用只读属性
print(p.age_group)  # 成年人

# ❌ 无法设置只读属性
# p.age_group = "青年"  # AttributeError!
```

### @property 的作用

```
┌─────────────────────────────────────────────────────────────┐
│              @property 的作用                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 让方法像属性一样调用（不用加括号）                     │
│                                                             │
│   2. 可以添加验证逻辑                                       │
│      在 setter 中验证数据有效性                             │
│                                                             │
│   3. 可以创建只读属性                                       │
│      只定义 getter，不定义 setter                           │
│                                                             │
│   4. 计算属性                                               │
│      动态计算并返回值                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
