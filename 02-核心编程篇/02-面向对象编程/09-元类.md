## 为什么需要元类？

### 问题场景

你在开发一个框架，要求所有模型类：
- 自动添加日志记录功能
- 自动注册到全局注册表
- 强制某些属性命名规范

**传统方式：手动在每个类添加**

```python
models_registry: dict[str, type] = {}

class User:
    _log_created = True
    
    def __init__(self, name: str) -> None:
        self.name = name
        if self._log_created:
            print(f"[LOG] User created: {name}")
    
    def _validate(self) -> None:
        if not self.name.islower():
            raise ValueError("name must be lowercase")

models_registry["User"] = User

class Product:
    _log_created = True
    
    def __init__(self, sku: str) -> None:
        self.sku = sku
        if self._log_created:
            print(f"[LOG] Product created: {sku}")
    
    def _validate(self) -> None:
        if not self.sku.islower():
            raise ValueError("sku must be lowercase")

models_registry["Product"] = Product
```

**问题：**
- 每个类都要重复写 `_log_created`、注册、验证逻辑
- 新增类时容易遗漏某个步骤
- 代码重复，维护困难

**元类方式：自动注入功能**

```python
models_registry: dict[str, type] = {}

class AutoModel(type):
    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        namespace["_log_created"] = True
        namespace["_validate"] = lambda self: None
        new_class = super().__new__(cls, name, bases, namespace)
        models_registry[name] = new_class
        return new_class

class User(metaclass=AutoModel):
    def __init__(self, name: str) -> None:
        self.name = name
        if self._log_created:
            print(f"[LOG] User created: {name}")

class Product(metaclass=AutoModel):
    def __init__(self, sku: str) -> None:
        self.sku = sku
        if self._log_created:
            print(f"[LOG] Product created: {sku}")

print(models_registry)
# {'User': <class 'User'>, 'Product': <class 'Product'>}
```

这就是元类的价值：**控制类的创建过程，自动注入功能**。

## L1 理解层：元类基础

### 类本身是对象

理解元类首先要理解：**类定义执行后，产生的是一个对象**。

```python
class User:
    name: str = ""

MyClass = User
obj = MyClass()
print(type(obj))
print(type(MyClass))
```

**关键代码说明：**

| 代码 | 含义 |
|------|------|
| `class User:` | 定义一个类，执行后产生"类对象" |
| `MyClass = User` | 类对象可以赋值给变量 |
| `obj = MyClass()` | 类对象可以创建实例 |
| `type(obj)` | obj 的类型是 User（类对象） |
| `type(MyClass)` | MyClass 的类型是 type（元类） |

### type 动态创建类

`type()` 有两种用法：

```python
print(type(42))
print(type("hello"))

MyClass = type("DynamicClass", (object,), {"attr": 100, "method": lambda self: self.attr})
obj = MyClass()
print(obj.attr)
print(obj.method())
print(type(MyClass))
print(type(obj))
```

**type() 创建类的语法：**

```
┌─────────────────────────────────────────────────────────────┐
│  type(name, bases, namespace)                               │
│                                                             │
│  name      → 类名（字符串）                                  │
│  bases     → 基类列表（元组）                                │
│  namespace → 属性和方法字典                                  │
│                                                             │
│  示例：                                                      │
│  type("Point", (), {"x": 0, "y": 0})                        │
│                                                             │
│  等价于：                                                    │
│  class Point:                                               │
│      x = 0                                                  │
│      y = 0                                                  │
└─────────────────────────────────────────────────────────────┘
```

**详细示例：**

```python
def get_info(self) -> str:
    return f"{self.name} ({self.age})"

Person = type(
    "Person",
    (object,),
    {
        "name": "",
        "age": 0,
        "get_info": get_info,
    },
)

p = Person()
p.name = "张三"
p.age = 25
print(p.get_info())
print(type(Person))
print(type(p))
```

### 自定义元类

自定义元类就是继承 `type`，重写 `__new__` 或 `__init__`。

```
┌─────────────────────────────────────────────────────────────┐
│  元类定义语法                                                │
│                                                             │
│  class MetaClassName(type):                                 │
│      def __new__(cls, name, bases, namespace):              │
│          # 修改 namespace 或添加属性                         │
│          return super().__new__(cls, name, bases, namespace)│
│                                                             │
│  class MyClass(metaclass=MetaClassName):                    │
│      # 元类 MetaClassName 创建这个类                         │
│      pass                                                   │
│                                                             │
│  流程：                                                      │
│  1. class MyClass 定义时                                    │
│  2. Python 调用 MetaClassName.__new__()                     │
│  3. MetaClassName 修改/添加属性                             │
│  4. 返回创建好的类对象                                       │
└─────────────────────────────────────────────────────────────┘
```

**最简示例：**

```python
class LogMeta(type):
    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        print(f"[元类] 正在创建类: {name}")
        return super().__new__(cls, name, bases, namespace)

class User(metaclass=LogMeta):
    pass

class Product(metaclass=LogMeta):
    pass

u = User()
p = Product()
```

**详细示例：自动添加属性**

```python
class AutoAttrMeta(type):
    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        namespace["_created_at"] = "2024-01-01"
        namespace["_class_name"] = name
        return super().__new__(cls, name, bases, namespace)

class Document(metaclass=AutoAttrMeta):
    title: str = ""
    
    def get_meta(self) -> dict[str, str]:
        return {
            "created_at": self._created_at,
            "class_name": self._class_name,
        }

doc = Document()
print(doc._created_at)
print(doc._class_name)
print(doc.get_meta())
print(Document._created_at)
print(Document._class_name)
```

**关键代码说明：**

| 代码 | 含义 |
|------|------|
| `cls` | 元类本身（LogMeta） |
| `name` | 正在创建的类名（"User"、"Product"） |
| `bases` | 基类列表（tuple） |
| `namespace` | 类的属性和方法字典 |
| `super().__new__(...)` | 调用 type.__new__ 完成类创建 |
| `metaclass=LogMeta` | 指定用哪个元类创建这个类 |

## L2 实践层：元类应用

### 实际应用场景

#### 场景1：自动注册类

```python
plugins: dict[str, type] = {}

class PluginMeta(type):
    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        new_class = super().__new__(cls, name, bases, namespace)
        if name != "BasePlugin":
            plugins[name.lower()] = new_class
        return new_class

class BasePlugin(metaclass=PluginMeta):
    def run(self) -> None:
        raise NotImplementedError

class PDFPlugin(BasePlugin):
    def run(self) -> None:
        print("处理 PDF 文件")

class ImagePlugin(BasePlugin):
    def run(self) -> None:
        print("处理图片文件")

print(plugins)

plugin = plugins["pdfplugin"]()
plugin.run()
```

**说明：**
- 所有 Plugin 子类自动注册到 `plugins` 字典
- BasePlugin 被排除（它是基类，不是实际插件）
- 用户可以通过名称查找并使用插件

#### 场景2：强制属性命名规范

```python
class UpperAttrMeta(type):
    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        uppercase_namespace: dict[str, object] = {}
        for key, value in namespace.items():
            if key.startswith("__") and key.endswith("__"):
                uppercase_namespace[key] = value
            elif callable(value):
                uppercase_namespace[key] = value
            else:
                uppercase_namespace[key.upper()] = value
        return super().__new__(cls, name, bases, uppercase_namespace)

class Config(metaclass=UpperAttrMeta):
    database_url = "localhost:5432"
    api_key = "secret123"
    timeout = 30

print(Config.DATABASE_URL)
print(Config.API_KEY)
print(Config.TIMEOUT)
```

**说明：**
- 元类将普通属性名转为大写
- 魔术方法和方法保持原名
- 强制命名规范，无需手动约定

#### 场景3：ORM 简化版

```python
class ModelMeta(type):
    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        fields: list[str] = []
        for key, value in namespace.items():
            if not key.startswith("_") and not callable(value):
                fields.append(key)
        namespace["_fields"] = fields
        new_class = super().__new__(cls, name, bases, namespace)
        return new_class

class Model(metaclass=ModelMeta):
    def to_dict(self) -> dict[str, object]:
        return {field: getattr(self, field, None) for field in self._fields}

class User(Model):
    name: str = ""
    email: str = ""
    age: int = 0

class Product(Model):
    sku: str = ""
    price: float = 0.0

print(User._fields)
print(Product._fields)

u = User()
u.name = "张三"
u.email = "zhang@example.com"
u.age = 25
print(u.to_dict())

p = Product()
p.sku = "ABC123"
p.price = 99.9
print(p.to_dict())
```

**说明：**
- 元类自动收集字段列表
- Model 提供通用的 `to_dict()` 方法
- 子类只需定义字段，无需手动维护字段列表

### 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| **框架开发时使用** | 元类是框架级工具 | Django ORM、ABC 抽象基类 |
| **用简单方案替代** | 大多数需求有更简单方案 | 装饰器、__init_subclass__、类装饰器 |
| **保持元类简单** | 复杂元类难以理解和调试 | 只做一件事 |
| **命名约定** | 元类名以 Meta 结尾 | `LogMeta`、`PluginMeta` |

### 反模式：不要这样做

```python
class CrazyMeta(type):
    def __new__(cls, name, bases, namespace):
        for i in range(100):
            namespace[f"auto_method_{i}"] = lambda self: i
        namespace["__getattr__"] = lambda self, name: f"动态属性: {name}"
        namespace["__setattr__"] = lambda self, name, value: print("禁止修改")
        return super().__new__(cls, name, bases, namespace)

class MyClass(metaclass=CrazyMeta):
    pass

obj = MyClass()
print(obj.auto_method_50())
print(obj.anything)
obj.x = 1
```

**问题：**
- 元类过于复杂，难以理解
- 动态注入大量方法，影响可读性
- 魔术行为过多，调试困难

```python
class SimpleMeta(type):
    def __new__(cls, name, bases, namespace):
        namespace["_version"] = "1.0"
        return super().__new__(cls, name, bases, namespace)

class MyClass(metaclass=SimpleMeta):
    pass

obj = MyClass()
print(MyClass._version)
```

**正确做法：**
- 元类只做一件事
- 明确的、可预测的行为
- 易于理解和调试

### 元类替代方案

大多数场景下，元类有更简单的替代方案：

```
┌─────────────────────────────────────────────────────────────┐
│          元类替代方案对比                                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 类装饰器                                                │
│  ─────────────────────────────────────────────              │
│  • 在类定义后修改类                                         │
│  • 更简单、更直观                                           │
│  • 适用：修改已定义的类                                     │
│                                                             │
│  2. __init_subclass__                                       │
│  ─────────────────────────────────────────────              │
│  • Python 3.6+ 引入                                         │
│  • 在父类中定义，子类创建时自动调用                         │
│  • 适用：子类初始化、注册                                   │
│                                                             │
│  3. 描述符                                                  │
│  ─────────────────────────────────────────────              │
│  • 控制属性访问行为                                         │
│  • 适用：属性验证、类型检查                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 替代方案1：类装饰器

```python
def add_version(cls: type) -> type:
    cls._version = "1.0"
    cls._created_at = "2024-01-01"
    return cls

def register_plugin(cls: type) -> type:
    if cls.__name__ != "BasePlugin":
        plugins[cls.__name__.lower()] = cls
    return cls

plugins: dict[str, type] = {}

@add_version
@register_plugin
class PDFPlugin:
    def run(self) -> None:
        print("处理 PDF")

print(PDFPlugin._version)
print(PDFPlugin._created_at)
print(plugins)
```

**对比元类：**
- 类装饰器更简单、更直观
- 可以组合多个装饰器
- 但无法在"类创建时"介入（类已创建后才执行）

#### 替代方案2：__init_subclass__

```python
plugins: dict[str, type] = {}

class BasePlugin:
    def __init_subclass__(cls, **kwargs: object) -> None:
        super().__init_subclass__(**kwargs)
        plugins[cls.__name__.lower()] = cls
    
    def run(self) -> None:
        raise NotImplementedError

class PDFPlugin(BasePlugin):
    def run(self) -> None:
        print("处理 PDF")

class ImagePlugin(BasePlugin):
    def run(self) -> None:
        print("处理图片")

print(plugins)
```

**对比元类：**
- `__init_subclass__` 是 Python 3.6+ 推荐方式
- 在父类定义，子类创建时自动调用
- 更简单，无需自定义元类
- 但只能在父类中定义，不如元类灵活

### 适用场景

| 场景 | 是否推荐元类 | 原因 |
|------|-------------|------|
| 框架开发（ORM、插件系统） | ✅ 推荐 | 需要深度控制类创建 |
| 自动注册子类 | ❌ 不推荐 | 用 `__init_subclass__` 更简单 |
| 添加类属性 | ❌ 不推荐 | 用类装饰器更简单 |
| 强制接口规范 | ❓ 可选 | ABC 抽象基类已用元类实现 |
| 动态修改类方法 | ❓ 可选 | 视复杂度决定 |
| 简单应用开发 | ❌ 不推荐 | 过度设计，增加复杂度 |

## L3 专家层：元类底层原理

### Python 类创建流程

```
┌─────────────────────────────────────────────────────────────┐
│          类创建流程                                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  class MyClass(Base):                                       │
│      attr = 100                                             │
│                                                             │
│  Python 执行步骤：                                          │
│                                                             │
│  1. 准备命名空间                                            │
│     namespace = {"attr": 100, "__module__": ...}           │
│                                                             │
│  2. 确定元类                                                │
│     • 如果指定 metaclass → 使用指定的                       │
│     • 如果基类有元类 → 继承基类的元类                       │
│     • 否则 → 使用 type                                      │
│                                                             │
│  3. 调用元类的 __prepare__                                  │
│     namespace = Meta.__prepare__(name, bases)               │
│     （返回字典，用于收集类属性）                             │
│                                                             │
│  4. 执行类体，填充 namespace                                │
│     class body → namespace                                  │
│                                                             │
│  5. 调用元类的 __new__                                      │
│     cls = Meta.__new__(Meta, name, bases, namespace)        │
│     （创建类对象）                                           │
│                                                             │
│  6. 调用元类的 __init__                                     │
│     Meta.__init__(cls, name, bases, namespace)              │
│     （初始化类对象）                                         │
│                                                             │
│  7. 返回类对象                                              │
│     MyClass = cls                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### __prepare__ 方法

`__prepare__` 在类创建最早期调用，返回命名空间容器。

```python
class OrderedMeta(type):
    @classmethod
    def __prepare__(
        cls,
        name: str,
        bases: tuple[type, ...],
        **kwargs: object,
    ) -> dict[str, object]:
        print(f"[__prepare__] 准备命名空间: {name}")
        return {}

    def __new__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> type:
        print(f"[__new__] 创建类: {name}")
        print(f"  namespace: {namespace}")
        return super().__new__(cls, name, bases, namespace)

    def __init__(
        cls,
        name: str,
        bases: tuple[type, ...],
        namespace: dict[str, object],
    ) -> None:
        print(f"[__init__] 初始化类: {name}")

class MyClass(metaclass=OrderedMeta):
    attr1 = 100
    attr2 = 200
```

**说明：**
- `__prepare__` 最先调用，准备命名空间
- 可以返回自定义字典（如 OrderedDict）保持定义顺序
- Python 3.6+ 类属性已默认保持定义顺序

### 元类继承规则

```python
class MetaA(type):
    pass

class MetaB(type):
    pass

class BaseA(metaclass=MetaA):
    pass

class BaseB(metaclass=MetaB):
    pass

class Child(BaseA, BaseB):
    pass
```

**运行结果：**

```
TypeError: metaclass conflict: 
the metaclass of a derived class must be a (non-strict) subclass 
of the metaclass of all its bases
```

**原因：**
- BaseA 的元类是 MetaA
- BaseB 的元类是 MetaB
- Child 继承两者，必须用兼容的元类

**解决方案：**

```python
class MetaA(type):
    pass

class MetaB(type):
    pass

class CombinedMeta(MetaA, MetaB):
    pass

class BaseA(metaclass=MetaA):
    pass

class BaseB(metaclass=MetaB):
    pass

class Child(BaseA, BaseB, metaclass=CombinedMeta):
    pass

print(type(Child))
```

### type 的元类是什么？

```python
print(type(type))
print(type.__class__)
```

**结果：** `type` 本身就是 `type` 的实例。

### 元类性能考量

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 元类 `__new__` | O(n) | n = 类属性数量 |
| 类创建 | O(n+m) | n=属性，m=基类 |
| 实例创建 | O(1) | 与普通类相同 |
| 实例方法调用 | O(1) | 与普通类相同 |

**性能影响：**

- 类创建时一次性开销（元类介入）
- 实例创建和使用无额外开销
- 不影响运行时性能

```python
import timeit

setup_normal = """
class NormalClass:
    attr = 100
"""

setup_meta = """
class MyMeta(type):
    def __new__(cls, name, bases, namespace):
        namespace['attr'] = 100
        return super().__new__(cls, name, bases, namespace)

class MetaClass(metaclass=MyMeta):
    pass
"""

normal_time = timeit.timeit("NormalClass()", setup=setup_normal, number=100000)
meta_time = timeit.timeit("MetaClass()", setup=setup_meta, number=100000)

print(f"普通类实例创建: {normal_time:.4f}s")
print(f"元类类实例创建: {meta_time:.4f}s")
print(f"差异: {abs(normal_time - meta_time):.4f}s")
```

### 设计动机

| 设计选择 | 原因 |
|----------|------|
| type 是所有类的元类 | 统一模型，简化实现 |
| 元类可以自定义 | 允许框架级扩展 |
| `__prepare__` 存在 | 提供最早期介入点 |
| `__init_subclass__` 引入 | 提供更简单的替代方案 |
