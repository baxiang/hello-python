# 面向对象基础

> **Python 版本要求：** 3.11+

本章讲解 Python 面向对象编程的基本概念，包括类与对象的关系、类的定义和对象的创建。

---

## 为什么需要面向对象？

### 问题场景

假设你要开发一个学生管理系统，需要管理大量学生的信息（姓名、年龄、成绩等）。

**使用面向过程的方式：**

```python
# 每个学生需要多个变量
student1_name = "张三"
student1_age = 18
student1_score = 85

student2_name = "李四"
student2_age = 17
student2_score = 92

# 每个学生都需要单独处理
def print_student(name: str, age: int, score: int) -> None:
    print(f"{name}, {age}岁, 成绩:{score}")

print_student(student1_name, student1_age, student1_score)
print_student(student2_name, student2_age, student2_score)
```

**问题：**
- 学生数量增加时，变量数量爆炸
- 相关数据没有组织在一起，容易出错
- 代码难以维护和扩展

**使用面向对象的方式：**

```python
class Student:
    def __init__(self, name: str, age: int, score: int) -> None:
        self.name = name
        self.age = age
        self.score = score
    
    def print_info(self) -> None:
        print(f"{self.name}, {self.age}岁, 成绩:{self.score}")

# 创建学生对象
students = [
    Student("张三", 18, 85),
    Student("李四", 17, 92),
]

# 统一处理
for student in students:
    student.print_info()
```

**优势：**
- 数据和行为封装在一起
- 代码更易维护和扩展
- 可以创建任意数量的学生对象

## 第一部分：面向过程 vs 面向对象

### 1.1 概念动机

面向对象是一种编程思想，它将数据和操作数据的方法封装在一起，形成"对象"。

```
┌─────────────────────────────────────────────────────────────┐
│              面向过程 vs 面向对象                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   面向过程：                                                 │
│   ✓ 关注"怎么做"（步骤）                                   │
│   ✓ 程序 = 函数 + 数据                                     │
│   ✓ 例如：洗菜() → 切菜() → 炒菜()                         │
│                                                             │
│   面向对象：                                                 │
│   ✓ 关注"谁来做"（对象）                                   │
│   ✓ 程序 = 对象 + 消息                                     │
│   ✓ 例如：厨师.洗菜()、厨师.切菜()、厨师.炒菜()            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 最简示例

```python
# ─────────────────────────────────────
# 面向过程：关注"怎么做"（步骤）
# ─────────────────────────────────────

def 洗菜() -> None:
    print("洗菜")

def 切菜() -> None:
    print("切菜")

def 炒菜() -> None:
    print("炒菜")

# 按步骤调用
洗菜()
切菜()
炒菜()


# ─────────────────────────────────────
# 面向对象：关注"谁来做"（对象）
# ─────────────────────────────────────

class 厨师:
    def 洗菜(self) -> None:
        print("洗菜")

    def 切菜(self) -> None:
        print("切菜")

    def 炒菜(self) -> None:
        print("炒菜")

# 创建对象并调用
chef = 厨师()
chef.洗菜()
chef.切菜()
chef.炒菜()
```

**核心理念：**
- **面向过程**：把问题分解为一系列步骤
- **面向对象**：找出问题域中的对象和它们的交互

### 1.3 详细讲解

```
类（Class）：模板/蓝图
│
├── 例如："Dog"类定义了狗的共性
│   - 属性：name, age, breed
│   - 方法：bark(), run()
│
▼
对象（Object）：具体的实例
│
├── my_dog = Dog("Buddy", 3)   → 具体的狗：Buddy，3 岁
├── your_dog = Dog("Max", 5)   → 具体的狗：Max，5 岁
└── his_dog = Dog("Lucy", 2)   → 具体的狗：Lucy，2 岁
```

**类比理解：**
- 类 = 月饼模具，对象 = 用模具做出的月饼
- 类 = 建筑图纸，对象 = 按图纸建好的房子
- 类 = 菜谱，对象 = 按菜谱做好的菜

## 第二部分：定义和创建类

### 2.1 概念动机

类是创建对象的模板，定义了对象应该有哪些属性和方法。

### 2.2 最简示例

**基本语法：**

```python
class ClassName:
    """类的文档字符串 - 说明类的用途"""

    def __init__(self, param1: str, param2: int) -> None:
        """初始化方法（构造函数）"""
        # 实例属性（每个实例独立）
        self.attribute1 = param1
        self.attribute2 = param2

    def method1(self) -> None:
        """实例方法"""
        pass
```

### 2.3 详细讲解

```python
class Dog:
    """表示狗的类"""

    # 类属性（所有实例共享）
    species: str = "Canis familiaris"

    def __init__(self, name: str, age: int, breed: str = "中华田园犬") -> None:
        """
        初始化方法
        参数:
            name: 名字
            age: 年龄
            breed: 品种（默认"中华田园犬"）
        """
        # 实例属性（每个实例独立）
        self.name: str = name
        self.age: int = age
        self.breed: str = breed

    def bark(self) -> None:
        """狗叫"""
        print(f"{self.name} 在叫：汪汪汪！")

    def run(self) -> None:
        """跑步"""
        print(f"{self.name} 正在奔跑！")

    def get_human_age(self) -> int:
        """计算人类年龄"""
        return self.age * 7


# ─────────────────────────────────────
# 创建和使用对象
# ─────────────────────────────────────

# 创建对象（实例化）
my_dog: Dog = Dog("Buddy", 3)
your_dog: Dog = Dog("Max", 5, "金毛")

# 访问属性
print(my_dog.name)      # Buddy
print(my_dog.species)   # Canis familiaris（类属性）

# 调用方法
my_dog.bark()             # Buddy 在叫：汪汪汪！
print(my_dog.get_human_age())  # 21
```

### 2.4 渐进复杂化

当类需要更多功能时，可以添加更多属性和方法：

```python
class Dog:
    """更完整的狗类"""
    
    species: str = "Canis familiaris"
    
    def __init__(self, name: str, age: int, breed: str = "中华田园犬", owner: str = "") -> None:
        self.name: str = name
        self.age: int = age
        self.breed: str = breed
        self.owner: str = owner
        self.energy: int = 100  # 能量值
    
    def bark(self, volume: str = "normal") -> None:
        """不同音量的叫声"""
        if volume == "loud":
            print(f"{self.name} 在大声叫：汪汪汪！")
        elif volume == "quiet":
            print(f"{self.name} 在小声叫：汪...")
        else:
            print(f"{self.name} 在叫：汪汪汪！")
    
    def run(self, distance: int) -> None:
        """跑步消耗能量"""
        if self.energy >= distance:
            self.energy -= distance
            print(f"{self.name} 跑了 {distance}米，剩余能量：{self.energy}")
        else:
            print(f"{self.name} 太累了，需要休息")
    
    def rest(self) -> None:
        """休息恢复能量"""
        self.energy = min(100, self.energy + 20)
        print(f"{self.name} 休息后能量：{self.energy}")
    
    def get_info(self) -> dict[str, str | int]:
        """获取完整信息"""
        return {
            "name": self.name,
            "age": self.age,
            "breed": self.breed,
            "owner": self.owner,
            "energy": self.energy,
        }
```

### 2.5 实际应用

创建多个狗对象并管理：

```python
# 创建狗群
dogs = [
    Dog("Buddy", 3, "金毛", "张三"),
    Dog("Max", 5, "拉布拉多", "李四"),
    Dog("Lucy", 2, "柯基", "王五"),
]

# 让每只狗活动
for dog in dogs:
    dog.bark()
    dog.run(30)
    print(dog.get_info())
    dog.rest()
```

## 第三部分：核心要点总结

### 3.1 核心概念表

| 概念 | 说明 | 示例 |
|------|------|------|
| **类** | 对象的模板 | `class Dog:` |
| **对象** | 类的具体实例 | `my_dog = Dog()` |
| **实例属性** | 每个实例独立 | `self.name` |
| **类属性** | 所有实例共享 | `species = "..."` |
| **`__init__`** | 初始化方法，创建对象时自动调用 | `def __init__(self):` |
| **self** | 指向当前实例对象 | 方法第一个参数 |

## L2 实践层：最佳实践

### 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| **类名使用大驼峰命名** | 符合PEP8规范，与函数名区分 | `class StudentManager:` |
| **方法和属性使用小写+下划线** | 符合PEP8规范，提高可读性 | `def get_student_info():` |
| **私有属性用单下划线前缀** | 约定俗成，表明内部使用 | `self._internal_data` |
| **类属性用于共享常量** | 所有实例共享，节省内存 | `MAX_SIZE = 100` |
| **在`__init__`中初始化所有属性** | 对象创建后状态完整，避免AttributeError | `self.name = name` |
| **使用类型注解** | 提高代码可读性，便于IDE检查 | `self.age: int = age` |
| **添加文档字符串** | 自动生成文档，帮助其他开发者 | `"""学生类"""` |

### 反模式：不要这样做

```python
# ❌ 类名使用小写或下划线
class student_manager:  # 不符合命名规范
    pass

# ❌ 属性在 __init__ 外部动态添加
class Student:
    def __init__(self, name: str) -> None:
        self.name = name

    def add_age(self, age: int) -> None:
        self.age = age  # 可能导致 AttributeError

s = Student("张三")
print(s.age)  # AttributeError! age 未在 __init__ 中初始化

# ❌ 使用可变对象作为类属性
class Config:
    default_settings: list[str] = []  # 所有实例共享同一个列表！

c1 = Config()
c1.default_settings.append("theme")
c2 = Config()
print(c2.default_settings)  # ['theme'] - 被污染了！

# ❌ 类做太多事情（上帝类）
class GodClass:
    """违反单一职责原则"""
    def manage_users(self) -> None:
        pass
    def process_orders(self) -> None:
        pass
    def send_emails(self) -> None:
        pass
    def generate_reports(self) -> None:
        pass
```

```python
# ✅ 正确做法

# 类名大驼峰
class StudentManager:
    pass

# 所有属性在 __init__ 中初始化
class Student:
    def __init__(self, name: str, age: int = 0) -> None:
        self.name: str = name
        self.age: int = age  # 提供默认值

# 类属性使用不可变对象或 None
class Config:
    default_settings: tuple[str, ...] = ()  # 不可变

    def __init__(self) -> None:
        self.settings: list[str] = []  # 实例属性

# 单一职责：每个类只做一件事
class UserManager:
    def manage_users(self) -> None:
        pass

class OrderProcessor:
    def process_orders(self) -> None:
        pass
```

### 常见陷阱

| 陷阱 | 问题 | 解决方案 |
|------|------|----------|
| 可变类属性 | 所有实例共享，修改互相影响 | 使用不可变类型或改为实例属性 |
| 忘记`self`参数 | 方法定义缺少self，调用报错 | 始终检查第一个参数是self |
| `__init__`返回值 | 初始化方法不应返回任何值 | 删除return语句 |
| 类属性与实例属性同名 | 通过实例修改时创建实例属性遮蔽类属性 | 通过类名访问类属性 |
| 属性未初始化 | 访问未设置的属性导致AttributeError | 在__init__中初始化所有属性 |

### 适用场景

| 场景 | 是否推荐 | 原因 |
|------|---------|------|
| 数据需要封装和行为关联 | ✅ 推荐 | OOP核心优势 |
| 多个实例共享相同结构 | ✅ 推荐 | 类作为模板 |
| 简单的数据传递 | ❌ 不推荐 | 用数据类或字典更简单 |
| 单一功能的工具函数 | ❌ 不推荐 | 用函数更直接 |
| 继承层次超过3层 | ❌ 不推荐 | 耦合过高，难以维护 |

## L3 专家层：底层原理

### Python 如何实现对象

```
对象内存布局：
┌─────────────────────────────────────────────────────────────┐
│  Python 对象的内部结构                                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   PyObject_HEAD（所有对象共享）：                            │
│   ┌─────────────────────────────────────┐                   │
│   │ ob_refcnt  │ 引用计数（垃圾回收）   │                   │
│   │ ob_type    │ 指向类型对象 PyTypeObject│                   │
│   └─────────────────────────────────────┘                   │
│                                                             │
│   实例对象的属性存储：                                        │
│   ┌─────────────────────────────────────┐                   │
│   │ __dict__   │ 属性字典 {'name': 'xxx'}│                   │
│   │ __weakref__│ 弱引用列表              │                   │
│   └─────────────────────────────────────┘                   │
│                                                             │
│   类对象（PyTypeObject）：                                   │
│   ┌─────────────────────────────────────┐                   │
│   │ tp_dict    │ 类属性和方法字典        │                   │
│   │ tp_bases   │ 基类列表（继承）        │                   │
│   │ tp_mro     │ 方法解析顺序            │                   │
│   └─────────────────────────────────────┘                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```python
# 演示对象内部结构
class Student:
    school: str = "第一中学"

    def __init__(self, name: str) -> None:
        self.name = name

s = Student("张三")

# 查看实例的属性字典
print(s.__dict__)  # {'name': '张三'}

# 查看类的属性字典
print(Student.__dict__.keys())
# 包含 school, __init__, __dict__, __weakref__ 等

# 查看对象的类型
print(type(s))  # <class 'Student'>
print(type(Student))  # <class 'type'> - 类本身是 type 的实例

# 查看 MRO
print(Student.__mro__)
# (<class 'Student'>, <class 'object'>)
```

### __new__ 与 __init__ 的区别

```
对象创建过程：
┌─────────────────────────────────────────────────────────────┐
│  __new__ vs __init__                                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   对象创建流程：                                              │
│                                                             │
│   1. __new__(cls, ...)                                       │
│      ┌─────────────────────────────────────┐                │
│      │ • 创建对象（分配内存）              │                │
│      │ • 返回新创建的对象实例              │                │
│      │ • 是静态方法（不需self）            │                │
│      │ • 通常继承自 object.__new__        │                │
│      └─────────────────────────────────────┘                │
│                         │                                    │
│                         ↓                                    │
│   2. __init__(self, ...)                                     │
│      ┌─────────────────────────────────────┐                │
│      │ • 初始化对象（设置属性）            │                │
│      │ • 不返回值（返回None）              │                │
│      │ • 是实例方法（第一个参数是self）    │                │
│      │ • 在 __new__ 返回后自动调用         │                │
│      └─────────────────────────────────────┘                │
│                                                             │
│   关系：MyClass() = __new__ + __init__                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```python
# 演示 __new__ 和 __init__ 的执行顺序
class Demo:
    def __new__(cls, *args, **kwargs) -> "Demo":
        print("1. __new__ 执行：创建对象")
        instance = super().__new__(cls)  # 调用父类创建对象
        print(f"2. __new__ 返回：{instance}")
        return instance

    def __init__(self, name: str) -> None:
        print("3. __init__ 执行：初始化对象")
        self.name = name
        print(f"4. __init__ 完成：{self.name}")

d = Demo("张三")
# 输出顺序：
# 1. __new__ 执行：创建对象
# 2. __new__ 返回：<Demo object>
# 3. __init__ 执行：初始化对象
# 4. __init__ 完成：张三
```

**__new__ 的实际应用：单例模式**

```python
class Singleton:
    """单例模式：使用 __new__ 控制"""

    _instance: "Singleton | None" = None

    def __new__(cls) -> "Singleton":
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            print("创建新实例")
        else:
            print("返回已有实例")
        return cls._instance

    def __init__(self) -> None:
        # 注意：每次调用都会执行 __init__
        print("__init__ 执行")

s1 = Singleton()  # 创建新实例, __init__ 执行
s2 = Singleton()  # 返回已有实例, __init__ 执行
print(s1 is s2)   # True - 同一个对象

# 改进版：避免 __init__ 多次执行
class SingletonImproved:
    _instance: "SingletonImproved | None" = None
    _initialized: bool = False

    def __new__(cls) -> "SingletonImproved":
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self) -> None:
        if not self._initialized:
            self._initialized = True
            print("初始化一次")
```

### 性能考量

| 操作 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 创建对象 | O(1) | O(k) | k = 属性数量 |
| 属性访问 `obj.attr` | O(1) 平均 | - | 字典查找 |
| 属性设置 `obj.attr = v` | O(1) 平均 | - | 字典写入 |
| 方法调用 `obj.method()` | O(1) | - | 查找后调用 |
| `__dict__` 扩容 | O(n) 偶尔 | - | 哈希表扩容时 |

```python
# 性能对比：类 vs 字典
import timeit

# 使用类
class DataClass:
    def __init__(self, a: int, b: int, c: int) -> None:
        self.a = a
        self.b = b
        self.c = c

# 使用字典
def create_dict(a: int, b: int, c: int) -> dict:
    return {"a": a, "b": b, "c": c}

# 测试创建速度
print("类创建:", timeit.timeit("DataClass(1, 2, 3)", globals=globals(), number=100000))
print("字典创建:", timeit.timeit("create_dict(1, 2, 3)", globals=globals(), number=100000))

# 测试访问速度
obj = DataClass(1, 2, 3)
d = create_dict(1, 2, 3)

print("类属性访问:", timeit.timeit("obj.a", globals=globals(), number=1000000))
print("字典访问:", timeit.timeit("d['a']", globals=globals(), number=1000000))

# 类访问略快于字典访问（__dict__ 是优化过的字典）
```

### 设计动机

| Python 设计选择 | 原因 | 其他语言对比 |
|-----------------|------|-------------|
| 一切皆对象 | 统一的类型系统，简化语义 | Java有原始类型 |
| `__init__`而非构造函数 | 分离创建和初始化，更灵活 | Java用构造函数 |
| 动态添加属性 | 灵活性，无需预定义所有属性 | Java需预定义 |
| `self`显式传递 | 明确方法属于实例，避免歧义 | Java隐式this |
| `__dict__`存储属性 | 灵活但略慢，可优化为`__slots__` | C++固定布局 |

```

                    ┌───────────────┐
```
