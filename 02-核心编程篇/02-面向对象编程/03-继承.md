# 继承

> **Python 版本要求：** 3.11+

本章讲解 Python 面向对象编程中的继承机制，包括继承基础、super() 的使用和多重继承。

## 为什么需要继承？

### 问题场景

假设你要开发一个动物管理系统，有多种动物：
- 狗：有名字、会叫、会跑、会捡球
- 猫：有名字、会叫、会跑、会抓老鼠
- 鸟：有名字、会叫、会飞

**不使用继承：**

```python
class Dog:
    def __init__(self, name: str) -> None:
        self.name = name
    
    def speak(self) -> str:
        return f"{self.name}：汪汪汪"
    
    def move(self) -> str:
        return f"{self.name}在跑"
    
    def fetch(self) -> str:
        return f"{self.name}在捡球"

class Cat:
    def __init__(self, name: str) -> None:
        self.name = name
    
    def speak(self) -> str:
        return f"{self.name}：喵喵喵"
    
    def move(self) -> str:
        return f"{self.name}在跑"
    
    def catch_mouse(self) -> str:
        return f"{self.name}在抓老鼠"

class Bird:
    def __init__(self, name: str) -> None:
        self.name = name
    
    def speak(self) -> str:
        return f"{self.name}：啾啾啾"
    
    def fly(self) -> str:
        return f"{self.name}在飞"

# 问题：
# 1. 每个类都重复定义 __init__ 和 name 属性
# 2. Dog 和 Cat 的 move() 方法完全相同
# 3. 修改共性需要修改所有类
```

**问题：**
- 大量重复代码
- 修改共性需要在多个地方修改
- 难以统一管理

**使用继承：**

```python
class Animal:
    """动物基类"""
    def __init__(self, name: str) -> None:
        self.name: str = name
    
    def speak(self) -> str:
        return f"{self.name}发出声音"
    
    def move(self) -> str:
        return f"{self.name}在移动"

class Dog(Animal):
    """狗：继承自动物"""
    def speak(self) -> str:
        return f"{self.name}：汪汪汪"
    
    def fetch(self) -> str:
        return f"{self.name}在捡球"

class Cat(Animal):
    """猫：继承自动物"""
    def speak(self) -> str:
        return f"{self.name}：喵喵喵"
    
    def catch_mouse(self) -> str:
        return f"{self.name}在抓老鼠"

class Bird(Animal):
    """鸟：继承自动物"""
    def speak(self) -> str:
        return f"{self.name}：啾啾啾"
    
    def move(self) -> str:
        return f"{self.name}在飞"

# 共性代码只在 Animal 中写一次
# 子类只关注自己的特性
# 修改 Animal.move() 自动影响所有子类
```

**优势：**
- 消除重复代码
- 易于维护和扩展
- 统一管理公共行为

## 第一部分：继承基础

### 1.1 概念动机

继承是面向对象的核心特性，它允许子类继承父类的特征和行为，可以：
- 自动拥有父类的属性和方法
- 添加自己特有的属性和方法
- 重写父类的方法

```
┌─────────────────────────────────────────────────────────────┐
│                      继承的作用                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   代码复用：                                                 │
│   ✓ 子类自动拥有父类的属性和方法                            │
│   ✓ 消除重复代码                                            │
│                                                             │
│   功能扩展：                                                 │
│   ✓ 子类可以添加新方法                                      │
│   ✓ 子类可以重写父类方法                                    │
│                                                             │
│   统一管理：                                                 │
│   ✓ 修改父类自动影响子类                                    │
│   ✓ 可以统一处理所有子类                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 最简示例

```python
# 父类（基类）
class Animal:
    def __init__(self, name: str) -> None:
        self.name: str = name

    def speak(self) -> None:
        print("动物在说话")

    def move(self) -> None:
        print("动物在移动")


# 子类（派生类）
class Dog(Animal):
    def __init__(self, name: str, breed: str) -> None:
        # 调用父类的初始化方法
        super().__init__(name)
        self.breed: str = breed

    # 重写父类方法
    def speak(self) -> None:
        print(f"{self.name} 在叫：汪汪汪！")

    # 子类特有方法
    def fetch(self) -> None:
        print(f"{self.name} 正在捡球")


# 使用
dog = Dog("Buddy", "金毛")

# 继承的方法
dog.move()   # 动物在移动

# 重写的方法
dog.speak()  # Buddy 在叫：汪汪汪！

# 子类特有方法
dog.fetch()  # Buddy 正在捡球

# 子类特有属性
print(dog.breed)  # 金毛
```

### 1.3 详细讲解

**继承语法：**

```python
class ParentClass:
    """父类"""
    def parent_method(self) -> None:
        pass

class ChildClass(ParentClass):
    """子类：继承自 ParentClass"""
    def child_method(self) -> None:
        pass
```

**继承的关键概念：**

| 概念 | 说明 | 示例 |
|------|------|------|
| **父类/基类** | 被继承的类 | `Animal` |
| **子类/派生类** | 继承父类的类 | `Dog(Animal)` |
| **继承** | 子类获得父类的属性和方法 | `Dog` 继承 `Animal` |
| **重写** | 子类重新定义父类的方法 | `Dog.speak()` 重写 `Animal.speak()` |
| **扩展** | 子类添加新的方法 | `Dog.fetch()` |

**继承层次：**

```
Animal（父类）
│
├── Dog（子类）
│   ├── 继承：name, speak(), move()
│   ├── 重写：speak()
│   └── 扩展：breed, fetch()
│
├── Cat（子类）
│   ├── 继承：name, speak(), move()
│   ├── 重写：speak()
│   └── 扩展：catch_mouse()
│
└── Bird（子类）
    ├── 继承：name, speak(), move()
    ├── 重写：speak(), move()
    └── 扩展：fly()
```

### 1.4 渐进复杂化

多层继承：

```python
class Animal:
    """动物基类"""
    def __init__(self, name: str, age: int) -> None:
        self.name: str = name
        self.age: int = age
    
    def eat(self) -> str:
        return f"{self.name}在吃东西"
    
    def sleep(self) -> str:
        return f"{self.name}在睡觉"

class Mammal(Animal):
    """哺乳动物"""
    def __init__(self, name: str, age: int, fur_color: str) -> None:
        super().__init__(name, age)
        self.fur_color: str = fur_color
    
    def breathe(self) -> str:
        return f"{self.name}在呼吸"

class Dog(Mammal):
    """狗"""
    def __init__(self, name: str, age: int, fur_color: str, breed: str) -> None:
        super().__init__(name, age, fur_color)
        self.breed: str = breed
    
    def bark(self) -> str:
        return f"{self.name}在叫：汪汪汪"
    
    def get_info(self) -> dict[str, str | int]:
        return {
            "name": self.name,
            "age": self.age,
            "fur_color": self.fur_color,
            "breed": self.breed,
        }

# Dog 继承了 Animal 和 Mammal 的所有属性和方法
dog = Dog("Buddy", 3, "golden", "金毛")
print(dog.eat())      # 来自 Animal
print(dog.breathe())  # 来自 Mammal
print(dog.bark())     # 来自 Dog
print(dog.get_info())
```

### 1.5 实际应用

员工管理系统：

```python
from datetime import date

class Employee:
    """员工基类"""
    
    def __init__(self, name: str, employee_id: str, hire_date: date) -> None:
        self.name: str = name
        self.employee_id: str = employee_id
        self.hire_date: date = hire_date
    
    def get_info(self) -> dict[str, str]:
        return {
            "name": self.name,
            "id": self.employee_id,
            "hire_date": str(self.hire_date),
        }
    
    def calculate_tenure(self) -> int:
        """计算工作年限"""
        today = date.today()
        return today.year - self.hire_date.year

class Manager(Employee):
    """经理"""
    
    def __init__(self, name: str, employee_id: str, hire_date: date, department: str) -> None:
        super().__init__(name, employee_id, hire_date)
        self.department: str = department
        self.team_size: int = 0
    
    def set_team_size(self, size: int) -> None:
        self.team_size = size
    
    def get_info(self) -> dict[str, str | int]:
        info = super().get_info()
        info["department"] = self.department
        info["team_size"] = self.team_size
        return info

class Developer(Employee):
    """开发人员"""
    
    def __init__(self, name: str, employee_id: str, hire_date: date, skills: list[str]) -> None:
        super().__init__(name, employee_id, hire_date)
        self.skills: list[str] = skills
    
    def add_skill(self, skill: str) -> None:
        self.skills.append(skill)
    
    def get_info(self) -> dict[str, str | list]:
        info = super().get_info()
        info["skills"] = self.skills
        return info

class SalesPerson(Employee):
    """销售人员"""
    
    def __init__(self, name: str, employee_id: str, hire_date: date, region: str) -> None:
        super().__init__(name, employee_id, hire_date)
        self.region: str = region
        self.sales_total: float = 0.0
    
    def record_sale(self, amount: float) -> None:
        self.sales_total += amount
    
    def get_info(self) -> dict[str, str | float]:
        info = super().get_info()
        info["region"] = self.region
        info["sales_total"] = self.sales_total
        return info

# 使用
from datetime import date

manager = Manager("张经理", "M001", date(2020, 1, 1), "研发部")
manager.set_team_size(10)

developer = Developer("李开发", "D001", date(2021, 6, 1), ["Python", "Java"])
developer.add_skill("Go")

sales = SalesPerson("王销售", "S001", date(2022, 3, 1), "华东")
sales.record_sale(50000)

# 统一处理所有员工
employees: list[Employee] = [manager, developer, sales]
for emp in employees:
    print(f"{emp.name}: {emp.calculate_tenure()}年工作经验")
    print(emp.get_info())
```

## 第二部分：super() 的使用

### 2.1 概念动机

`super()` 用于调用父类的方法，常用于：
- 在子类 `__init__` 中调用父类的初始化
- 在重写的方法中保留父类的行为

### 2.2 最简示例

```python
class Animal:
    def __init__(self, name: str) -> None:
        self.name: str = name

    def speak(self) -> None:
        print(f"{self.name} 发出声音")


class Dog(Animal):
    def __init__(self, name: str, breed: str) -> None:
        # 使用 super() 调用父类方法（推荐）
        super().__init__(name)
        self.breed: str = breed

    def speak(self) -> None:
        # 先调用父类方法，再添加自己的行为
        super().speak()
        print(f"{self.name} 汪汪叫")


class Cat(Animal):
    def __init__(self, name: str, color: str) -> None:
        super().__init__(name)
        self.color: str = color

    def speak(self) -> None:
        # 完全覆盖父类方法
        print(f"{self.name} 喵喵叫")


# 使用
dog = Dog("Buddy", "金毛")
dog.speak()
# 输出：
# Buddy 发出声音
# Buddy 汪汪叫

cat = Cat("咪咪", "白色")
cat.speak()
# 输出：
# 咪咪 喵喵叫
```

### 2.3 详细讲解

**super() 的两种用法：**

1. **在 `__init__` 中调用父类初始化：**

```python
class Parent:
    def __init__(self, name: str) -> None:
        self.name: str = name

class Child(Parent):
    def __init__(self, name: str, age: int) -> None:
        # 先调用父类初始化
        super().__init__(name)
        # 再添加子类属性
        self.age: int = age
```

2. **在重写方法中保留父类行为：**

```python
class Parent:
    def method(self) -> str:
        return "父类行为"

class Child(Parent):
    def method(self) -> str:
        # 保留父类行为
        parent_result = super().method()
        # 添加子类行为
        return f"{parent_result} + 子类行为"
```

**注意事项：**

```python
class A:
    def __init__(self) -> None:
        print("A.__init__")

class B(A):
    def __init__(self) -> None:
        print("B.__init__")
        super().__init__()  # 调用 A.__init__

class C(A):
    def __init__(self) -> None:
        print("C.__init__")
        super().__init__()  # 调用 A.__init__

class D(B, C):
    def __init__(self) -> None:
        print("D.__init__")
        super().__init__()  # 按 MRO 调用

d = D()
# 输出顺序（取决于 MRO）：
# D.__init__
# B.__init__
# C.__init__
# A.__init__
```

### 2.4 渐进复杂化

多层继承中的 super()：

```python
class Level1:
    def __init__(self, name: str) -> None:
        self.name: str = name
        print("Level1 初始化")
    
    def process(self) -> str:
        return "Level1处理"

class Level2(Level1):
    def __init__(self, name: str, level2_attr: str) -> None:
        super().__init__(name)
        self.level2_attr: str = level2_attr
        print("Level2 初始化")
    
    def process(self) -> str:
        result = super().process()
        return f"{result} → Level2处理"

class Level3(Level2):
    def __init__(self, name: str, level2_attr: str, level3_attr: str) -> None:
        super().__init__(name, level2_attr)
        self.level3_attr: str = level3_attr
        print("Level3 初始化")
    
    def process(self) -> str:
        result = super().process()
        return f"{result} → Level3处理"

obj = Level3("test", "L2", "L3")
# 输出：
# Level1 初始化
# Level2 初始化
# Level3 初始化

print(obj.process())
# 输出：Level1处理 → Level2处理 → Level3处理
```

## 第三部分：多重继承

### 3.1 概念动机

Python 支持多重继承，一个子类可以继承多个父类。这允许组合多个类的功能。

```
┌─────────────────────────────────────────────────────────────┐
│              多重继承                                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   优点：                                                     │
│   ✓ 组合多个类的功能                                        │
│   ✓ 灵活设计                                                │
│                                                             │
│   注意：                                                     │
│   ✓ MRO（方法解析顺序）决定调用顺序                         │
│   ✓ 设计时需避免菱形继承问题                                │
│   ✓ 优先使用组合而非多重继承                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 最简示例

```python
class Flyable:
    """会飞的类"""
    def fly(self) -> None:
        print("在空中飞翔")


class Swimmable:
    """会游泳的类"""
    def swim(self) -> None:
        print("在水中游泳")


# 鸭子：继承多个父类
class Duck(Flyable, Swimmable):
    def __init__(self, name: str) -> None:
        self.name: str = name

    def introduce(self) -> None:
        print(f"我是{self.name}，我会：")
        self.fly()
        self.swim()


# 使用
duck = Duck("唐老鸭")
duck.introduce()
# 我是唐老鸭，我会：
# 在空中飞翔
# 在水中游泳
```

### 3.3 详细讲解

**MRO（方法解析顺序）：**

**MRO：** 多重继承时，Python按特定顺序查找方法。

```
MRO规则：
┌─────────────────────────────────────────────────────────────┐
│  方法解析顺序（MRO）                                          │
│                                                              │
│  MRO = Method Resolution Order                               │
│                                                              │
│  规则：                                                       │
│  1. 从子类到父类依次查找                                       │
│  2. 同级父类按继承声明顺序                                     │
│  3. 每个类只出现一次（避免重复调用）                            │
│  4. 使用 C3 算法计算                                          │
│                                                              │
│  查看MRO：                                                    │
│  • 类名.__mro__                                               │
│  • 类名.mro()                                                 │
│                                                              │
│  示例：class D(B, C)                                          │
│  MRO: D → B → C → A → object                                  │
│       先查D，再查B，再查C，再查A，最后object                   │
└─────────────────────────────────────────────────────────────┘
```

### 最简示例

```python
class A:
    def method(self) -> None:
        print("A")

class B(A):
    pass

class C(A):
    pass

class D(B, C):
    pass

print(D.__mro__)
# D → B → C → A → object
```

### 关键代码解释

| 概念 | 含义 | 示例 |
|------|------|------|
| `__mro__` | MRO 元组 | 查看方法查找顺序 |
| `D(B, C)` | 继承顺序 | 先查 B，后查 C |
| C3算法 | 计算规则 | 保证每个类只调用一次 |

---

### 菱形继承详解（核心）

**菱形继承：** 多个子类继承同一个父类，形成菱形结构。

```
菱形继承结构：
┌─────────────────────────────────────────────────────────────┐
│  菱形继承（Diamond Problem）                                   │
│                                                              │
│         A                                                    │
│        / \                                                   │
│       B   C                                                   │
│        \ /                                                   │
│         D                                                    │
│                                                              │
│  问题：D 继承 B 和 C，B 和 C 都继承 A                          │
│        调用 A 的方法时，会执行几次？                            │
│                                                              │
│  ⚠️ Python 解决方案：MRO 保证每个类只调用一次                  │
│                                                              │
│  D 的 MRO: D → B → C → A → object                             │
│                                                              │
│  执行顺序：                                                    │
│  1. D.method() → 调用 super()                                │
│  2. B.method() → 调用 super()                                │
│  3. C.method() → 调用 super()                                │
│  4. A.method() → 返回（只执行一次！）                         │
│                                                              │
│  结果链：A → C → B → D                                        │
└─────────────────────────────────────────────────────────────┘
```

### 最简示例

```python
class A:
    def method(self) -> str:
        return "A"

class B(A):
    def method(self) -> str:
        return super().method() + " → B"

class C(A):
    def method(self) -> str:
        return super().method() + " → C"

class D(B, C):
    def method(self) -> str:
        return super().method() + " → D"

d = D()
print(d.method())
# A → C → B → D（每个类只调用一次）
```

### 关键代码解释

| 代码 | 执行过程 | 说明 |
|------|---------|------|
| `D(B, C)` | 先B后C | 继承顺序决定MRO |
| `super().method()` | 按MRO调用 | 不是调用直接父类 |
| `A → C → B → D` | 结果链 | A只执行一次 |

### 详细示例

```python
# 查看 MRO
print(D.__mro__)
# (<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>)

# 调用方法时按 MRO 顺序查找
d = D()
d.method()  # A → C → B → D
```

Mixin 模式（推荐的多重继承方式）：

```python
class LoggingMixin:
    """日志混入类"""
    
    def log(self, message: str) -> None:
        print(f"[{self.__class__.__name__}] {message}")

class SerializableMixin:
    """序列化混入类"""
    
    def to_dict(self) -> dict[str, str | int]:
        return {
            key: value for key, value in self.__dict__.items()
            if not key.startswith("_")
        }

class ComparableMixin:
    """比较混入类"""
    
    def __eq__(self, other: object) -> bool:
        if not isinstance(other, self.__class__):
            return False
        return self.__dict__ == other.__dict__

class User(LoggingMixin, SerializableMixin, ComparableMixin):
    """用户类：组合多个 Mixin"""
    
    def __init__(self, name: str, age: int) -> None:
        self.name: str = name
        self.age: int = age
    
    def update_age(self, new_age: int) -> None:
        self.log(f"年龄从 {self.age} 更新为 {new_age}")
        self.age = new_age

# 使用
user1 = User("Alice", 25)
user2 = User("Alice", 25)
user3 = User("Bob", 30)

user1.update_age(26)  # [User] 年龄从 25 更新为 26
print(user1.to_dict())  # {'name': 'Alice', 'age': 26}
print(user1 == user2)  # False（年龄不同）
print(user2 == user3)  # False
```

from typing import Protocol

class CombatMixin:
    """战斗能力"""
    
    def attack(self) -> str:
        return f"{self.name} 攻击！"
    
    def defend(self) -> str:
        return f"{self.name} 防御！"

class MagicMixin:
    """魔法能力"""
    
    def cast_spell(self, spell_name: str) -> str:
        return f"{self.name} 施放 {spell_name}！"
    
    def meditate(self) -> str:
        return f"{self.name}冥想恢复魔法值"

class StealthMixin:
    """潜行能力"""
    
    def hide(self) -> str:
        return f"{self.name}潜入阴影中"
    
    def sneak_attack(self) -> str:
        return f"{self.name}发动突袭！"

class Character:
    """角色基类"""
    
    def __init__(self, name: str, level: int = 1) -> None:
        self.name: str = name
        self.level: int = level
        self.hp: int = 100 * level
    
    def level_up(self) -> None:
        self.level += 1
        self.hp = 100 * self.level

class Warrior(Character, CombatMixin):
    """战士：角色 + 战斗能力"""

class Mage(Character, MagicMixin):
    """法师：角色 + 魔法能力"""

class Rogue(Character, StealthMixin, CombatMixin):
    """盗贼：角色 + 潜行 + 战斗"""

class Paladin(Character, CombatMixin, MagicMixin):
    """圣骑士：角色 + 战斗 + 魔法"""

# 使用
warrior = Warrior("战士张三", 5)
mage = Mage("法师李四", 3)
rogue = Rogue("盗贼王五", 4)
paladin = Paladin("圣骑士赵六", 6)

print(warrior.attack())
print(mage.cast_spell("火球术"))
print(rogue.sneak_attack())
print(paladin.attack())
print(paladin.cast_spell("圣光术"))
