本章讲解面向对象设计的基本原则，包括组合与继承的选择和最佳实践。

## 为什么需要设计原则？

### 问题场景

假设你要开发一个游戏角色系统，角色有不同的能力组合：

**糟糕的设计（过度继承）：**

```python
class Character:
    def move(self) -> str:
        return "移动"

class FlyingCharacter(Character):
    def fly(self) -> str:
        return "飞翔"

class SwimmingCharacter(Character):
    def swim(self) -> str:
        return "游泳"

class RunningCharacter(Character):
    def run(self) -> str:
        return "奔跑"

# 问题：如何创建既会飞又会游泳的角色？
# 需要创建 FlyingSwimmingCharacter
class FlyingSwimmingCharacter(FlyingCharacter, SwimmingCharacter):
    pass

# 问题：如何创建既会飞又会跑还会游泳的角色？
# 需要创建 FlyingRunningSwimmingCharacter
# 继承层次越来越复杂，呈指数增长！
```

**问题：**
- 继承层次过深，难以维护
- 组合爆炸（FlyingSwimming, FlyingRunning, SwimmingRunning...）
- 修改父类影响所有子类
- 代码复用困难
- 灵活性差

**更好的设计（使用组合）：**

```python
from typing import Optional

class FlyBehavior:
    """飞行行为"""
    def fly(self) -> str:
        return "在空中飞翔"

class SwimBehavior:
    """游泳行为"""
    def swim(self) -> str:
        return "在水中游泳"

class RunBehavior:
    """奔跑行为"""
    def run(self) -> str:
        return "在地面奔跑"

class Character:
    """角色类：使用组合"""
    
    def __init__(self, name: str) -> None:
        self.name: str = name
        self.fly_behavior: Optional[FlyBehavior] = None
        self.swim_behavior: Optional[SwimBehavior] = None
        self.run_behavior: Optional[RunBehavior] = None
    
    def set_fly(self, fly: FlyBehavior) -> None:
        self.fly_behavior = fly
    
    def set_swim(self, swim: SwimBehavior) -> None:
        self.swim_behavior = swim
    
    def set_run(self, run: RunBehavior) -> None:
        self.run_behavior = run
    
    def perform_fly(self) -> str:
        if self.fly_behavior:
            return self.fly_behavior.fly()
        return "不会飞"
    
    def perform_swim(self) -> str:
        if self.swim_behavior:
            return self.swim_behavior.swim()
        return "不会游泳"
    
    def perform_run(self) -> str:
        if self.run_behavior:
            return self.run_behavior.run()
        return "不会跑"

# 灵活组合，无需创建大量子类
duck = Character("鸭子")
duck.set_fly(FlyBehavior())
duck.set_swim(SwimBehavior())

horse = Character("马")
horse.set_run(RunBehavior())

dragon = Character("龙")
dragon.set_fly(FlyBehavior())
dragon.set_swim(SwimBehavior())
dragon.set_run(RunBehavior())

# 动态修改能力
duck.set_run(RunBehavior())  # 鸭子学会跑！
```

**优势：**
- 灵活组合行为
- 易于扩展新行为
- 降低耦合度
- 动态修改能力
- 符合 SOLID 原则

## 第一部分：组合 vs 继承

### 1.1 概念动机

组合和继承是代码复用的两种方式，选择正确的复用方式是良好设计的关键。

```
┌─────────────────────────────────────────────────────────────┐
│              组合 vs 继承                                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   继承（is-a 关系）：                                        │
│   ✓ "是一个"的关系                                          │
│   ✓ Dog is-a Animal                                         │
│   ✓ 子类自动拥有父类的属性和方法                            │
│   ✓ 高耦合度                                                │
│                                                             │
│   组合（has-a 关系）：                                       │
│   ✓ "有一个"的关系                                          │
│   ✓ Car has-a Engine                                        │
│   ✓ 对象包含其他对象                                        │
│   ✓ 低耦合度                                                │
│                                                             │
│   设计原则：                                                 │
│   ✓ 组合优于继承（Favor composition over inheritance）      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 最简示例

**继承示例：**

```python
class Animal:
    def __init__(self, name: str) -> None:
        self.name: str = name

    def breathe(self) -> str:
        return f"{self.name} 在呼吸"

class Dog(Animal):
    def bark(self) -> str:
        return f"{self.name} 在叫"

# Dog is-a Animal
dog = Dog("Buddy")
print(dog.breathe())  # 继承的方法
print(dog.bark())     # 自己的方法
```

**组合示例：**

```python
class Engine:
    """发动机类"""
    def start(self) -> str:
        return "发动机启动"

class Wheel:
    """轮子类"""
    def rotate(self) -> str:
        return "轮子转动"

class Car:
    """汽车类 - 使用组合"""
    def __init__(self, brand: str) -> None:
        self.brand: str = brand
        # 组合：汽车"有"发动机和轮子
        self.engine: Engine = Engine()
        self.wheels: list[Wheel] = [Wheel() for _ in range(4)]

    def drive(self) -> str:
        result = [f"{self.brand} 汽车："]
        result.append(self.engine.start())
        for wheel in self.wheels:
            result.append(wheel.rotate())
        return "\n".join(result)


# 使用
car = Car("特斯拉")
print(car.drive())
# 特斯拉 汽车：
# 发动机启动
# 轮子转动
# 轮子转动
# 轮子转动
# 轮子转动
```

### 1.3 详细讲解

**组合 vs 继承对比表：**

| | 继承 | 组合 |
|---|---|---|
| **关系** | "is-a"（是一个） | "has-a"（有一个） |
| **示例** | 狗 is-a 动物 | 汽车 has-a 发动机 |
| **代码** | `class Dog(Animal):` | `self.engine = Engine()` |
| **耦合度** | 高 | 低 |
| **灵活性** | 低（编译时确定） | 高（运行时可变） |
| **扩展性** | 需要创建新子类 | 只需添加新组件 |
| **测试性** | 困难（继承链） | 简单（独立组件） |

**选择指南：**

```
┌─────────────────────────────────────────────────────────────┐
│              组合 vs 继承 选择指南                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   使用继承：                                                 │
│   ✓ 确实存在 "is-a" 关系                                   │
│   ✓ 需要重写或扩展父类行为                                 │
│   ✓ 例如：Dog is-a Animal                                  │
│   ✓ 例如：Rectangle is-a Shape                             │
│                                                             │
│   使用组合：                                                 │
│   ✓ 存在 "has-a" 关系                                      │
│   ✓ 需要灵活替换组件                                       │
│   ✓ 需要降低耦合度                                         │
│   ✓ 例如：Car has-a Engine                                 │
│   ✓ 例如：User has-a Address                               │
│                                                             │
│   设计原则：                                                 │
│   ✓ 组合优于继承                                            │
│   ✓ 继承层次不宜过深（建议不超过 3 层）                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 渐进复杂化

策略模式（使用组合实现多态）：

```python
from typing import Protocol

class SortStrategy(Protocol):
    """排序策略协议"""
    def sort(self, data: list[int]) -> list[int]:
        ...

class BubbleSort:
    """冒泡排序"""
    def sort(self, data: list[int]) -> list[int]:
        arr = data.copy()
        n = len(arr)
        for i in range(n):
            for j in range(0, n - i - 1):
                if arr[j] > arr[j + 1]:
                    arr[j], arr[j + 1] = arr[j + 1], arr[j]
        return arr

class QuickSort:
    """快速排序"""
    def sort(self, data: list[int]) -> list[int]:
        if len(data) <= 1:
            return data
        pivot = data[len(data) // 2]
        left = [x for x in data if x < pivot]
        middle = [x for x in data if x == pivot]
        right = [x for x in data if x > pivot]
        return self.sort(left) + middle + self.sort(right)

class MergeSort:
    """归并排序"""
    def sort(self, data: list[int]) -> list[int]:
        if len(data) <= 1:
            return data
        mid = len(data) // 2
        left = self.sort(data[:mid])
        right = self.sort(data[mid:])
        return self._merge(left, right)
    
    def _merge(self, left: list[int], right: list[int]) -> list[int]:
        result = []
        i, j = 0, 0
        while i < len(left) and j < len(right):
            if left[i] <= right[j]:
                result.append(left[i])
                i += 1
            else:
                result.append(right[j])
                j += 1
        result.extend(left[i:])
        result.extend(right[j:])
        return result

class Sorter:
    """排序器：使用组合"""
    
    def __init__(self, strategy: SortStrategy) -> None:
        self.strategy: SortStrategy = strategy
    
    def set_strategy(self, strategy: SortStrategy) -> None:
        """动态更换策略"""
        self.strategy = strategy
    
    def sort(self, data: list[int]) -> list[int]:
        """使用当前策略排序"""
        return self.strategy.sort(data)

# 使用
data = [64, 34, 25, 12, 22, 11, 90]

sorter = Sorter(BubbleSort())
print(f"冒泡排序: {sorter.sort(data)}")

sorter.set_strategy(QuickSort())
print(f"快速排序: {sorter.sort(data)}")

sorter.set_strategy(MergeSort())
print(f"归并排序: {sorter.sort(data)}")
```

### 1.5 实际应用

日志系统：

```python
from abc import ABC, abstractmethod
from datetime import datetime
from typing import Any

class LogHandler(ABC):
    """日志处理器抽象类"""
    
    @abstractmethod
    def write(self, message: str) -> None:
        pass

class ConsoleHandler(LogHandler):
    """控制台处理器"""
    
    def write(self, message: str) -> None:
        print(message)

class FileHandler(LogHandler):
    """文件处理器"""
    
    def __init__(self, file_path: str) -> None:
        self.file_path: str = file_path
    
    def write(self, message: str) -> None:
        with open(self.file_path, 'a') as f:
            f.write(message + '\n')

class EmailHandler(LogHandler):
    """邮件处理器"""
    
    def __init__(self, email: str) -> None:
        self.email: str = email
    
    def write(self, message: str) -> None:
        # 模拟发送邮件
        print(f"发送邮件到 {self.email}: {message}")

class Logger:
    """日志器：使用组合"""
    
    def __init__(self, name: str) -> None:
        self.name: str = name
        self.handlers: list[LogHandler] = []
    
    def add_handler(self, handler: LogHandler) -> None:
        """添加处理器"""
        self.handlers.append(handler)
    
    def remove_handler(self, handler: LogHandler) -> None:
        """移除处理器"""
        self.handlers.remove(handler)
    
    def log(self, level: str, message: str) -> None:
        """记录日志"""
        timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        formatted = f"[{timestamp}] [{self.name}] [{level}] {message}"
        
        for handler in self.handlers:
            handler.write(formatted)
    
    def info(self, message: str) -> None:
        self.log("INFO", message)
    
    def warning(self, message: str) -> None:
        self.log("WARNING", message)
    
    def error(self, message: str) -> None:
        self.log("ERROR", message)

# 使用
logger = Logger("MyApp")

# 组合多个处理器
logger.add_handler(ConsoleHandler())
logger.add_handler(FileHandler("app.log"))

# 记录日志会输出到所有处理器
logger.info("应用启动")
logger.warning("内存使用率高")
logger.error("连接失败")

# 动态添加处理器
logger.add_handler(EmailHandler("admin@example.com"))
logger.error("严重错误")  # 同时输出到控制台、文件和邮件
```

## 第二部分：SOLID 原则简介

### 2.1 概念动机

SOLID 是面向对象设计的五大基本原则，帮助创建可维护、可扩展的软件。

```
┌─────────────────────────────────────────────────────────────┐
│              SOLID 原则                                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   S - 单一职责原则（Single Responsibility）                 │
│       一个类只做一件事                                      │
│                                                             │
│   O - 开放封闭原则（Open/Closed）                           │
│       对扩展开放，对修改封闭                                │
│                                                             │
│   L - 里氏替换原则（Liskov Substitution）                   │
│       子类可以替换父类                                      │
│                                                             │
│   I - 接口隔离原则（Interface Segregation）                 │
│       接口要小而专一                                        │
│                                                             │
│   D - 依赖倒置原则（Dependency Inversion）                  │
│       依赖抽象而非具体实现                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 单一职责原则（SRP）

**一个类只做一件事**

```python
# ❌ 违反单一职责：一个类做太多事
class User:
    def __init__(self, name: str, email: str) -> None:
        self.name: str = name
        self.email: str = email

    def save(self) -> None:
        # 保存到数据库
        pass

    def send_email(self, message: str) -> None:
        # 发送邮件
        pass

    def generate_report(self) -> str:
        # 生成报告
        return ""

# 问题：User 类承担了数据、持久化、邮件、报告等多个职责


# ✅ 遵循单一职责：每个类只做一件事
class User:
    """用户：只负责用户数据"""
    def __init__(self, name: str, email: str) -> None:
        self.name: str = name
        self.email: str = email

class UserRepository:
    """用户仓库：只负责数据持久化"""
    def save(self, user: User) -> None:
        # 保存到数据库
        print(f"保存用户: {user.name}")

class EmailService:
    """邮件服务：只负责发送邮件"""
    def send(self, user: User, message: str) -> None:
        # 发送邮件
        print(f"发送邮件到 {user.email}: {message}")

class ReportGenerator:
    """报告生成器：只负责生成报告"""
    def generate(self, user: User) -> str:
        # 生成报告
        return f"用户报告: {user.name}"

# 使用
user = User("张三", "zhang@example.com")
UserRepository().save(user)
EmailService().send(user, "欢迎")
print(ReportGenerator().generate(user))
```

### 2.3 开放封闭原则（OCP）

**对扩展开放，对修改封闭**

```python
from abc import ABC, abstractmethod

# ❌ 违反开放封闭：每次添加新形状都要修改函数
def calculate_area(shape_type: str, **params) -> float:
    if shape_type == "rectangle":
        return params["width"] * params["height"]
    elif shape_type == "circle":
        return 3.14 * params["radius"] ** 2
    # 每次添加新形状都要修改这个函数！


# ✅ 遵循开放封闭：通过扩展而非修改
class Shape(ABC):
    """形状抽象基类"""
    
    @abstractmethod
    def get_area(self) -> float:
        pass

class Rectangle(Shape):
    def __init__(self, width: float, height: float) -> None:
        self.width: float = width
        self.height: float = height
    
    def get_area(self) -> float:
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius: float) -> None:
        self.radius: float = radius
    
    def get_area(self) -> float:
        return 3.14 * self.radius ** 2

# 添加新形状无需修改现有代码
class Triangle(Shape):
    def __init__(self, base: float, height: float) -> None:
        self.base: float = base
        self.height: float = height
    
    def get_area(self) -> float:
        return 0.5 * self.base * self.height

def calculate_total_area(shapes: list[Shape]) -> float:
    """无需修改即可处理新形状"""
    return sum(shape.get_area() for shape in shapes)
```

### 2.4 里氏替换原则（LSP）

**子类可以替换父类**

```python
# ❌ 违反里氏替换
class Bird:
    def fly(self) -> str:
        return "飞翔"

class Penguin(Bird):
    def fly(self) -> str:
        raise Exception("企鹅不会飞！")  # 破坏了父类的行为

def make_bird_fly(bird: Bird) -> str:
    return bird.fly()

# make_bird_fly(Penguin())  # 抛出异常！


# ✅ 遵循里氏替换：正确的设计
class Bird:
    """鸟类"""
    def move(self) -> str:
        return "移动"

class FlyingBird(Bird):
    """会飞的鸟"""
    def fly(self) -> str:
        return "飞翔"
    
    def move(self) -> str:
        return self.fly()

class NonFlyingBird(Bird):
    """不会飞的鸟"""
    def walk(self) -> str:
        return "行走"
    
    def move(self) -> str:
        return self.walk()

class Sparrow(FlyingBird):
    """燕子"""
    pass

class Penguin(NonFlyingBird):
    """企鹅"""
    pass

def make_bird_move(bird: Bird) -> str:
    """所有鸟都能移动"""
    return bird.move()

print(make_bird_move(Sparrow()))   # 飞翔
print(make_bird_move(Penguin()))   # 行走
```

### 2.5 接口隔离原则（ISP）

**接口要小而专一**

```python
# ❌ 违反接口隔离：一个接口包含太多方法
class Worker:
    """工人接口：太大"""
    def work(self) -> str:
        pass
    
    def eat(self) -> str:
        pass
    
    def sleep(self) -> str:
        pass

class Robot(Worker):
    """机器人：被迫实现不需要的方法"""
    def work(self) -> str:
        return "工作"
    
    def eat(self) -> str:
        raise Exception("机器人不吃饭")
    
    def sleep(self) -> str:
        raise Exception("机器人不睡觉")


# ✅ 遵循接口隔离：拆分为小接口
class Workable:
    """可工作的"""
    def work(self) -> str:
        pass

class Eatable:
    """可吃饭的"""
    def eat(self) -> str:
        pass

class Sleepable:
    """可睡觉的"""
    def sleep(self) -> str:
        pass

class Human(Workable, Eatable, Sleepable):
    """人类：实现所有接口"""
    def work(self) -> str:
        return "工作"
    
    def eat(self) -> str:
        return "吃饭"
    
    def sleep(self) -> str:
        return "睡觉"

class Robot(Workable):
    """机器人：只实现需要的接口"""
    def work(self) -> str:
        return "工作"
```

### 2.6 依赖倒置原则（DIP）

**依赖抽象而非具体实现**

```python
# ❌ 违反依赖倒置：依赖具体实现
class Light:
    """电灯"""
    def turn_on(self) -> None:
        print("灯亮")
    
    def turn_off(self) -> None:
        print("灯灭")

class Switch:
    """开关：依赖具体的 Light 类"""
    def __init__(self, light: Light) -> None:
        self.light: Light = light
    
    def press(self) -> None:
        self.light.turn_on()


# ✅ 遵循依赖倒置：依赖抽象
from abc import ABC, abstractmethod

class Switchable(ABC):
    """可开关的抽象"""
    
    @abstractmethod
    def turn_on(self) -> None:
        pass
    
    @abstractmethod
    def turn_off(self) -> None:
        pass

class Light(Switchable):
    """电灯：实现抽象"""
    def turn_on(self) -> None:
        print("灯亮")
    
    def turn_off(self) -> None:
        print("灯灭")

class Fan(Switchable):
    """风扇：实现抽象"""
    def turn_on(self) -> None:
        print("风扇启动")
    
    def turn_off(self) -> None:
        print("风扇停止")

class Switch:
    """开关：依赖抽象"""
    def __init__(self, device: Switchable) -> None:
        self.device: Switchable = device
        self.is_on: bool = False
    
    def press(self) -> None:
        if self.is_on:
            self.device.turn_off()
            self.is_on = False
        else:
            self.device.turn_on()
            self.is_on = True

# 可以控制任何实现了 Switchable 的设备
light_switch = Switch(Light())
fan_switch = Switch(Fan())

light_switch.press()  # 灯亮
fan_switch.press()    # 风扇启动
```

## 第三部分：最佳实践

### 3.1 设计原则

```
┌─────────────────────────────────────────────────────────────┐
│              面向对象最佳实践                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   命名规范：                                                 │
│   ✓ 类名用大驼峰：MyClassName                               │
│   ✓ 方法和属性用小写+下划线：my_method                      │
│   ✓ 私有属性用双下划线：__private                           │
│   ✓ 常量用大写+下划线：MAX_SIZE                             │
│                                                             │
│   设计原则：                                                 │
│   ✓ 优先使用组合而非继承                                    │
│   ✓ 单一职责：一个类只做一件事                              │
│   ✓ 开放封闭：对扩展开放，对修改封闭                        │
│   ✓ 接口隔离：接口要小而专一                                │
│                                                             │
│   避免：                                                     │
│   ✗ 上帝类（一个类做太多事）                                │
│   ✗ 过度继承（建议不超过 3 层）                             │
│   ✗ 直接访问私有属性                                        │
│   ✗ 使用可变对象作为类属性                                  │
│   ✗ 循环依赖                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

class MyClass:
    """类的文档字符串"""

    # 3. 公共方法

        pass
    
    # 5. 私有方法
    def __private_method(self) -> None:
        pass
    
    # 6. @property

    def computed_property(self) -> int:
        return self._internal_state
    
    # 7. 类方法
    @classmethod
    def from_string(cls, data: str) -> "MyClass":
        return cls(data)
    
    # 8. 静态方法

    def utility_function(value: int) -> bool:
        return value > 0

### 3.3 实际应用示例

电商订单系统：

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from datetime import datetime
from typing import Optional
from enum import Enum

class OrderStatus(Enum):
    """订单状态"""
    PENDING = "pending"
    PAID = "paid"
    SHIPPED = "shipped"
    DELIVERED = "delivered"
    CANCELLED = "cancelled"

@dataclass
class Product:
    """产品"""
    id: str
    name: str
    price: float

@dataclass
class Customer:
    """客户"""
    id: str
    name: str
    email: str

class PaymentProcessor(ABC):
    """支付处理器抽象"""
    
    @abstractmethod
    def process(self, amount: float) -> bool:
        pass

class CreditCardPayment(PaymentProcessor):
    """信用卡支付"""
    
    def process(self, amount: float) -> bool:
        print(f"信用卡支付: {amount}")
        return True

class PayPalPayment(PaymentProcessor):
    """PayPal 支付"""
    
    def process(self, amount: float) -> bool:
        print(f"PayPal 支付: {amount}")
        return True

class NotificationService(ABC):
    """通知服务抽象"""
    
    @abstractmethod
    def send(self, customer: Customer, message: str) -> None:
        pass

class EmailNotification(NotificationService):
    """邮件通知"""
    
    def send(self, customer: Customer, message: str) -> None:
        print(f"发送邮件到 {customer.email}: {message}")

class SMSNotification(NotificationService):
    """短信通知"""
    
    def send(self, customer: Customer, message: str) -> None:
        print(f"发送短信: {message}")

class Order:
    """订单类"""
    
    def __init__(
        self,
        customer: Customer,
        products: list[Product],
        payment_processor: PaymentProcessor,
        notification_service: NotificationService,
    ) -> None:
        self.id: str = f"ORD-{datetime.now().strftime('%Y%m%d%H%M%S')}"
        self.customer: Customer = customer
        self.products: list[Product] = products
        self.status: OrderStatus = OrderStatus.PENDING
        self.payment_processor: PaymentProcessor = payment_processor
        self.notification_service: NotificationService = notification_service
        self.created_at: datetime = datetime.now()
    
    @property
    def total_amount(self) -> float:
        """计算总金额"""
        return sum(p.price for p in self.products)
    
    def pay(self) -> bool:
        """支付"""
        if self.status != OrderStatus.PENDING:
            print("订单已支付或已取消")
            return False
        
        if self.payment_processor.process(self.total_amount):
            self.status = OrderStatus.PAID
            self.notification_service.send(
                self.customer,
                f"订单 {self.id} 支付成功，金额: {self.total_amount}"
            )
            return True
        return False
    
    def ship(self) -> None:
        """发货"""
        if self.status == OrderStatus.PAID:
            self.status = OrderStatus.SHIPPED
            self.notification_service.send(
                self.customer,
                f"订单 {self.id} 已发货"
            )
    
    def deliver(self) -> None:
        """送达"""
        if self.status == OrderStatus.SHIPPED:
            self.status = OrderStatus.DELIVERED
            self.notification_service.send(
                self.customer,
                f"订单 {self.id} 已送达"
            )
    
    def cancel(self) -> None:
        """取消"""
        if self.status in [OrderStatus.PENDING, OrderStatus.PAID]:
            self.status = OrderStatus.CANCELLED
            self.notification_service.send(
                self.customer,
                f"订单 {self.id} 已取消"
            )
    
    def get_info(self) -> dict:
        """获取订单信息"""
        return {
            "id": self.id,
            "customer": self.customer.name,
            "products": [p.name for p in self.products],
            "total": self.total_amount,
            "status": self.status.value,
            "created_at": str(self.created_at),
        }

# 使用
customer = Customer("C001", "张三", "zhang@example.com")
products = [
    Product("P001", "笔记本", 29.99),
    Product("P002", "钢笔", 15.99),
]

# 使用组合而非继承
order = Order(
    customer,
    products,
    payment_processor=CreditCardPayment(),
    notification_service=EmailNotification(),
)

print(order.get_info())
order.pay()

# 可以动态替换组件
order2 = Order(
    customer,
    products,
    payment_processor=PayPalPayment(),
    notification_service=SMSNotification(),
)
order2.pay()
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `payment_processor: PaymentProcessor` | 依赖抽象接口而非具体实现 | `Order` 不关心是信用卡还是 PayPal，只要实现 `process` 接口即可（依赖倒置原则） |
| `notification_service: NotificationService` | 通知服务也通过接口注入 | 便于替换为短信/邮件/推送，无需修改 `Order` 类（开放封闭原则） |
| `Order(... payment_processor=CreditCardPayment(), ...)` | 构造时注入依赖 | 依赖注入使测试时可传入 Mock 对象，不依赖真实支付和通知服务 |
| `order2 = Order(... payment_processor=PayPalPayment(), ...)` | 创建另一订单用不同实现 | 组合优于继承的体现：无需创建 `CreditCardOrder`/`PayPalOrder` 子类，直接替换组件 |
