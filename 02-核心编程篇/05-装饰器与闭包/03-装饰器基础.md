## 为什么需要装饰器？

### 实际场景引入

想象你正在开发一个 Web 应用，需要对多个函数添加日志：

```python
# 问题场景：每个函数都要添加日志代码
def get_user(user_id: int) -> dict:
    print(f"[LOG] 调用 get_user, 参数: {user_id}")  # 重复代码
    result = {"id": user_id, "name": "Alice"}
    print(f"[LOG] get_user 返回: {result}")  # 重复代码
    return result

def create_user(name: str, email: str) -> dict:
    print(f"[LOG] 调用 create_user, 参数: {name}, {email}")  # 重复代码
    result = {"name": name, "email": email}
    print(f"[LOG] create_user 返回: {result}")  # 重复代码
    return result

def delete_user(user_id: int) -> bool:
    print(f"[LOG] 调用 delete_user, 参数: {user_id}")  # 重复代码
    result = True
    print(f"[LOG] delete_user 返回: {result}")  # 重复代码
    return result
```

**痛点**：
- 大量重复的日志代码
- 业务逻辑被淹没在日志代码中
- 修改日志格式需要改很多地方

**解决思路**：如果能把日志逻辑抽离出来，在不修改原函数的情况下添加功能，问题就解决了！

## 第一部分：什么是装饰器

### 概念动机

装饰器（Decorator）是一种特殊的函数，它可以在不修改原函数代码的情况下，动态地给函数添加功能。

```
┌─────────────────────────────────────────┐
│             装饰器的概念                │
├─────────────────────────────────────────┤
│                                         │
│  装饰器（Decorator）                    │
│  ─────────────────                      │
│  是一个函数，用于修改其他函数的行为    │
│                                         │
│  特点：                                 │
│  • 不修改原函数的代码                  │
│  • 不改变原函数的调用方式              │
│  • 在运行时动态添加功能                │
│                                         │
│  类比：                                 │
│  给房子装修                             │
│  • 房子 = 原函数                        │
│  • 装修 = 装饰器                        │
│  • 装修后房子功能增强，但结构不变       │
│                                         │
└─────────────────────────────────────────┘
```

### 最简示例

```python
from typing import Callable, TypeVar

T = TypeVar('T')

def my_decorator(func: Callable[..., T]) -> Callable[..., T]:
    """简单的装饰器"""
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        print("函数执行前")
        result = func(*args, **kwargs)
        print("函数执行后")
        return result
    
    return wrapper

@my_decorator
def say_hello() -> None:
    print("Hello!")

say_hello()
# 函数执行前
# Hello!
# 函数执行后
```

### 关键代码解释（核心）

| 代码 | 含义 | 执行过程 |
|------|------|---------|
| `func` | 原函数 | 被装饰的函数 |
| `wrapper` | 包装函数 | 替代原函数执行 |
| `@decorator` | 装饰语法 | `func = decorator(func)` |
| `return wrapper` | 返回新函数 | 原函数被替换 |

---

### 装饰器等价写法（核心）

**等价写法：** `@decorator` 语法本质是函数替换。

```
装饰器等价写法：
┌─────────────────────────────────────────────────────────────┐
│  @语法 vs 手动包装                                             │
│                                                              │
│  方式1：@语法（推荐）                                          │
│  ─────────────────────────────                               │
│  @my_decorator                                                │
│  def say_hello():                                             │
│      print("Hello!")                                          │
│                                                              │
│  方式2：手动包装（等价）                                       │
│  ─────────────────────────────                               │
│  def say_hello():                                             │
│      print("Hello!")                                          │
│  say_hello = my_decorator(say_hello)                         │
│                                                              │
│  ⚠️ 执行过程：                                                 │
│  1. Python 定义原函数 say_hello                               │
│  2. 调用 my_decorator(say_hello)                              │
│ 3. my_decorator 返回 wrapper                                  │
│  4. say_hello 现在指向 wrapper                                │
│                                                              │
│  调用 say_hello() 时：                                         │
│  ─────────────────────────────                               │
│  实际调用的是 wrapper()                                        │
│  wrapper() 内部调用原函数                                      │
└─────────────────────────────────────────────────────────────┘
```

### 最简示例

```python
# 方式 1：使用 @ 语法
@my_decorator
def say_hello() -> None:
    print("Hello!")

# 方式 2：手动包装（完全等价）
def say_hello() -> None:
    print("Hello!")

say_hello = my_decorator(say_hello)
```

### 关键代码解释

| 步骤 | 操作 | 结果 |
|------|------|------|
| `@my_decorator` | 装饰语法 | Python 解析为函数替换 |
| `my_decorator(say_hello)` | 调用装饰器 | 返回 wrapper 函数 |
| `say_hello = wrapper` | 函数替换 | 原函数名指向新函数 |

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def logger(func: Callable[..., T]) -> Callable[..., T]:
    """日志装饰器：记录函数调用信息"""
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        print(f"[LOG] 调用 {func.__name__}, 参数: {args}, {kwargs}")
        result = func(*args, **kwargs)
        print(f"[LOG] {func.__name__} 返回: {result}")
        return result
    return wrapper

@logger
def get_user(user_id: int) -> dict:
    return {"id": user_id, "name": "Alice"}

@logger
def create_user(name: str, email: str) -> dict:
    return {"name": name, "email": email}

@logger
def delete_user(user_id: int) -> bool:
    return True

# 使用
get_user(1)
# [LOG] 调用 get_user, 参数: (1,), {}
# [LOG] get_user 返回: {'id': 1, 'name': 'Alice'}
```

## 第二部分：实用装饰器示例

### 计时装饰器

```python
import time
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def timer(func: Callable[..., T]) -> Callable[..., T]:
    """计时装饰器：测量函数执行时间"""
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} 执行时间：{end_time - start_time:.4f}秒")
        return result
    return wrapper

@timer
def slow_function() -> None:
    time.sleep(1)
    print("任务完成")

slow_function()
# 任务完成
# slow_function 执行时间：1.0012 秒
```

### 权限验证装饰器

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

class User:
    def __init__(self, name: str, is_authenticated: bool):
        self.name = name
        self.is_authenticated = is_authenticated

def require_auth(func: Callable[..., T]) -> Callable[..., T]:
    """需要登录验证"""
    @wraps(func)
    def wrapper(user: User, *args: tuple, **kwargs: dict) -> T:
        if not user.is_authenticated:
            raise PermissionError("用户未登录")
        return func(user, *args, **kwargs)
    return wrapper

@require_auth
def view_profile(user: User) -> str:
    return f"查看 {user.name} 的个人资料"

# 使用
alice = User("Alice", is_authenticated=True)
print(view_profile(alice))  # 查看 Alice 的个人资料

guest = User("Guest", is_authenticated=False)
# view_profile(guest)  # PermissionError: 用户未登录
```

### 重试装饰器

```python
import time
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def retry(times: int = 3, delay: float = 1.0) -> Callable[[Callable[..., T]], Callable[..., T]]:
    """重试装饰器"""
    def decorator(func: Callable[..., T]) -> Callable[..., T]:
        @wraps(func)
        def wrapper(*args: tuple, **kwargs: dict) -> T:
            last_exception: Exception | None = None
            for attempt in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_exception = e
                    print(f"尝试 {attempt + 1}/{times} 失败：{e}")
                    if attempt < times - 1:
                        time.sleep(delay)
            raise last_exception
        return wrapper
    return decorator

@retry(times=3, delay=0.5)
def unstable_operation() -> str:
    import random
    if random.random() < 0.7:
        raise ConnectionError("网络错误")
    return "成功"

# result = unstable_operation()  # 可能会重试几次
```

### 缓存装饰器

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def cache(func: Callable[..., T]) -> Callable[..., T]:
    """简单缓存装饰器"""
    cached: dict[tuple, T] = {}
    
    @wraps(func)
    def wrapper(*args: tuple) -> T:
        if args in cached:
            print(f"缓存命中: {args}")
            return cached[args]
        result = func(*args)
        cached[args] = result
        return result
    
    return wrapper

@cache
def expensive_calculation(n: int) -> int:
    print(f"计算 {n}...")
    return n * n

print(expensive_calculation(5))  # 计算 5... → 25
print(expensive_calculation(5))  # 缓存命中: (5,) → 25
```

## 第三部分：@wraps 的作用

### 概念动机

当使用装饰器时，原函数的元信息（如 `__name__`、`__doc__`）会被包装函数覆盖。`@wraps` 装饰器可以保留这些信息。

### 最简示例

```python
from functools import wraps

# ❌ 不使用 @wraps
def decorator_without_wraps(func: Callable) -> Callable:
    def wrapper(*args: tuple, **kwargs: dict):
        return func(*args, **kwargs)
    return wrapper

@decorator_without_wraps
def my_func() -> None:
    """这是 my_func 的文档"""
    pass

print(my_func.__name__)  # wrapper（不是 my_func！）
print(my_func.__doc__)   # None（文档丢失！）

# ✅ 使用 @wraps
def decorator_with_wraps(func: Callable) -> Callable:
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict):
        return func(*args, **kwargs)
    return wrapper

@decorator_with_wraps
def my_func() -> None:
    """这是 my_func 的文档"""
    pass

print(my_func.__name__)  # my_func
print(my_func.__doc__)   # 这是 my_func 的文档
```

### 详细讲解

```python
from functools import wraps
from typing import Callable, TypeVar

T = TypeVar('T')

def proper_decorator(func: Callable[..., T]) -> Callable[..., T]:
    """正确的装饰器写法"""
    @wraps(func)  # 重要：保留原函数的元信息
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        """包装函数的文档"""
        print(f"调用 {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@proper_decorator
def calculate(a: int, b: int) -> int:
    """计算两个数的和"""
    return a + b

# 元信息被正确保留
print(calculate.__name__)  # calculate
print(calculate.__doc__)   # 计算两个数的和
```

### 实际应用：保持调试信息

```python
from functools import wraps
from typing import Callable, TypeVar
import traceback

T = TypeVar('T')

def debug(func: Callable[..., T]) -> Callable[..., T]:
    """调试装饰器：保留函数名用于错误追踪"""
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        try:
            return func(*args, **kwargs)
        except Exception as e:
            print(f"错误发生在 {func.__name__}")
            print(f"文件: {func.__code__.co_filename}")
            print(f"行号: {func.__code__.co_firstlineno}")
            raise
    return wrapper

@debug
def divide(a: int, b: int) -> float:
    return a / b

# divide(1, 0)  # 错误发生在 divide
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `@wraps(func)` | 保留被装饰函数的元信息 | 不加此行，`divide.__name__` 会变成 `wrapper`，调试时无法识别真正出错的函数 |
| `func.__code__.co_filename` | 获取函数定义所在文件 | `__code__` 是函数的代码对象，`co_filename` 直接指向源文件路径，便于定位错误位置 |
| `func.__code__.co_firstlineno` | 获取函数定义起始行号 | 配合文件路径，精确指示函数定义的位置，比只打印异常消息更有用 |
| `except Exception as e: print(...); raise` | 打印调试信息后重新抛出 | 装饰器负责记录信息，但不吞掉异常，让调用方仍能感知和处理错误 |

## L2 实践层：最佳实践

### 装饰器最佳实践

#### 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| 始终使用 `@wraps(func)` | 保留原函数元信息，便于调试 | `@wraps(func)` 放在 wrapper 定义前 |
| 保持 wrapper 签名一致 | 使用 `*args, **kwargs` 兼容所有参数 | `def wrapper(*args, **kwargs): ...` |
| 单一职责原则 | 每个装饰器只做一件事 | 不要混合日志、计时、权限等多种功能 |
| 装饰器命名清晰 | 使用动词或描述功能的名称 | `@timer`, `@log_execution`, `@require_auth` |
| 无副作用 | 不改变原函数返回值（除非是装饰器目的） | `return func(*args, **kwargs)` 保持原返回值 |

#### 反模式：不使用 @wraps

```python
# ❌ 不使用 @wraps（丢失元信息）
def timer_wrong(func):
    def wrapper(*args, **kwargs):
        import time
        start = time.time()
        result = func(*args, **kwargs)
        print(f"耗时: {time.time() - start:.2f}s")
        return result
    return wrapper

@timer_wrong
def calculate_sum(numbers: list) -> int:
    """计算列表元素的和"""
    return sum(numbers)

# 元信息丢失
print(calculate_sum.__name__)  # 'wrapper'（应该是 'calculate_sum'）
print(calculate_sum.__doc__)   # None（应该是 '计算列表元素的和'）

# 问题：
# 1. 调试时看到的函数名是 wrapper，无法定位原函数
# 2. 文档字符串丢失，IDE 无法提供帮助
# 3. traceback 中显示的是 wrapper，难以追踪错误来源
```

```python
# ✅ 正确做法：使用 @wraps
from functools import wraps

def timer_correct(func):
    @wraps(func)  # 保留原函数的元信息
    def wrapper(*args, **kwargs):
        import time
        start = time.time()
        result = func(*args, **kwargs)
        print(f"耗时: {time.time() - start:.2f}s")
        return result
    return wrapper

@timer_correct
def calculate_sum(numbers: list) -> int:
    """计算列表元素的和"""
    return sum(numbers)

# 元信息保留
print(calculate_sum.__name__)  # 'calculate_sum'
print(calculate_sum.__doc__)   # '计算列表元素的和'
```

#### 反模式：装饰器改变返回值

```python
# ❌ 装饰器意外改变返回值
def log_wrong(func):
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}")
        func(*args, **kwargs)  # 忘记 return！
        print(f"完成 {func.__name__}")
    return wrapper

@log_wrong
def get_data() -> dict:
    return {"status": "ok"}

result = get_data()
print(result)  # None（应该是 {"status": "ok"}）

# 问题：
# 1. wrapper 忘记返回原函数的结果
# 2. 所有被装饰的函数都返回 None
# 3. 破坏了原函数的行为契约
```

```python
# ✅ 正确做法：保留返回值
from functools import wraps

def log_correct(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}")
        result = func(*args, **kwargs)  # 保存结果
        print(f"完成 {func.__name__}")
        return result  # 返回原函数结果
    return wrapper

@log_correct
def get_data() -> dict:
    return {"status": "ok"}

result = get_data()
print(result)  # {"status": "ok"}
```

#### 反模式：装饰器做太多事

```python
# ❌ 一个装饰器做太多事情
def do_everything(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        # 日志
        print(f"调用 {func.__name__}")
        # 计时
        import time
        start = time.time()
        # 权限检查
        if not kwargs.get('user'):
            raise PermissionError("未登录")
        # 缓存
        cache_key = str(args)
        if cache_key in wrapper.cache:
            return wrapper.cache[cache_key]
        # 执行
        result = func(*args, **kwargs)
        # 后处理
        wrapper.cache[cache_key] = result
        print(f"耗时: {time.time() - start:.2f}s")
        return result
    wrapper.cache = {}
    return wrapper

# 问题：
# 1. 功能耦合，难以单独测试
# 2. 某个功能有问题影响整体
# 3. 无法灵活组合不同功能
# 4. 代码复杂，难以维护
```

```python
# ✅ 正确做法：单一职责，组合使用
from functools import wraps
import time

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"耗时: {time.time() - start:.2f}s")
        return result
    return wrapper

def logger(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}")
        result = func(*args, **kwargs)
        print(f"完成 {func.__name__}")
        return result
    return wrapper

def require_auth(func):
    @wraps(func)
    def wrapper(user, *args, **kwargs):
        if not user.is_authenticated:
            raise PermissionError("未登录")
        return func(user, *args, **kwargs)
    return wrapper

# 组合使用（按需选择）
@timer
@logger
def process_data(data):
    return data * 2

@require_auth
@timer
def admin_action(user, action):
    return f"执行 {action}"
```

### @wraps 的重要性详解

#### 为什么必须用 @wraps

```
@wraps 保留的元信息：
┌───────────────────────────────────────────────────────────────┐
│  functools.wraps(func) 复制的属性                              │
├───────────────────────────────────────────────────────────────┤
│                                                                │
│  __name__      → 函数名称                                      │
│  __doc__       → 文档字符串                                    │
│  __module__    → 所属模块                                      │
│  __annotations__ → 类型注解                                    │
│  __qualname__  → 完整限定名                                    │
│  __wrapped__   → 指向原函数（新增）                            │
│                                                                │
│  影响：                                                         │
│  ─────────────────────────────────────────                    │
│  • traceback 显示正确的函数名                                  │
│  • help() 显示正确的文档                                       │
│  • IDE 自动补全显示正确信息                                    │
│  • 单元测试定位正确的函数                                      │
│  • introspection 工具正确识别                                  │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

```python
from functools import wraps

def decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator
def example(x: int, y: int) -> int:
    """计算两个数的和"""
    return x + y

# 验证 @wraps 的效果
print(example.__name__)       # 'example'
print(example.__doc__)        # '计算两个数的和'
print(example.__annotations__) # {'x': int, 'y': int, 'return': int}
print(example.__module__)     # '__main__'
print(example.__wrapped__)    # <function example at ...>（原函数）

# 可以通过 __wrapped__ 访问原函数
print(example.__wrapped__(1, 2))  # 3（直接调用原函数，绕过装饰器）
```

#### 不用 @wraps 的后果

def decorator_no_wraps(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@decorator_no_wraps
def important_function(x: int) -> int:

    # （应该是 important_function！）

## L3 专家层：底层原理

### 装饰器内部实现

#### Python 如何实现

装饰器本质是函数替换，在模块加载时执行：

```
装饰器执行流程：
┌───────────────────────────────────────────────────────────────┐
│  模块加载阶段                                                   │
├───────────────────────────────────────────────────────────────┤
│                                                                │
│  1. Python 解析 @decorator 语法                                │
│     ↓                                                          │
│  2. 定义原函数 my_func                                         │
│     ↓                                                          │
│  3. 调用 decorator(my_func)                                    │
│     ↓                                                          │
│  4. decorator 返回 wrapper 函数                                │
│     ↓                                                          │
│  5. my_func = wrapper（函数名被重新绑定）                      │
│                                                                │
│  调用阶段：                                                     │
│  ─────────────────────────────────────────                    │
│  my_func(args)                                                 │
│     ↓                                                          │
│  实际执行 wrapper(args)                                        │
│     ↓                                                          │
│  wrapper 内部调用 func(args)（原函数）                         │
│     ↓                                                          │
│  wrapper 处理后返回结果                                        │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

```python
# 演示装饰器执行时机
print("=== 模块开始加载 ===")

def decorator(func):
    print(f"装饰器执行: 正在装饰 {func.__name__}")
    def wrapper(*args, **kwargs):
        print(f"wrapper 执行: 调用 {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

print("=== 函数定义前 ===")

@decorator  # 此处立即执行 decorator(my_func)
def my_func():
    print("my_func 执行")
    return "结果"

print("=== 函数定义后 ===")

# 输出顺序：
# === 模块开始加载 ===
# === 函数定义前 ===
# 装饰器执行: 正在装饰 my_func
# === 函数定义后 ===

# 验证 my_func 已被替换
print(f"my_func 现在是: {my_func.__name__}")  # wrapper

# 调用时执行 wrapper
result = my_func()
# wrapper 执行: 调用 my_func
# my_func 执行
```

### functools.wraps 原理

#### Python 如何实现

`wraps` 本身也是一个装饰器，内部使用 `update_wrapper`：

```
functools.wraps 实现原理：
┌───────────────────────────────────────────────────────────────┐
│  wraps(func) 的工作流程                                        │
├───────────────────────────────────────────────────────────────┤
│                                                                │
│  1. wraps 是 partial(update_wrapper, wrapped=func)            │
│     → 预绑定了 func 的 update_wrapper                          │
│                                                                │
│  2. @wraps(func) 应用到 wrapper                                │
│     → 执行 update_wrapper(wrapper, func)                      │
│                                                                │
│  3. update_wrapper 复制属性：                                  │
│     for attr in WRAPPER_UPDATES:                               │
│         setattr(wrapper, attr, getattr(func, attr))            │
│                                                                │
│  WRAPPER_UPDATES = (                                           │
│      '__module__', '__name__', '__qualname__',                 │
│      '__annotations__', '__doc__'                              │
│  )                                                              │
│                                                                │
│  4. 设置 __wrapped__ 属性：                                     │
│     wrapper.__wrapped__ = func                                 │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

```python
# 查看 wraps 的实现
import functools
import inspect

print(functools.wraps)
# functools.partial(<function update_wrapper at ...>, ...)

# 手动实现 wraps 功能
def manual_wraps(source_func):
    """手动实现 wraps 的功能"""
    def decorator(wrapper_func):
        # 复制属性
        for attr in ('__module__', '__name__', '__qualname__',
                     '__annotations__', '__doc__'):
            try:
                setattr(wrapper_func, attr, getattr(source_func, attr))
            except AttributeError:
                pass
        # 设置 __wrapped__
        wrapper_func.__wrapped__ = source_func
        return wrapper_func
    return decorator

def my_decorator(func):
    @manual_wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def test_func(x: int) -> int:
    """测试函数"""
    return x + 1

print(test_func.__name__)      # 'test_func'
print(test_func.__doc__)       # '测试函数'
print(test_func.__wrapped__)   # <function test_func at ...>
```

#### 性能考量

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 装饰器应用 | O(1) | 模块加载时一次性执行 |
| @wraps 复制属性 | O(5) | 复制 5 个固定属性 |
| 装饰后函数调用 | O(1) + 原函数 | 多一层 wrapper 调用 |
| __wrapped__ 访问 | O(1) | 直接引用原函数 |

```python
import timeit

# 测试装饰器的性能开销
def original_func(n):
    return sum(range(n))

def decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

def decorator_with_wraps(func):
    import functools
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

decorated = decorator(original_func)
decorated_wrapped = decorator_with_wraps(original_func)

# 性能对比
n = 1000
iterations = 100000

print("原函数:", timeit.timeit(lambda: original_func(n), number=iterations))
print("装饰后:", timeit.timeit(lambda: decorated(n), number=iterations))
print("装饰后+wraps:", timeit.timeit(lambda: decorated_wrapped(n), number=iterations))

# 结论：
# 1. 装饰器调用有轻微开销（多一层函数调用）
# 2. @wraps 的开销只在定义时发生，不影响调用性能
# 3. 实际项目中，装饰器开销通常可忽略
```

#### 设计动机

Python 为什么这样设计装饰器：

| 设计选择 | 原因 | 影响 |
|----------|------|------|
| @ 语法 | 比 `func = decorator(func)` 更直观 | 代码可读性更好 |
| 函数替换而非包装 | 简单且统一 | 所有函数行为一致 |
| `__wrapped__` 属性 | 保留对原函数的访问 | 可绕过装饰器，支持 unwrap |
| wraps 复制元信息 | 保持调试和文档正常 | traceback、help 正常工作 |

```python
# __wrapped__ 的实际用途
from functools import wraps

def debug_only(func):
    """仅在调试模式下生效的装饰器"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        import os
        if os.environ.get('DEBUG'):
            print(f"[DEBUG] {func.__name__}({args}, {kwargs})")
        return func(*args, **kwargs)
    return wrapper

@debug_only
def process(data):
    return data.upper()

# 生产环境：绕过装饰器，直接调用原函数
result = process.__wrapped__("hello")
# 不打印调试信息

# 调试环境：正常调用，带调试输出
import os
os.environ['DEBUG'] = '1'
result = process("hello")
# [DEBUG] process(('hello',), {})
```
