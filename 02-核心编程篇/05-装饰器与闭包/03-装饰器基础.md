## 为什么需要装饰器？

### 实际场景引入

想象你正在开发一个 Web 应用，需要对多个函数添加日志：

```python
# 问题场景：每个函数都要添加日志代码
def get_user(user_id: int) -> dict:
    print(f"[LOG] 调用 get_user, 参数: {user_id}")  # 重复代码
    result = {"id": user_id, "name": "Alice"}
    print(f"[LOG] get_user 返回: {result}")  # 重复代码
    return result

def create_user(name: str, email: str) -> dict:
    print(f"[LOG] 调用 create_user, 参数: {name}, {email}")  # 重复代码
    result = {"name": name, "email": email}
    print(f"[LOG] create_user 返回: {result}")  # 重复代码
    return result

def delete_user(user_id: int) -> bool:
    print(f"[LOG] 调用 delete_user, 参数: {user_id}")  # 重复代码
    result = True
    print(f"[LOG] delete_user 返回: {result}")  # 重复代码
    return result
```

**痛点**：
- 大量重复的日志代码
- 业务逻辑被淹没在日志代码中
- 修改日志格式需要改很多地方

**解决思路**：如果能把日志逻辑抽离出来，在不修改原函数的情况下添加功能，问题就解决了！

## 第一部分：什么是装饰器

### 概念动机

装饰器（Decorator）是一种特殊的函数，它可以在不修改原函数代码的情况下，动态地给函数添加功能。

```
┌─────────────────────────────────────────┐
│             装饰器的概念                │
├─────────────────────────────────────────┤
│                                         │
│  装饰器（Decorator）                    │
│  ─────────────────                      │
│  是一个函数，用于修改其他函数的行为    │
│                                         │
│  特点：                                 │
│  • 不修改原函数的代码                  │
│  • 不改变原函数的调用方式              │
│  • 在运行时动态添加功能                │
│                                         │
│  类比：                                 │
│  给房子装修                             │
│  • 房子 = 原函数                        │
│  • 装修 = 装饰器                        │
│  • 装修后房子功能增强，但结构不变       │
│                                         │
└─────────────────────────────────────────┘
```

### 最简示例

```python
from typing import Callable, TypeVar

T = TypeVar('T')

def my_decorator(func: Callable[..., T]) -> Callable[..., T]:
    """简单的装饰器"""
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        print("函数执行前")
        result = func(*args, **kwargs)
        print("函数执行后")
        return result
    
    return wrapper

@my_decorator
def say_hello() -> None:
    print("Hello!")

say_hello()
# 函数执行前
# Hello!
# 函数执行后
```

### 关键代码解释（核心）

| 代码 | 含义 | 执行过程 |
|------|------|---------|
| `func` | 原函数 | 被装饰的函数 |
| `wrapper` | 包装函数 | 替代原函数执行 |
| `@decorator` | 装饰语法 | `func = decorator(func)` |
| `return wrapper` | 返回新函数 | 原函数被替换 |

---

### 装饰器等价写法（核心）

**等价写法：** `@decorator` 语法本质是函数替换。

```
装饰器等价写法：
┌─────────────────────────────────────────────────────────────┐
│  @语法 vs 手动包装                                             │
│                                                              │
│  方式1：@语法（推荐）                                          │
│  ─────────────────────────────                               │
│  @my_decorator                                                │
│  def say_hello():                                             │
│      print("Hello!")                                          │
│                                                              │
│  方式2：手动包装（等价）                                       │
│  ─────────────────────────────                               │
│  def say_hello():                                             │
│      print("Hello!")                                          │
│  say_hello = my_decorator(say_hello)                         │
│                                                              │
│  ⚠️ 执行过程：                                                 │
│  1. Python 定义原函数 say_hello                               │
│  2. 调用 my_decorator(say_hello)                              │
│ 3. my_decorator 返回 wrapper                                  │
│  4. say_hello 现在指向 wrapper                                │
│                                                              │
│  调用 say_hello() 时：                                         │
│  ─────────────────────────────                               │
│  实际调用的是 wrapper()                                        │
│  wrapper() 内部调用原函数                                      │
└─────────────────────────────────────────────────────────────┘
```

### 最简示例

```python
# 方式 1：使用 @ 语法
@my_decorator
def say_hello() -> None:
    print("Hello!")

# 方式 2：手动包装（完全等价）
def say_hello() -> None:
    print("Hello!")

say_hello = my_decorator(say_hello)
```

### 关键代码解释

| 步骤 | 操作 | 结果 |
|------|------|------|
| `@my_decorator` | 装饰语法 | Python 解析为函数替换 |
| `my_decorator(say_hello)` | 调用装饰器 | 返回 wrapper 函数 |
| `say_hello = wrapper` | 函数替换 | 原函数名指向新函数 |

回到最初的场景，用装饰器解决：

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def logger(func: Callable[..., T]) -> Callable[..., T]:
    """日志装饰器：记录函数调用信息"""
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        print(f"[LOG] 调用 {func.__name__}, 参数: {args}, {kwargs}")
        result = func(*args, **kwargs)
        print(f"[LOG] {func.__name__} 返回: {result}")
        return result
    return wrapper

@logger
def get_user(user_id: int) -> dict:
    return {"id": user_id, "name": "Alice"}

@logger
def create_user(name: str, email: str) -> dict:
    return {"name": name, "email": email}

@logger
def delete_user(user_id: int) -> bool:
    return True

# 使用
get_user(1)
# [LOG] 调用 get_user, 参数: (1,), {}
# [LOG] get_user 返回: {'id': 1, 'name': 'Alice'}
```

## 第二部分：实用装饰器示例

### 计时装饰器

```python
import time
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def timer(func: Callable[..., T]) -> Callable[..., T]:
    """计时装饰器：测量函数执行时间"""
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} 执行时间：{end_time - start_time:.4f}秒")
        return result
    return wrapper

@timer
def slow_function() -> None:
    time.sleep(1)
    print("任务完成")

slow_function()
# 任务完成
# slow_function 执行时间：1.0012 秒
```

### 权限验证装饰器

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

class User:
    def __init__(self, name: str, is_authenticated: bool):
        self.name = name
        self.is_authenticated = is_authenticated

def require_auth(func: Callable[..., T]) -> Callable[..., T]:
    """需要登录验证"""
    @wraps(func)
    def wrapper(user: User, *args: tuple, **kwargs: dict) -> T:
        if not user.is_authenticated:
            raise PermissionError("用户未登录")
        return func(user, *args, **kwargs)
    return wrapper

@require_auth
def view_profile(user: User) -> str:
    return f"查看 {user.name} 的个人资料"

# 使用
alice = User("Alice", is_authenticated=True)
print(view_profile(alice))  # 查看 Alice 的个人资料

guest = User("Guest", is_authenticated=False)
# view_profile(guest)  # PermissionError: 用户未登录
```

### 重试装饰器

```python
import time
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def retry(times: int = 3, delay: float = 1.0) -> Callable[[Callable[..., T]], Callable[..., T]]:
    """重试装饰器"""
    def decorator(func: Callable[..., T]) -> Callable[..., T]:
        @wraps(func)
        def wrapper(*args: tuple, **kwargs: dict) -> T:
            last_exception: Exception | None = None
            for attempt in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_exception = e
                    print(f"尝试 {attempt + 1}/{times} 失败：{e}")
                    if attempt < times - 1:
                        time.sleep(delay)
            raise last_exception
        return wrapper
    return decorator

@retry(times=3, delay=0.5)
def unstable_operation() -> str:
    import random
    if random.random() < 0.7:
        raise ConnectionError("网络错误")
    return "成功"

# result = unstable_operation()  # 可能会重试几次
```

### 缓存装饰器

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def cache(func: Callable[..., T]) -> Callable[..., T]:
    """简单缓存装饰器"""
    cached: dict[tuple, T] = {}
    
    @wraps(func)
    def wrapper(*args: tuple) -> T:
        if args in cached:
            print(f"缓存命中: {args}")
            return cached[args]
        result = func(*args)
        cached[args] = result
        return result
    
    return wrapper

@cache
def expensive_calculation(n: int) -> int:
    print(f"计算 {n}...")
    return n * n

print(expensive_calculation(5))  # 计算 5... → 25
print(expensive_calculation(5))  # 缓存命中: (5,) → 25
```

## 第三部分：@wraps 的作用

### 概念动机

当使用装饰器时，原函数的元信息（如 `__name__`、`__doc__`）会被包装函数覆盖。`@wraps` 装饰器可以保留这些信息。

### 最简示例

```python
from functools import wraps

# ❌ 不使用 @wraps
def decorator_without_wraps(func: Callable) -> Callable:
    def wrapper(*args: tuple, **kwargs: dict):
        return func(*args, **kwargs)
    return wrapper

@decorator_without_wraps
def my_func() -> None:
    """这是 my_func 的文档"""
    pass

print(my_func.__name__)  # wrapper（不是 my_func！）
print(my_func.__doc__)   # None（文档丢失！）

# ✅ 使用 @wraps
def decorator_with_wraps(func: Callable) -> Callable:
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict):
        return func(*args, **kwargs)
    return wrapper

@decorator_with_wraps
def my_func() -> None:
    """这是 my_func 的文档"""
    pass

print(my_func.__name__)  # my_func
print(my_func.__doc__)   # 这是 my_func 的文档
```

### 详细讲解

```python
from functools import wraps
from typing import Callable, TypeVar

T = TypeVar('T')

def proper_decorator(func: Callable[..., T]) -> Callable[..., T]:
    """正确的装饰器写法"""
    @wraps(func)  # 重要：保留原函数的元信息
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        """包装函数的文档"""
        print(f"调用 {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@proper_decorator
def calculate(a: int, b: int) -> int:
    """计算两个数的和"""
    return a + b

# 元信息被正确保留
print(calculate.__name__)  # calculate
print(calculate.__doc__)   # 计算两个数的和
```

### 实际应用：保持调试信息

```python
from functools import wraps
from typing import Callable, TypeVar
import traceback

T = TypeVar('T')

def debug(func: Callable[..., T]) -> Callable[..., T]:
    """调试装饰器：保留函数名用于错误追踪"""
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        try:
            return func(*args, **kwargs)
        except Exception as e:
            print(f"错误发生在 {func.__name__}")
            print(f"文件: {func.__code__.co_filename}")
            print(f"行号: {func.__code__.co_firstlineno}")
            raise
    return wrapper

@debug
def divide(a: int, b: int) -> float:
    return a / b

# divide(1, 0)  # 错误发生在 divide
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `@wraps(func)` | 保留被装饰函数的元信息 | 不加此行，`divide.__name__` 会变成 `wrapper`，调试时无法识别真正出错的函数 |
| `func.__code__.co_filename` | 获取函数定义所在文件 | `__code__` 是函数的代码对象，`co_filename` 直接指向源文件路径，便于定位错误位置 |
| `func.__code__.co_firstlineno` | 获取函数定义起始行号 | 配合文件路径，精确指示函数定义的位置，比只打印异常消息更有用 |
| `except Exception as e: print(...); raise` | 打印调试信息后重新抛出 | 装饰器负责记录信息，但不吞掉异常，让调用方仍能感知和处理错误 |
