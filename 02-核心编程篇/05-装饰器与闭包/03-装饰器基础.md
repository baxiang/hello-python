## 为什么需要装饰器？

### 问题场景：API 函数需要日志

继续 Web API 系统，每个处理函数都需要记录日志和耗时：

```python
# ❌ 方案一：每个函数重复添加日志代码
def handle_users(req: Request) -> Response:
    print(f"[LOG] 调用 handle_users, 参数: {req}")  # 重复
    start = time.time()
    result = {"users": ["Alice", "Bob"]}
    elapsed = time.time() - start
    print(f"[LOG] handle_users 返回, 耗时: {elapsed:.3f}s")  # 重复
    return Response(200, result)

def handle_products(req: Request) -> Response:
    print(f"[LOG] 调用 handle_products, 参数: {req}")  # 重复
    start = time.time()
    result = {"products": ["Apple"]}
    elapsed = time.time() - start
    print(f"[LOG] handle_products 返回, 耗时: {elapsed:.3f}s")  # 重复
    return Response(200, result)

# ✅ 方案二：装饰器统一添加日志
@timer
@logger
def handle_users(req: Request) -> Response:
    return Response(200, {"users": ["Alice", "Bob"]})

@timer
@logger
def handle_products(req: Request) -> Response:
    return Response(200, {"products": ["Apple"]})
```

**问题：如何不修改原函数代码，就能添加日志功能？**

## 生活类比：礼物包装

把装饰器想象成礼物包装纸：

```
┌─────────────────────────────────────────────────────────────┐
│  装饰器 = 礼物包装纸                                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  原函数 = 礼物                                                │
│  • 有核心功能（如 handle_users 处理请求）                    │
│  • 不改变内在                                                │
│                                                             │
│  装饰器 = 包装纸                                              │
│  • 包裹礼物（包装原函数）                                     │
│  • 增加外观功能（日志、计时）                                 │
│  • 不改变礼物本身                                            │
│                                                             │
│  wrapper = 包装后的礼物                                       │
│  • 外面是包装纸（装饰器添加的功能）                           │
│  • 里面是礼物（原函数）                                       │
│  • 拿起来还是礼物，但多了包装                                 │
│                                                             │
│  示例：                                                      │
│  ─────────────────────                                      │
│  @logger                      # 包装纸                      │
│  def handle_users(req):       # 礼物                        │
│      return process(req)                                     │
│                                                             │
│  调用 handle_users(req) 时：                                 │
│  • 先执行包装纸功能（打印日志）                               │
│  • 再执行礼物功能（处理请求）                                 │
│  • 最后再执行包装纸功能（打印日志）                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**一句话：**
> **装饰器** = 不修改原函数，动态添加功能的包装函数

## 装饰器原理

### @语法等价写法

```
┌─────────────────────────────────────────────────────────────┐
│  @语法 vs 手动包装                                             │
│                                                             │
│  方式1：@语法（推荐）                                          │
│  ─────────────────────────────                               │
│  @logger                                                     │
│  def handle_users(req):                                      │
│      return process(req)                                     │
│                                                             │
│  方式2：手动包装（等价）                                       │
│  ─────────────────────────────                               │
│  def handle_users(req):                                      │
│      return process(req)                                     │
│  handle_users = logger(handle_users)                        │
│                                                             │
│  ⚠️ 执行过程：                                                 │
│  1. Python 定义原函数 handle_users                           │
│  2. 调用 logger(handle_users)                                │
│  3. logger 返回 wrapper                                      │
│  4. handle_users 现在指向 wrapper                            │
│                                                             │
│  调用 handle_users(req) 时：                                  │
│  ─────────────────────────────                               │
│  实际调用的是 wrapper(req)                                    │
│  wrapper 内部调用原函数                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 最简示例

```python
from typing import Callable, TypeVar
from functools import wraps

T = TypeVar('T')

def my_decorator(func: Callable[..., T]) -> Callable[..., T]:
    """简单装饰器"""
    @wraps(func)  # 保留原函数的元信息
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        print("函数执行前")
        result: T = func(*args, **kwargs)
        print("函数执行后")
        return result
    return wrapper

@my_decorator
def say_hello(name: str) -> str:
    return f"Hello, {name}!"

print(say_hello("Alice"))
# 函数执行前
# Hello, Alice!
# 函数执行后
```

## 装饰器执行时间线

```
┌─────────────────────────────────────────────────────────────┐
│  装饰器执行时间线                                              │
│                                                             │
│  代码：                                                       │
│  @logger                                                     │
│  def handle_users(req):                                      │
│      return process(req)                                     │
│                                                             │
│  handle_users(request)                                       │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 时间线                                               │   │
│  │                                                      │   │
│  │  1. 定义阶段（加载模块时）                           │   │
│  │     • Python 定义 handle_users                       │   │
│  │     • 调用 logger(handle_users)                      │   │
│  │     • logger 返回 wrapper                            │   │
│  │     • handle_users = wrapper                        │   │
│  │        │                                             │   │
│  │        ↓                                             │   │
│  │  2. 调用阶段（运行时）                               │   │
│  │     handle_users(request)                            │   │
│  │     实际执行 wrapper(request)                        │   │
│  │        │                                             │   │
│  │        ↓                                             │   │
│  │  3. wrapper 执行                                     │   │
│  │     print("[LOG] 调用...")                           │   │
│  │        │                                             │   │
│  │        ↓                                             │   │
│  │  4. 调用原函数                                       │   │
│  │     result = func(request)                           │   │
│  │     • func 是被保存的原 handle_users                │   │
│  │        │                                             │   │
│  │        ↓                                             │   │
│  │  5. wrapper 继续执行                                 │   │
│  │     print("[LOG] 返回...")                           │   │
│  │     return result                                     │   │
│  │                                                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ⚠️ 关键：定义时就完成包装，调用时执行 wrapper                  │
│  ⚠️ 关键：原函数 func 被保存在闭包中                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 贯穿实战：日志与计时装饰器

### @logger 装饰器

```python
from typing import Callable, TypeVar, Any
from functools import wraps
from datetime import datetime

T = TypeVar('T')

def logger(func: Callable[..., T]) -> Callable[..., T]:
    """日志装饰器
    
    贯穿项目：Web API 请求处理系统
    功能：自动记录函数调用日志
    """
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        # 前置日志
        timestamp: str = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        print(f"[{timestamp}] 调用 {func.__name__}")
        
        # 执行原函数
        result: T = func(*args, **kwargs)
        
        # 后置日志
        print(f"[{timestamp}] {func.__name__} 返回: {result}")
        
        return result
    return wrapper

# 使用
@logger
def handle_users(req: dict[str, Any]) -> dict[str, Any]:
    return {"users": ["Alice", "Bob"]}

@logger
def handle_products(req: dict[str, Any]) -> dict[str, Any]:
    return {"products": ["Apple", "Banana"]}

handle_users({"path": "/users"})
# [2024-01-19 10:00:00] 调用 handle_users
# [2024-01-19 10:00:00] handle_users 返回: {'users': [...]}
```

### @timer 装饰器

```python
import time
from typing import Callable, TypeVar

T = TypeVar('T')

def timer(func: Callable[..., T]) -> Callable[..., T]:
    """计时装饰器
    
    贯穿项目：Web API 请求处理系统
    功能：自动记录函数执行耗时
    """
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        start: float = time.time()
        result: T = func(*args, **kwargs)
        elapsed: float = time.time() - start
        print(f"{func.__name__} 耗时: {elapsed:.3f}s")
        return result
    return wrapper

# 使用
@timer
def process_data(data: list[int]) -> list[int]:
    return [x * 2 for x in data]

process_data([1, 2, 3, 4, 5])
# process_data 耗时: 0.001s
```

### 多个装饰器叠加

```python
from typing import Callable, TypeVar, Any
from functools import wraps

T = TypeVar('T')

def logger(func: Callable[..., T]) -> Callable[..., T]:
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        print(f"[LOG] {func.__name__} 开始")
        result = func(*args, **kwargs)
        print(f"[LOG] {func.__name__} 结束")
        return result
    return wrapper

def timer(func: Callable[..., T]) -> Callable[..., T]:
    @wraps(func)
    def wrapper(*args: tuple, **kwargs: dict) -> T:
        start = time.time()
        result = func(*args, **kwargs)
        print(f"[TIMER] {func.__name__}: {time.time() - start:.3f}s")
        return result
    return wrapper

# 多装饰器叠加（从下往上执行）
@logger   # 外层
@timer    # 内层
def handle_request(req: dict[str, Any]) -> dict[str, Any]:
    return {"status": "ok"}

handle_request({"path": "/users"})
# [TIMER] handle_request 开始
# [LOG] handle_request 开始
# 原函数执行
# [LOG] handle_request 结束
# [TIMER] handle_request 结束: 0.001s
```

**执行顺序：**

```
@logger
@timer
def func():
    ...

等价于：
func = logger(timer(func))

调用顺序：
wrapper_logger → wrapper_timer → func
```

## functools.wraps 的作用

### 为什么需要 wraps

```python
from typing import Callable

def bad_decorator(func: Callable) -> Callable:
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@bad_decorator
def my_func() -> str:
    """这是原函数的文档"""
    return "result"

print(my_func.__name__)  # "wrapper"（错误！）
print(my_func.__doc__)   # None（错误！）

# ✅ 使用 wraps 保留元信息
from functools import wraps

def good_decorator(func: Callable) -> Callable:
    @wraps(func)  # 复制原函数的元信息
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@good_decorator
def my_func() -> str:
    """这是原函数的文档"""
    return "result"

print(my_func.__name__)  # "my_func"（正确）
print(my_func.__doc__)   # "这是原函数的文档"（正确）
```

---

## 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| **用 @wraps(func)** | 保留原函数元信息 | `@wraps(func)` |
| **装饰器只做一件事** | 保持简单 | `@timer` 只计时 |
| **wrapper 返回原函数结果** | 不改变行为 | `return func(...)` |
| **支持 *args, **kwargs** | 通用性 | `wrapper(*args, **kwargs)` |

回答以下问题，检查你是否掌握了核心概念：

1. `@decorator` 等价于什么手动写法？
2. 装饰器在什么时候完成包装？
3. 多个装饰器叠加时的执行顺序是什么？
4. `@wraps(func)` 的作用是什么？
5. wrapper 函数为什么要返回原函数的结果？

**答案**：

1. `func = decorator(func)`
2. 定义阶段（模块加载时），不是调用时
3. 从下往上包装，从外往里执行
4. 保留原函数的 `__name__`、`__doc__` 等元信息
5. 保持原函数行为不变，装饰器只添加功能
