# 函数进阶

> **Python 版本要求**：本章内容需要 Python 3.11 或更高版本

本章讲解 Python 函数的高级特性，包括函数作为对象和嵌套函数。

---

## 为什么需要函数进阶？

### 实际场景引入

想象你正在开发一个数据处理系统：

```python
# 问题场景：如何灵活地处理不同的计算？
def process_data(data: list[int], operation: str) -> list[int]:
    """根据操作类型处理数据"""
    if operation == "double":
        return [x * 2 for x in data]
    elif operation == "square":
        return [x ** 2 for x in data]
    elif operation == "negate":
        return [-x for x in data]
    # 每次新增操作都要修改这个函数...
```

**痛点**：
- 每增加一个操作就要修改 `process_data` 函数
- 代码变得越来越长，难以维护
- 无法动态添加新操作

**解决思路**：如果能把函数当作参数传递，问题就迎刃而解了！

## 第一部分：函数是对象

### 概念动机

在 Python 中，函数不仅是代码块，更是**一等公民**（First-class Citizen）。这意味着函数可以像普通数据一样被操作。

```
┌─────────────────────────────────────────┐
│       Python 函数是"一等公民"           │
├─────────────────────────────────────────┤
│                                         │
│  一等公民（First-class Citizen）        │
│  ─────────────────────────────          │
│  可以在程序中任意位置使用的对象         │
│                                         │
│  函数可以：                             │
│  ────────────                           │
│  ✅ 赋值给变量                          │
│  ✅ 作为参数传递给其他函数              │
│  ✅ 作为函数的返回值                    │
│  ✅ 存储在数据结构中（列表、字典）      │
│                                         │
└─────────────────────────────────────────┘
```

### 最简示例

```python
# 函数可以赋值给变量
def greet(name: str) -> str:
    return f"Hello, {name}!"

say_hello = greet  # 函数赋值
print(say_hello("Alice"))  # Hello, Alice!
```

### 详细讲解

```python
# ─────────────────────────────────────
# 1. 赋值给变量
# ─────────────────────────────────────
def greet(name: str) -> str:
    return f"Hello, {name}!"

say_hello = greet
print(say_hello("Alice"))  # Hello, Alice!

# ─────────────────────────────────────
# 2. 作为参数传递
# ─────────────────────────────────────
def execute(func: callable, value: str) -> str:
    """执行传入的函数"""
    return func(value)

result = execute(greet, "David")
print(result)  # Hello, David!

# ─────────────────────────────────────
# 3. 作为返回值
# ─────────────────────────────────────
def get_operation(op: str) -> callable:
    """返回不同的操作函数"""
    if op == "add":
        return lambda a, b: a + b
    elif op == "mul":
        return lambda a, b: a * b
    return lambda a, b: None

add_func = get_operation("add")
print(add_func(3, 5))  # 8

# ─────────────────────────────────────
# 4. 存储在数据结构中
# ─────────────────────────────────────
def add(a: int, b: int) -> int: 
    return a + b

def sub(a: int, b: int) -> int: 
    return a - b

def mul(a: int, b: int) -> int: 
    return a * b

op_dict: dict[str, callable] = {"+": add, "-": sub, "*": mul}
print(op_dict["+"](10, 5))  # 15
```

### 渐进复杂化

回到最初的场景，现在可以用函数作为参数来解决：

```python
from typing import Callable

def process_data(data: list[int], operation: Callable[[int], int]) -> list[int]:
    """使用传入的函数处理数据"""
    return [operation(x) for x in data]

# 定义不同的操作
def double(x: int) -> int:
    return x * 2

def square(x: int) -> int:
    return x ** 2

def negate(x: int) -> int:
    return -x

# 使用
data = [1, 2, 3, 4, 5]
print(process_data(data, double))   # [2, 4, 6, 8, 10]
print(process_data(data, square))   # [1, 4, 9, 16, 25]
print(process_data(data, negate))   # [-1, -2, -3, -4, -5]

# 甚至可以用 lambda
print(process_data(data, lambda x: x + 100))  # [101, 102, 103, 104, 105]
```

### 实际应用

```python
from typing import Callable

# 策略模式：灵活切换排序算法
def sort_users(users: list[dict], key_func: Callable[[dict], str | int]) -> list[dict]:
    """按指定键排序用户"""
    return sorted(users, key=key_func)

users = [
    {"name": "Alice", "age": 30},
    {"name": "Bob", "age": 25},
    {"name": "Charlie", "age": 35},
]

# 按年龄排序
by_age = sort_users(users, lambda u: u["age"])
print(by_age)  # [{'name': 'Bob', 'age': 25}, ...]

# 按名字排序
by_name = sort_users(users, lambda u: u["name"])
print(by_name)  # [{'name': 'Alice', 'age': 30}, ...]
```

## 第二部分：嵌套函数

### 概念动机

有时我们需要在函数内部定义辅助函数，这些辅助函数只在外层函数内部使用，不需要暴露给外部。

### 最简示例

```python
def outer() -> None:
    """外层函数"""
    print("进入外层函数")

    def inner() -> None:
        """内层函数"""
        print("  进入内层函数")
        print("  离开内层函数")

    inner()  # 调用内层函数
    print("离开外层函数")

outer()
# 进入外层函数
#   进入内层函数
#   离开内层函数
# 离开外层函数

# ❌ 内层函数在外部不可见
# inner()  # NameError!
```

### 详细讲解

嵌套函数的一个常见用途是**代码组织和封装**：

```python
def process_file(filename: str) -> dict[str, int]:
    """处理文件并返回统计信息"""
    
    def count_lines(content: str) -> int:
        """统计行数"""
        return len(content.splitlines())
    
    def count_words(content: str) -> int:
        """统计单词数"""
        return len(content.split())
    
    def count_chars(content: str) -> int:
        """统计字符数"""
        return len(content)
    
    with open(filename, 'r') as f:
        content = f.read()
    
    return {
        "lines": count_lines(content),
        "words": count_words(content),
        "chars": count_chars(content),
    }
```

### 渐进复杂化

嵌套函数还可以用于**延迟执行**：

```python
from typing import Callable

def create_timer() -> Callable[[], float]:
    """创建一个计时器"""
    import time
    start_time = time.time()
    
    def get_elapsed() -> float:
        """获取已经过的时间"""
        return time.time() - start_time
    
    return get_elapsed

timer = create_timer()
# ... 做一些操作 ...
print(f"已过时间: {timer():.2f} 秒")
```

### 实际应用

```python
from typing import Callable

def validate_input(rules: dict[str, Callable[[str], bool]]) -> Callable[[dict], dict]:
    """创建输入验证器"""
    
    def validate(data: dict) -> dict:
        """验证数据"""
        errors = []
        for field, validator in rules.items():
            if field in data:
                if not validator(data[field]):
                    errors.append(f"字段 '{field}' 验证失败")
            else:
                errors.append(f"缺少字段 '{field}'")
        return {"valid": len(errors) == 0, "errors": errors}
    
    return validate

# 创建验证器
user_validator = validate_input({
    "name": lambda x: len(x) >= 2,
    "email": lambda x: "@" in x,
    "age": lambda x: x.isdigit() and int(x) >= 0,
})

# 使用验证器
result = user_validator({"name": "Alice", "email": "alice@example.com", "age": "25"})
print(result)  # {'valid': True, 'errors': []}
```

## 第三部分：LEGB 作用域规则

### 概念动机

理解变量查找顺序对于编写正确的嵌套函数至关重要。

```
┌─────────────────────────────────────────┐
│         LEGB 作用域查找规则             │
├─────────────────────────────────────────┤
│                                         │
│  L - Local（局部作用域）                │
│      当前函数内部的变量                 │
│                                         │
│  E - Enclosing（封闭作用域）            │
│      外层嵌套函数的变量                 │
│                                         │
│  G - Global（全局作用域）               │
│      模块级别的变量                     │
│                                         │
│  B - Built-in（内置作用域）             │
│      Python 内置的变量和函数            │
│                                         │
│  查找过程：                             │
│  Local → Enclosing → Global → Built-in │
│                                         │
└─────────────────────────────────────────┘
```

### 最简示例

```python
x = "全局变量"  # Global

def outer() -> None:
    y = "外层变量"  # Enclosing

    def inner() -> None:
        z = "内层变量"  # Local
        print(f"inner: {z}")    # Local
        print(f"inner: {y}")    # Enclosing
        print(f"inner: {x}")    # Global

    inner()

outer()
```

### 详细讲解

```python
# 完整的 LEGB 演示
x = "global"

def outer() -> None:
    y = "enclosing"
    
    def inner() -> None:
        z = "local"
        
        # 查找顺序：Local → Enclosing → Global → Built-in
        print(z)  # local (Local)
        print(y)  # enclosing (Enclosing)
        print(x)  # global (Global)
        print(len)  # <built-in function len> (Built-in)
    
    inner()
```

### 渐进复杂化：变量遮蔽

```python
x = "global"

def outer() -> None:
    x = "enclosing"  # 遮蔽全局 x
    
    def inner() -> None:
        x = "local"  # 遮蔽外层 x
        print(x)  # local
    
    inner()
    print(x)  # enclosing

outer()
print(x)  # global
```

### 实际应用

```python
# 配置管理：使用作用域隔离配置
DEFAULT_CONFIG = {"debug": False, "log_level": "INFO"}

def create_service(config: dict) -> Callable[[], dict]:
    """创建服务，配置被封闭在闭包中"""
    # 合并默认配置和用户配置
    final_config = {**DEFAULT_CONFIG, **config}
    
    def get_config() -> dict:
        """获取当前配置"""
        return final_config.copy()
    
    return get_config

service_a = create_service({"debug": True})
service_b = create_service({"log_level": "DEBUG"})

print(service_a())  # {'debug': True, 'log_level': 'INFO'}
print(service_b())  # {'debug': False, 'log_level': 'DEBUG'}
```

## L2 实践层：最佳实践

### 一等公民的应用场景

#### 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| 策略模式用函数代替类 | 函数更简洁，无需定义类层次结构 | `sorted(data, key=lambda x: x.name)` |
| 高阶函数接收函数参数 | 提高代码复用性，避免重复逻辑 | `process_items(items, transform_func)` |
| 函数工厂返回定制函数 | 闭包封装配置，避免全局变量 | `create_validator(min=0, max=100)` |
| 用字典映射替代 if-else | 扩展新功能无需修改原代码 | `operations = {'add': add, 'sub': sub}` |

#### 反模式：不要这样做

```python
# ❌ 用字符串参数选择操作（不易扩展）
def calculate(data: list, operation: str) -> list:
    if operation == "double":
        return [x * 2 for x in data]
    elif operation == "square":
        return [x ** 2 for x in data]
    elif operation == "negate":
        return [-x for x in data]
    # 每次新增操作都要修改这里！

# 问题：
# 1. 违反开闭原则——新增操作必须修改原函数
# 2. if-else 链越来越长，难以维护
# 3. 无法动态添加新操作
```

```python
# ✅ 正确做法：函数作为参数
from typing import Callable

def calculate(data: list[int], operation: Callable[[int], int]) -> list[int]:
    return [operation(x) for x in data]

# 新增操作无需修改 calculate
def cube(x: int) -> int:
    return x ** 3

calculate([1, 2, 3], cube)  # 直接使用，无需改原函数
```

#### 反模式：滥用全局变量

```python
# ❌ 用全局变量传递状态
current_operation = "add"

def process(a: int, b: int) -> int:
    if current_operation == "add":
        return a + b
    elif current_operation == "sub":
        return a - b

# 问题：
# 1. 全局变量容易被意外修改
# 2. 多线程环境下不安全
# 3. 函数依赖隐式状态，难以测试
```

```python
# ✅ 正确做法：函数返回函数（工厂模式）
from typing import Callable

def create_operation(op: str) -> Callable[[int, int], int]:
    operations = {
        "add": lambda a, b: a + b,
        "sub": lambda a, b: a - b,
    }
    return operations[op]

add_func = create_operation("add")
print(add_func(3, 5))  # 8，状态封装在闭包中
```

#### 适用场景

| 场景 | 是否推荐 | 原因 |
|------|---------|------|
| 回调函数 | ✅ 推荐 | GUI 事件处理、异步任务完成回调 |
| 策略模式 | ✅ 推荐 | 排序、过滤、转换等可变策略 |
| 函数工厂 | ✅ 推荐 | 创建带配置的定制函数 |
| 简单业务逻辑 | ✅ 推荐 | 替代复杂的类层次结构 |
| 状态复杂的场景 | ❌ 不推荐 | 用类更合适，可维护多个属性 |
| 需要继承的场景 | ❌ 不推荐 | 函数无法继承，用类实现 |

#### 嵌套函数最佳实践

| 做法 | 原因 | 示例 |
|------|------|------|
| 辅助函数用嵌套定义 | 避免污染模块命名空间 | `process_file` 内定义 `count_lines` |
| 嵌套层级不超过 2 层 | 过深嵌套难以阅读和理解 | `outer → inner` 即可，避免 `outer → middle → inner` |
| 复杂逻辑提取为独立函数 | 提高可测试性和复用性 | 工具函数放在模块级别 |

#### 反模式：嵌套过深

```python
# ❌ 嵌套层级过深（难以阅读）
def outer():
    def middle():
        def inner():
            def deep():
                def very_deep():
                    return "太难理解了"
                return very_deep()
            return deep()
        return inner()
    return middle()

# 问题：
# 1. 难以追踪变量来源（LEGB 查找复杂）
# 2. 调试困难，栈帧信息混乱
# 3. 代码可读性极差
```

```python
# ✅ 正确做法：拆分为独立函数
def very_deep() -> str:
    return "清晰易懂"

def deep() -> str:
    return very_deep()

def inner() -> str:
    return deep()

def outer() -> str:
    return inner()
```

## L3 专家层：底层原理

### 函数对象的内部结构

#### Python 如何实现

在 Python 中，函数是对象，拥有多个内部属性：

```
函数对象内部结构：
┌───────────────────────────────────────────────────────────────┐
│  函数对象（function）                                           │
├───────────────────────────────────────────────────────────────┤
│                                                                │
│  __code__      → 代码对象（字节码、参数信息）                   │
│  __globals__   → 全局命名空间字典                              │
│  __name__      → 函数名称字符串                                │
│  __defaults__  → 默认参数值元组                                │
│  __closure__   → 闭包变量元组（如果有）                        │
│  __annotations__ → 类型注解字典                                │
│  __doc__       → 文档字符串                                    │
│  __kwdefaults__ → 关键字参数默认值字典                         │
│                                                                │
│  调用过程：                                                     │
│  1. 创建栈帧（frame）                                          │
│  2. 从 __code__ 加载字节码                                     │
│  3. 在 __globals__ 中查找全局变量                              │
│  4. 从 __closure__ 获取闭包变量                                │
│  5. 执行字节码，返回结果                                       │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

```python
# 演示函数对象的内部属性
def example(a: int, b: int = 10) -> int:
    """示例函数"""
    x = 5
    return a + b + x

# 查看函数对象属性
print(example.__name__)       # 'example'
print(example.__doc__)        # '示例函数'
print(example.__defaults__)   # (10,)
print(example.__annotations__) # {'a': int, 'b': int, 'return': int}

# __code__ 属性包含代码对象
code = example.__code__
print(code.co_name)           # 'example'
print(code.co_argcount)       # 2（参数数量）
print(code.co_varnames)       # ('a', 'b', 'x')（局部变量名）
print(code.co_consts)         # (None, 5)（常量）
```

### __code__ 属性详解

#### Python 如何实现

`__code__` 是函数的核心，包含编译后的字节码：

```
__code__ 代码对象结构：
┌───────────────────────────────────────────────────────────────┐
│  代码对象（code）                                               │
├───────────────────────────────────────────────────────────────┤
│                                                                │
│  co_name       → 函数/代码块名称                               │
│  co_filename   → 源文件路径                                   │
│  co_firstlineno → 起始行号                                    │
│  co_argcount   → 位置参数数量                                 │
│  co_kwonlyargcount → 仅关键字参数数量                          │
│  co_varnames   → 局部变量名元组                                │
│  co_freevars   → 自由变量名元组（闭包用）                      │
│  co_cellvars   → 被嵌套函数引用的变量名元组                    │
│  co_consts     → 常量元组                                     │
│  co_names      → 全局名称元组                                 │
│  co_code       → 字节码指令序列（bytes）                       │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

```python
import dis

def add(a: int, b: int) -> int:
    return a + b

# 查看字节码
dis.dis(add)
# 输出：
#   2           0 LOAD_FAST                0 (a)
#               2 LOAD_FAST                1 (b)
#               4 BINARY_ADD
#               6 RETURN_VALUE

# 解释字节码：
# LOAD_FAST 0  → 加载局部变量 a（索引 0）
# LOAD_FAST 1  → 加载局部变量 b（索引 1）
# BINARY_ADD   → 执行加法操作
# RETURN_VALUE → 返回结果

# 查看代码对象的详细信息
code = add.__code__
print(f"参数数量: {code.co_argcount}")      # 2
print(f"局部变量: {code.co_varnames}")      # ('a', 'b')
print(f"字节码长度: {len(code.co_code)}")   # 8（每条指令 2 字节）
```

#### 性能考量

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 函数调用 | O(1) + 函数体 | 创建栈帧有固定开销 |
| 局部变量访问 | O(1) | LOAD_FAST 直接索引，最快 |
| 全局变量访问 | O(1) | LOAD_GLOBAL 查字典，稍慢 |
| 闭包变量访问 | O(1) | LOAD_DEREF 通过闭包元组 |

```python
import timeit

# 测试局部变量 vs 全局变量访问速度
GLOBAL_VAR = 100

def local_access():
    local_var = 100
    return local_var + 1

def global_access():
    return GLOBAL_VAR + 1

print("局部变量访问:", timeit.timeit(local_access, number=1000000))
print("全局变量访问:", timeit.timeit(global_access, number=1000000))
# 局部变量访问更快（约快 15-20%）
```

#### 设计动机

Python 为什么把函数设计为对象：

| 设计选择 | 原因 | 影响 |
|----------|------|------|
| 函数是一等公民 | 支持函数式编程范式 | 可传递、返回、存储函数 |
| 函数有 __code__ 属性 | 允许运行时检查和修改 | 动态分析、调试工具可访问字节码 |
| 函数有 __closure__ 属性 | 支持闭包语义 | 闭包变量持久保存 |
| 函数调用创建栈帧 | 支持递归和调试 | traceback 可追踪调用链 |

#### 知识关联

```
函数对象知识关联：
                    ┌───────────────┐
                    │   字节码      │
                    │   dis模块     │
                    └───────────────┘
                          │
                          ↓
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│   __code__    │────→│   函数对象    │────→│   __closure__ │
│   代码对象    │     │   function    │     │   闭包变量    │
└───────────────┘     └───────────────┘     └───────────────┘
                          │
                          ↓
                    ┌───────────────┐
                    │   栈帧        │
                    │   frame       │
                    │   (调用时创建) │
                    └───────────────┘
                          │
                          ↓
                    ┌───────────────┐
                    │   traceback   │
                    │   异常追踪    │
                    └───────────────┘
```
