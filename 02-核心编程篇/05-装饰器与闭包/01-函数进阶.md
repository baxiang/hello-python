## 为什么需要函数进阶？

### 问题场景：API 路由分发

继续 Web API 系统，需要根据不同路径调用不同处理函数：

```python
# ❌ 方案一：硬编码分支判断
def handle_request(path: str, data: dict) -> dict:
    if path == "/users":
        return {"users": ["Alice", "Bob"]}
    elif path == "/products":
        return {"products": ["Apple", "Banana"]}
    elif path == "/orders":
        return {"orders": []}
    # 每次新增路由都要修改这个函数！
    
# ✅ 方案二：函数作为参数传递
def handle_request(path: str, handler: Callable, data: dict) -> dict:
    return handler(data)  # 直接调用传入的处理函数
```

**问题：如何把函数当作参数传递？函数是一等公民意味着什么？**

## 生活类比：快递分拣中心

把函数想象成快递包裹：

```
┌─────────────────────────────────────────────────────────────┐
│  函数 = 快递包裹                                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  普通变量 = 普通包裹                                          │
│  • 存放数据                                                  │
│  • 传递给其他函数                                            │
│                                                             │
│  函数 = 特殊包裹                                              │
│  • 存放"处理逻辑"                                           │
│  • 可以传递给其他函数                                        │
│  • 可以从函数返回                                            │
│                                                             │
│  高阶函数 = 分拣中心                                          │
│  • 接收包裹（函数作为参数）                                   │
│  • 分发包裹（函数作为返回值）                                 │
│                                                             │
│  示例：                                                      │
│  ─────────────────────                                      │
│  def route(handler):  # 分拣中心                            │
│      return handler(data)  # 执行包裹里的逻辑               │
│                                                             │
│  route(handle_users)  # 传入handle_users包裹               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**一句话：**
> 函数是**一等公民**，可以像数据一样传递、存储、返回

## 函数是一等公民

### 四大能力

```
┌─────────────────────────────────────────────────────────────┐
│  函数的一等公民能力                                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 赋值给变量                                               │
│  ─────────────────                                          │
│  say_hello = greet                                          │
│  say_hello("Alice")  # 间接调用                             │
│                                                             │
│  2. 作为参数传递                                             │
│  ─────────────────                                          │
│  execute(greet, "Alice")  # 函数作参数                      │
│                                                             │
│  3. 作为返回值                                               │
│  ─────────────────                                          │
│  def get_handler(path):                                     │
│      return handle_users  # 返回函数                        │
│                                                             │
│  4. 存储在数据结构                                           │
│  ─────────────────                                          │
│  routes = {"GET": handle_get, "POST": handle_post}          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 示例代码

```python
from typing import Callable

# ─────────────────────────────────────
# 1. 赋值给变量
# ─────────────────────────────────────
def greet(name: str) -> str:
    return f"Hello, {name}!"

say_hello: Callable[[str], str] = greet
print(say_hello("Alice"))  # Hello, Alice!

# ─────────────────────────────────────
# 2. 作为参数传递
# ─────────────────────────────────────
def execute(func: Callable[[str], str], value: str) -> str:
    return func(value)

print(execute(greet, "Bob"))  # Hello, Bob!

# ─────────────────────────────────────
# 3. 作为返回值
# ─────────────────────────────────────
def get_operation(op: str) -> Callable[[int, int], int]:
    if op == "add":
        return lambda a, b: a + b
    elif op == "mul":
        return lambda a, b: a * b
    return lambda a, b: 0

add_func: Callable[[int, int], int] = get_operation("add")
print(add_func(3, 5))  # 8

# ─────────────────────────────────────
# 4. 存储在数据结构
# ─────────────────────────────────────
def add(a: int, b: int) -> int:
    return a + b

def sub(a: int, b: int) -> int:
    return a - b

operations: dict[str, Callable[[int, int], int]] = {
    "+": add,
    "-": sub
}
print(operations["+"](10, 5))  # 15
```

## 贯穿实战：路由分发系统

### 场景说明

Web API 系统需要根据 URL 路径调用不同处理函数：

```
┌─────────────────────────────────────────────────────────────┐
│  路由分发架构                                                 │
│                                                             │
│  请求：GET /users                                            │
│         │                                                   │
│         ↓                                                   │
│  ┌─────────────┐                                            │
│  │ 路由表      │ routes = {                                 │
│  │ (dict)      │   "/users": handle_users,                 │
│  │             │   "/products": handle_products,           │
│  │             │   "/orders": handle_orders                │
│  │             │ }                                          │
│  └─────────────┘                                            │
│         │                                                   │
│         ↓                                                   │
│  routes["/users"](request)                                  │
│         │                                                   │
│         ↓                                                   │
│  ┌─────────────┐                                            │
│  │ handle_users│ 执行处理函数                               │
│  │ (函数)      │                                            │
│  └─────────────┘                                            │
│         │                                                   │
│         ↓                                                   │
│  返回：{"users": ["Alice", "Bob"]}                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 实现路由分发

```python
from typing import Callable, Any
from dataclasses import dataclass

@dataclass
class Request:
    """模拟 HTTP 请求"""
    path: str
    method: str
    data: dict[str, Any]

@dataclass
class Response:
    """模拟 HTTP 响应"""
    status: int
    data: dict[str, Any]

# 定义处理函数
def handle_users(req: Request) -> Response:
    """处理用户请求"""
    if req.method == "GET":
        return Response(200, {"users": ["Alice", "Bob", "Charlie"]})
    elif req.method == "POST":
        return Response(201, {"created": req.data})
    return Response(405, {"error": "Method not allowed"})

def handle_products(req: Request) -> Response:
    """处理商品请求"""
    return Response(200, {"products": ["Apple", "Banana"]})

def handle_orders(req: Request) -> Response:
    """处理订单请求"""
    return Response(200, {"orders": [{"id": 1, "user": "Alice"}]})

# 路由表：路径 → 处理函数
routes: dict[str, Callable[[Request], Response]] = {
    "/users": handle_users,
    "/products": handle_products,
    "/orders": handle_orders,
}

def dispatch(req: Request) -> Response:
    """路由分发器
    
    贯穿项目：Web API 请求处理系统
    核心功能：根据路径调用对应处理函数
    """
    handler: Callable[[Request], Response] | None = routes.get(req.path)
    
    if handler is None:
        return Response(404, {"error": "Not found"})
    
    return handler(req)  # 函数作为变量被调用

# 使用示例
req1 = Request("/users", "GET", {})
req2 = Request("/products", "GET", {})
req3 = Request("/unknown", "GET", {})

print(dispatch(req1))  # Response(status=200, data={'users': [...]})
print(dispatch(req2))  # Response(status=200, data={'products': [...]})
print(dispatch(req3))  # Response(status=404, data={'error': 'Not found'})
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `routes: dict[str, Callable]` | 路径→函数映射 | 函数作为字典值存储 |
| `handler = routes.get(req.path)` | 查找处理函数 | 函数作为变量取出 |
| `return handler(req)` | 调用处理函数 | 函数作为变量调用 |
| `Callable[[Request], Response]` | 类型提示 | 明确函数签名 |

## 嵌套函数与 LEGB 规则

### 嵌套函数

```python
def outer(x: int) -> Callable[[int], int]:
    """外层函数"""
    def inner(y: int) -> int:
        """内层函数，引用外层变量 x"""
        return x + y
    return inner  # 返回内层函数

add5: Callable[[int], int] = outer(5)
print(add5(3))  # 8（inner 记住了 x=5）
```

### LEGB 变量查找顺序

```
┌─────────────────────────────────────────────────────────────┐
│  LEGB 查找顺序                                                 │
│                                                             │
│  L → Local（局部）                                           │
│  │  当前函数内的变量                                          │
│  │                                                          │
│  E → Enclosed（嵌套外层）                                    │
│  │  外层函数的变量（闭包变量）                                │
│  │                                                          │
│  G → Global（全局）                                          │
│  │  模块级别的变量                                            │
│  │                                                          │
│  B → Built-in（内置）                                        │
│  │  Python 内置名称（如 print、len）                         │
│  │                                                          │
│  查找顺序：L → E → G → B                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```python
x: str = "global"

def outer() -> Callable[[], str]:
    x: str = "enclosed"
    def inner() -> str:
        x: str = "local"
        return x
    return inner

print(outer()())  # "local"（L优先）

# 移除 inner 中的 local
def outer2() -> Callable[[], str]:
    x: str = "enclosed"
    def inner() -> str:
        return x  # 查找 E 层
    return inner

print(outer2()())  # "enclosed"
```

## 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| **用 Callable 类型提示** | 明确函数签名 | `Callable[[int], str]` |
| **函数字典代替分支** | 避免长 if-elif | `routes[path](req)` |
| **返回函数而非调用** | 惰性执行 | `return handler` |
| **保持函数纯度** | 易测试、易理解 | 避免修改外部状态 |
