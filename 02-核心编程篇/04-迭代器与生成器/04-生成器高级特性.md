## 第一部分：为什么需要生成器高级特性？

### 1.1 实际场景引入

假设你需要实现一个双向通信的协程，可以接收外部数据并处理：

```python
from typing import Generator, Optional

# 尝试一：普通生成器只能单向输出
def simple_counter() -> Generator[int, None, None]:
    count: int = 0
    while True:
        yield count
        count += 1

counter = simple_counter()
print(next(counter))  # 0
print(next(counter))  # 1
# ❌ 无法从外部重置或修改计数！

# 尝试二：使用 send() 实现双向通信
def advanced_counter() -> Generator[int, Optional[int], None]:
    count: int = 0
    while True:
        # yield 既可以输出值，也可以接收外部传入的值
        received: Optional[int] = yield count
        if received is not None:
            count = received  # 接收外部重置
        else:
            count += 1

counter = advanced_counter()
print(next(counter))       # 0（启动生成器）
print(next(counter))       # 1（继续计数）
print(counter.send(10))    # 10（重置计数）
print(next(counter))       # 11（继续计数）
```

**问题**：如何实现生成器与外部代码的双向通信？

### 1.2 概念动机

生成器有四个核心方法：

```
┌─────────────────────────────────────────┐
│        生成器的四个方法                  │
├─────────────────────────────────────────┤
│                                         │
│  next(gen)                              │

│  触发 finally 块执行                    │
│                                         │
└─────────────────────────────────────────┘
```

## 第二部分：send() 方法

### 2.1 最简示例：双向通信

```python
from typing import Generator

def accumulator() -> Generator[int, int, None]:
    """累加器生成器"""
    total: int = 0
    while True:
        # yield 表达式的值是外部 send() 传入的值
        value: int = yield total
        total += value

# 使用
acc: Generator[int, int, None] = accumulator()

# 第 1 步：启动生成器（必须先用 next() 或 send(None)）
print(next(acc))        # 0

# 第 2 步：发送值
print(acc.send(10))     # 10
print(acc.send(20))     # 30
print(acc.send(5))      # 35
```

### 2.2 详细讲解：send() 执行流程

```
acc = accumulator()   # 创建生成器
         │
    next(acc)
         ▼
执行到 yield total
返回 0，暂停在 value = yield total
         │
    acc.send(10)
         ▼
value = 10
total = 0 + 10 = 10
继续循环，回到 yield total
返回 10，暂停
         │
    acc.send(20)
         ▼
value = 20
total = 10 + 20 = 30
继续循环，回到 yield total
返回 30，暂停
```

### 2.3 渐进复杂化：带状态的协程

```python
from typing import Generator, Optional

def state_machine() -> Generator[str, Optional[str], None]:
    """状态机生成器"""
    state: str = "初始状态"
    while True:
        command: Optional[str] = yield f"当前状态：{state}"
        if command == "start":
            state = "运行中"
        elif command == "pause":
            state = "已暂停"
        elif command == "stop":
            state = "已停止"
        elif command is not None:
            state = f"未知命令：{command}"

# 使用
sm: Generator[str, Optional[str], None] = state_machine()
print(next(sm))           # 当前状态：初始状态
print(sm.send("start"))   # 当前状态：运行中
print(sm.send("pause"))   # 当前状态：已暂停
print(sm.send("stop"))    # 当前状态：已停止
```

## 第三部分：throw() 方法

### 3.1 最简示例：向生成器抛出异常

```python
from typing import Generator

def my_generator() -> Generator[int, None, None]:
    """带异常处理的生成器"""
    try:
        print("生成器开始")
        yield 1
        yield 2
        yield 3
    except ValueError as e:
        print(f"捕获到 ValueError: {e}")
        yield -1
    finally:
        print("生成器结束")

# 使用
gen: Generator[int, None, None] = my_generator()
print(next(gen))  # 生成器开始 \n 1
print(next(gen))  # 2

# 向生成器抛出异常
gen.throw(ValueError("测试异常"))
# 捕获到 ValueError: 测试异常 \n -1
```

### 3.2 渐进复杂化：错误恢复

```python
from typing import Generator

def resilient_generator() -> Generator[int, None, None]:
    """可恢复的生成器"""
    count: int = 0
    while True:
        try:
            value: int = yield count
            count += value
        except ValueError as e:
            print(f"错误恢复：{e}")
            # 恢复后继续执行
        except GeneratorExit:
            print("生成器关闭")
            raise

# 使用
gen: Generator[int, None, None] = resilient_generator()
print(next(gen))              # 0
print(gen.send(5))            # 5
gen.throw(ValueError("测试")) # 错误恢复：测试
print(gen.send(3))            # 8
gen.close()                   # 生成器关闭
```

## 第四部分：close() 方法

### 4.1 最简示例：关闭生成器

```python
from typing import Generator

def infinite_generator() -> Generator[int, None, None]:
    """无限生成器"""
    try:
        i: int = 0
        while True:
            yield i
            i += 1
    finally:
        print("生成器被清理")

gen: Generator[int, None, None] = infinite_generator()
print(next(gen))  # 0
print(next(gen))  # 1

# 关闭生成器
gen.close()
# 输出：生成器被清理

# 关闭后再调用会抛出 StopIteration
# next(gen)  # ❌ StopIteration
```

### 4.2 详细讲解：资源清理

```python
from typing import Generator

def file_reader(filename: str) -> Generator[str, None, None]:
    """文件读取生成器，带资源清理"""
    file = None
    try:
        file = open(filename, "r", encoding="utf-8")
        for line in file:
            yield line.strip()
    finally:
        if file:
            file.close()
            print(f"文件 {filename} 已关闭")

# 使用
reader: Generator[str, None, None] = file_reader("data.txt")
print(next(reader))  # 第一行
print(next(reader))  # 第二行

# 提前关闭
reader.close()  # 文件 data.txt 已关闭
```

## 第五部分：yield from

### 5.1 最简示例：委托生成器

```python
from typing import Generator, Iterable

def sub_generator() -> Generator[int, None, None]:
    """子生成器"""
    yield 1
    yield 2
    yield 3

def main_generator() -> Generator[int, None, None]:
    """主生成器"""
    yield "开始"
    yield from sub_generator()  # 委托给子生成器
    yield "结束"

# 使用
for value in main_generator():
    print(value)
# 开始
# 1
# 2
# 3
# 结束
```

### 5.2 详细讲解：yield from 语法

```python
from typing import Generator, Iterable, TypeVar

T = TypeVar("T")

# yield from 等价于：
def manual_delegate(iterable: Iterable[T]) -> Generator[T, None, None]:
    for item in iterable:
        yield item

# 使用 yield from 更简洁：
def simple_delegate(iterable: Iterable[T]) -> Generator[T, None, None]:
    yield from iterable
```

### 5.3 渐进复杂化：展平嵌套列表

```python
from typing import Generator, Any, Union

def flatten(nested: list[Union[Any, list[Any]]]) -> Generator[Any, None, None]:
    """递归展平嵌套列表"""
    for item in nested:
        if isinstance(item, list):
            yield from flatten(item)  # 递归委托
        else:
            yield item

# 使用
nested: list[Any] = [1, [2, 3], [4, [5, 6]], 7]
print(list(flatten(nested)))  # [1, 2, 3, 4, 5, 6, 7]
```

## 第六部分：实际应用场景

### 6.1 协程实现生产者-消费者模式

```python
from typing import Generator

def consumer() -> Generator[None, int, None]:
    """消费者"""
    print("消费者：等待数据...")
    while True:
        data: int = yield
        print(f"消费者：处理数据 {data}")

def producer(consumer_gen: Generator[None, int, None]) -> None:
    """生产者"""
    consumer_gen.send(None)  # 启动消费者
    for i in range(5):
        print(f"生产者：生产数据 {i}")
        consumer_gen.send(i)
    consumer_gen.close()

# 使用
cons = consumer()
producer(cons)
# 生产者：生产数据 0
# 消费者：处理数据 0
# ...
```

### 6.2 带状态的解析器

```python
from typing import Generator, Optional

def parser() -> Generator[str, Optional[str], None]:
    """带状态的解析器"""
    state: str = "waiting_header"
    buffer: list[str] = []
    
    while True:
        line: Optional[str] = yield f"状态：{state}"
        
        if line is None:
            continue
        
        if state == "waiting_header":
            if line.startswith("HEADER:"):
                state = "reading_data"
                yield f"找到头部：{line}"
        
        elif state == "reading_data":
            if line == "END":
                state = "done"
                yield f"数据：{''.join(buffer)}"
            else:
                buffer.append(line)

# 使用
p = parser()
print(next(p))  # 状态：waiting_header
print(p.send("HEADER:START"))  # 找到头部：HEADER:START
```

### 6.3 异步任务调度器

```python
from typing import Generator, Any
from collections import deque

def task_a() -> Generator[str, None, None]:
    """任务 A"""
    for i in range(3):
        yield f"Task A: step {i}"

def task_b() -> Generator[str, None, None]:
    """任务 B"""
    for i in range(2):
        yield f"Task B: step {i}"

def scheduler(tasks: list[Generator[str, None, None]]) -> None:
    """简单任务调度器"""
    queue: deque[Generator[str, None, None]] = deque(tasks)
    
    while queue:
        task: Generator[str, None, None] = queue.popleft()
        try:
            result: str = next(task)
            print(result)
            queue.append(task)  # 重新加入队列
        except StopIteration:
            pass

# 使用
scheduler([task_a(), task_b()])
# Task A: step 0
# Task B: step 0
# Task A: step 1
# Task B: step 1
# Task A: step 2
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `queue: deque[Generator[...]] = deque(tasks)` | 用 deque 管理任务队列 | `deque` 的 `popleft`/`append` 均为 O(1)，比 `list.pop(0)` 更高效 |
| `task = queue.popleft()` | 取出队头任务 | 轮询调度：先进先出，确保所有任务均匀推进 |
| `result = next(task)` | 驱动生成器执行一步 | `next()` 让生成器运行到下一个 `yield`，实现协作式调度 |
| `queue.append(task)` | 将未完成的任务重新入队 | 任务还有剩余步骤则放回队尾，等待下一轮调度 |
| `except StopIteration: pass` | 生成器耗尽时忽略 | `StopIteration` 表示任务完成，不再入队，让调度器自然缩小直到队列为空 |

## L2 实践层：send/throw/close 使用场景

### 推荐做法

| 做法 | 原因 | 示例 |
|------|------|------|
| **send() 用于双向通信** | 实现协程式的数据交换 | `result = yield value; gen.send(data)` |
| **throw() 用于错误恢复** | 让生成器有机会处理外部异常 | `gen.throw(ValueError, "错误")` |
| **close() 用于资源清理** | 确保生成器释放资源 | `gen.close()` 或 `finally` 块 |
| **yield from 简化委托** | 避免手动处理子生成器的值 | `yield from sub_gen()` |
| **用 try/finally 包裹 yield** | 确保 close() 时执行清理 | `try: yield x; finally: cleanup()` |
| **协程用 @types.coroutine 或 async** | 明确标识协程意图 | `async def coro(): await ...` |

### 反模式：不要这样做

```python
# ❌ send() 未先启动生成器
def accumulator():
    total = 0
    while True:
        value = yield total
        total += value

acc = accumulator()
acc.send(10)  # TypeError: can't send non-None value to a just-started generator

# ✅ 正确做法：先启动生成器
acc = accumulator()
next(acc)  # 或 acc.send(None)，启动生成器
acc.send(10)  # 正常工作
```

```python
# ❌ throw() 后不处理 GeneratorExit
def bad_generator():
    try:
        yield 1
        yield 2
    except GeneratorExit:
        yield 3  # 错误：GeneratorExit 后不能 yield

gen = bad_generator()
next(gen)
gen.close()  # RuntimeError: generator ignored GeneratorExit

# ✅ 正确做法：GeneratorExit 后直接退出
def good_generator():
    try:
        yield 1
        yield 2
    except GeneratorExit:
        print("清理资源")
        # 不能 yield，直接结束
    finally:
        print("最终清理")
```

```python
# ❌ 不使用 finally 导致资源泄漏
def file_reader(filename):
    f = open(filename)
    for line in f:
        yield line
    # 如果生成器被提前关闭，文件不会关闭！

reader = file_reader("data.txt")
next(reader)  # 读一行
reader.close()  # 文件可能未关闭

# ✅ 正确做法：使用 finally 确保清理
def safe_file_reader(filename):
    f = open(filename)
    try:
        for line in f:
            yield line
    finally:
        f.close()
        print("文件已关闭")

reader = safe_file_reader("data.txt")
next(reader)
reader.close()  # 触发 finally，文件关闭
```

```python
# ❌ yield from 嵌套层级过深
def deep_nest():
    yield from (yield from (yield from sub_gen()))
    # 难以理解和调试

# ✅ 正确做法：保持扁平或明确命名
def flat_delegate():
    yield "开始"
    yield from level1()
    yield "结束"

def level1():
    yield from level2()

def level2():
    yield from sub_gen()
```

│                                                             │

### 适用场景表

| 场景 | 是否推荐 | 推荐方法 | 原因 |
|------|---------|---------|------|
| 生产者-消费者模式 | ✅ 推荐 | send() + yield | 双向通信高效 |
| 状态机实现 | ✅ 推荐 | send() 传递命令 | 外部控制状态切换 |
| 错误恢复场景 | ✅ 推荐 | throw() + try/except | 生成器可恢复 |
| 资源管理（文件、连接） | ✅ 推荐 | close() + finally | 确保清理 |
| 嵌套数据展平 | ✅ 推荐 | yield from | 简洁递归 |
| 简单数据遍历 | ❌ 不推荐 send/throw | for 循环 | 不需要额外功能 |
| 异步编程 | ✅ 推荐 | async/await | 原生协程更清晰 |

### 常见陷阱

```python
# 陷阱1：忘记 send() 需要先启动
def coro():
    result = yield "ready"
    print(f"收到：{result}")

c = coro()
c.send("data")  # TypeError！

# 解决：先启动
c = coro()
print(next(c))  # "ready"
c.send("data")  # 正常执行
```

```python
# 陷阱2：close() 后尝试继续使用
def gen():
    yield 1
    yield 2

g = gen()
next(g)  # 1
g.close()
next(g)  # StopIteration（已关闭）

# 解决：close 是最终操作，不要再使用
```

```python
# 陷阱3：yield from 不处理 StopIteration.value
def sub_gen():
    yield 1
    return "完成"

def main_gen():
    result = yield from sub_gen()
    # result = "完成"，但如果不接收就丢失了

# 解决：接收 yield from 的返回值
def proper_main_gen():
    result = yield from sub_gen()
    yield f"子生成器返回：{result}"
```

## L3 专家层：协程基础与生成器状态机

### Python 如何实现协程

```
生成器协程机制：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  协程 = 可以暂停和恢复的函数                                 │
│  ─────────────────────────                                  │
│  • yield 暂停执行                                           │
│  • send() 恢复并传入值                                      │
│  • 双向数据流                                               │
│                                                             │
│  协程 vs 函数：                                              │
│  ─────────────────                                          │
│  函数：调用 → 执行 → 返回 → 结束                            │
│  协程：调用 → 暂停 → 恢复 → 暂停 → ... → 结束              │
│                                                             │
│  协程 vs 线程：                                              │
│  ─────────────────                                          │
│  • 协程：单线程，协作式调度（主动让出）                      │
│  • 线程：多线程，抢占式调度（系统切换）                      │
│  • 协程无并发问题，线程需要锁                                │
│                                                             │
│  yield 表达式的值：                                          │
│  ─────────────────                                          │
│  result = yield value                                       │
│         │         │                                         │
│         │         └── 返回给调用方的值                       │
│         └── send() 传入的值（首次 yield 返回 None）          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```python
# 演示协程的双向通信
from typing import Generator

def coroutine_example() -> Generator[str, str, None]:
    """协程示例：双向通信"""
    print("协程启动")
    while True:
        # yield 返回值给调用方，同时接收 send() 的值
        received = yield "等待输入"
        print(f"收到：{received}")
        if received == "quit":
            break
    print("协程结束")

# 使用
coro = coroutine_example()
print(next(coro))       # 启动，输出"等待输入"
print(coro.send("hello"))  # 发送"hello"，输出"等待输入"
print(coro.send("world"))  # 发送"world"，输出"等待输入"
coro.send("quit")       # 发送"quit"，协程结束
```

### 生成器状态机模式

生成器状态机实现：

│  状态机核心：                                                │
│  ─────────────────                                          │
│  • 每个状态对应一个代码块                                    │
│  • yield 暂停等待外部事件                                   │
│  • send() 传入触发状态转换的事件                            │

│  状态转换流程：                                              │
│  ─────────────────                                          │
│  状态A ──[事件1]──→ 状态B ──[事件2]──→ 状态C               │
│    │                  │                  │                 │

│  while True:                                                │
│      event = yield f"当前状态：{state}"                     │
│      if event == "触发A":                                   │
│          state = "状态A"                                    │

│          state = "状态B"                                    │

```python
from typing import Generator, Optional

def state_machine() -> Generator[str, Optional[str], None]:
    """网络连接状态机"""
    state = "DISCONNECTED"

    while True:
        event = yield f"状态：{state}"

        if state == "DISCONNECTED":
            if event == "connect":
                state = "CONNECTING"

        elif state == "CONNECTING":
            if event == "connected":
                state = "CONNECTED"
            elif event == "error":
                state = "ERROR"

        elif state == "CONNECTED":
            if event == "disconnect":
                state = "DISCONNECTING"
            elif event == "error":
                state = "ERROR"

        elif state == "DISCONNECTING":
            if event == "done":
                state = "DISCONNECTED"

        elif state == "ERROR":
            if event == "retry":
                state = "CONNECTING"
            elif event == "abort":
                state = "DISCONNECTED"

# 使用状态机
sm = state_machine()
print(next(sm))           # 状态：DISCONNECTED
print(sm.send("connect"))  # 状态：CONNECTING
print(sm.send("connected")) # 状态：CONNECTED
print(sm.send("disconnect")) # 状态：DISCONNECTING
print(sm.send("done"))     # 状态：DISCONNECTED
```

### yield from 的完整行为

```
yield from 完整语义：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  yield from ITERABLE                                        │
│  ─────────────────────────                                  │
│  • 自动委托所有 next/send/throw/close 操作                  │
│  • 子生成器返回值成为 yield from 的结果                     │
│                                                             │
│  操作传递：                                                  │
│  ─────────────────                                          │
│  next(outer)     → next(inner)                              │
│  outer.send(v)   → inner.send(v)                            │
│  outer.throw(e)  → inner.throw(e)                           │
│  outer.close()   → inner.close()                            │
│                                                             │
│  返回值传递：                                                │
│  ─────────────────                                          │
│  inner 的 return 值 → outer 的 yield from 结果              │
│                                                             │
│  手动实现等价：                                              │
│  ─────────────────                                          │
│  _i = iter(iterable)                                        │
│  try:                                                       │
│      while True:                                            │
│          _v = next(_i)                                      │
│          try:                                               │
│              _s = yield _v                                  │
│          except GeneratorExit:                              │
│              _i.close()                                     │
│              raise                                          │
│          _v = _i.send(_s)                                   │
│  except StopIteration as _e:                                │
│      _r = _e.value  # 子生成器返回值                        │
│  # _r 是 yield from 的结果                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```python
# yield from 的返回值机制
from typing import Generator

def sub_generator() -> Generator[int, None, str]:
    """子生成器，有返回值"""
    yield 1
    yield 2
    return "子生成器完成"  # 返回值

def main_generator() -> Generator[int, None, None]:
    """主生成器"""
    result = yield from sub_generator()
    print(f"子生成器返回：{result}")
    yield "主生成器继续"

# 使用
gen = main_generator()
print(next(gen))  # 1
print(next(gen))  # 2
print(next(gen))  # 输出"子生成器返回：子生成器完成" 后 yield "主生成器继续"
```

### 性能考量

| 操作 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| send() 调用 | O(1) | O(1) | 直接恢复帧 |
| throw() 调用 | O(1) + 异常处理 | O(异常栈) | 抛出异常到 yield 点 |
| close() 调用 | O(1) + finally | O(清理资源) | 触发 finally 块 |
| yield from 值传递 | O(1) | O(1) | 直接传递，无额外开销 |
| yield from 返回值 | O(1) | O(1) | StopIteration.value |

```python
# 性能对比：yield from vs 手动循环
import timeit
from typing import Generator, Iterator

# yield from 方式
def yield_from_chain(iterables: list[Iterator[int]]) -> Generator[int, None, None]:
    for it in iterables:
        yield from it

# 手动循环方式
def manual_chain(iterables: list[Iterator[int]]) -> Generator[int, None, None]:
    for it in iterables:
        for item in it:
            yield item

# 性能测试
iterables = [range(1000) for _ in range(100)]

yf_time = timeit.timeit(lambda: list(yield_from_chain(iterables)), number=100)
manual_time = timeit.timeit(lambda: list(manual_chain(iterables)), number=100)

print(f"yield from 时间：{yf_time:.3f}s")
print(f"手动循环时间：{manual_time:.3f}s")
# yield from 通常更快或相近，但代码更简洁
```

### 设计动机

| 设计选择 | 原因 | 替代方案对比 |
|----------|------|-------------|
| send() 双向通信 | 实现协程的基础 | 回调函数模式更复杂 |
| throw() 外部异常 | 生成器可处理外部错误 | 只能内部处理异常 |
| close() 资源清理 | 确保生成器释放资源 | del 不保证及时执行 |
| yield from 委托 | 简化嵌套生成器代码 | 手动循环冗长易错 |
| GeneratorExit 专用异常 | 区分正常关闭和错误 | StopIteration 不够明确 |
