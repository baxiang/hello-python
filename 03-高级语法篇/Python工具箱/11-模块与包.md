# 第 11 章 - 模块与包

本章讲解 Python 模块与包的概念、导入机制、自定义模块创建以及包的组织结构。

---

## 第一部分：模块基础

### 11.1 什么是模块

#### 概念说明

模块是一个包含 Python 代码的 `.py` 文件，可以被其他程序导入使用。

**模块的作用：**

```
┌─────────────────────────────────────────────────────────────┐
│                    模块的作用                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 代码组织                                               │
│      将相关功能放在同一文件中，便于管理                     │
│                                                             │
│   2. 代码复用                                               │
│      一个模块可以被多个程序导入使用                         │
│                                                             │
│   3. 命名空间隔离                                           │
│      每个模块有独立的命名空间，避免命名冲突                 │
│                                                             │
│   4. 维护方便                                               │
│      修改模块代码，所有导入该模块的程序自动受益             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 11.2 import 语法

#### 概念说明

Python 提供多种导入方式，可根据需要选择。

**基本导入方式：**

```python
# 方式 1：导入整个模块
import math
print(math.sqrt(16))  # 4.0

# 方式 2：导入模块中的特定函数
from math import sqrt, pi
print(sqrt(16))  # 4.0
print(pi)        # 3.14159...

# 方式 3：导入模块中的所有内容（不推荐）
from math import *
print(sqrt(16))  # 可能造成命名冲突

# 方式 4：使用别名
import math as m
print(m.sqrt(16))  # 4.0

from math import sqrt as square_root
print(square_root(16))  # 4.0
```

**导入顺序建议：**

```python
# 标准库
import os
import sys
from datetime import datetime

# 第三方库
import numpy as np
import pandas as pd

# 本地模块
from myproject import utils
from myproject.models import User
```

### 11.3 `__name__` 变量

#### 概念说明

`__name__` 变量用于判断模块是被直接运行还是被导入。

**示例代码：**

```python
# my_module.py

def main():
    print("程序主逻辑执行")

# 判断是否被直接运行
if __name__ == '__main__':
    main()
else:
    print("模块被导入")
```

**运行结果：**

```bash
# 直接运行
python my_module.py
# 输出：程序主逻辑执行

# 作为模块导入
python -c "import my_module"
# 输出：模块被导入
```

**常见用法：**

```
┌─────────────────────────────────────────────────────────────┐
│              __name__ 的常见用法                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 测试代码                                               │
│      将测试代码放在 if __name__ == '__main__' 块中          │
│      导入时不会执行测试                                     │
│                                                             │
│   2. 模块演示                                               │
│      提供模块使用示例                                       │
│                                                             │
│   3. 命令行入口                                             │
│      作为脚本运行时执行特定逻辑                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 第二部分：自定义模块

### 11.4 创建模块

#### 概念说明

任何 `.py` 文件都可以作为模块被导入。

**示例：创建计算器模块**

```python
# calculator.py

"""简单计算器模块"""

def add(a, b):
    """加法"""
    return a + b

def subtract(a, b):
    """减法"""
    return a - b

def multiply(a, b):
    """乘法"""
    return a * b

def divide(a, b):
    """除法"""
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b

# 模块级常量
PI = 3.14159
E = 2.71828

# 测试代码
if __name__ == '__main__':
    print(f"1 + 2 = {add(1, 2)}")
    print(f"10 / 2 = {divide(10, 2)}")
```

**导入使用：**

```python
# main.py
import calculator

result = calculator.add(5, 3)
print(result)  # 8

print(calculator.PI)  # 3.14159
```

### 11.5 模块搜索路径

#### 概念说明

Python 按照特定顺序搜索模块。

**搜索顺序：**

```python
import sys

# 查看模块搜索路径
for path in sys.path:
    print(path)
```

**输出示例：**

```
/Users/user/project        # 当前目录
/usr/lib/python3.11        # 标准库目录
/usr/lib/python3.11/site-packages  # 第三方库目录
...
```

**添加自定义路径：**

```python
import sys
sys.path.append('/path/to/my/modules')

# 现可以导入该路径下的模块
import my_module
```

## 第三部分：包（Package）

### 11.6 包的结构

#### 概念说明

包是一个包含多个模块的目录，必须有 `__init__.py` 文件。

**包的目录结构：**

```
my_package/
├── __init__.py          # 包初始化文件
├── module_a.py          # 模块 A
├── module_b.py          # 模块 B
└── subpackage/          # 子包
    ├── __init__.py
    └── module_c.py
```

**`__init__.py` 的作用：**

```python
# my_package/__init__.py

"""我的包"""

# 包级常量
VERSION = '1.0.0'

# 导入子模块，简化导入路径
from .module_a import func_a
from .module_b import func_b

# 导入子包
from .subpackage import module_c
```

---

### 11.7 包的导入

#### 概念说明

包可以有多种导入方式。

**导入方式：**

```python
# 导入整个包
import my_package
my_package.func_a()

# 导入包中的模块
import my_package.module_a
my_package.module_a.func_a()

# 导入包中的函数
from my_package import func_a
func_a()

# 导入子包中的模块
from my_package.subpackage import module_c
module_c.func_c()

# 导入子包中的函数
from my_package.subpackage.module_c import func_c
func_c()
```

---

### 11.8 相对导入与绝对导入

#### 概念说明

包内部的模块可以使用相对导入或绝对导入。

**绝对导入：**

```python
# my_package/module_a.py
from my_package.module_b import func_b  # 绝对导入
```

**相对导入：**

```python
# my_package/module_a.py
from .module_b import func_b      # 同级模块
from .subpackage import module_c  # 子包
from .. import other_package      # 上级包（父包同级）
```

**相对导入符号：**

| 符号 | 含义 |
|------|------|
| `.` | 当前包 |
| `..` | 上级包 |
| `...` | 上两级包 |

## 第四部分：pip 包管理

### 11.9 pip 基本命令

#### 概念说明

pip 是 Python 的包管理工具，用于安装和管理第三方库。

**常用命令：**

```bash
# 安装包
pip install requests
pip install requests==2.28.0  # 指定版本
pip install requests>=2.28.0  # 最低版本

# 卸载包
pip uninstall requests

# 升级包
pip install --upgrade requests

# 查看已安装的包
pip list

# 查看包详情
pip show requests

# 搜索包
pip search requests  # 已禁用，使用 pypi.org 搜索

# 导出依赖
pip freeze > requirements.txt

# 从 requirements.txt 安装
pip install -r requirements.txt
```

---

### 11.10 requirements.txt

#### 概念说明

`requirements.txt` 文件记录项目依赖，便于环境复现。

**文件格式：**

```txt
# requirements.txt
requests==2.28.0
numpy>=1.21.0
pandas>=1.3.0,<2.0.0
flask>=2.0
```

**版本 specifier：**

| 符号 | 含义 | 示例 |
|------|------|------|
| `==` | 精确版本 | `requests==2.28.0` |
| `>=` | 最低版本 | `numpy>=1.21.0` |
| `<=` | 最高版本 | `numpy<=1.23.0` |
| `>` | 大于 | `numpy>1.21.0` |
| `<` | 小于 | `numpy<1.23.0` |
| `~=` | 兼容版本 | `numpy~=1.21.0`（>=1.21.0, <1.22.0） |
