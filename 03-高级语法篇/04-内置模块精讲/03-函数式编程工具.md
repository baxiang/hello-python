# 函数式编程工具 (functools)

> **本章基于 Python 3.11+**
>
> `functools` 模块专门用于**高阶函数**（即操作函数的函数）。它是编写装饰器、缓存优化和构建复杂函数逻辑的神器。

---

## 1. @lru_cache: 缓存装饰器

**场景**：函数的计算成本很高，或者存在大量重复输入。使用 LRU (Least Recently Used) 缓存可以极大地提升性能。

### 案例：斐波那契数列
**未优化版**：计算 `fib(40)` 会非常慢（指数级复杂度）。

```python
import time

def fib(n):
    if n < 2: return n
    return fib(n-1) + fib(n-2)

start = time.time()
print(fib(40))
print(f"耗时：{time.time() - start:.4f}s")  # 耗时约 15-20 秒
```

**优化版**：加上 `@lru_cache`，耗时瞬间变为 0 秒（因为每个 `fib(n)` 只计算一次）。

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def fib_cached(n):
    if n < 2: return n
    return fib_cached(n-1) + fib_cached(n-2)

start = time.time()
print(fib_cached(40))
print(f"耗时：{time.time() - start:.6f}s")  # 耗时约 0.00005s
```

> **参数**：`maxsize=128` 表示最多缓存 128 个不同的参数组合结果。设为 `None` 则无限制。
> **提示**：缓存的函数参数必须是**可哈希的**（如 int, str, tuple），不能是 list 或 dict。

## 2. reduce: 归约累积

**场景**：将序列中的元素通过二元函数累积，最终得到一个单值（如求和、连乘、拼接）。

### 案例：连乘
`reduce` 接收一个函数和一个序列。
步骤：`func(func(func(a, b), c), d)`

```python
from functools import reduce

numbers = [1, 2, 3, 4, 5]

# 计算阶乘 1*2*3*4*5
result = reduce(lambda x, y: x * y, numbers)
print(result) # 120
```

### 案例：合并字典列表
将多个小字典合并为一个大字典。

```python
dicts = [{"a": 1}, {"b": 2}, {"a": 3, "c": 4}]

# 后面的字典会覆盖前面的键
merged = reduce(lambda d1, d2: {**d1, **d2}, dicts)

print(merged) # {'a': 3, 'b': 2, 'c': 4}
```

> **注意**：对于简单的求和/求最大值，优先使用内置的 `sum()` 或 `max()`，它们的效率更高。`reduce` 更适合自定义复杂的累积逻辑。

## 3. partial: 偏函数（冻结参数）

**场景**：你有一个包含很多参数的函数，但在某个特定场景下，其中几个参数的值是固定的。你可以“冻结”这些参数，生成一个新函数，从而简化调用。

### 案例：简化 int 转换
`int()` 函数有个 `base` 参数，默认是 10。如果你想频繁转换二进制字符串。

```python
from functools import partial

# 原生写法
print(int("1001", base=2)) 
print(int("1010", base=2))

# 使用 partial
binary_int = partial(int, base=2)

print(binary_int("1001")) # 9
print(binary_int("1010")) # 10
```

### 案例：回调函数传参
在 GUI 编程或网络请求回调中，回调函数通常只接受固定参数（如 `request`），但你想传递额外数据。

```python
import requests

def fetch_data(url, session):
    return session.get(url)

session = requests.Session()

# 创建一个只接受 url 的函数，session 已绑定
get_with_session = partial(fetch_data, session=session)

# 后续只需传 url
# get_with_session("https://example.com") 
```

## 4. @wraps: 装饰器的最佳实践

**场景**：编写自定义装饰器时，如果不加处理，被装饰函数的元数据（如 `__name__`, `__doc__`）会被装饰器内部的 wrapper 函数覆盖。

### 问题示例
```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def say_hello():
    """Say hello to the world."""
    print("Hello!")

print(say_hello.__name__)  # 输出 wrapper (丢失了原始函数名)
print(say_hello.__doc__)   # 输出 None (丢失了文档)
```

### 解决：使用 @wraps
`@wraps` 会自动将原始函数的 `__module__`, `__name__`, `__qualname__`, `__annotations__`, `__doc__`, `__dict__` 拷贝给 wrapper。

```python
from functools import wraps

def my_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def say_hello():
    """Say hello to the world."""
    print("Hello!")

print(say_hello.__name__)  # say_hello (恢复了)
print(say_hello.__doc__)   # Say hello to the world. (恢复了)
```

> **建议**：编写任何装饰器时，**必须**使用 `@wraps`，否则会影响日志记录和调试工具的行为。

## 5. singledispatch: 函数重载

**场景**：Python 不支持原生的函数重载（同名不同参），但可以通过 `@singledispatch` 根据第一个参数的类型分发到不同的处理函数。

```python
from functools import singledispatch

@singledispatch
def process(data):
    """默认处理"""
    raise NotImplementedError("Unsupported type")

@process.register
def _(data: int):
    print(f"处理整数：{data}")

@process.register
def _(data: list):
    print(f"处理列表：包含 {len(data)} 个元素")

@process.register
def _(data: str):
    print(f"处理字符串：{data.upper()}")

process(10)          # 处理整数：10
process([1, 2, 3])   # 处理列表：包含 3 个元素
process("hello")     # 处理字符串：HELLO
```

> **原理**：根据传入的第一个参数的类型，动态查找注册的函数版本。
