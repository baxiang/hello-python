# 函数式编程工具 (functools)

> **本章基于 Python 3.11+**
>
> `functools` 模块专门用于**高阶函数**（即操作函数的函数）。它是编写装饰器、缓存优化和构建复杂函数逻辑的神器。

---

## 1. @lru_cache: 缓存装饰器

**场景**：函数的计算成本很高，或者存在大量重复输入。使用 LRU (Least Recently Used) 缓存可以极大地提升性能。

### 案例：斐波那契数列
**未优化版**：计算 `fib(40)` 会非常慢（指数级复杂度）。

```python
import time

def fib(n):
    if n < 2: return n
    return fib(n-1) + fib(n-2)

start = time.time()
print(fib(40))
print(f"耗时：{time.time() - start:.4f}s")  # 耗时约 15-20 秒
```

**优化版**：加上 `@lru_cache`，耗时瞬间变为 0 秒（因为每个 `fib(n)` 只计算一次）。

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def fib_cached(n):
    if n < 2: return n
    return fib_cached(n-1) + fib_cached(n-2)

start = time.time()
print(fib_cached(40))
print(f"耗时：{time.time() - start:.6f}s")  # 耗时约 0.00005s
```

> **参数**：`maxsize=128` 表示最多缓存 128 个不同的参数组合结果。设为 `None` 则无限制。
> **提示**：缓存的函数参数必须是**可哈希的**（如 int, str, tuple），不能是 list 或 dict。

### @cache：无限制缓存（Python 3.9+）

`@cache` 是 `@lru_cache(maxsize=None)` 的简写，适合需要永久缓存的场景。

```python
from functools import cache

@cache
def factorial(n: int) -> int:
    """计算阶乘，结果永久缓存"""
    if n < 2:
        return 1
    return n * factorial(n - 1)

print(factorial(10))  # 第一次计算
print(factorial(10))  # 直接从缓存读取
print(factorial.cache_info())  # CacheInfo(hits=1, misses=1, maxsize=None, currsize=11)
```

**@cache vs @lru_cache：**

| 装饰器 | 缓存大小 | 适用场景 |
|--------|---------|----------|
| `@lru_cache(maxsize=128)` | 有限制，LRU淘汰 | 参数组合多，需控制内存 |
| `@lru_cache(maxsize=None)` | 无限制 | 参数组合少，需永久缓存 |
| `@cache` | 无限制（等价上面） | 语法更简洁 |

## 2. @cached_property：属性缓存（Python 3.8+）

**场景**：类中某个属性计算成本高（如数据库查询、网络请求），但多次访问时希望只计算一次。

### 案例：延迟加载配置

```python
from functools import cached_property
import time

class ConfigManager:
    def __init__(self, config_path: str):
        self.config_path = config_path
    
    @cached_property
    def config(self) -> dict:
        """配置只加载一次，后续访问直接返回缓存"""
        print("加载配置文件...")
        time.sleep(1)  # 模拟耗时操作
        return {"host": "localhost", "port": 8080}
    
    @cached_property
    def db_connection(self) -> str:
        """数据库连接信息缓存"""
        print("建立数据库连接...")
        time.sleep(0.5)
        return "connected"

config = ConfigManager("config.yaml")

print(config.config)  # 第一次访问：加载配置文件...
print(config.config)  # 第二次访问：直接返回缓存，不打印

print(config.db_connection)  # 第一次访问：建立数据库连接...
print(config.db_connection)  # 第二次访问：直接返回缓存
```

### 与 @property 的区别

```python
class WithProperty:
    @property
    def expensive(self) -> int:
        print("每次访问都计算")
        return sum(range(10000))

class WithCachedProperty:
    @cached_property
    def expensive(self) -> int:
        print("只计算一次")
        return sum(range(10000))

wp = WithProperty()
print(wp.expensive)  # 打印：每次访问都计算
print(wp.expensive)  # 打印：每次访问都计算

wcp = WithCachedProperty()
print(wcp.expensive)  # 打印：只计算一次
print(wcp.expensive)  # 不打印，直接返回缓存
```

### 清除缓存

```python
from functools import cached_property

class Service:
    @cached_property
    def api_token(self) -> str:
        print("获取API令牌")
        return "token_abc123"

service = Service()
print(service.api_token)  # 获取API令牌
print(service.api_token)  # 缓存

# 清除缓存（重新计算）
del service.api_token
print(service.api_token)  # 再次获取API令牌
```

> **注意**：`@cached_property` 是**线程安全**的，计算过程中其他线程会等待。

## 2. reduce: 归约累积

**场景**：将序列中的元素通过二元函数累积，最终得到一个单值（如求和、连乘、拼接）。

### 案例：连乘
`reduce` 接收一个函数和一个序列。
步骤：`func(func(func(a, b), c), d)`

```python
from functools import reduce

numbers = [1, 2, 3, 4, 5]

# 计算阶乘 1*2*3*4*5
result = reduce(lambda x, y: x * y, numbers)
print(result) # 120
```

### 案例：合并字典列表
将多个小字典合并为一个大字典。

```python
dicts = [{"a": 1}, {"b": 2}, {"a": 3, "c": 4}]

# 后面的字典会覆盖前面的键
merged = reduce(lambda d1, d2: {**d1, **d2}, dicts)

print(merged) # {'a': 3, 'b': 2, 'c': 4}
```

> **注意**：对于简单的求和/求最大值，优先使用内置的 `sum()` 或 `max()`，它们的效率更高。`reduce` 更适合自定义复杂的累积逻辑。

## 3. partial: 偏函数（冻结参数）

**场景**：你有一个包含很多参数的函数，但在某个特定场景下，其中几个参数的值是固定的。你可以“冻结”这些参数，生成一个新函数，从而简化调用。

### 案例：简化 int 转换
`int()` 函数有个 `base` 参数，默认是 10。如果你想频繁转换二进制字符串。

```python
from functools import partial

# 原生写法
print(int("1001", base=2)) 
print(int("1010", base=2))

# 使用 partial
binary_int = partial(int, base=2)

print(binary_int("1001")) # 9
print(binary_int("1010")) # 10
```

### 案例：回调函数传参
在 GUI 编程或网络请求回调中，回调函数通常只接受固定参数（如 `request`），但你想传递额外数据。

```python
import requests

def fetch_data(url, session):
    return session.get(url)

session = requests.Session()

# 创建一个只接受 url 的函数，session 已绑定
get_with_session = partial(fetch_data, session=session)

# 后续只需传 url
# get_with_session("https://example.com") 
```

## 4. @wraps: 装饰器的最佳实践

**场景**：编写自定义装饰器时，如果不加处理，被装饰函数的元数据（如 `__name__`, `__doc__`）会被装饰器内部的 wrapper 函数覆盖。

### 问题示例
```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def say_hello():
    """Say hello to the world."""
    print("Hello!")

print(say_hello.__name__)  # 输出 wrapper (丢失了原始函数名)
print(say_hello.__doc__)   # 输出 None (丢失了文档)
```

### 解决：使用 @wraps
`@wraps` 会自动将原始函数的 `__module__`, `__name__`, `__qualname__`, `__annotations__`, `__doc__`, `__dict__` 拷贝给 wrapper。

```python
from functools import wraps

def my_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def say_hello():
    """Say hello to the world."""
    print("Hello!")

print(say_hello.__name__)  # say_hello (恢复了)
print(say_hello.__doc__)   # Say hello to the world. (恢复了)
```

> **建议**：编写任何装饰器时，**必须**使用 `@wraps`，否则会影响日志记录和调试工具的行为。

## 5. singledispatch: 函数重载

**场景**：Python 不支持原生的函数重载（同名不同参），但可以通过 `@singledispatch` 根据第一个参数的类型分发到不同的处理函数。

```python
from functools import singledispatch

@singledispatch
def process(data):
    """默认处理"""
    raise NotImplementedError("Unsupported type")

@process.register
def _(data: int):
    print(f"处理整数：{data}")

@process.register
def _(data: list):
    print(f"处理列表：包含 {len(data)} 个元素")

@process.register
def _(data: str):
    print(f"处理字符串：{data.upper()}")

process(10)          # 处理整数：10
process([1, 2, 3])   # 处理列表：包含 3 个元素
process("hello")     # 处理字符串：HELLO
```

> **原理**：根据传入的第一个参数的类型，动态查找注册的函数版本。

### 何时使用？
*   **计算慢？** -> 试 `lru_cache` 或 `cache`
*   **属性计算贵？** -> 用 `cached_property`
*   **参数多？** -> 试 `partial`
*   **写装饰器？** -> 必用 `wraps`
*   **处理累积逻辑？** -> 试 `reduce`
*   **需要类型分发？** -> 用 `singledispatch`
