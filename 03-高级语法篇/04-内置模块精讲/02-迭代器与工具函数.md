# 迭代器与工具函数 (itertools)

> **本章基于 Python 3.11+**
>
> `itertools` 是 Python 标准库中最强大的模块之一。它提供了一系列用于处理**迭代器**的高效工具。
>
> **核心特性**：
> 1.  **惰性求值**：所有函数返回的都是迭代器，不立即产生结果，极度**节省内存**。
> 2.  **C 语言速度**：底层由 C 语言实现，比纯 Python 的 `for` 循环快得多。

---

## 1. 核心概念：什么是惰性迭代器？

对比原生列表推导式：
```python
# 列表推导式：立即生成 1 亿个数字，内存瞬间爆炸
nums = [x for x in range(100_000_000)]
```

使用 `itertools`：
```python
import itertools
import sys

# count：生成器，返回一个迭代器，不占用内存
counter = itertools.count(start=0, step=2)

print(type(counter))       # <class 'itertools.count'>
print(sys.getsizeof(counter)) # 非常小，只存储状态

# 只有当你去取数据（循环或 next）时，它才会计算
print(next(counter)) # 0
print(next(counter)) # 2
print(next(counter)) # 4
```

## 2. 常用工具函数分类

### 2.1 无限迭代器
如果不配合 `break` 或 `islice` 使用，它们会永远运行下去。

#### `count(start, step)`
生成无限递增序列。

```python
import itertools

# 模拟带索引的循环（比 range 更灵活，支持浮点数）
counter = itertools.count(start=1.5, step=0.5)

print(next(counter)) # 1.5
print(next(counter)) # 2.0
print(next(counter)) # 2.5
```

#### `cycle(iterable)`
无限循环遍历一个序列（常用于轮询）。

```python
servers = ["Server A", "Server B", "Server C"]
lb = itertools.cycle(servers)

for _ in range(5):
    print(f"请求发往 -> {next(lb)}")
    
# 输出：
# A -> B -> C -> A -> B
```

#### `repeat(object, times)`
重复同一个对象。如果不指定 `times`，则无限重复。

```python
squares = map(pow, range(10), itertools.repeat(2))
# 等同于 map(lambda x: x**2, range(10))，但 repeat 更快
print(list(squares)) # [0, 1, 4, 9, 16, ...]
```

### 2.2 序列处理与组合

#### `chain(*iterables)`
将多个迭代器“首尾相连”。

**场景**：遍历多个文件，或者遍历多个列表。

```python
list1 = [1, 2]
list2 = [3, 4]
list3 = [5]

# 原生写法需要多次 for 循环或 list1 + list2（耗内存）
for x in itertools.chain(list1, list2, list3):
    print(x) # 1, 2, 3, 4, 5
```

#### `chain.from_iterable(iterables)`
**扁平化神器**。接受一个可迭代对象（里面包含了多个迭代器）。

**场景**：二维列表转一维列表。

```python
matrix = [[1, 2], [3, 4], [5]]
flattened = list(itertools.chain.from_iterable(matrix))

print(flattened) # [1, 2, 3, 4, 5]
```

#### `groupby(iterable, key)`
将连续且 `key` 相同的元素分组。
**注意**：使用前通常需要先对数据进行**排序**。

**场景**：按日期分组订单，或者按类型分组用户。

```python
from itertools import groupby

data = [
    {"name": "A", "type": "X"},
    {"name": "B", "type": "X"},
    {"name": "C", "type": "Y"},
]

# 必须按 key 排序，否则相同类型的会被分开
data.sort(key=lambda x: x["type"])

for key, group in groupby(data, key=lambda x: x["type"]):
    print(f"类型：{key}")
    print(list(group))
    
# 输出：
# 类型：X -> [{'name': 'A', 'type': 'X'}, {'name': 'B', 'type': 'X'}]
# 类型：Y -> [{'name': 'C', 'type': 'Y'}]
```

#### `islice(iterable, start, stop, step)`
对迭代器进行切片，但不像列表切片那样复制数据（省内存）。

```python
large_iter = range(1000)
first_5 = list(itertools.islice(large_iter, 5))
print(first_5) # [0, 1, 2, 3, 4]
```

### 2.3 排列与组合

处理数学中的排列组合问题，常用于算法题或暴力破解。

#### `product(*iterables)`
笛卡尔积（多层嵌套循环的替身）。

**场景**：生成所有可能的颜色/尺寸组合。

```python
colors = ["Red", "Blue"]
sizes = ["S", "M", "L"]

# 原生写法：for c in colors: for s in sizes: ...
combos = list(itertools.product(colors, sizes))

print(combos)
# [('Red', 'S'), ('Red', 'M'), ('Red', 'L'), ('Blue', 'S'), ...]
```

#### `permutations(iterable, r)`
排列（**有序**的）。
```python
# ABC, ACB 是不同的
perms = list(itertools.permutations(['A', 'B', 'C'], 2))
print(perms)
# [('A', 'B'), ('A', 'C'), ('B', 'A'), ('B', 'C'), ('C', 'A'), ('C', 'B')]
```

#### `combinations(iterable, r)`
组合（**无序**的）。
```python
# AB, BA 视为相同，只保留 AB
combs = list(itertools.combinations(['A', 'B', 'C'], 2))
print(combs)
# [('A', 'B'), ('A', 'C'), ('B', 'C')]
```

## 3. 实战案例：高效的数据清洗

假设有一堆脏数据，包含嵌套列表和需要过滤的空值。

```python
data = [
    [1, 2, None],
    [3, None, 4],
    [],
    [5, 6]
]

# 目标：展平并去除 None

# 步骤 1：展平
flat = itertools.chain.from_iterable(data)

# 步骤 2：过滤 (使用 filterfalse 去除 None)
cleaned = list(itertools.filterfalse(lambda x: x is None, flat))

print(cleaned) # [1, 2, 3, 4, 5, 6]
```

**关键代码说明：**

| 代码 | 含义 | 为什么这样写 |
|------|------|-------------|
| `itertools.chain.from_iterable(data)` | 将二维列表惰性展平为一维 | `chain.from_iterable` 不创建新列表，逐个 yield 元素，内存占用是 O(1) 而非 O(n) |
| `itertools.filterfalse(lambda x: x is None, flat)` | 过滤掉 None 元素 | `filterfalse` 保留使函数返回 `False` 的元素，即保留非 None 的值；比 `filter(lambda x: x is not None, ...)` 更语义化 |
| `flat = chain.from_iterable(data)` 再传给 `filterfalse` | 管道式串联 | `flat` 是惰性迭代器，`filterfalse` 继续惰性消费它，整条管道直到 `list()` 才真正执行 |
