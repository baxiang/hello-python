# 数据容器进阶 (collections)

> **本章基于 Python 3.11+**
>
> 原生的 `list`, `dict`, `tuple` 虽然好用，但在某些特定场景下（如计数、去重、复杂配置合并），原生数据结构会显得笨重且低效。
>
> `collections` 模块提供了高性能的替代方案，底层由 C 语言实现，是进阶 Python 开发的必修课。

## 1. Counter: 计数器

**场景：** 统计列表中元素出现的频率，或者找出出现次数最多的 Top N 元素。

### 原生写法（笨重）
```python
words = ["apple", "banana", "apple", "orange", "banana", "apple"]
counts = {}
for w in words:
    counts[w] = counts.get(w, 0) + 1

print(counts)
# 找出最多的前 2 个需要排序，代码复杂...
```

### Counter 写法（优雅）
```python
from collections import Counter

words = ["apple", "banana", "apple", "orange", "banana", "apple"]
c = Counter(words)

print(c)  # Counter({'apple': 3, 'banana': 2, 'orange': 1})

# 获取出现次数最多的 2 个
top_2 = c.most_common(2)
print(top_2)  # [('apple', 3), ('banana', 2)]
```

### 进阶用法：数学运算
```python
c1 = Counter(a=3, b=1)
c2 = Counter(a=1, b=2)

print(c1 + c2)   # Counter({'a': 4, 'b': 3}) - 相加
print(c1 - c2)   # Counter({'a': 2})      - 相减（只保留正数）
```

## 2. defaultdict: 带默认值的字典

**场景：** 当字典的 value 是列表或字典时，初始化通常很繁琐（`if key not in dict: dict[key] = []`）。

### 痛点
```python
users = [("Alice", "HR"), ("Bob", "IT"), ("Alice", "IT")]
dept_map = {}
for name, dept in users:
    if name not in dept_map:
        dept_map[name] = []
    dept_map[name].append(dept)
```

### defaultdict 解法
```python
from collections import defaultdict

# int 默认返回 0
# list 默认返回 []
# dict 默认返回 {}

dept_map = defaultdict(list)

for name, dept in users:
    dept_map[name].append(dept)  # 自动处理初始化！

print(dict(dept_map))
# {'Alice': ['HR', 'IT'], 'Bob': ['IT']}
```

## 3. namedtuple: 轻量级数据模型

**场景：** 定义一个只有属性没有方法的简单对象（如坐标点、数据库行）。
虽然 Python 3.7+ 有了 `dataclasses`，但 `namedtuple` 依然因其**不可变性**和**内存占用极小**而被广泛使用。

### 原生 Tuple 的问题
```python
user = (1, "Alice", 20)
print(user[0])  # ID 是 1？还是索引 0？可读性差
```

### namedtuple 解法
```python
from collections import namedtuple

# 定义类型
User = namedtuple('User', ['id', 'name', 'age'])

# 创建实例
u = User(id=1, name="Alice", age=20)

# 访问属性
print(u.name)   # Alice
print(u.age)    # 20

# 也可以像普通 tuple 一样索引
print(u[1])     # Alice
```

## 4. deque: 双向队列

**场景：** 需要频繁在序列的**头部**或**尾部**进行添加/删除操作。

### 为什么不用 list？
`list` 是基于数组实现的，在头部插入/删除 (`pop(0)`, `insert(0, x)`) 需要移动整个数组，时间复杂度为 **O(n)**。
`deque` 是基于双向链表实现的，两端操作均为 **O(1)**。

### 使用示例
```python
from collections import deque

d = deque([1, 2, 3])

# 尾部操作（同 list）
d.append(4)      # [1, 2, 3, 4]
d.pop()          # [1, 2, 3]

# 头部操作（高效！）
d.appendleft(0)  # [0, 1, 2, 3]
d.popleft()      # [1, 2, 3]

# 固定长度缓存 (maxlen)
# 当队列满时，新元素加入会自动挤出最老的元素
cache = deque(maxlen=3)
cache.append(1)
cache.append(2)
cache.append(3)
cache.append(4)  # 1 被挤出去了

print(cache)     # deque([2, 3, 4], maxlen=3)
```

## 5. ChainMap: 配置合并

**场景：** 处理多层级的配置（如：默认配置 -> 环境配置 -> 用户自定义配置），查找时优先查找自定义配置。

```python
from collections import ChainMap

defaults = {'theme': 'light', 'font': 'Arial'}
env_config = {'theme': 'dark'}
user_config = {'font': 'Times New Roman'}

# 优先级：user > env > defaults
config = ChainMap(user_config, env_config, defaults)

print(config['theme'])  # 'dark' (来自 env_config，user 没设)
print(config['font'])   # 'Times New Roman' (来自 user_config)
print(config['debug'])  # KeyError (都没设)

# 动态修改会作用到第一个映射中
config['debug'] = True
print(user_config)      # {'font': 'Times New Roman', 'debug': True}
```
