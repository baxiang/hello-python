# 数据结构与算法 (heapq, bisect)

> **本章基于 Python 3.11+**
>
> Python 的基础数据结构是列表，但在处理**优先级队列**、**快速查找**等算法场景时，原生列表的效率较低（O(n)）。
>
> 本章节介绍两个底层基于 C 语言实现的算法模块：`heapq`（堆）和 `bisect`（二分法），它们能将相关操作的复杂度降低到 **O(log n)**。

---

## 1. heapq: 堆队列算法

**场景**：需要从一组数据中快速获取**最大/最小值**，或者维护一个 Top N 列表。

### 为什么不用 `list.sort()`？
*   列表排序时间复杂度为 **O(n log n)**。
*   堆的操作（插入、弹出）复杂度仅为 **O(log n)**，非常适合数据频繁变动且只需关注极值的场景。

### 1.1 核心概念：小顶堆
`heapq` 默认提供**小顶堆**（Min-Heap）：列表的第一个元素 `list[0]` 永远是最小的。

```python
import heapq

data = [5, 1, 9, 3, 7]

# 将列表转换为堆 (O(n) 复杂度)
heapq.heapify(data)

print(data[0])  # 1 (最小值，无需完整排序)
```

### 1.2 基本操作：heappush 与 heappop

```python
import heapq

heap = []

# 插入元素 (O(log n))
heapq.heappush(heap, 5)
heapq.heappush(heap, 1)
heapq.heappush(heap, 10)

print(heap)  # [1, 5, 10] (虽然看起来有序，但内部结构是堆，不是完全有序列表)

# 弹出最小值 (O(log n))
smallest = heapq.heappop(heap)
print(smallest)  # 1
print(heap)      # [5, 10]
```

### 1.3 实战：Top K 问题

使用 `nlargest` 和 `nsmallest`，它们比 `sorted(data)[:k]` 更高效。

```python
import heapq

nums = [12, 45, 2, 89, 33, 5]

# 最大的 3 个数
print(heapq.nlargest(3, nums))  # [89, 45, 33]

# 最小的 3 个数
print(heapq.nsmallest(3, nums)) # [2, 5, 12]

# 高级用法：带 key 比较 (如获取最贵的 2 个商品)
products = [{"name": "A", "price": 10}, {"name": "B", "price": 50}, {"name": "C", "price": 30}]
top_expensive = heapq.nlargest(2, products, key=lambda p: p["price"])
print(top_expensive) 
```

## 2. bisect: 二分查找算法

**场景**：在**已排序**的列表中查找元素的插入位置，保持列表有序。

> **前提**：列表必须是有序的！否则结果无意义。

### 2.1 查找插入位置
如果你想在列表中插入一个元素，但不知道插在哪里才能保持有序。

```python
import bisect

scores = [10, 20, 30, 40]

# 找到插入点
pos = bisect.bisect(scores, 25)
print(pos) # 2 (插入到 20 和 30 之间)

# 插入并保持有序
bisect.insort(scores, 25)
print(scores) # [10, 20, 25, 30]
```

### 2.2 实战：根据分数定级
利用 `bisect_right` 配合索引查找，实现高效的区间映射。

```python
import bisect

def grade(score):
    breakpoints = [60, 70, 80, 90]
    grades = 'FDCBA'
    i = bisect.bisect_right(breakpoints, score)
    return grades[i]

print(grade(55)) # F
print(grade(65)) # D
print(grade(95)) # A
```

## 3. 性能对比实验

我们来对比一下原生列表和这些模块的性能差异。

### 场景：查找列表中第 3 小的元素

| 方法 | 代码示例 | 复杂度 | 说明 |
|------|----------|--------|------|
| **排序法** | `sorted(data)[2]` | O(n log n) | 对所有数据排序，浪费计算 |
| **heapq 法** | `heapq.nsmallest(3, data)[-1]` | O(n log k) | **推荐**，当 k 远小于 n 时极快 |

### 场景：在有序列表中插入 10 万个元素

| 方法 | 代码示例 | 复杂度 | 说明 |
|------|----------|--------|------|
| **暴力插入** | `data.append(x); data.sort()` | O(n * n log n) | 慢到无法接受 |
| **Bisect 插入**| `bisect.insort(data, x)` | O(n) | 虽然仍是 O(n) 因为列表移动，但比全排序快得多 |

## 4. 综合实战：任务调度系统

模拟一个简单的优先级任务队列。优先级越高的任务（数字越小），越先执行。

```python
import heapq
from dataclasses import dataclass, field

@dataclass(order=True)
class Task:
    priority: int
    name: str = field(compare=False) # name 不参与比较
    data: str = field(compare=False)

queue = []

# 1. 添加任务 (随机优先级)
tasks = [
    Task(2, "发邮件", "Hello"),
    Task(5, "备份数据库", "DB Backup"),
    Task(1, "紧急修复", "Bug Fix"),
    Task(3, "生成报表", "Report"),
]

for t in tasks:
    heapq.heappush(queue, t)

# 2. 执行任务
while queue:
    task = heapq.heappop(queue)
    print(f"执行任务: {task.name} (优先级: {task.priority})")

# 输出顺序：
# 1. 紧急修复 (1)
# 2. 发邮件 (2)
# 3. 生成报表 (3)
# 4. 备份数据库 (5)
```
