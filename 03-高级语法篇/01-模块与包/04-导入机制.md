# 导入机制（详细版）

> Python 3.11+

## 第一部分：导入路径

### 4.1 绝对导入与相对导入

#### 实际场景

你的项目结构如下：

```
myproject/
├── __init__.py
├── core/
│   ├── __init__.py
│   ├── config.py
│   └── database.py
├── services/
│   ├── __init__.py
│   └── user_service.py
└── utils/
    ├── __init__.py
    └── helpers.py
```

在 `services/user_service.py` 里，你想导入 `core/config.py` 里的配置。应该怎么写导入语句？

---

## 绝对导入

### 概念说明

绝对导入使用完整的模块路径，从项目根目录开始。

```python
# myproject/core/config.py
from __future__ import annotations
from typing import Final
from pathlib import Path

class Config:
    """应用配置"""
    DEBUG: Final[bool] = True
    DATABASE_URL: Final[str] = "postgresql://localhost/mydb"
    BASE_DIR: Final[Path] = Path(__file__).parent.parent
    
    @classmethod
    def get_database_path(cls) -> Path:
        """获取数据库文件路径"""
        return cls.BASE_DIR / "data" / "app.db"


# 在 models/user.py 中导入 core/config.py
from myproject.core.config import Config
from myproject.core import config
import myproject.core.config
```

### 优点与缺点

```
┌─────────────────────────────────────────────────────────────┐
│              绝对导入的优缺点                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   优点：                                                     │
│   ✓ 路径清晰，易于理解                                      │
│   ✓ 不受当前模块位置影响                                    │
│   ✓ IDE 支持良好（跳转、重构）                              │
│                                                             │
│   缺点：                                                     │
│   ✗ 包名变更时需要修改多处                                  │
│   ✗ 深层嵌套时路径较长                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 相对导入

### 概念说明

相对导入使用点号表示相对位置，只能在包内部使用。

```
┌─────────────────────────────────────────────────────────────┐
│                  相对导入符号含义                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   .   表示当前目录（当前包）                                │
│   ..  表示上一级目录（父包）                                │
│   ... 表示上两级目录（祖父包）                              │
│                                                             │
│   示例：                                                    │
│   from . import module          # 导入同级模块              │
│   from .module import func      # 从同级模块导入函数        │
│   from .. import parent_module  # 导入父级模块              │
│   from ..parent import func     # 从父级模块导入函数        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**项目结构示例：**

```
myproject/
├── __init__.py
├── core/
│   ├── __init__.py
│   ├── config.py
│   └── exceptions.py
└── models/
    ├── __init__.py
    ├── user.py
    └── product.py
```

**相对导入示例：**

```python
# 在 core/exceptions.py 中导入 config.py（同级模块）
from .config import Config

# 在 models/user.py 中导入 core/config.py（跨包导入）
from ..core.config import Config

# 在 models/__init__.py 中导入子模块
from .user import User
from .product import Product
```

### 相对导入符号

| 符号 | 含义 | 示例 |
|------|------|------|
| `.` | 当前包 | `from .module import func` |
| `..` | 上级包 | `from ..module import func` |
| `...` | 上两级包 | `from ...module import func` |

### 重要限制

```
┌─────────────────────────────────────────────────────────────┐
│              相对导入的重要限制                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ⚠️ 相对导入只能在包内部使用                               │
│                                                             │
│   ⚠️ 不能在顶层脚本（直接运行的 .py 文件）中使用            │
│                                                             │
│   ⚠️ 直接运行包含相对导入的模块会报错                       │
│                                                             │
│   错误示例：                                                │
│   # mypackage/module.py                                     │
│   from . import other_module  # 相对导入                    │
│                                                             │
│   # 直接运行                                                │
│   python mypackage/module.py                                │
│   # ImportError: attempted relative import with no known    │
│   # parent package                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 优点与缺点

```
┌─────────────────────────────────────────────────────────────┐
│              相对导入的优缺点                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   优点：                                                     │
│   ✓ 包名变更时无需修改                                      │
│   ✓ 路径简洁                                                │
│                                                             │
│   缺点：                                                     │
│   ✗ 只能在包内部使用                                        │
│   ✗ 直接运行模块会报错                                      │
│   ✗ IDE 支持较差                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 导入陷阱

### 陷阱 1：相对导入与直接运行

```python
# myproject/models/user.py
from ..core.config import Config  # 相对导入

# 直接运行会报错
# python myproject/models/user.py
# ImportError: attempted relative import with no known parent package
```

**解决方案：**

```bash
# 方式 1：使用 -m 运行
python -m myproject.models.user

# 方式 2：在项目根目录运行
cd /path/to/project
python -c "from myproject.models.user import User"
```

### 陷阱 2：循环导入

```python
# myproject/models/user.py
from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from myproject.models.product import Product


class User:
    """用户模型"""
    
    def __init__(self, user_id: int, name: str) -> None:
        self.user_id = user_id
        self.name = name
    
    def get_products(self) -> list[Product]:
        """获取用户的产品列表"""
        from myproject.models.product import Product
        return []


# myproject/models/product.py
from __future__ import annotations


class Product:
    """产品模型"""
    
    def __init__(self, product_id: int, name: str) -> None:
        self.product_id = product_id
        self.name = name
```

**解决方案：**

```python
from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from myproject.models.product import Product


# 方案 1：延迟导入
class User:
    def get_product(self) -> Product:
        from myproject.models.product import Product
        return Product()


# 方案 2：重构代码
# models/__init__.py
from .user import User
from .product import Product


# 方案 3：使用 TYPE_CHECKING（推荐用于类型提示）
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from models.product import Product


class User:
    def get_products(self) -> list[Product]:
        from models.product import Product
        return [Product()]
```

### 陷阱 3：模块名与标准库冲突

```python
# 创建了 email.py
# email.py
def send_email():
    pass

# main.py
import email  # 导入的是标准库 email，而不是自定义模块！
email.send_email()  # AttributeError
```

**解决方案：**

```python
# 重命名模块，避免与标准库冲突
# my_email.py
def send_email():
    pass

# 或使用别名
import email as my_email_module
```

### 陷阱 4：导入顺序问题

```python
# models/__init__.py
from .user import User
from .product import Product

# models/user.py
from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from models.product import Product


class User:
    def get_products(self) -> list[Product]:
        from models.product import Product
        return []
```

**解决方案：**

```python
# models/user.py
from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from models.product import Product


class User:
    def get_products(self) -> list[Product]:
        from models.product import Product
        return []
```

## 导入的最佳实践

### 推荐做法

```python
from __future__ import annotations
from typing import TYPE_CHECKING, Final
from pathlib import Path

# 1. 优先使用绝对导入
from myproject.core.config import Config

# 2. 包内部可以使用相对导入
from .config import Config

# 3. 导入顺序
# 标准库
import os
import sys
from typing import List, Dict, Final

# 第三方库
import numpy as np
import pandas as pd

# 本地模块
from myproject.core import config
from myproject.models import User

# 4. 避免循环导入
# 使用延迟导入或重构代码结构

# 5. 使用 __all__ 控制导出
__all__: Final[list[str]] = ["User", "Product"]

# 6. 使用 TYPE_CHECKING 进行类型提示
if TYPE_CHECKING:
    from myproject.models.product import Product


def process(user: User) -> list[Product]:
    from myproject.models.product import Product
    return []
```
