# 导入机制

本章讲解 Python 的导入机制，包括绝对导入、相对导入以及常见的导入陷阱。

---

## 绝对导入

### 概念说明

绝对导入使用完整的模块路径，从项目根目录开始。

```python
# 项目结构
myproject/
├── __init__.py
├── core/
│   ├── __init__.py
│   └── config.py
└── models/
    ├── __init__.py
    └── user.py

# 在 models/user.py 中导入 core/config.py
from myproject.core.config import Config
from myproject.core import config
import myproject.core.config
```

### 优点与缺点

```
┌─────────────────────────────────────────────────────────────┐
│              绝对导入的优缺点                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   优点：                                                     │
│   ✓ 路径清晰，易于理解                                      │
│   ✓ 不受当前模块位置影响                                    │
│   ✓ IDE 支持良好（跳转、重构）                              │
│                                                             │
│   缺点：                                                     │
│   ✗ 包名变更时需要修改多处                                  │
│   ✗ 深层嵌套时路径较长                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 相对导入

### 概念说明

相对导入使用点号表示相对位置，只能在包内部使用。

```
┌─────────────────────────────────────────────────────────────┐
│                  相对导入符号含义                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   .   表示当前目录（当前包）                                │
│   ..  表示上一级目录（父包）                                │
│   ... 表示上两级目录（祖父包）                              │
│                                                             │
│   示例：                                                    │
│   from . import module          # 导入同级模块              │
│   from .module import func      # 从同级模块导入函数        │
│   from .. import parent_module  # 导入父级模块              │
│   from ..parent import func     # 从父级模块导入函数        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**项目结构示例：**

```
myproject/
├── __init__.py
├── core/
│   ├── __init__.py
│   ├── config.py
│   └── exceptions.py
└── models/
    ├── __init__.py
    ├── user.py
    └── product.py
```

**相对导入示例：**

```python
# 在 core/exceptions.py 中导入 config.py（同级模块）
from .config import Config

# 在 models/user.py 中导入 core/config.py（跨包导入）
from ..core.config import Config

# 在 models/__init__.py 中导入子模块
from .user import User
from .product import Product
```

### 相对导入符号

| 符号 | 含义 | 示例 |
|------|------|------|
| `.` | 当前包 | `from .module import func` |
| `..` | 上级包 | `from ..module import func` |
| `...` | 上两级包 | `from ...module import func` |

### 重要限制

```
┌─────────────────────────────────────────────────────────────┐
│              相对导入的重要限制                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ⚠️ 相对导入只能在包内部使用                               │
│                                                             │
│   ⚠️ 不能在顶层脚本（直接运行的 .py 文件）中使用            │
│                                                             │
│   ⚠️ 直接运行包含相对导入的模块会报错                       │
│                                                             │
│   错误示例：                                                │
│   # mypackage/module.py                                     │
│   from . import other_module  # 相对导入                    │
│                                                             │
│   # 直接运行                                                │
│   python mypackage/module.py                                │
│   # ImportError: attempted relative import with no known    │
│   # parent package                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 优点与缺点

```
┌─────────────────────────────────────────────────────────────┐
│              相对导入的优缺点                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   优点：                                                     │
│   ✓ 包名变更时无需修改                                      │
│   ✓ 路径简洁                                                │
│                                                             │
│   缺点：                                                     │
│   ✗ 只能在包内部使用                                        │
│   ✗ 直接运行模块会报错                                      │
│   ✗ IDE 支持较差                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 导入陷阱

### 陷阱 1：相对导入与直接运行

```python
# myproject/models/user.py
from ..core.config import Config  # 相对导入

# 直接运行会报错
# python myproject/models/user.py
# ImportError: attempted relative import with no known parent package
```

**解决方案：**

```bash
# 方式 1：使用 -m 运行
python -m myproject.models.user

# 方式 2：在项目根目录运行
cd /path/to/project
python -c "from myproject.models.user import User"
```

### 陷阱 2：循环导入

```python
# models/user.py
from models.product import Product

# models/product.py
from models.user import User

# 循环导入错误！
```

**解决方案：**

```python
# 方案 1：延迟导入
# models/user.py
def get_product():
    from models.product import Product
    return Product()

# 方案 2：重构代码
# models/__init__.py
from .user import User
from .product import Product

# 方案 3：使用 TYPE_CHECKING
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from models.product import Product

class User:
    def get_product(self) -> 'Product':
        from models.product import Product
        return Product()
```

### 陷阱 3：模块名与标准库冲突

```python
# 创建了 email.py
# email.py
def send_email():
    pass

# main.py
import email  # 导入的是标准库 email，而不是自定义模块！
email.send_email()  # AttributeError
```

**解决方案：**

```python
# 重命名模块，避免与标准库冲突
# my_email.py
def send_email():
    pass

# 或使用别名
import email as my_email_module
```

### 陷阱 4：导入顺序问题

```python
# models/__init__.py
from .user import User
from .product import Product

# models/user.py
from models import Product  # 错误！Product 还未导入

class User:
    def get_products(self) -> list['Product']:
        return []
```

**解决方案：**

```python
# models/user.py
from __future__ import annotations  # Python 3.7+

class User:
    def get_products(self) -> list['Product']:
        from models.product import Product
        return []
```

## 导入的最佳实践

### 推荐做法

```python
# 1. 优先使用绝对导入
from myproject.core.config import Config

# 2. 包内部可以使用相对导入
from .config import Config

# 3. 导入顺序
# 标准库
import os
import sys
from typing import List, Dict

# 第三方库
import numpy as np
import pandas as pd

# 本地模块
from myproject.core import config
from myproject.models import User

# 4. 避免循环导入
# 使用延迟导入或重构代码结构

# 5. 使用 __all__ 控制导出
__all__ = ['User', 'Product']
```
