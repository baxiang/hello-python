# 第 03 章 - 模块管理

> 良好的模块组织是构建可维护 Python 项目的关键。

---

## 第一部分 - Python 导入系统

### 1.1 导入机制

```python
# 导入整个模块
import math
result = math.sqrt(16)

# 导入特定成员
from math import sqrt, pi
result = sqrt(16)

# 导入并重命名
from math import sqrt as square_root
import numpy as np

# 相对导入（包内）
from . import sibling_module
from ..parent_package import module
from ...grandparent import module
```

### 1.2 导入搜索路径

```python
import sys

# 查看导入搜索路径
print(sys.path)

# 临时添加路径
sys.path.insert(0, '/path/to/module')

# 推荐：使用 PYTHONPATH 环境变量
# export PYTHONPATH=/path/to/module:$PYTHONPATH
```

### 1.3 __init__.py 的作用

```
my_package/
├── __init__.py      # 使目录成为包
├── module_a.py
├── module_b.py
└── subpackage/
    ├── __init__.py
    └── module_c.py
```

```python
# __init__.py 可以定义包的内容
from .module_a import public_function
from .module_b import AnotherClass

__all__ = ['public_function', 'AnotherClass']  # 控制 * 导入的内容
__version__ = '1.0.0'
```

### 3.3 避免的陷阱

```python
# ❌ 避免：循环导入
# module_a.py
from module_b import func_b
def func_a():
    pass

# module_b.py
from module_a import func_a  # 循环导入！
def func_b():
    pass

# ✅ 解决：重构代码结构或使用延迟导入
# module_a.py
def func_a():
    from module_b import func_b  # 延迟导入
    return func_b()
```

## 第四部分 - 包的分发

### 4.1 准备发布

```python
# src/my_package/__init__.py
__version__ = "1.0.0"
__author__ = "Your Name"
__email__ = "your.email@example.com"

from .core import main_function
from .utils import helper_function

__all__ = ["main_function", "helper_function"]
```

### 4.2 构建包

```bash
# 安装构建工具
uv add --dev build twine

# 构建分发包
python -m build

# 检查包
twine check dist/*

# 上传到 PyPI
twine upload dist/*

# 上传到 TestPyPI（测试）
twine upload --repository testpypi dist/*
```

### 4.3 发布到 PyPI

1. 注册 [PyPI](https://pypi.org/) 账号
2. 创建 API token
3. 配置 `~/.pypirc`

```ini
[distutils]
index-servers =
    pypi
    testpypi

[pypi]
username = __token__
password = pypi-xxx

[testpypi]
repository = https://test.pypi.org/legacy/
username = __token__
password = pypi-xxx
```

## 最佳实践

### 1. 使用 src 布局

```
✅ src/my_package/    # 推荐
❌ my_package/        # 避免
```

### 2. 保持模块简洁

```python
# ✅ 每个模块专注于单一职责
# database.py - 只处理数据库相关
# api.py - 只处理 API 相关

# ❌ 避免大杂烩
# utils.py - 包含所有东西
```

### 3. 明确导出内容

```python
# __init__.py
__all__ = ['PublicClass', 'public_function']

from .internal import PublicClass, public_function, _internal_helper  # _internal_helper 不会导出
```

### 4. 使用语义化版本

```toml
# pyproject.toml
version = "1.2.3"  # MAJOR.MINOR.PATCH
```
