# 第 01 章 - 包管理工具

> Python 包管理是工程化开发的基础，本章介绍主流包管理工具的使用。

---

## 第一部分 - pip 包管理器

### 1.1 pip 简介

pip 是 Python 官方推荐的包管理工具，用于安装、卸载和管理 Python 包。

#### 基本命令

```bash
# 安装包
pip install package_name

# 安装指定版本
pip install package_name==1.0.0

# 升级包
pip install --upgrade package_name

# 卸载包
pip uninstall package_name

# 列出已安装的包
pip list

# 查看包信息
pip show package_name

# 导出依赖
pip freeze > requirements.txt

# 从文件安装依赖
pip install -r requirements.txt
```

### 1.2 虚拟环境

使用虚拟环境可以隔离项目依赖，避免包冲突。

#### venv 使用

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境 (macOS/Linux)
source venv/bin/activate

# 激活虚拟环境 (Windows)
venv\Scripts\activate

# 退出虚拟环境
deactivate
```

## 第二部分 - uv 包管理器

### 2.1 uv 简介

uv 是由 Astral 开发的新一代 Python 包管理器，用 Rust 编写，速度极快。

#### 安装 uv

```bash
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 使用 pip 安装
pip install uv
```

### 2.2 uv 基本使用

```bash
# 初始化新项目
uv init my-project
cd my-project

# 添加依赖
uv add requests
uv add flask@2.0.0

# 添加开发依赖
uv add --dev pytest black ruff

# 运行脚本
uv run python main.py

# 运行已安装的命令行工具
uv run pytest

# 创建虚拟环境
uv venv

# 激活虚拟环境
source .venv/bin/activate  # macOS/Linux
.venv\Scripts\activate     # Windows
```

### 2.3 uv 项目结构

```
my-project/
├── pyproject.toml    # 项目配置和依赖
├── uv.lock          # 锁定文件
├── .venv/           # 虚拟环境
└── src/
    └── my_project/
        └── __init__.py
```

### 2.4 pyproject.toml 示例

```toml
[project]
name = "my-project"
version = "0.1.0"
description = "My Python Project"
requires-python = ">=3.11"
dependencies = [
    "requests>=2.31.0",
    "flask>=3.0.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.4.0",
    "black>=23.0.0",
    "ruff>=0.1.0",
]

[tool.uv]
dev-dependencies = [
    "pytest>=7.4.0",
    "black>=23.0.0",
    "ruff>=0.1.0",
]
```

## 第三部分 - conda 包管理器

### 3.1 conda 简介

conda 是跨平台的包管理和环境管理工具，特别适合数据科学和机器学习项目。

#### 安装 conda

推荐安装 [Miniconda](https://docs.conda.io/en/latest/miniconda.html) 或 [Anaconda](https://www.anaconda.com/)。

### 3.2 conda 基本使用

```bash
# 创建环境
conda create -n myenv python=3.11

# 激活环境
conda activate myenv

# 退出环境
conda deactivate

# 安装包
conda install package_name

# 从 conda-forge 安装
conda install -c conda-forge package_name

# 列出环境
conda env list

# 导出环境
conda env export > environment.yml

# 从文件创建环境
conda env create -f environment.yml

# 删除环境
conda env remove -n myenv
```

### 3.3 environment.yml 示例

```yaml
name: myenv
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.11
  - numpy>=1.24
  - pandas>=2.0
  - pip
  - pip:
    - requests>=2.31.0
```

## 第四部分 - 包管理器对比

| 特性 | pip | uv | conda |
|------|-----|----|-------|
| 速度 | 慢 | 极快 | 中等 |
| 语言 | Python | Rust | Python |
| 虚拟环境 | 需配合 venv | 内置 | 内置 |
| 非 Python 包 | ❌ | ❌ | ✅ |
| 锁定文件 | 需额外工具 | ✅ uv.lock | ✅ environment.yml |
| 适用场景 | 通用 | 现代 Python 项目 | 数据科学/ML |

## 最佳实践

### 1. 始终使用虚拟环境

```bash
# 使用 uv (推荐)
uv init my-project
cd my-project
uv add requests

# 或使用 venv
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. 锁定依赖版本

```toml
# pyproject.toml
dependencies = [
    "requests>=2.31.0,<3.0.0",  # 指定版本范围
]
```

### 3. 分离生产和开发依赖

```toml
# 生产依赖
dependencies = ["flask>=3.0.0"]

# 开发依赖
[tool.uv.dev-dependencies]
dev = ["pytest>=7.4.0", "black>=23.0.0"]
```

### 4. 使用锁定文件

```bash
# uv 自动生成 uv.lock，确保可重复构建
git add uv.lock
```
