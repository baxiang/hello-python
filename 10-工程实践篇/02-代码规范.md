# 第 02 章 - 代码规范

> 良好的代码规范是团队协作和项目可维护性的基础。

---

## 第一部分 - PEP 8 风格指南

### 1.1 代码布局

#### 缩进

```python
# ✅ 使用 4 个空格缩进
def hello():
    print("Hello")
    if True:
        print("World")

# ❌ 避免使用 Tab
def hello():
	print("Hello")  # Tab 缩进
```

#### 行长度

```python
# ✅ 每行不超过 79 字符（代码）或 72 字符（文档）
result = (
    some_function(arg1, arg2, arg3)
    + another_function(arg4, arg5)
)

# ❌ 避免过长的行
result = some_function(arg1, arg2, arg3) + another_function(arg4, arg5) + yet_another_function(arg6)
```

#### 空行

```python
# ✅ 顶层函数/类之间空两行
def function_one():
    pass


def function_two():
    pass


class MyClass:
    pass


def function_three():
    pass

# ✅ 方法之间空一行
class MyClass:
    def method_one(self):
        pass
    
    def method_two(self):
        pass
```

### 1.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量/函数 | snake_case | `user_name`, `calculate_total()` |
| 类名 | PascalCase | `UserProfile`, `DataProcessor` |
| 常量 | UPPER_CASE | `MAX_SIZE`, `API_KEY` |
| 私有变量 | _prefix | `_internal_value` |
| 私有方法 | _prefix | `_internal_method()` |

```python
# ✅ 好的命名
user_name = "Alice"
MAX_RETRIES = 3

def calculate_total():
    pass

class DataProcessor:
    pass

# ❌ 避免的命名
userName = "Alice"  # camelCase
UserName = "Alice"  # PascalCase for variable
max_retries = 3     # snake_case for constant
```

### 1.3 导入规范

```python
# ✅ 导入顺序：标准库 → 第三方库 → 本地模块
import os
import sys
from pathlib import Path

import requests
from flask import Flask

from .utils import helper
from .models import User

# ✅ 每个导入一行
import os
import sys

# ❌ 避免在一行导入多个模块
import os, sys

# ✅ 避免使用通配符导入
from module import specific_function  # ✅
from module import *                 # ❌
```

## 第二部分 - 类型提示

### 2.1 基本类型注解

```python
from typing import List, Dict, Optional, Union

# 基本类型
def greet(name: str) -> str:
    return f"Hello, {name}"

def add(a: int, b: int) -> int:
    return a + b

# 容器类型
def process_items(items: List[str]) -> Dict[str, int]:
    return {item: len(item) for item in items}

# 可选类型
def find_user(user_id: int) -> Optional[dict]:
    # 可能返回 None
    pass

# 联合类型
def parse_value(value: Union[str, int]) -> str:
    return str(value)

# Python 3.10+ 新语法
def parse_value_new(value: str | int) -> str:
    return str(value)
```

### 2.2 复杂类型注解

```python
from typing import Callable, Tuple, Any, TypeVar

# 函数类型
def apply(func: Callable[[int, int], int], x: int, y: int) -> int:
    return func(x, y)

# 元组类型
def get_coords() -> Tuple[float, float]:
    return (10.5, 20.3)

# 任意类型（尽量避免）
def process(data: Any) -> Any:
    return data

# 类型变量
T = TypeVar('T')

def first_item(items: list[T]) -> T:
    return items[0]
```

### 2.3 类类型注解

```python
from typing import Self

class Person:
    def __init__(self, name: str, age: int) -> None:
        self.name = name
        self.age = age
    
    def greet(self) -> str:
        return f"Hi, I'm {self.name}"
    
    # Python 3.11+ Self 类型
    def with_age(self, age: int) -> Self:
        self.age = age
        return self

# 前向引用
class TreeNode:
    def __init__(self, left: 'TreeNode | None' = None) -> None:
        self.left = left
```

## 第三部分 - 代码格式化工具

### 3.1 Black 格式化器

```bash
# 安装
uv add --dev black

# 格式化单个文件
black file.py

# 格式化整个项目
black .

# 检查格式（不修改）
black --check .

# 指定行长度
black --line-length 88 .
```

#### Black 格式化前后对比

```python
# 格式化前
def my_function( a,b,c ):
    x=[1,2,3]
    return a+b+c

# 格式化后
def my_function(a, b, c):
    x = [1, 2, 3]
    return a + b + c
```

### 3.2 Ruff Linter

```bash
# 安装
uv add --dev ruff

# 检查代码
ruff check .

# 自动修复
ruff check --fix .

# 格式化（替代 Black）
ruff format .
```

#### pyproject.toml 配置

```toml
[tool.ruff]
line-length = 88
target-version = "py311"

[tool.ruff.lint]
select = [
    "E",   # pycodestyle errors
    "W",   # pycodestyle warnings
    "F",   # Pyflakes
    "I",   # isort
    "B",   # flake8-bugbear
    "C4",  # flake8-comprehensions
]
ignore = [
    "E501",  # line too long (由 Black 处理)
]

[tool.ruff.lint.isort]
known-first-party = ["my_package"]
```

### 3.3 isort 导入排序

```bash
# 安装
uv add --dev isort

# 排序导入
isort file.py
isort .
```

## 第五部分 - 预提交钩子

### 5.1 pre-commit 配置

```bash
# 安装
uv add --dev pre-commit

# 初始化
pre-commit install
```

### 5.2 .pre-commit-config.yaml

```yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files

  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.1.6
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

  - repo: https://github.com/psf/black
    rev: 23.11.0
    hooks:
      - id: black
        language_version: python3.11
```

## 最佳实践清单

### ✅ 必须遵守

1. 遵循 PEP 8 风格指南
2. 使用类型提示
3. 编写文档字符串
4. 使用自动格式化工具

### ✅ 推荐做法

1. 使用 pre-commit 钩子
2. 在 CI 中运行代码检查
3. 保持函数简短（< 50 行）
4. 避免过深的嵌套（< 4 层）
