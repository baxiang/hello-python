## RNN 核心概念

### 为什么需要 RNN？

**传统神经网络的问题：**

```
问题：处理序列数据

输入："我爱你" → 情感分析

传统方法：
把"我"、"爱"、"你"分别输入网络
❌ 丢失了词之间的顺序和关系！
```

**RNN 的解决方案：**

```
RNN 处理序列：
t=1: 输入"我" + 初始状态 → 隐藏状态1
t=2: 输入"爱" + 隐藏状态1 → 隐藏状态2
t=3: 输入"你" + 隐藏状态2 → 隐藏状态3

✅ 保留了词之间的顺序关系！
```

### RNN 结构示意

```
时间步 t=1      t=2      t=3
    ↓          ↓         ↓
   输入        输入      输入
    │          │         │
    ▼          ▼         ▼
  ┌────┐    ┌────┐    ┌────┐
  │ RNN│───→│ RNN│───→│ RNN│
  └────┘    └────┘    └────┘
    │          │         │
    ▼          ▼         ▼
   输出        输出      输出

核心思想：当前时刻的输出依赖于当前输入和前一时刻的状态
```

### LSTM 解决长序列问题

**普通 RNN 的问题：**

```
句子："我今天早上吃了一个非常非常...美味的苹果"
                                    ↑
                                 很远才出现
❌ 普通RNN会忘记开头的"我吃"
```

**LSTM 的解决方案：**

```
LSTM = 长短期记忆网络
- 遗忘门：决定忘记哪些信息
- 输入门：决定记住哪些新信息
- 输出门：决定输出哪些信息

✅ 可以记住长距离的依赖关系！
```

## 实战示例

**文本分类任务：**

```python
import torch.nn as nn

class TextRNN(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.rnn = nn.LSTM(embed_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, 2)  # 二分类
    
    def forward(self, x):
        embedded = self.embedding(x)         # [batch, seq_len, embed_dim]
        _, (hidden, _) = self.rnn(embedded)  # hidden: [1, batch, hidden_dim]
        output = self.fc(hidden.squeeze(0))  # [batch, 2]
        return output
```
