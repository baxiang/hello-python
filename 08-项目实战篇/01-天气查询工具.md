# 项目一：天气查询工具

本项目通过调用 OpenWeatherMap 的免费天气 API，构建一个命令行天气查询工具。你将学会如何发送 HTTP 请求、解析 JSON 数据、设计类结构，以及分层处理各类异常。

---

## 第一部分：项目概述

### 1.1 项目背景与目标

日常生活中，我们经常需要查询某个城市的天气。浏览器可以做到，但作为程序员，我们更希望能用代码实现同样的功能——这就是 API（应用程序接口）的价值所在。

**API 是什么？** 你可以把 API 想象成饭店的"点菜窗口"：你告诉服务员（API）你想要什么（城市名称），服务员把你的需求传给厨房（天气数据服务器），厨房做好后把菜（JSON 数据）端出来给你。你不需要知道厨房内部怎么工作，只需要按照菜单的格式下单就好。

本项目目标：

- 理解 HTTP 请求的工作原理
- 学会使用 `requests` 第三方库
- 掌握 JSON 数据的解析方法
- 学习分层异常处理的思路
- 体验从零到完整可用程序的开发过程

### 1.2 功能清单

| 功能 | 描述 |
|------|------|
| 按城市名查询 | 输入中英文城市名，返回当前天气 |
| 按经纬度查询 | 输入纬度和经度，返回该位置天气 |
| 格式化展示 | 温度、体感温度、湿度、风速、天气描述 |
| 日出日落时间 | 将 Unix 时间戳转换为本地可读时间 |
| 交互式循环 | 持续查询，输入 quit/exit/q 退出 |
| 错误友好提示 | 网络错误、城市不存在等均有清晰提示 |

## 第三部分：核心概念讲解

### 3.1 requests 库基础

#### 概念说明

`requests` 是 Python 最流行的 HTTP 请求库。HTTP（超文本传输协议）是浏览器和服务器之间通信的语言。当你在浏览器地址栏输入一个网址，浏览器就向服务器发送了一个 **GET 请求**，服务器把页面内容返回给浏览器。

`requests.get()` 做的事情完全相同——只不过不是浏览器在发请求，而是你的 Python 程序在发。

**类比：** 就像打电话点外卖。你拨出电话（发送请求），告诉店家你要什么（URL + 参数），店家做好后把东西送到你手上（返回响应）。`response.status_code` 就像外卖的配送状态：200 表示"已送达"，404 表示"找不到"，401 表示"没权限（没付钱）"。

#### 示例代码

```python
import requests

# 最简单的 GET 请求
# requests.get() 接收一个 URL，返回一个 Response 对象
response = requests.get("https://httpbin.org/get")

# 查看响应状态码：200 = 成功，404 = 找不到，500 = 服务器错误
print(response.status_code)  # 200

# 查看响应正文（文本形式）
print(response.text)

# 带参数的 GET 请求
# params 字典会自动拼接到 URL 后面，变成 ?key1=value1&key2=value2
params = {
    "q": "Beijing",       # 城市名
    "appid": "你的API Key",
    "units": "metric",    # 摄氏度（imperial=华氏度，standard=开尔文）
    "lang": "zh_cn",      # 返回中文天气描述
}
url = "https://api.openweathermap.org/data/2.5/weather"
response = requests.get(url, params=params)

# 实际请求的完整 URL（供调试参考）
print(response.url)
# 输出类似：https://api.openweathermap.org/data/2.5/weather?q=Beijing&appid=...

# 设置超时时间很重要！
# 如果不设置，网络故障时程序会永远卡住
response = requests.get(url, params=params, timeout=10)  # 10秒超时
```

### 3.2 JSON 解析

#### 概念说明

API 返回的数据通常是 **JSON 格式**（JavaScript Object Notation，JavaScript 对象表示法）。JSON 是一种文本格式，长得和 Python 字典很像，但本质上只是一串文字。

`response.json()` 这个方法会把那串 JSON 文字**解析**成 Python 字典，这样你就可以用 `data["key"]` 的方式取出里面的值了。

**类比：** 想象 API 返回了一封写在纸上的信（JSON 字符串），`response.json()` 就是帮你把信读出来，整理成一张表格（Python 字典），让你随时查阅。

#### 示例代码

```python
import requests

url = "https://api.openweathermap.org/data/2.5/weather"
params = {"q": "Beijing", "appid": "你的API Key", "units": "metric", "lang": "zh_cn"}
response = requests.get(url, params=params, timeout=10)

# .json() 将响应体（JSON 字符串）转换为 Python 字典
data = response.json()

# OpenWeatherMap 返回的 JSON 结构（简化版）：
# {
#   "name": "Beijing",
#   "main": {
#       "temp": 12.5,          ← 当前温度
#       "feels_like": 10.2,    ← 体感温度
#       "humidity": 45,        ← 湿度（百分比）
#   },
#   "weather": [               ← 注意这是一个列表！
#       {
#           "description": "晴",
#           "icon": "01d"
#       }
#   ],
#   "wind": {
#       "speed": 3.5           ← 风速（米/秒）
#   },
#   "sys": {
#       "sunrise": 1704920000, ← 日出时间（Unix 时间戳）
#       "sunset":  1704958000, ← 日落时间（Unix 时间戳）
#   }
# }

# 取出各字段的方法：
city_name = data["name"]                            # 城市名："Beijing"
temperature = data["main"]["temp"]                  # 嵌套字典：12.5
description = data["weather"][0]["description"]     # 列表第一项再取字段："晴"
humidity = data["main"]["humidity"]                 # 湿度：45
wind_speed = data["wind"]["speed"]                  # 风速：3.5

print(f"城市：{city_name}")
print(f"温度：{temperature}°C，天气：{description}")
print(f"湿度：{humidity}%，风速：{wind_speed} 米/秒")

# 安全取值：用 .get() 防止字段不存在时报 KeyError
feels_like = data["main"].get("feels_like", "N/A")  # 若没有该字段则返回 "N/A"
```

### 3.3 异常处理分层

#### 概念说明

网络程序比本地程序更容易出错，而且出错的原因有很多种：

- **网络层错误**：断网、DNS 解析失败、连接超时 → `requests.exceptions.RequestException`
- **API 业务错误**：城市不存在（404）、API Key 错误（401）、超出调用限额（429）→ HTTP 状态码非 200
- **数据解析错误**：API 返回了意外的数据格式，导致取字段时报 `KeyError` 或 `TypeError`

**为什么要分层处理？** 就像医院分科室：感冒去内科，骨折去骨科，眼睛不舒服去眼科。把不同类型的错误交给对应的处理逻辑，用户才能看到清晰、有帮助的错误信息，而不是一堆令人看不懂的 Python 报错。

#### 示例代码

```python
import requests

def fetch_weather(city: str, api_key: str) -> dict | None:
    """
    演示三层异常处理：
    第一层：网络连接问题
    第二层：API 返回的业务错误
    第三层：数据结构解析问题
    """
    url = "https://api.openweathermap.org/data/2.5/weather"
    params = {"q": city, "appid": api_key, "units": "metric", "lang": "zh_cn"}

    # ── 第一层：捕获网络层异常 ────────────────────────────────────
    # requests 所有异常都继承自 RequestException，所以可以统一捕获
    try:
        response = requests.get(url, params=params, timeout=10)
    except requests.exceptions.ConnectionError:
        # 原因：断网、DNS 解析失败、服务器拒绝连接
        print("错误：无法连接到网络，请检查你的网络设置。")
        return None
    except requests.exceptions.Timeout:
        # 原因：服务器响应太慢，超过了 timeout=10 秒
        print("错误：请求超时，服务器响应太慢，请稍后重试。")
        return None
    except requests.exceptions.RequestException as e:
        # 兜底：其他所有 requests 相关异常
        print(f"错误：网络请求失败 → {e}")
        return None

    # ── 第二层：检查 API 业务错误（HTTP 状态码）────────────────────
    # 此时网络请求已成功，但 API 可能返回了错误状态码
    if response.status_code == 401:
        print("错误：API Key 无效或未激活，请检查你的 Key 是否正确。")
        return None
    elif response.status_code == 404:
        print(f"错误：找不到城市 '{city}'，请检查拼写（建议用英文城市名）。")
        return None
    elif response.status_code == 429:
        print("错误：API 调用次数超出限额，请等待一分钟后重试。")
        return None
    elif response.status_code != 200:
        print(f"错误：API 返回异常状态码 {response.status_code}。")
        return None

    # ── 第三层：处理数据解析错误 ────────────────────────────────────
    # 此时状态码为 200，但 JSON 结构可能不符合预期
    try:
        data = response.json()
        # 尝试访问核心字段，提前验证数据完整性
        _ = data["main"]["temp"]
        _ = data["weather"][0]["description"]
        return data
    except (KeyError, IndexError, TypeError) as e:
        # KeyError：字段不存在；IndexError：列表为空；TypeError：类型不对
        print(f"错误：解析天气数据时出错，数据格式异常 → {e}")
        return None
```

## 第四部分：完整代码实现

### 4.1 完整的 weather.py

```python
"""
天气查询工具
功能：通过 OpenWeatherMap API 查询城市天气
用法：python weather.py
依赖：pip install requests python-dotenv
"""

import os
import time
from datetime import datetime, timezone, timedelta

import requests
from dotenv import load_dotenv


class WeatherChecker:
    """
    天气查询器类

    将所有天气相关操作封装在一个类里，好处是：
    1. API Key 只需保存一次（在 __init__ 里），不用每次调用都传
    2. 相关功能集中在一起，代码更清晰
    3. 未来想扩展（比如加缓存），只需修改这个类即可
    """

    # OpenWeatherMap 当前天气 API 地址（类属性，所有实例共用）
    BASE_URL = "https://api.openweathermap.org/data/2.5/weather"

    def __init__(self, api_key: str) -> None:
        """
        初始化天气查询器

        参数：
            api_key: OpenWeatherMap 的 API Key
        """
        if not api_key:
            raise ValueError("API Key 不能为空，请先在 .env 文件中设置 OPENWEATHER_API_KEY")
        self.api_key = api_key
        # 公共请求参数（每次请求都需要）
        self._base_params = {
            "appid": self.api_key,
            "units": "metric",   # 温度单位：摄氏度
            "lang": "zh_cn",     # 天气描述语言：中文
        }

    def get_weather(self, city: str) -> dict | None:
        """
        按城市名称查询当前天气

        参数：
            city: 城市名称，推荐使用英文（如 "Beijing"、"Shanghai"）
                  也支持中文，但部分城市可能查不到

        返回：
            成功时返回包含天气数据的字典，失败时返回 None
        """
        # 构造本次请求的参数（在公共参数基础上加上城市名）
        params = {**self._base_params, "q": city}

        # 步骤一：发送 HTTP GET 请求
        try:
            response = requests.get(self.BASE_URL, params=params, timeout=10)
        except requests.exceptions.ConnectionError:
            print("  [错误] 无法连接到网络，请检查网络设置。")
            return None
        except requests.exceptions.Timeout:
            print("  [错误] 请求超时（超过 10 秒），服务器可能暂时不可用。")
            return None
        except requests.exceptions.RequestException as e:
            print(f"  [错误] 网络请求异常：{e}")
            return None

        # 步骤二：检查 HTTP 状态码
        if not self._check_status(response, city):
            return None

        # 步骤三：解析 JSON 数据
        return self._parse_response(response)

    def get_weather_by_coordinates(self, lat: float, lon: float) -> dict | None:
        """
        按地理坐标（经纬度）查询当前天气

        参数：
            lat: 纬度（-90 到 90，正数为北纬，负数为南纬）
            lon: 经度（-180 到 180，正数为东经，负数为西经）

        返回：
            成功时返回包含天气数据的字典，失败时返回 None

        使用示例：
            # 北京：纬度 39.9042，经度 116.4074
            data = checker.get_weather_by_coordinates(39.9042, 116.4074)
        """
        # 验证经纬度范围
        if not (-90 <= lat <= 90):
            print(f"  [错误] 纬度 {lat} 超出范围（应在 -90 到 90 之间）。")
            return None
        if not (-180 <= lon <= 180):
            print(f"  [错误] 经度 {lon} 超出范围（应在 -180 到 180 之间）。")
            return None

        params = {
            **self._base_params,
            "lat": lat,
            "lon": lon,
        }

        try:
            response = requests.get(self.BASE_URL, params=params, timeout=10)
        except requests.exceptions.ConnectionError:
            print("  [错误] 无法连接到网络，请检查网络设置。")
            return None
        except requests.exceptions.Timeout:
            print("  [错误] 请求超时，请稍后重试。")
            return None
        except requests.exceptions.RequestException as e:
            print(f"  [错误] 网络请求异常：{e}")
            return None

        if not self._check_status(response, f"({lat}, {lon})"):
            return None

        return self._parse_response(response)

    def _check_status(self, response: requests.Response, location: str) -> bool:
        """
        检查 HTTP 响应状态码，打印对应错误信息

        这是一个"私有方法"（以下划线开头），只在类内部使用。
        把状态码检查逻辑提取出来，避免在 get_weather 和
        get_weather_by_coordinates 中重复写同样的代码。

        参数：
            response: requests 返回的响应对象
            location: 查询的位置（用于错误提示）

        返回：
            True 表示状态码正常（200），False 表示有错误
        """
        status = response.status_code
        if status == 200:
            return True
        elif status == 401:
            print("  [错误] API Key 无效。请确认：")
            print("         1. .env 文件中 OPENWEATHER_API_KEY 是否正确")
            print("         2. 新注册的 Key 是否已等待 10 分钟激活")
        elif status == 404:
            print(f"  [错误] 找不到位置 '{location}'。")
            print("         提示：城市名请使用英文，例如 Beijing、Shanghai、Chengdu")
        elif status == 429:
            print("  [错误] API 调用次数超出每分钟限额（60次），请稍后重试。")
        else:
            # 对于其他未知状态码，尝试从响应体提取 API 错误信息
            try:
                err_msg = response.json().get("message", "未知错误")
            except Exception:
                err_msg = response.text
            print(f"  [错误] API 返回错误（状态码 {status}）：{err_msg}")
        return False

    def _parse_response(self, response: requests.Response) -> dict | None:
        """
        将响应体解析为 Python 字典，并验证关键字段存在

        参数：
            response: 状态码已确认为 200 的响应对象

        返回：
            成功时返回数据字典，解析失败时返回 None
        """
        try:
            data = response.json()
            # 提前验证关键字段，避免在 display_weather 里才发现数据残缺
            required_keys = [
                ("main", "temp"),
                ("main", "humidity"),
                ("weather",),
            ]
            for keys in required_keys:
                obj = data
                for k in keys:
                    obj = obj[k]  # 如果字段不存在，会抛出 KeyError
            # 验证 weather 是非空列表
            if not data["weather"]:
                raise ValueError("weather 字段为空列表")
            return data
        except (KeyError, IndexError) as e:
            print(f"  [错误] 数据字段缺失，API 返回了意外的格式：{e}")
            return None
        except ValueError as e:
            print(f"  [错误] 数据校验失败：{e}")
            return None

    def format_time(self, timestamp: int, timezone_offset: int = 0) -> str:
        """
        将 Unix 时间戳转换为人类可读的时间字符串

        什么是 Unix 时间戳？
        Unix 时间戳是从 1970年1月1日 00:00:00 UTC 到现在经过的秒数。
        例如 1704067200 表示 2024-01-01 00:00:00 UTC。
        OpenWeatherMap 返回的日出日落时间就是这种格式。

        参数：
            timestamp:       Unix 时间戳（整数，单位：秒）
            timezone_offset: API 返回的时区偏移量（单位：秒）
                             例如北京时间 UTC+8 对应 28800 秒

        返回：
            格式化的时间字符串，例如 "06:51:23"
        """
        # 先将时间戳转为 UTC 时间，再加上时区偏移得到本地时间
        utc_time = datetime.fromtimestamp(timestamp, tz=timezone.utc)
        local_tz = timezone(timedelta(seconds=timezone_offset))
        local_time = utc_time.astimezone(local_tz)
        return local_time.strftime("%H:%M:%S")

    def display_weather(self, data: dict) -> None:
        """
        格式化打印天气信息到终端

        参数：
            data: get_weather() 或 get_weather_by_coordinates() 返回的字典
        """
        # 从嵌套字典中提取各字段
        # .get() 方法：字段不存在时返回默认值，避免 KeyError
        city_name     = data.get("name", "未知城市")
        country       = data.get("sys", {}).get("country", "")
        temp          = data["main"]["temp"]
        feels_like    = data["main"].get("feels_like", temp)
        temp_min      = data["main"].get("temp_min", temp)
        temp_max      = data["main"].get("temp_max", temp)
        humidity      = data["main"]["humidity"]
        description   = data["weather"][0].get("description", "未知")
        wind_speed    = data.get("wind", {}).get("speed", 0)
        wind_deg      = data.get("wind", {}).get("deg", 0)
        visibility    = data.get("visibility", None)  # 单位：米，可能不存在
        timezone_off  = data.get("timezone", 0)       # 时区偏移（秒）
        sunrise_ts    = data.get("sys", {}).get("sunrise")
        sunset_ts     = data.get("sys", {}).get("sunset")

        # 将风向角度转换为中文方向
        wind_direction = self._deg_to_direction(wind_deg)

        # 将能见度从米转换为千米
        visibility_str = f"{visibility / 1000:.1f} 千米" if visibility else "N/A"

        # 日出日落时间格式化
        sunrise_str = self.format_time(sunrise_ts, timezone_off) if sunrise_ts else "N/A"
        sunset_str  = self.format_time(sunset_ts,  timezone_off) if sunset_ts  else "N/A"

        # 打印格式化结果（使用 ASCII 框线让输出更整洁）
        print()
        print("┌─────────────────────────────────────────┐")
        print(f"│  {city_name}, {country}".ljust(42) + "│")
        print("├─────────────────────────────────────────┤")
        print(f"│  天气状况：{description}".ljust(42) + "│")
        print(f"│  当前温度：{temp:.1f}°C（体感 {feels_like:.1f}°C）".ljust(42) + "│")
        print(f"│  最低 / 最高：{temp_min:.1f}°C / {temp_max:.1f}°C".ljust(42) + "│")
        print(f"│  相对湿度：{humidity}%".ljust(42) + "│")
        print(f"│  风速风向：{wind_speed} 米/秒，{wind_direction}".ljust(42) + "│")
        print(f"│  能见度：{visibility_str}".ljust(42) + "│")
        print("├─────────────────────────────────────────┤")
        print(f"│  日出：{sunrise_str}   日落：{sunset_str}".ljust(42) + "│")
        print("└─────────────────────────────────────────┘")
        print()

    @staticmethod
    def _deg_to_direction(deg: float) -> str:
        """
        将风向角度（0-360度）转换为中文方位

        角度说明：0/360°= 正北，90°= 正东，180°= 正南，270°= 正西

        这是一个静态方法（@staticmethod），因为它不需要访问实例属性（self），
        是一个纯粹的工具函数，只做计算，不依赖类的状态。
        """
        directions = [
            "北风", "东北风", "东风", "东南风",
            "南风", "西南风", "西风", "西北风",
        ]
        # 将 360 度均分为 8 个方向，每个方向占 45 度
        # (deg + 22.5) // 45 计算出对应的方向索引
        index = int((deg + 22.5) // 45) % 8
        return directions[index]


def main() -> None:
    """
    主函数：交互式命令行循环

    程序入口，负责：
    1. 加载 .env 文件中的 API Key
    2. 创建 WeatherChecker 实例
    3. 持续读取用户输入，查询天气，直到用户主动退出
    """
    # 加载 .env 文件中的环境变量
    # load_dotenv() 会查找当前目录或父目录的 .env 文件
    load_dotenv()
    api_key = os.getenv("OPENWEATHER_API_KEY", "")

    if not api_key:
        print("错误：未找到 API Key。")
        print("请在项目目录下创建 .env 文件，内容如下：")
        print("  OPENWEATHER_API_KEY=你的API Key")
        print("注册地址：https://openweathermap.org/api")
        return

    # 创建天气查询器实例
    try:
        checker = WeatherChecker(api_key)
    except ValueError as e:
        print(f"初始化失败：{e}")
        return

    print("=" * 45)
    print("        欢迎使用天气查询工具")
    print("  输入城市名查询天气（推荐英文名称）")
    print("  输入 quit / exit / q 退出程序")
    print("=" * 45)

    # 主循环：持续接受用户输入
    while True:
        try:
            # input() 会阻塞，等待用户输入后按回车
            city = input("\n请输入城市名称：").strip()
        except KeyboardInterrupt:
            # 用户按下 Ctrl+C 时，input() 会抛出 KeyboardInterrupt
            print("\n\n已退出，再见！")
            break

        # 检查是否为退出命令（不区分大小写）
        if city.lower() in ("quit", "exit", "q", "退出"):
            print("再见！")
            break

        # 忽略空输入
        if not city:
            print("  提示：请输入城市名称，不能为空。")
            continue

        # 查询天气
        print(f"  正在查询 '{city}' 的天气……")
        weather_data = checker.get_weather(city)

        # 只有查询成功才显示结果
        if weather_data:
            checker.display_weather(weather_data)


# Python 惯例：只有直接运行这个文件时才执行 main()
# 如果这个文件被其他模块 import，则不会自动执行
if __name__ == "__main__":
    main()
```

## 第五部分：在没有真实 API Key 时测试

### 5.1 为什么要用 Mock 测试

在开发过程中，你可能遇到这些情况：

- API Key 还没激活（新注册需要等 10 分钟）
- 网络不通（在地铁、飞机上写代码）
- 不想消耗 API 配额
- 需要测试错误处理逻辑（比如模拟 404 响应）

这时候可以用 Python 标准库里的 `unittest.mock` 来**模拟** API 的响应——让代码"以为"自己真的收到了 API 数据，而实际上根本没有发网络请求。

**类比：** 就像飞行员在模拟舱里练习，所有仪表读数都是模拟的，但训练效果和真机完全一样。

#### 概念说明

`unittest.mock.patch` 是 Python 标准库提供的"打补丁"工具。它可以在测试期间临时替换某个函数或对象，测试结束后自动恢复原样。

我们要替换的是 `requests.get`，让它不发真实请求，而是直接返回我们准备好的假数据。

#### 示例代码

```python
"""
天气查询工具的 Mock 测试
文件：test_weather_mock.py
运行：python test_weather_mock.py
无需真实 API Key，无需联网
"""

import json
import unittest
from unittest.mock import patch, MagicMock

# 导入我们要测试的类
# 假设 weather.py 和本文件在同一目录
from weather import WeatherChecker


# 准备一份"假的"API 响应数据，模拟 OpenWeatherMap 返回的真实 JSON 结构
MOCK_WEATHER_DATA = {
    "name": "Beijing",
    "main": {
        "temp": 8.5,
        "feels_like": 5.2,
        "temp_min": 5.0,
        "temp_max": 11.0,
        "humidity": 38,
    },
    "weather": [
        {
            "id": 800,
            "main": "Clear",
            "description": "晴",
            "icon": "01d",
        }
    ],
    "wind": {
        "speed": 4.1,
        "deg": 315,   # 315 度 = 西北风
    },
    "visibility": 10000,  # 10 千米
    "sys": {
        "country": "CN",
        "sunrise": 1704928800,
        "sunset": 1704965400,
    },
    "timezone": 28800,    # UTC+8，北京时间
}


class TestWeatherChecker(unittest.TestCase):
    """
    WeatherChecker 类的单元测试

    每个 test_ 开头的方法都是一个独立的测试用例。
    unittest 框架会自动发现并运行它们。
    """

    def setUp(self):
        """
        setUp 在每个测试方法运行前自动调用。
        用于准备测试所需的公共资源。
        """
        # 使用假的 API Key 创建实例（因为我们会 mock 掉网络请求，所以 Key 不重要）
        self.checker = WeatherChecker("fake-api-key-for-testing")

    # ── 测试用例一：正常查询成功 ────────────────────────────────────
    @patch("weather.requests.get")  # 将 weather.py 里的 requests.get 替换为 Mock
    def test_get_weather_success(self, mock_get):
        """测试：城市名查询成功时，应返回包含正确数据的字典"""

        # 配置 Mock 对象的行为：
        # 当代码调用 requests.get(...) 时，返回我们的假响应
        mock_response = MagicMock()
        mock_response.status_code = 200                   # 模拟状态码 200
        mock_response.json.return_value = MOCK_WEATHER_DATA  # 模拟 .json() 的返回值
        mock_get.return_value = mock_response              # 让 get() 返回这个假响应

        # 调用被测方法
        result = self.checker.get_weather("Beijing")

        # 断言：返回值不为 None
        self.assertIsNotNone(result)
        # 断言：返回的城市名正确
        self.assertEqual(result["name"], "Beijing")
        # 断言：温度字段存在且正确
        self.assertEqual(result["main"]["temp"], 8.5)

        # 断言：requests.get 确实被调用了一次（验证代码真的发了请求）
        mock_get.assert_called_once()

    # ── 测试用例二：城市不存在（404）────────────────────────────────
    @patch("weather.requests.get")
    def test_get_weather_city_not_found(self, mock_get):
        """测试：API 返回 404 时，get_weather 应返回 None"""
        mock_response = MagicMock()
        mock_response.status_code = 404
        mock_response.json.return_value = {"cod": "404", "message": "city not found"}
        mock_get.return_value = mock_response

        result = self.checker.get_weather("不存在的城市XYZ")

        # 404 时应该返回 None
        self.assertIsNone(result)

    # ── 测试用例三：API Key 无效（401）──────────────────────────────
    @patch("weather.requests.get")
    def test_get_weather_invalid_api_key(self, mock_get):
        """测试：API 返回 401 时，get_weather 应返回 None"""
        mock_response = MagicMock()
        mock_response.status_code = 401
        mock_response.json.return_value = {"cod": 401, "message": "Invalid API key"}
        mock_get.return_value = mock_response

        result = self.checker.get_weather("Beijing")
        self.assertIsNone(result)

    # ── 测试用例四：网络超时 ─────────────────────────────────────────
    @patch("weather.requests.get")
    def test_get_weather_timeout(self, mock_get):
        """测试：网络超时时，get_weather 应返回 None 而不是抛出异常"""
        import requests as req
        # 让 requests.get() 直接抛出 Timeout 异常
        mock_get.side_effect = req.exceptions.Timeout("连接超时")

        result = self.checker.get_weather("Beijing")
        self.assertIsNone(result)

    # ── 测试用例五：format_time 方法 ─────────────────────────────────
    def test_format_time(self):
        """测试：Unix 时间戳转换是否正确（不需要网络，直接测试）"""
        # 1704067200 = 2024-01-01 00:00:00 UTC
        # 加上 UTC+8 偏移（28800秒）= 2024-01-01 08:00:00
        result = self.checker.format_time(1704067200, timezone_offset=28800)
        self.assertEqual(result, "08:00:00")

    # ── 测试用例六：按经纬度查询 ─────────────────────────────────────
    @patch("weather.requests.get")
    def test_get_weather_by_coordinates_success(self, mock_get):
        """测试：经纬度查询成功时，应返回正确数据"""
        mock_response = MagicMock()
        mock_response.status_code = 200
        mock_response.json.return_value = MOCK_WEATHER_DATA
        mock_get.return_value = mock_response

        # 北京的经纬度
        result = self.checker.get_weather_by_coordinates(39.9042, 116.4074)
        self.assertIsNotNone(result)

    # ── 测试用例七：经纬度范围验证 ───────────────────────────────────
    def test_invalid_latitude(self):
        """测试：纬度超出范围时，应返回 None（不应发出网络请求）"""
        # 纬度必须在 -90 到 90 之间，91 是非法值
        result = self.checker.get_weather_by_coordinates(91.0, 116.4)
        self.assertIsNone(result)

    def test_invalid_longitude(self):
        """测试：经度超出范围时，应返回 None"""
        result = self.checker.get_weather_by_coordinates(39.9, 181.0)
        self.assertIsNone(result)

    # ── 测试用例八：风向转换 ─────────────────────────────────────────
    def test_deg_to_direction(self):
        """测试：风向角度转换是否正确"""
        # 使用类名直接调用静态方法
        self.assertEqual(WeatherChecker._deg_to_direction(0),   "北风")
        self.assertEqual(WeatherChecker._deg_to_direction(90),  "东风")
        self.assertEqual(WeatherChecker._deg_to_direction(180), "南风")
        self.assertEqual(WeatherChecker._deg_to_direction(270), "西风")
        self.assertEqual(WeatherChecker._deg_to_direction(315), "西北风")
        self.assertEqual(WeatherChecker._deg_to_direction(360), "北风")  # 360 = 0


if __name__ == "__main__":
    # 运行所有测试，verbosity=2 表示打印每个测试用例的名称和结果
    unittest.main(verbosity=2)
```

### 练习一：天气预警（高温/低温提醒）

在 `display_weather()` 方法末尾添加逻辑：当温度超过 35°C 时打印高温预警，低于 0°C 时打印低温预警。

# 在 display_weather() 末尾添加
if temp >= 35:

    print("  ⚠  低温预警：气温低于 0°C，注意保暖，小心路面结冰！")
