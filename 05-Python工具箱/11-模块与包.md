# 第 11 章 - 模块与包（详细版）

## 第一部分：模块基础

### 11.1 什么是模块

#### 概念说明

**模块（Module）** 是一个包含 Python 代码的文件（`.py` 后缀），可以包含：
- 函数（functions）
- 类（classes）
- 变量（variables）
- 常量（constants）

```
┌─────────────────────────────────────────┐
│            模块的概念图                 │
├─────────────────────────────────────────┤
│                                         │
│   大型程序                              │
│  ┌─────────────────────────────────┐    │
│  │  main.py                        │    │
│  │                                 │    │
│  │  import math      ← 导入模块    │    │
│  │  import utils     ← 导入模块    │    │
│  │  from config import API_KEY     │    │
│  └─────────────────────────────────┘    │
│           │         │         │         │
│           ▼         ▼         ▼         │
│  ┌───────────┐ ┌─────────┐ ┌─────────┐ │
│  │ math.py   │ │utils.py │ │config.py│ │
│  │ (标准库)  │ │(自定义) │ │(自定义) │ │
│  └───────────┘ └─────────┘ └─────────┘ │
│                                         │
│  好处：代码复用、便于维护、分工合作     │
│                                         │
└─────────────────────────────────────────┘
```

#### 为什么使用模块

```
┌─────────────────────────────────────────┐
│          使用模块的好处                 │
├─────────────────────────────────────────┤
│                                         │
│  代码复用                               │
│     写一次，到处使用                    │
│                                         │
│  组织代码                               │
│     相关功能放在一个文件中              │
│                                         │
│  团队协作                               │
│     不同人负责不同模块                  │
│                                         │
│  使用标准库                             │
│     直接使用 Python 提供的强大功能      │
│                                         │
│  使用第三方库                           │
│     站在巨人的肩膀上                    │
│                                         │
└─────────────────────────────────────────┘
```

#### 模块化编程的类比

```
┌─────────────────────────────────────────┐
│         模块化编程 ≈ 组装电脑           │
├─────────────────────────────────────────┤
│                                         │
│   不用模块的方式：                      │
│   ┌─────────────────────────────────┐   │
│   │  所有代码写在一个文件里          │   │
│   │  像一个巨大的毛坯房              │   │
```

### 11.2 导入模块的四种方式

#### 方式一：导入整个模块

```python
# 语法：import 模块名

import math

# 使用时需要加上模块名前缀
print(math.sqrt(16))    # 4.0
print(math.pi)          # 3.141592653589793
print(math.ceil(4.2))   # 5（向上取整）
print(math.floor(4.8))  # 4（向下取整）
```

**图解：**
```
import math

┌─────────────────────────────────┐
│           math 模块              │
├─────────────────────────────────│
│  sqrt()  ← math.sqrt() 调用它   │
│  pi      ← math.pi 访问它       │
│  ceil()                         │
│  floor()                        │
│  sin(), cos(), tan()            │
│  log(), exp()                   │
│  ...                            │
└─────────────────────────────────┘
      ↑
      │
   需要加前缀
```

#### 方式二：导入特定内容

```python
# 语法：from 模块名 import 内容 1, 内容 2

from math import sqrt, pi

# 使用时不需要加模块名前缀
print(sqrt(16))    # 4.0
print(pi)          # 3.141592653589793

# ❌ 以下会报错，因为没有导入
# print(math.ceil(4.2))  # NameError!
```

**图解：**
```
from math import sqrt, pi

┌─────────────────────────────────┐
│           math 模块              │
├─────────────────────────────────│
│  sqrt()  ──────────────→ 直接导入│
│  pi      ──────────────→ 直接导入│
│  ceil()  (未导入，不可用)        │
│  floor() (未导入，不可用)        │
└─────────────────────────────────┘

直接使用：sqrt(16) ✓
```

#### 方式三：导入所有内容

```python
# 语法：from 模块名 import *

from math import *

# 所有公开内容都可以直接使用
print(sqrt(16))
print(pi)
print(ceil(4.2))

# ⚠️ 不推荐使用！原因：
# 1. 不知道有哪些名字被导入
# 2. 可能覆盖自己定义的名字
# 3. 代码可读性差
```

**警告：**
```
┌─────────────────────────────────────────┐
│          ⚠️ 不推荐使用 import *          │
├─────────────────────────────────────────┤
│                                         │
│  问题 1：名字冲突                         │
│  ─────────────────────                  │
│  # 你自己定义了 sum 函数                 │
│  def sum(a, b):                         │
│      return a + b                       │
│                                         │
│  # 从某模块导入所有内容                  │
│  from some_module import *              │
│                                         │
│  # ❌ 你的 sum 函数被覆盖了！             │
│                                         │
│  ─────────────────────────────────      │
│                                         │
│  问题 2：不知道有哪些名字                │
│  阅读代码时，不知道哪些名字可用          │
│                                         │
│  ─────────────────────────────────      │
│                                         │
│  ✅ 推荐做法：明确导入需要的内容         │
│  from math import sqrt, pi              │
│                                         │
└─────────────────────────────────────────┘
```

#### 方式四：使用别名

```python
# 给模块起别名
import math as m

print(m.sqrt(16))    # 4.0
print(m.pi)          # 3.141592653589793

# 给导入的内容起别名
from math import sqrt as square_root

print(square_root(16))  # 4.0

# 常见用例：导入第三方库
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
```

**别名使用场景：**
```
┌─────────────────────────────────────────┐
│            别名的使用场景               │
├─────────────────────────────────────────┤
│                                         │
│  场景 1：模块名太长                     │
│  ─────────────────────────              │
│  import very_long_module_name as vlm    │
│                                         │
│  场景 2：名字冲突                        │
│  ─────────────────────────              │
│  # 两个模块都有 process 函数             │
│  from module1 import process as p1      │
│  from module2 import process as p2      │
│                                         │
│  场景 3：约定俗成的习惯                 │
│  ─────────────────────────              │
│  import numpy as np      # 标准用法     │
│  import pandas as pd     # 标准用法     │
│                                         │
└─────────────────────────────────────────┘
```

## 第三部分：自定义模块

### 11.8 创建和使用自定义模块

#### 创建模块文件

```
项目结构：
my_project/
├── main.py          # 主程序
└── mymodule.py      # 自定义模块
```

```python
# mymodule.py

# 模块文档字符串
"""
这是一个示例模块
包含一些实用的函数和常量
"""

# 常量
PI = 3.141592653589793
VERSION = "1.0.0"

# 函数
def greet(name):
    """向指定的人打招呼"""
    return f"Hello, {name}!"

def add(a, b):
    """计算两个数的和"""
    return a + b

def circle_area(radius):
    """计算圆的面积"""
    return PI * radius ** 2

# 类
class Person:
    """表示一个人的类"""

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        """自我介绍"""
        return f"我是{self.name}，今年{self.age}岁"
```

#### 使用自定义模块

```python
# main.py

# 方式 1：导入整个模块
import mymodule

print(mymodule.greet("Alice"))  # Hello, Alice!
print(mymodule.add(3, 5))       # 8
print(mymodule.PI)              # 3.141592653589793

# 方式 2：导入特定内容
from mymodule import greet, add

print(greet("Bob"))  # Hello, Bob!
print(add(10, 20))   # 30

# 方式 3：使用别名
import mymodule as mm

print(mm.greet("Charlie"))  # Hello, Charlie!

# 导入类
from mymodule import Person

person = Person("David", 30)
print(person.introduce())  # 我是 David，今年 30 岁
```

### 11.9 `__name__` 和 `__main__`

#### 理解 `__name__`

```python
# module_test.py

print(f"__name__ 的值是：{__name__}")

def hello():
    print("Hello from module!")

# 只有在直接运行此文件时才执行
if __name__ == "__main__":
    print("这是主程序，运行测试代码")
    hello()
```

**不同运行方式的输出：**
```
情况 1：直接运行
$ python module_test.py
__name__ 的值是：__main__
这是主程序，运行测试代码
Hello from module!

情况 2：被导入
# 在另一个文件中
import module_test
__name__ 的值是：module_test
（不会打印"这是主程序..."）
```

#### 图解 `__name__`

```
┌─────────────────────────────────────────┐
│       __name__ 的工作原理               │
├─────────────────────────────────────────┤
│                                         │
│  每个 Python 文件都有一个 __name__ 变量  │
│                                         │
│  直接运行文件时：                       │
│  ──────────────────                    │
│  __name__ = "__main__"                 │
│                                         │
│  被导入时：                             │
│  ──────────────────                    │
│  __name__ = "模块名"                    │
│                                         │
│  ─────────────────────────────────      │
│                                         │
│  if __name__ == "__main__":            │
│      # 只有直接运行时才执行的代码       │
│      # 通常用于测试或演示              │
│                                         │
└─────────────────────────────────────────┘
```

#### 实际应用

```python
# calculator.py

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b

# 测试代码（只有直接运行时才执行）
if __name__ == "__main__":
    print("测试计算器模块")
    print(f"add(2, 3) = {add(2, 3)}")       # 5
    print(f"subtract(5, 2) = {subtract(5, 2)}")  # 3
    print(f"multiply(4, 3) = {multiply(4, 3)}")  # 12
    print(f"divide(10, 2) = {divide(10, 2)}")    # 5.0
```

## 第四部分：包（Package）

### 11.10 什么是包

#### 包的概念

```
┌─────────────────────────────────────────┐
│         包的概念图解                    │
├─────────────────────────────────────────┤
│                                         │
│  模块：一个.py 文件                      │
│  ──────────────────                    │
│  mymodule.py                            │
│                                         │
│  包：包含多个模块的目录                  │
│  ──────────────────                    │
│  mypackage/                             │
│  ├── __init__.py                        │
│  ├── module1.py                         │
│  └── module2.py                         │
│                                         │
│  包可以包含子包                          │
│  ──────────────────                    │
│  mypackage/                             │
│  ├── __init__.py                        │
│  ├── module1.py                         │
│  └── subpackage/                        │
│      ├── __init__.py                    │
│      └── module3.py                     │
│                                         │
└─────────────────────────────────────────┘
```

#### 包的结构

```
my_package/
│
├── __init__.py          # 包的初始化文件（必需）
├── utils.py             # 工具模块
├── helpers.py           # 辅助模块
│
├── database/            # 子包
│   ├── __init__.py
│   ├── mysql.py
│   └── postgres.py
│
└── web/                 # 子包
    ├── __init__.py
    ├── server.py
    └── client.py
```

### 11.11 `__init__.py` 的作用

```python
# my_package/__init__.py

# 作用 1：标识这是一个包
# （Python 3.3+ 中不是必需的，但推荐使用）

# 作用 2：包的初始化代码
print("my_package 已加载")

# 作用 3：定义 __all__ 控制导入内容
__all__ = ['utils', 'helpers']

# 作用 4：导入内容到包级别，方便使用
from .utils import greet
from .helpers import add

# 作用 5：定义包的版本等信息
__version__ = "1.0.0"
__author__ = "Your Name"
```

#### 使用包

```python
# 方式 1：导入包中的模块
from my_package import utils
utils.greet("Alice")

# 方式 2：导入包中的特定内容
from my_package.utils import greet
greet("Alice")

# 方式 3：导入子包
from my_package.database import mysql

# 方式 4：使用包的简写（如果 __init__.py 中导入了）
from my_package import greet
greet("Alice")
```

#### 相对导入

```python
# my_package/module1.py

# 从同级模块导入
from . import module2
from .module2 import some_function

# 从父包导入
from .. import something
from ..parent_module import func

# 从子包导入
from .subpackage import module3
```

**相对导入符号说明：**
```
.      - 当前目录（当前包）
..     - 上级目录（父包）
...    - 上上级目录
.模块名 - 当前包中的模块
```

## 第五部分：第三方包管理

### 11.12 pip - Python 包管理器

#### 基本命令

```bash
# ─────────────────────────────────────
# 安装包
# ─────────────────────────────────────

# 安装最新版本的包
pip install requests

# 安装指定版本
pip install requests==2.28.0

# 安装最低版本
pip install "requests>=2.20.0"

# 安装版本范围
pip install "requests>=2.20.0,<3.0.0"

# ─────────────────────────────────────
# 升级包
# ─────────────────────────────────────

# 升级到最新版本
pip install --upgrade requests

# ─────────────────────────────────────
# 卸载包
# ─────────────────────────────────────

pip uninstall requests

# ─────────────────────────────────────
# 查看信息
# ─────────────────────────────────────

# 列出已安装的包
pip list

# 查看包的详细信息
pip show requests
# Name: requests
# Version: 2.28.0
# Summary: Python HTTP for Humans.

# ─────────────────────────────────────
# 导出和还原
# ─────────────────────────────────────

# 导出已安装的包列表
pip freeze > requirements.txt

# 从 requirements.txt 安装
pip install -r requirements.txt
```

#### requirements.txt 格式

```txt
# requirements.txt 示例

# 精确版本
requests==2.28.0
flask==2.0.1

# 最低版本
numpy>=1.20.0

# 版本范围
pandas>=1.0.0,<2.0.0

# 不包含特定版本（避免 bug 版本）
django>=3.0,!=3.1.0
```

## 本章总结

### 知识点回顾

| 知识点 | 要点 | 常用示例 |
|--------|------|----------|
| 导入模块 | `import`、`from...import` | `import math` |

### 常见错误提醒

```
┌─────────────────────────────────────────┐
│          常见错误及解决方法             │
├─────────────────────────────────────────┤
│                                         │
│  ❌ 忘记加模块前缀                      │
│  import math                            │

│  # 不要命名你的文件为 random.py         │
│  # 会导致导入标准库时出问题             │
│                                         │
│  ─────────────────────────────────      │
│                                         │
│  ❌ 循环导入                             │
│  # a.py 导入 b，b 又导入 a               │
│  # 解决方法：重构代码，提取公共部分     │
│                                         │
│  ─────────────────────────────────      │
│                                         │
│  ❌ 相对导入错误                        │
│  # 相对导入只能在包内使用               │
│  # 直接运行的文件不能用相对导入         │
│                                         │
└─────────────────────────────────────────┘
```

### 最佳实践

```
┌─────────────────────────────────────────┐
│            最佳实践建议                 │
├─────────────────────────────────────────┤
│                                         │
│  导入顺序                               │
│  1. 标准库（import os）                 │
│  2. 第三方库（import requests）         │
│  3. 本地模块（import mymodule）         │
│  每组之间用空行分隔                     │
│                                         │
│  使用 requirements.txt                  │
│  记录项目依赖，方便他人安装             │
│                                         │
│  使用虚拟环境                           │
│  python -m venv venv                    │
│  隔离项目依赖                           │
│                                         │
│  给模块写文档字符串                     │
│  """模块的说明文档"""                  │
│                                         │
│  明确导入，不用 import *                │
│  from module import func1, func2        │
│                                         │
└─────────────────────────────────────────┘
```
