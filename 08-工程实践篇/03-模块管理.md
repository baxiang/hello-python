# 第 03 章 - 模块管理

> 良好的模块组织是构建可维护 Python 项目的关键。

---

## 第一部分 - 导入系统

### 1.1 导入方式

```python
# 1. 导入整个模块
import math
result = math.sqrt(16)

# 2. 导入特定成员
from math import sqrt, pi
result = sqrt(16)

# 3. 导入并重命名
from math import sqrt as square_root
import numpy as np

# 4. 相对导入（包内）
from . import sibling_module
from ..parent_package import module
```

### 1.2 导入搜索路径

```python
import sys

# 查看搜索路径
print(sys.path)

# 添加路径（不推荐运行时修改）
sys.path.insert(0, '/path/to/module')

# 推荐：使用 PYTHONPATH 环境变量
# export PYTHONPATH=/path/to/module:$PYTHONPATH
```

### 1.3 __init__.py 的作用

```
my_package/
├── __init__.py      # 使目录成为包
├── module_a.py
└── subpackage/
    ├── __init__.py
    └── module_b.py
```

```python
# __init__.py - 定义包的公共接口
from .module_a import public_function
from .subpackage.module_b import AnotherClass

__version__ = '1.0.0'
__all__ = ['public_function', 'AnotherClass']  # 控制 * 导入
```

## 第二部分 - 项目结构

### 2.1 标准项目结构

```
my_project/
├── pyproject.toml       # 项目配置和依赖
├── README.md            # 项目说明
├── LICENSE              # 许可证
├── .gitignore           # Git 忽略
│
├── src/
│   └── my_package/      # 源代码
│       ├── __init__.py
│       ├── core.py
│       └── utils.py
│
├── tests/               # 测试目录
│   ├── __init__.py
│   └── test_core.py
│
└── docs/                # 文档目录
    └── index.md
```

### 2.2 src 布局的优势

```
✅ src/my_package/    # 推荐
   - 避免导入本地目录而非安装包
   - 测试导入时使用已安装的包
   - 符合打包最佳实践

❌ my_package/        # 避免（扁平布局）
   - 可能与本地目录冲突
   - 测试时导入行为不一致
```

### 2.3 pyproject.toml 精简配置

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "my-package"
version = "0.1.0"
description = "一个有用的 Python 包"
requires-python = ">=3.11"
dependencies = [
    "requests>=2.31.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.4.0",
    "black>=23.0.0",
    "ruff>=0.1.0",
]

[tool.hatch.build.targets.wheel]
packages = ["src/my_package"]
```

### 3.1 按功能分组

```

│   ├── routes.py
│   └── handlers.py
└── utils/          # 工具函数

```

### 3.2 按业务领域分组

```
my_package/
├── __init__.py
├── user/           # 用户领域
│   ├── __init__.py
│   ├── models.py
│   └── services.py
├── product/        # 产品领域
│   ├── __init__.py
│   ├── models.py
│   └── services.py
└── order/          # 订单领域
    ├── __init__.py
    ├── models.py
    └── services.py
```

### 3.3 避免循环导入

```python
# ❌ 错误：循环导入
# module_a.py
from module_b import func_b
def func_a():
    return func_b()

# module_b.py
from module_a import func_a  # 循环导入！
def func_b():
    return func_a()

# ✅ 解决：延迟导入
# module_a.py
def func_a():
    from module_b import func_b  # 需要时再导入
    return func_b()
```

## 第四部分 - 包的分发

### 4.1 准备发布

```python
# src/my_package/__init__.py
__version__ = "1.0.0"
__author__ = "Your Name"

from .core import main_function
from .utils import helper_function

__all__ = ["main_function", "helper_function"]
```

### 4.2 构建和发布

```bash
# 1. 安装构建工具
uv add --dev build twine

# 2. 构建分发包
python -m build

# 3. 检查包
twine check dist/*

# 4. 上传到 PyPI
twine upload dist/*

# 5. 上传到 TestPyPI（测试）
twine upload --repository testpypi dist/*
```

### 4.3 配置 PyPI

```ini
# ~/.pypirc
[pypi]
username = __token__
password = pypi-xxx

[testpypi]
repository = https://test.pypi.org/legacy/
username = __token__
password = pypi-xxx
```

## 最佳实践

| 实践 | 说明 |
|------|------|
| 使用 src 布局 | `src/my_package/` 避免导入冲突 |
| 保持模块简洁 | 每个模块专注于单一职责 |
| 明确导出内容 | 使用 `__all__` 控制公共接口 |
| 语义化版本 | `version = "1.2.3"` (MAJOR.MINOR.PATCH) |
| 避免循环导入 | 重构代码或使用延迟导入 |
